# <a name="dimensions-and-measurements-available-in-call-quality-dashboard-cqd"></a>CQD(통화 품질 대시보드)에서 사용할 수 있는 차원 및 측정값

Microsoft Teams 및 비즈니스용 Skype Online용 CQD(통화 품질 대시보드)를 사용하면 이러한 서비스로 이루어진 통화의 통화 품질을 더 잘 이해할 수 있습니다. 이 항목에서는 CQD를 통해 표시되는 차원 및 측정값에 대한 자세한 정보를 제공합니다. CQD에 대한 자세한 내용은 [CQD를 사용하여 Microsoft Teams 통화 및 모임 품질 관리를](quality-of-experience-review-guide.md) 참조하세요.

## <a name="first-and-second-endpoint-classification"></a>첫 번째 및 두 번째 엔드포인트 분류

CQD의 많은 차원 및 측정값은 첫 번째 또는 초로 레이블이 지정됩니다. 다음 논리는 스트림 또는 호출에 관련된 엔드포인트가 먼저 레이블이 지정된 엔드포인트를 결정합니다.

- 서버 엔드포인트(AV MCU, 중재 서버 등)는 서버가 스트림 또는 호출에 관여할 때 첫 번째 엔드포인트로 간주됩니다.
- 스트림이 두 서버 엔드포인트 사이에 있지 않으면 클라이언트 엔드포인트는 두 번째 엔드포인트로 간주됩니다.
- 두 엔드포인트가 동일한 형식인 경우 순서가 일관되도록 사용자 에이전트 범주의 내부 순서에 따라 첫 번째와 두 번째 엔드포인트가 설정됩니다.

예를 들어 여기에서 각 행은 스트림에 관련된 사용자 에이전트 쌍을 나타냅니다.

|:--- |:--- |:--- |:--- |:--- |
|AV-MCU |OC(비즈니스용 Skype 클라이언트) |AV-MCU |OC(비즈니스용 Skype 클라이언트) |사실 |
|OC(비즈니스용 Skype 클라이언트) |AV-MCU |AV-MCU |OC(비즈니스용 Skype 클라이언트) |FALSE |

## <a name="dimensions"></a>차원

차원 정보는 CQD 포털에 업로드된 데이터를 부분적으로 기반으로 합니다. 많은 차원 값을 필터로 사용할 수도 있습니다.

### <a name="dimension-data-types-and-units"></a>차원 데이터 형식 및 단위

#### <a name="boolean"></a>부울

부울 값은 항상 True 또는 False입니다. 경우에 따라 True를 1로 나타낼 수도 있고 False를 0으로 나타낼 수도 있습니다.

#### <a name="range"></a>범위

범위 또는 값 그룹으로 제공되는 차원은 다음 형식을 사용하여 표시됩니다.

 _\<sort order string\> [\<lower bound inclusive\> - \<upper bound exclusive\>_

예를 들어 Duration(분) 차원은 값 범위로 보고된 값이 있는 호출 기간(초)을 나타냅니다.

|기간(분) |해석 방법 |
|:--- |:--- |
|062: [0 - 0) |스트림 기간 = 0분 |
|064: [1 - 2) |1분 < = 스트림 기간 < 2분 |
|065: [2 - 3) |2분 < = 스트림 지속 시간 < 3분 |
|066: [3–4) |3분 < = 스트림 지속 시간 < 4분 |
|  | |

데이터를 \<sort order string> 표시할 때 정렬 순서를 제어하는 데 사용되며 필터링에 사용할 수 있습니다. 예를 들어 "065"< 기간(분)에 대한 필터는 기간이 2분 미만인 스트림을 표시합니다(필터가 예상대로 작동하려면 선행 '0'이 필요). 정렬 순서 문자열의 실제 값은 중요하지 않습니다.

> [!NOTE]
> 지정된 차원에 대해 잘못된 것으로 보이는 범위를 확인할 수 있습니다. 예를 들어 100이 Wifi 신호 강도에 대해 가능한 최대 값인 경우 082: [100 - 110) 범위의 호출을 보여 주는 Wifi 신호 강도입니다. 이는 CQD 데이터 모델의 범위에 숫자가 할당되는 방식 때문입니다. 정수 값이 99이면 081: [90 - 100) 범위에서 계산됩니다. 해당 값이 100이면 082: [100 - 110) 범위에서 계산됩니다. 이는 보고되는 100% 이상의 Wifi 신호 강도 값이 있음을 나타내지 않습니다.

#### <a name="enumeration-strings"></a>열거형 문자열

CQD에서 사용하는 문자열은 데이터 파일에서 파생되는 경우가 많으며 허용되는 길이 내의 거의 모든 문자 조합일 수 있습니다. 일부 차원은 문자열처럼 보이지만 미리 정의된 값의 짧은 목록 중 하나일 수 있으므로 열거형이며 true 문자열이 아닙니다. 일부 열거형 문자열도 쌍으로 사용됩니다.

#### <a name="enumeration-pair"></a>열거형 쌍

열거형 쌍으로 제공되는 차원은 다음 형식을 사용하여 표시됩니다.

 _\<enumeration value from one end point\> : \<enumeration value from the other endpoint\>_

열거형 값의 순서는 일관되지만 첫 번째 또는 두 번째 엔드포인트의 순서를 반영하지는 않습니다.

예를 들어 네트워크 연결 세부 정보 쌍에는 두 엔드포인트에 대한 네트워크 연결 세부 정보 값이 표시됩니다.

|네트워크 연결 세부 정보 쌍 |해석 방법 |
|:--- |:--- |
|유선 : 유선 |첫 번째 및 두 번째 엔드포인트는 모두 유선 이더넷 연결을 사용했습니다. |
|유선 : wifi |첫 번째 엔드포인트는 유선 이더넷 연결을 사용했고 두 번째 엔드포인트는 Wi-Fi 연결에 사용되거나 두 번째 엔드포인트는 유선 이더넷 연결을 사용했으며 첫 번째 엔드포인트는 Wi-Fi 연결에 사용되었습니다. |
|: wifi |첫 번째 엔드포인트는 WiFi 연결을 사용했으며 두 번째 엔드포인트에서 사용하는 네트워크 연결을 알 수 없거나 두 번째 엔드포인트가 WiFi 연결을 사용했으며 첫 번째 엔드포인트에서 사용하는 네트워크 연결을 알 수 없습니다. |
| | |

#### <a name="blank-values"></a>빈 값

아래 표에는 차원이 비어 있을 수 있는 가능한 이유가 나와 있습니다. QoE Record Available 차원이 false이면 많은 차원과 측정값이 비어 있습니다. 이는 일반적으로 호출이 성공적으로 설정되지 않았거나 클라이언트가 서비스에 원격 분석을 보내지 못한 경우에 발생합니다.

### <a name="notes-on-measurements"></a>측정에 대한 참고 사항

#### <a name="accuracy-limitations"></a>정확도 제한 사항
특정 사용자 및 호출 수 측정값은 개수를 계산하기 위해 데이터 집합에 대해 고유한 countif 연산을 수행하는 데 의존합니다. 현재 작업이 수행되는 행 수에 따라 고유 countif 작업에 내재된 최대 0.2% 오류가 있습니다. 가장 정확한 볼륨의 경우 스트림 개수 측정값은 이 고유 countif 작업에 의존하지 않으므로 사용해야 합니다. 데이터 볼륨을 줄이기 위한 필터링은 오류를 줄일 수 있지만 고유한 호출 및 사용자 수에서 이 오류 원본을 제거하지 못할 수 있습니다. 이 제한에 대한 자세한 내용은 [dcount 집계 함수](/azure/data-explorer/kusto/query/dcount-aggfunction)에서 찾을 수 있습니다.

## <a name="filters"></a>필터

많은 차원 및 측정값을 필터로 사용할 수도 있습니다. 쿼리에서 필터를 사용하여 차원 또는 측정값을 선택하여 쿼리에 정보를 추가하거나 포함하는 것과 같은 방식으로 정보를 제거할 수 있습니다.
