# <a name="private-channels-in-microsoft-teams"></a>Microsoft 팀의 개인 채널

Microsoft 팀의 개인 채널은 팀 내에서 공동 작업을 위해 중요 한 공간을 만듭니다. 소유자 또는 개인 채널의 구성원 인 팀의 사용자만 채널에 액세스할 수 있습니다. 게스트를 비롯 한 모든 사용자는 이미 팀의 구성원 인 경우 개인 채널의 구성원으로 추가 될 수 있습니다.

개인 채널을 사용 하 여 특정 프로젝트에 할당 된 사용자 그룹 간에 공동 작업을 수행 하 고 관리할 추가 팀을 만들 필요 없이 해당 사용자에 게 사용자 지정을 수행할 수 있습니다.

예를 들어 개인 채널은 다음과 같은 시나리오에서 유용 합니다.

- 팀의 사용자 그룹은 별도의 팀을 만들 필요 없이 공동 작업을 수행 하기 위해 중요 한 공간을 사용 하 고 있습니다.
- 팀 구성원의 하위 집합은 개인 채널을 통해 예산, resourcing, 전략적 포지셔닝 등의 중요 한 정보를 토론할 수 있도록 합니다.

잠금 아이콘은 개인 채널을 나타냅니다. 개인 채널의 구성원만이 추가 된 비공개 채널을 보고 참가할 수 있습니다.

![팀의 비공개 채널 스크린샷](media/private-channels-in-teams.png)

## <a name="what-you-need-to-know-about-private-channels"></a>개인 채널에 대해 알아야 할 사항

현재 개인 채널은 커넥터 및 탭 (스트림, Planner, 양식 제외)을 지원 합니다. 메시징 확장과 봇을 포함 하 여 개인 채널에 대 한 전체 앱 지원이 진행 되 고 있습니다.

각 팀은 개인 채널을 최대 30 개까지 포함할 수 있으며 각 개인 채널은 최대 250 구성원을 가질 수 있습니다. 30 개의 개인 채널 한도가 팀 당 200 표준 채널 한도에 추가 됩니다.

> [!NOTE]
> 개인 채널에 대 한 기능을 지속적으로 추가 하 여 앱, 채널 모임 및 대규모 팀의 개인 채널 크기 조정에 대 한 최신 정보를 다시 확인 합니다.

|예     |예         |아니요         |기존 팀에서 개인 채널을 만듭니다.         |
|예     |아니요         |아니요         |기존 팀에서 채널을 만듭니다.         |
|아니요     |아니요         |아니요         |새 팀을 만드는 것이 좋습니다.         |

개인 채널이 만들어지면 상위 팀에 연결 되며 다른 팀으로 이동할 수 없습니다. 또한 개인 채널을 표준 채널로 변환할 수 없으며 그 반대의 경우도 마찬가지입니다.

## <a name="private-channel-creation-and-membership"></a>개인 채널 만들기 및 구성원 자격

### <a name="who-can-create-private-channels"></a>개인 채널을 만들 수 있는 사람은 누구 인가요?

기본적으로 모든 팀 소유자나 팀 멤버가 개인 채널을 만들 수 있습니다. 관람객이 만들 수 없습니다. 개인 채널을 만드는 기능은 팀 수준 및 조직 수준에서 관리할 수 있습니다.

- 팀 소유자는 팀에 대 한 **설정** 탭에서 구성원이 개인 채널을 만들 수 있도록 설정 또는 해제 하도록 설정할 수 있습니다.
- 관리자는 [정책을](teams-policies.md) 사용 하 여 조직에서 개인 채널을 만들 수 있는 사용자를 제어할 수 있습니다.

개인 채널을 만든 사용자는 개인 채널 소유자 이며 개인 채널 소유자만 직접 사용자를 추가 하거나 제거할 수 있습니다. 개인 채널 소유자는 게스트를 포함 하 여 자신이 만든 개인 채널에 팀 구성원을 추가할 수 있습니다. 개인 채널의 구성원에 게는 보안 대화 공간이 있으며, 새 구성원이 추가 되 면 해당 개인 채널의 모든 대화 (이전 대화)를 볼 수 있습니다.

### <a name="what-happens-when-a-team-member-leaves-or-is-removed-from-a-team"></a>팀 구성원이 팀에서 탈퇴 하거나 제거 되 면 어떻게 되나요?

팀 구성원이 팀에서 탈퇴 하거나 제거 하는 경우 팀의 모든 비공개 채널 에서도 해당 사용자를 제거 하거나 제거할 수 있습니다. 사용자가 팀에 다시 추가 되는 경우 팀의 개인 채널에 다시 추가 되어야 합니다.

### <a name="what-happens-when-a-private-channel-owner-is-removed-from-a-private-channel"></a>비공개 채널 소유자가 개인 채널에서 제거 되 면 어떻게 되나요?

개인 채널 소유자는 하나 이상의 개인 채널의 마지막 소유자 인 경우 팀 클라이언트를 통해 제거할 수 없습니다.

개인 채널 소유자가 조직을 떠나는 경우 또는 팀과 연결 된 Office 365 그룹에서 해당 사용자가 제거 된 경우 개인 채널의 구성원이 자동으로 개인 채널 소유자로 승격 됩니다.

### <a name="what-can-team-owners-and-team-members-see-in-a-private-channel"></a>팀 소유자와 팀 구성원은 개인 채널에서 볼 수 있는 항목

팀 소유자는 팀의 모든 비공개 채널 이름을 볼 수 있으며 팀의 모든 비공개 채널을 삭제할 수도 있습니다. (삭제 한 후 30 일 내에 삭제 된 개인 채널을 복원할 수 있습니다.) 팀 소유자는 개인 채널의 파일을 볼 수 없으며 개인 채널의 구성원 인 경우에만 비공개 채널이 있는 대화 및 구성원 목록이 표시 됩니다.

다음 표에서는 개인 채널의 내용을 볼 수 있는 사용자를 보여 줍니다.

|항목과  |팀 소유자가 볼 수 있음 |팀 구성원이 볼 수 있습니다. |
|---------|---------|---------|
|이름 및 설명    |팀의 모든 개인 채널         |에 추가 된 개인 채널만         |
|대화 및 탭     |개인 채널에 추가 된 경우에만         |개인 채널에 추가 된 경우에만         |
|파일 및 콘텐츠    |개인 채널에 추가 된 경우에만        |개인 채널에 추가 된 경우에만         |
|개인 채널 소유자    |팀의 모든 개인 채널        |개인 채널에 추가 된 경우에만         |
|마지막 작업 시간 스탬프  |팀의 모든 개인 채널       |개인 채널에 추가 된 경우에만         |

### <a name="manage-private-channel-membership-and-settings"></a>개인 채널 멤버 자격 및 설정 관리

각 개인 채널에는 구성원을 추가 및 제거 하 고, 탭을 추가 하 고, 전체 채널에 대 한 @mentioning 하는 기능을 비롯 하 여 고유한 설정이 있습니다. 이러한 설정은 상위 팀 설정과 독립적입니다. 개인 채널이 생성 되 면 상위 팀 설정에 관계 없이 설정이 변경 될 수 있는 것으로 가정 하 여 해당 설정을 상속 합니다.

개인 채널 소유자는 **채널 관리**를 클릭 한 다음 **구성원** 및 **설정** 탭을 사용 하 여 구성원을 추가 하거나 제거 하 고 설정을 편집할 수 있습니다.

## <a name="manage-the-life-cycle-of-private-channels"></a>개인 채널의 수명 주기 관리

조직의 개인 채널 수명 주기를 관리 하는 방법에 대 한 지침은 [팀에서 비공개 채널의 수명 주기 관리](private-channels-life-cycle-management.md) 를 참조 하세요. 여기에는 조직의 사용자가 개인 채널을 만들 수 있는지 여부, 팀 소유자를 대신 하 여 개인 채널을 만드는 방법, 보관 및 감사 용도로 모든 개인 채널 메시지 목록 가져오기, 기타 관리 작업 등이 포함 됩니다.  

## <a name="private-channel-sharepoint-sites"></a>개인 채널 SharePoint 사이트

각 개인 채널에는 파일 공유 및 빠른 프로 비전에 최적화 된 고유한 SharePoint 사이트 모음이 있습니다. 별도의 사이트 모음은 팀 소유자가 사이트 모음 내의 모든 자산에 액세스할 수 있는 팀 사이트와 비교 하 여 개인 채널 파일에 대 한 액세스를 개인 채널의 구성원 만으로 제한 하도록 하는 것입니다. 이러한 사이트 모음은 기본적으로 문서 라이브러리를 사용 하 여 만들어지며 [사이트 관리 인터페이스](https://support.office.com/article/Enable-or-disable-site-collection-features-A2F2A5C2-093D-4897-8B7F-37F86D83DF04)를 통해 모든 기능을 갖춘 사이트 모음으로 쉽게 향상 될 수 있습니다. 각 사이트 모음은 상위 팀의 사이트 모음과 동일한 지역에서 만들어집니다. 이러한 경량 사이트에는 PowerShell 및 Graph API를 통해 쉽게 관리할 수 있는 사용자 지정 서식 파일 ID 인 "TEAMCHANNEL # 0"이 있습니다.  디자인에 따라 SharePoint 관리 센터에 표시 되지 않습니다.

테 넌 트 당 더 많은 수의 사이트 모음을 수용 하기 위해 한도가 50만에서 200만로 늘어났습니다. 개인 채널 사이트 모음은 데이터 분류를 동기화 하 고 상위 팀의 사이트 모음에서 게스트 액세스 권한을 상속 합니다.  사이트 모음 소유자 및 구성원 그룹에 대 한 멤버 자격은 팀 내의 개인 채널 구성원과 동기화 된 상태로 유지 됩니다. SharePoint Online의 소유자 또는 구성원 그룹의 구성원 자격에 대 한 변경 내용은 자동으로 4 시간 내에 개인 채널 구성원으로 되돌려집니다. 특정 사용자가 개인 채널 메시지에 액세스할 필요 없이 문서에 액세스 해야 하는 경우에는 사이트의 방문자 그룹 또는 소유자와 구성원을 구분 하는 새 그룹에 추가 합니다.

팀은 개인 채널 SharePoint 사이트 모음의 수명 주기를 관리 합니다. 팀 외부에서 사이트 모음을 삭제 한 경우 백그라운드 작업은 개인 채널이 여전히 활성 상태인 동안 4 시간 내에 사이트를 복원 합니다. 사이트를 삭제 하 고 하드 삭제 한 경우 개인 채널에 대해 새 사이트 모음이 프로 비전 됩니다.

개인 채널이 나 개인 채널이 포함 된 팀이 복원 되 면 사이트 모음이 복원 됩니다. 개인 채널 사이트 모음이 복원 되 고 개인 채널에 대 한 30 일 일시 삭제 창이 지난 경우 사이트 모음은 독립 실행형 사이트 모음으로 작동 합니다.

## <a name="private-channel-message-compliance-records"></a>개인 채널 메시지 준수 레코드

개인 채널로 전송 된 메시지의 레코드는 그룹 사서함이 아닌 모든 비공개 채널 구성원의 사서함으로 전달 됩니다. 레코드의 제목은 전송 된 개인 채널을 나타내도록 서식이 지정 됩니다.

개인 채널 메시지에 대해 eDiscovery 검색을 수행 하는 방법에 대 한 자세한 내용은 [개인 채널 eDiscovery](ediscovery-investigation.md#ediscovery-of-private-channels)를 참조 하세요.

## <a name="considerations-around-access-in-private-channels"></a>개인 채널의 액세스 관련 고려 사항

개인 채널에서 새 OneNote 전자 필기장을 만든 경우이 동작은 사용자가 개인 채널 SharePoint 사이트의 다른 항목에 대 한 액세스를 공유 하는 것과 동일 하므로 계속 해 서 전자 필기장에 액세스할 수 있습니다.

사용자에 게 SharePoint를 통해 개인 채널의 전자 필기장에 대 한 액세스 권한이 부여 되는 경우 팀 또는 개인 채널에서 사용자를 제거 해도 전자 필기장에 대 한 사용자의 액세스는 제거 되지 않습니다.

기존 전자 필기장이 개인 채널에 탭으로 추가 되는 경우 개인 채널에 대 한 액세스는 변경 되지 않습니다. 이는 다음을 의미 합니다.

- 개인 채널의 모든 사람이 기본적으로 전자 필기장에 액세스할 수 있는 것은 아닙니다. 이는 다른 팀의 SharePoint 사이트와 같이 전자 필기장이 호스팅되는 위치에 대 한 액세스 권한이 없을 수 있기 때문입니다.
- 개인 채널의 구성원이 아닌 사용자는 전자 필기장을 볼 수 있습니다.
