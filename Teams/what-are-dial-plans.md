# <a name="what-are-dial-plans"></a>다이얼 플랜이 무엇인가요?

다이얼 플랜은 전화 권한 부여 및 음성 라우팅을 위해 개별 사용자의 전화 접속 전화 번호를 대체 형식(일반적으로 E.164)으로 변환하는 명명된 정규화 규칙 집합입니다.

다이얼 플랜은 다양한 형식으로 표현된 전화 번호를 대체 형식으로 변환하는 방법을 정의하는 하나 이상의 정규화 규칙으로 구성됩니다. 동일한 다이얼 문자열을 다른 다이얼 플랜에서 다르게 해석하고 번역할 수 있으므로 지정된 사용자에게 할당된 다이얼 플랜에 따라 동일한 전화 걸기 번호가 번역되고 다르게 라우팅될 수 있습니다. 최대 1,000개의 테넌트 다이얼 플랜이 있을 수 있습니다.

[테넌트 다이얼 플랜을 만들고 관리하려면 다이얼 플랜 만들기 및 관리를](create-and-manage-dial-plans.md) 참조하세요.

## <a name="tenant-dial-plan-scope"></a>테넌트 다이얼 플랜 범위

다이얼 플랜의 범위는 다이얼 플랜을 적용할 수 있는 계층 수준을 결정합니다. 클라이언트는 사용자가 Teams에 로그인할 때 자동으로 제공되는 프로비저닝 설정을 통해 적절한 다이얼 플랜을 받습니다. 관리자는 Microsoft Teams 관리 센터 또는 원격 PowerShell을 사용하여 다이얼 플랜 범위 수준을 관리하고 할당할 수 있습니다.

Teams에는 서비스 범위 및 테넌트 범위(조직용)라는 두 가지 유형의 다이얼 플랜이 있습니다. 전화 시스템을 사용할 수 있는 모든 국가 또는 지역에 대해 서비스 범위 다이얼 플랜이 정의됩니다. 각 사용자에게 할당된 사용 위치와 일치하는 서비스 국가 다이얼 플랜이 자동으로 할당됩니다. 서비스 국가 다이얼 플랜은 변경할 수 없지만 서비스 국가 다이얼 플랜을 보강하는 테넌트 범위 다이얼 플랜을 만들 수 있습니다. 클라이언트가 프로비전되면 서비스 국가 다이얼 플랜과 적절하게 범위가 지정된 테넌트 다이얼 플랜의 조합인 "효과적인 다이얼 플랜"을 얻습니다. 따라서 서비스 국가 다이얼 플랜에 이미 있을 수 있으므로 테넌트 다이얼 플랜의 모든 정규화 규칙을 정의할 필요는 없습니다.

테넌트 다이얼 플랜은 테넌트 범위 또는 사용자 범위의 두 가지 범위로 더 세분화될 수 있습니다. 테넌트가 사용자 범위 다이얼 플랜을 정의하고 할당하는 경우 해당 사용자는 사용자의 서비스 국가 다이얼 플랜 및 할당된 사용자 다이얼 플랜의 효과적인 다이얼 플랜으로 프로비전됩니다. 테넌트가 테넌트 범위 다이얼 플랜을 정의하지만 사용자 범위 다이얼 플랜을 할당하지 않는 경우 해당 사용자는 사용자의 서비스 국가 다이얼 플랜 및 테넌트 다이얼 플랜의 유효 다이얼 플랜으로 프로비전됩니다.

다음은 Teams의 다이얼 플랜 상속 모델입니다.

![Teams에서 다이얼 플랜을 상속하는 방법.](media/b2744f33-ebbd-4c23-bfba-1747312ab178.png)

다음은 가능한 유효 다이얼 플랜입니다.

 **서비스 국가** 테넌트 범위 다이얼 플랜이 정의되어 있지 않고 프로비전된 사용자에게 테넌트 사용자 범위 다이얼 플랜이 할당되지 않은 경우 사용자는 사용 위치와 연결된 서비스 국가에 매핑된 유효 다이얼 플랜을 받게 됩니다.

 **테넌트 전역 - 서비스 국가** 테넌트 사용자 다이얼 플랜이 정의되었지만 사용자에게 할당되지 않은 경우 프로비전된 사용자는 병합된 테넌트 다이얼 플랜과 사용 위치와 연결된 서비스 국가 다이얼 플랜으로 구성된 유효 다이얼 플랜을 받게 됩니다.

 **테넌트 사용자 - 서비스 국가** 테넌트 사용자 다이얼 플랜이 정의되고 사용자에게 할당된 경우 프로비전된 사용자는 병합된 테넌트 사용자 다이얼 플랜과 사용 위치와 연결된 서비스 국가 다이얼 플랜으로 구성된 유효 다이얼 플랜을 받게 됩니다.

[테넌트 다이얼 플랜을 만들려면 다이얼 플랜 만들기 및 관리를](create-and-manage-dial-plans.md) 참조하세요.

> [!NOTE]
> 전화 걸기 번호에 다이얼 플랜 정규화 규칙이 적용되지 않는 시나리오에서는 다이얼링된 문자열이 "+CC"를 앞에 추가하도록 계속 정규화됩니다. 여기서 CC는 전화 걸기 사용자의 사용 위치의 국가 코드입니다. 이는 통화 플랜, 직접 라우팅 및 PSTN 컨퍼런스 전화 접속 시나리오에 적용됩니다. 또한 테넌트 다이얼 플랜 정규화 규칙에서 "+"로 시작하지 않는 숫자가 발생하는 경우 호출 서비스는 테넌트 다이얼 플랜에 따라 Teams 클라이언트에서 받은 번호를 정규화하려고 시도하며, 일치하지 않는 경우 지역 다이얼 플랜에서 해당 번호를 정규화합니다. 이중 정규화를 방지하려면 직접 라우팅 고객이 숫자를 정규화하여 +를 포함하고 트렁크 번역 규칙을 사용하여 +를 제거하는 것이 좋습니다. 

## <a name="planning-for-tenant-dial-plans"></a>테넌트 다이얼 플랜 계획

사용자 지정 다이얼 플랜을 계획하려면 다음 단계를 수행합니다.

- **1단계** 사용자 전화 걸기 환경을 향상시키기 위해 사용자 지정 다이얼 플랜이 필요한지 여부를 결정합니다. 일반적으로 확장 또는 약식 전국 전화 걸기와 같은 비 E.164 전화 걸기를 지원해야 합니다.

- **2단계** 테넌트 전역 또는 테넌트 사용자 범위 다이얼 플랜이 필요한지 또는 둘 다 필요한지 확인합니다. 사용자가 다른 로컬 전화 걸기 요구 사항이 있는 경우 사용자 범위 다이얼 플랜이 필요합니다.

- **3단계** 각 필수 다이얼 플랜에 대해 유효한 번호 패턴을 식별합니다. 서비스 수준 국가 다이얼 플랜에 정의되지 않은 숫자 패턴만 필요합니다.

- **4단계** 다이얼 플랜 이름을 지정하기 위한 조직 차원의 체계를 개발합니다. 표준 명명 체계를 채택하면 조직 전체에서 일관성을 보장하고 유지 관리 및 업데이트를 더 쉽게 수행할 수 있습니다.


## <a name="creating-your-new-dial-plan"></a>새 다이얼 플랜 만들기

새 다이얼 플랜을 만들 때 필요한 정보를 입력해야 합니다.

### <a name="name-and-simple-name"></a>이름 및 단순 이름

사용자 다이얼 플랜의 경우 다이얼 플랜이 할당될 사용자를 식별하는 설명이 포함된 이름을 지정해야 합니다. 다이얼 플랜 단순 이름은 다이얼 플랜 이름에서 파생된 문자열로 미리 채워집니다. 단순 이름 필드는 편집할 수 있으므로 다이얼 플랜에 대한 보다 설명적인 명명 규칙을 만들 수 있습니다. 단순 이름 값은 비워 둘 수 없으며 고유해야 합니다. 모범 사례는 전체 조직에 대한 명명 규칙을 개발한 다음 모든 사이트 및 사용자 간에 이 규칙을 일관되게 사용하는 것입니다.

### <a name="description"></a>설명

해당 다이얼 플랜이 적용되는 지리적 위치 또는 사용자 그룹의 일반적이고 인식 가능한 이름을 입력하는 것이 좋습니다.

### <a name="external-access-prefix"></a>외부 액세스 접두사
<a name="bkexternalprefix"> </a>

사용자가 외부 줄을 가져오기 위해 하나 이상의 추가 선행 숫자(예: 9)로 전화를 걸어야 하는 경우 최대 4자(#, *, 0-9)의 외부 액세스 접두사로 지정할 수 있습니다.

> [!NOTE]
> 외부 액세스 접두사로 지정하는 경우 접두사에 맞게 추가 정규화 규칙을 만들 필요가 없습니다.

[테넌트 다이얼 플랜을 만들려면 다이얼 플랜 만들기 및 관리를](create-and-manage-dial-plans.md) 참조하세요.

## <a name="normalization-rules"></a>정규화 규칙
<a name="bknormalizationrule"> </a>

정규화 규칙은 다양한 형식으로 표현된 전화 번호를 번역하는 방법을 정의합니다. 동일한 숫자 문자열은 전화를 거는 로캘에 따라 다르게 해석되고 번역될 수 있습니다. 사용자가 약식 내부 또는 외부 번호로 전화를 걸 수 있어야 하는 경우 정규화 규칙이 필요할 수 있습니다.

다이얼 플랜에 하나 이상의 정규화 규칙을 할당해야 합니다. 정규화 규칙은 위에서 아래로 일치하므로 테넌트 다이얼 플랜에 표시되는 순서가 중요합니다. 예를 들어 테넌트 다이얼 플랜에 10개의 정규화 규칙이 있는 경우 첫 번째 정규화 규칙부터 전화 접속 번호 일치 논리가 시도됩니다. 일치 항목이 없는 경우 두 번째 정규화 규칙 등입니다. 일치가 이루어지면 해당 규칙이 사용되며 정의된 다른 규칙과 일치하도록 노력하지 않습니다. 지정된 테넌트 다이얼 플랜에는 최대 50가지 정규화 규칙이 있을 수 있습니다.

### <a name="determining-the-required-normalization-rules"></a>필요한 정규화 규칙 결정

테넌트 다이얼 플랜은 지정된 사용자의 서비스 국가 다이얼 플랜과 효과적으로 병합되므로 필요한 테넌트 다이얼 플랜 정규화 규칙을 결정하기 위해 서비스 국가 다이얼 플랜의 정규화 규칙을 평가해야 할 수 있습니다. 이 용도로 **Get-CsEffectiveTenantDialPlan** cmdlet을 사용할 수 있습니다. cmdlet은 사용자의 ID를 입력 매개 변수로 사용하고 사용자에게 적용되는 모든 정규화 규칙을 반환합니다.

### <a name="creating-normalization-rules"></a>정규화 규칙 만들기
<a name="createrule"> </a>

정규화 규칙은 .NET Framework 정규식을 사용하여 서버에서 다이얼 문자열을 E.164 형식으로 변환하는 데 사용하는 숫자 일치 패턴을 지정합니다. 일치 항목의 정규식을 지정하고 일치 항목을 찾을 때 수행할 변환을 지정하여 정규화 규칙을 만들 수 있습니다. 완료되면 테스트 번호를 입력하여 정규화 규칙이 예상대로 작동하는지 확인할 수 있습니다.

.NET Framework 정규식을 사용하는 방법에 대한 자세한 내용은 [.NET Framework 정규식을 참조하세요](/dotnet/standard/base-types/regular-expressions).

테넌 [트 다이얼 플랜](create-and-manage-dial-plans.md) 에 대한 정규화 규칙을 만들고 관리하려면 다이얼 플랜 만들기 및 관리를 참조하세요.

> [!NOTE]
> 첫 번째 토큰을 선택 사항으로 사용하는 정규화 규칙은 현재 3pip 디바이스에서 지원되지 않습니다(예: Polycom VVX 601 모델). 3pip 디바이스에서 선택 사항으로 정규화 규칙을 적용하려면 정규화 규칙이 아닌 두 개의 정규화 규칙을 만들어야 합니다. 예를 들어 규칙 ^0? (999)$는 (999)$(번역:$1) 및 ^0(999)$(번역:$1)의 두 규칙으로 바꿔야 합니다.


### <a name="sample-normalization-rules"></a>샘플 정규화 규칙

다음 표에서는 .NET Framework 정규식으로 작성된 샘플 정규화 규칙을 보여 줍니다. 샘플은 예제일 뿐이며 사용자 고유의 정규화 규칙을 만들기 위한 규범적인 참조가 아닙니다.

<a name="regularexpression"> </a>
 **.NET Framework 정규식을 사용하는 정규화 규칙**

| 규칙 이름<br/> | 설명<br/> | 숫자 패턴<br/> | 번역<br/> | 예제<br/> |
|:-----|:-----|:-----|:-----|:-----|
|4digitExtension  <br/> |4자리 확장을 변환합니다.  <br/> |^(\\d{4})$  <br/> |+1425555$1  <br/> |0100은 +14255550100  <br/> |
|5digitExtension  <br/> |5자리 확장을 변환합니다.  <br/> |^5(\\d{4})$  <br/> |+1425555$1  <br/> |50100은 +14255550100  <br/> |
|7digitcallingRedmond  <br/> |7자리 숫자를 Redmond 로컬 숫자로 변환합니다.  <br/> |^(\\d{7})$  <br/> |+1425$1  <br/> |5550100은 +14255550100  <br/>|
|RedmondOperator  <br/> |0을 Redmond 연산자로 변환합니다.  <br/> |^0$  <br/> |+14255550100  <br/> |0은 +14255550100  <br/> |
|RedmondSitePrefix  <br/> |온-넷 접두사(6) 및 Redmond 사이트 코드(222)를 사용하여 숫자를 변환합니다.  <br/> |^6222(\\d{4})$  <br/> |+1425555$1  <br/> |62220100 +14255550100  <br/> |
|5digitRange  <br/> |3-7 사이의 숫자 범위부터 시작하여 5자리 확장을 변환합니다.  <br/> |^([3-7]\\d{4})$  <br/> |+142555$1 <br/> |54567은 +14255554567  <br/> |
|PrefixAdded  <br/> |첫 번째 숫자와 세 번째 숫자에 제한이 있는 9자리 숫자 앞에 국가 접두사 추가  <br/> |^([2-9]\\d\\[2-9]\\d{6})$  <br/> |1$1  <br/> |4255554567 14255554567 변환됩니다.  <br/> |
