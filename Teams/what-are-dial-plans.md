# <a name="what-are-dial-plans"></a>다이얼 플랜이 무엇인가요?

다이얼 플랜은 통화 권한 부여 및 통화 라우팅을 위해 개별 사용자가 전화 걸기 전화 번호를 대체 형식(일반적으로 E.164)으로 변환하는 명명된 정규화 규칙 집합입니다.

다이얼 플랜은 다양한 형식으로 표현된 전화 번호를 대체 형식으로 변환하는 방법을 정의하는 하나 이상의 정규화 규칙으로 구성됩니다. 동일한 다이얼 문자열이 다른 다이얼 플랜에서 다르게 해석 및 번역될 수 있으므로 지정된 사용자에게 할당된 다이얼 플랜에 따라 동일한 전화 걸기 번호가 변환 및 라우팅될 수 있습니다. 최대 1,000대의 테넌트 다이얼 플랜이 있을 수 있습니다.

[테넌트](create-and-manage-dial-plans.md) 다이얼 플랜을 만들고 관리하기 위해 다이얼 플랜 만들기 및 관리를 참조합니다.

## <a name="tenant-dial-plan-scope"></a>테넌트 다이얼 플랜 범위

다이얼 플랜의 범위에 따라 다이얼 플랜을 적용할 수 있는 계층 수준이 결정됩니다. 클라이언트는 사용자가 Teams에 로그인할 때 자동으로 제공되는 프로비전 설정을 통해 적절한 다이얼 플랜을 얻습니다. 관리자는 Microsoft Teams 관리 센터 또는 원격 PowerShell을 사용하여 다이얼 플랜 범위 수준을 관리하고 할당할 수 있습니다.

Teams에는 서비스 범위 및 테넌트 범위(조직용)의 두 가지 전화 플랜 유형이 있습니다. 전화 시스템을 사용할 수 있는 모든 국가 또는 지역에 대해 서비스 범위 다이얼 플랜이 정의됩니다. 각 사용자에게 할당된 사용 위치와 일치하는 서비스 국가 다이얼 플랜이 자동으로 할당됩니다. 서비스 국가 다이얼 플랜은 변경할 수 없지만 서비스 국가 다이얼 플랜을 보강하는 테넌트 범위 다이얼 플랜을 만들 수 있습니다. 클라이언트가 프로비전될 때 서비스 국가 다이얼 플랜과 적절한 범위의 테넌트 다이얼 플랜의 조합인 "유효 다이얼 플랜"을 얻습니다. 따라서 서비스 국가 다이얼 플랜에 이미 있을 수 있으므로 테넌트 다이얼 플랜에서 모든 정규화 규칙을 정의할 필요가 없습니다.

테넌트 다이얼 플랜은 테넌트 범위 또는 사용자 범위의 두 범위로 세분화할 수 있습니다. 테넌트가 사용자 범위 다이얼 플랜을 정의하고 할당하는 경우 해당 사용자는 사용자의 서비스 국가 다이얼 플랜 및 할당된 사용자 다이얼 플랜의 효과적인 다이얼 플랜으로 프로비전됩니다. 테넌트가 테넌트 범위 다이얼 플랜을 정의하지만 사용자 범위 다이얼 플랜을 할당하지 않는 경우 해당 사용자는 사용자의 서비스 국가 다이얼 플랜 및 테넌트 다이얼 플랜의 효과적인 다이얼 플랜으로 프로비전됩니다.

다음은 Teams에서 다이얼 플랜의 상속 모델입니다.

![Teams에서 다이얼 플랜을 상속하는 방법](media/b2744f33-ebbd-4c23-bfba-1747312ab178.png)

다음은 가능한 유효 다이얼 플랜입니다.

 **서비스 국가** 테넌트 범위 다이얼 플랜이 정의되지 않은 경우 테넌트 사용자 범위 다이얼 플랜이 프로비전된 사용자에게 할당되지 않은 경우 사용자는 사용 위치와 연결된 서비스 국가에 매핑된 효과적인 다이얼 플랜을 받게 됩니다.

 **테넌트 전역 - 서비스 국가** 테넌트 사용자 다이얼 플랜이 정의되어 있지만 사용자에게 할당되지 않은 경우 프로비전된 사용자는 병합된 테넌트 다이얼 플랜과 사용 위치와 연결된 서비스 국가 다이얼 플랜으로 구성된 효과적인 다이얼 플랜을 받게 됩니다.

 **테넌트 사용자 - 서비스 국가** 테넌트 사용자 다이얼 플랜이 정의되고 사용자에게 할당된 경우 프로비전된 사용자는 병합된 테넌트 사용자 다이얼 플랜 및 사용 위치와 연결된 서비스 국가 다이얼 플랜으로 구성된 효과적인 다이얼 플랜을 받게 됩니다.

테넌트 다이얼 [플랜을 만들 수](create-and-manage-dial-plans.md) 있도록 다이얼 플랜 만들기 및 관리를 참조합니다.

> [!NOTE]
> 전화 걸기 번호에 다이얼 플랜 정규화 규칙이 적용되지 않습니다. CC가 전화 걸기 사용자의 사용 위치 국가 코드인 "+CC"를 추가하도록 전화 걸기 문자열은 여전히 정규화됩니다. 이는 통화 요금제, 직접 라우팅 및 PSTN 전화 회의 전화 걸기 시나리오에 적용됩니다.

## <a name="planning-for-tenant-dial-plans"></a>테넌트 다이얼 플랜 계획

사용자 지정 다이얼 플랜을 계획하기 위해 다음 단계를 수행합니다.

- **1단계** 사용자 전화 걸기 환경을 개선하기 위해 사용자 지정 다이얼 플랜이 필요한지 여부를 결정하세요. 일반적으로 내선 번호 또는 약식 국가 전화 걸기 같은 비 E.164 다이얼링을 지원해야 합니다.

- **2단계** 테넌트 전역 또는 테넌트 사용자 범위 다이얼 플랜이 필요한지 또는 둘 다 필요한지 여부를 판단합니다. 사용자가 다른 로컬 전화 걸기 요구 사항이 있는 경우 사용자 범위 다이얼 플랜이 필요합니다.

- **3단계** 필요한 각 다이얼 플랜에 대해 유효한 번호 패턴을 식별합니다. 서비스 수준 국가 다이얼 플랜에 정의되지 않은 숫자 패턴만 필요합니다.

- **4단계** 전화 요금제의 이름을 정하기 위한 조직 전체 체계를 개발합니다. 표준 이름 체계를 채택하면 조직 전체의 일관성을 보장하고 유지 관리 및 업데이트를 더 쉽게 만들 수 있습니다.


## <a name="creating-your-new-tenant-dial-plan"></a>새 테넌트 다이얼 플랜 만들기

새 다이얼 플랜을 만들 때 필요한 정보를 넣어야 합니다.

### <a name="name-and-simple-name"></a>이름 및 단순 이름

사용자 다이얼 플랜의 경우 다이얼 플랜을 할당할 사용자를 식별하는 설명이 있는 이름을 지정해야 합니다. 다이얼 플랜 단순 이름은 다이얼 플랜 이름에서 파생된 문자열로 미리 채워진 것입니다. 간단한 이름 필드를 편집할 수 있습니다. 따라서 다이얼 플랜에 대해 보다 설명적인 명명 규칙을 만들 수 있습니다. 단순 이름 값은 비어 있을 수 없습니다. 고유해야 합니다. 모범 사례는 전체 조직에 대한 이름 규칙을 개발한 다음 모든 사이트와 사용자에 걸쳐 일관되게 이 규칙을 사용하는 것입니다.

### <a name="description"></a>설명

해당 다이얼 플랜이 적용되는 지리적 위치 또는 사용자 그룹의 일반적인 인식 가능한 이름을 입력하는 것이 좋습니다.

### <a name="external-access-prefix"></a>외부 액세스 prefix
<a name="bkexternalprefix"> </a>

사용자가 외부 줄을 얻기 위해 한 자리 이상의 앞 자릿수(예: 9)에 전화를 걸 필요가 있는 경우 최대 4자(#, *, 0-9)의 외부 액세스 액세스를 지정할 수 있습니다.

> [!NOTE]
> 외부 액세스 사전을 지정하는 경우, 추가 정규화 규칙을 만들어서 해당 조건에 맞춰 지정할 필요가 없습니다.

테넌트 다이얼 [플랜을 만들 수](create-and-manage-dial-plans.md) 있도록 다이얼 플랜 만들기 및 관리를 참조합니다.

## <a name="normalization-rules"></a>정규화 규칙
<a name="bknormalizationrule"> </a>

정규화 규칙은 다양한 형식으로 표현된 전화 번호를 번역하는 방법을 정의합니다. 동일한 숫자 문자열이 전화 걸기 로지스에 따라 다르게 해석 및 번역될 수 있습니다. 사용자가 약어 내부 또는 외부 번호로 전화를 걸 수 있도록 해야 하는 경우 정규화 규칙이 필요할 수 있습니다.

다이얼 플랜에 하나 이상의 정규화 규칙을 할당해야 합니다. 정규화 규칙은 위쪽에서 아래쪽으로 일치하기 때문에 테넌트 다이얼 플랜에 나타나는 순서가 중요합니다. 예를 들어 테넌트 다이얼 플랜에 10개 정규화 규칙이 있는 경우 첫 번째 정규화 규칙부터 전화 걸기 번호 일치 논리가 시도됩니다. 일치하는 규칙이 없는 경우 두 번째 규칙이 있는 경우 두 번째 규칙부터 시도합니다. 일치하는 규칙이 만들어지면 해당 규칙이 사용되어 정의된 다른 규칙과 일치하지 않습니다. 주어진 테넌트 다이얼 플랜에는 최대 50개 정규화 규칙이 있을 수 있습니다.

### <a name="determining-the-required-normalization-rules"></a>필요한 정규화 규칙 결정

테넌트 다이얼 플랜은 특정 사용자의 서비스 국가 다이얼 플랜과 효과적으로 병합될 수 있기 때문에 필요한 테넌트 다이얼 플랜 정규화 규칙을 결정하기 위해 서비스 국가 다이얼 플랜의 정규화 규칙을 평가해야 할 수 있습니다. **Get-CsEffectiveTenantDialPlan** cmdlet을 이 용도로 사용할 수 있습니다. cmdlet은 사용자 ID를 입력 매개 변수로 사용하며 사용자에게 적용되는 모든 정규화 규칙을 반환합니다.

### <a name="creating-normalization-rules"></a>정규화 규칙 만들기
<a name="createrule"> </a>

정규화 규칙은 .NET Framework 정규식을 사용하여 서버에서 다이얼 문자열을 E.164 형식으로 변환하는 데 사용하는 숫자 일치 패턴을 지정합니다. 정규화 규칙은 일치에 대한 정규식을 지정하고 일치가 발견될 때 수행될 번역을 지정하여 만들 수 있습니다. 완료되면 테스트 번호를 입력하여 정규화 규칙이 예상대로 작동하는지 확인할 수 있습니다.

.NET Framework 정규식을 사용하는 자세한 내용은 .NET Framework 정규식을 [참조합니다.](https://go.microsoft.com/fwlink/p/?linkId=140927)

[테넌트](create-and-manage-dial-plans.md) 다이얼 플랜에 대한 정규화 규칙을 만들고 관리하기 위해 다이얼 플랜 만들기 및 관리를 참조합니다.

> [!NOTE]
> 첫 번째 토큰을 선택 사항으로 사용하여 정규화 규칙은 현재 3pip 디바이스에서 지원되지 않습니다(예: Polycom VVX 601 모델). 3pip 디바이스에서 선택 사항으로 정규화 규칙을 적용하려는 경우 하나 대신 두 개의 정규화 규칙을 만들어야 합니다. 예를 들어 규칙 ^0? (999)$는 (999)$(Translation:$1) 및 ^0(999)$(Translation:$1)의 두 규칙으로 대체해야 합니다.

|5digitRange  <br/> |3-7 자릿수 범위로 시작하는 5자리 확장을 변환합니다.  <br/> |^([3-7] \\ d {4} )$  <br/> |+142555$1 <br/> |54567이 +14255554567로 변환됩니다.  <br/> |
|PrefixAdded  <br/> |첫 번째 및 세 번째 숫자에 제한이 있는 9자리 숫자 앞에 국가 앞에 국가를 추가합니다.  <br/> |^([2-9] \\ \\ d[2-9] \\ d {6} )$  <br/> |1$1  <br/> |4255554567은 14255554567로 변환됩니다.  <br/> |
|NoTranslation  <br/> |5자리를 일치하지만 번역은 일치하지 않습니다.  <br/> |^( \\ d {5} )$  <br/> |$1  <br/> |34567은 34567로 변환됩니다.  <br/> |

 **위에 표시된 정규화 규칙을 기반으로 하는 Redmond 다이얼 플랜입니다.**

 다음 표에서는 이전 표에 표시된 정규화 규칙에 따라 미국, 워싱턴, 레드몬드에 대한 샘플 다이얼 플랜을 보여 줄 수 있습니다.
