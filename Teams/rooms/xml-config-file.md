# <a name="manage-a-microsoft-teams-rooms-console-settings-remotely-with-an-xml-configuration-file"></a>XML 구성 파일을 사용하여 원격으로 Microsoft Teams 회의실 콘솔 설정 관리

이 문서에서는 사용자 지정 테마 적용을 포함하여 Microsoft Teams 회의실 장치에서 사용하는 기본 설정의 원격 관리에 대해 논의합니다. 마스터 설정 파일을 만드는 방법과 원격 관리 디바이스에 필요한 경우 이를 저장하는 방법에 대한 토론 링크를 제공합니다.
  
마스터 XML 파일을 업데이트하고 관리되는 본체에 복사본을 보내 원격 관리 장치의 기본 설정을 변경할 수 있습니다. XML 구성 파일을 사용하여 Microsoft Teams 회의실 콘솔에서 사용자 지정 테마를 구현할 수도 있습니다.
  
## <a name="create-an-xml-configuration-file"></a>XML 구성 파일 만들기

모든 텍스트 편집기를 사용하여 설정 파일을 만들 수 있습니다. **XML Elements** 테이블은 (필수 파일 이름) 구성 SkypeSettings.xml 샘플에 표시된 요소에 대해 설명하고 있습니다.
  
```XML
<SkypeSettings>
    <AutoScreenShare>true</AutoScreenShare>
    <HideMeetingName>true</HideMeetingName>
    <UserAccount>
        <SkypeSignInAddress>RanierConf@contoso.com</SkypeSignInAddress>
        <ExchangeAddress>RanierConf@contoso.com</ExchangeAddress>
        <ModernAuthEnabled>false</ModernAuthEnabled>
        <DomainUsername>Seattle\RanierConf</DomainUsername>
        <Password>password</Password>
        <ConfigureDomain>domain1, domain2</ConfigureDomain>
    </UserAccount>
    <IsTeamsDefaultClient>false</IsTeamsDefaultClient>
    <BluetoothAdvertisementEnabled>true</BluetoothAdvertisementEnabled>
    <AutoAcceptProximateMeetingInvitations>false</AutoAcceptProximateMeetingInvitations>
    <SkypeMeetingsEnabled>false</SkypeMeetingsEnabled>
    <TeamsMeetingsEnabled>true</TeamsMeetingsEnabled>
    <WebExMeetingsEnabled>true</WebExMeetingsEnabled>
    <UseCustomInfoForThirdPartyMeetings>true</UseCustomInfoForThirdPartyMeetings>
    <CustomDisplayNameForThirdPartyMeetings>Rainier Conference Room</CustomDisplayNameForThirdPartyMeetings>
    <CustomDisplayEmailForThirdPartyMeetings>RanierConf@contoso.com</CustomDisplayEmailForThirdPartyMeetings> 
    <DualScreenMode>true</DualScreenMode>
    <DuplicateIngestDefault>false</DuplicateIngestDefault>
    <DisableTeamsAudioSharing>false</DisableTeamsAudioSharing>
    <CortanaEnabled>false</CortanaEnabled>
    <CortanaWakewordEnabled>false</CortanaWakewordEnabled>
    <SendLogs>
        <EmailAddressForLogsAndFeedback>RanierConf@contoso.com</EmailAddressForLogsAndFeedback>
        <SendLogsAndFeedback>true</SendLogsAndFeedback>
    </SendLogs>
    <Devices>
        <MicrophoneForCommunication>Microsoft LifeChat LX-6000</MicrophoneForCommunication>
        <SpeakerForCommunication>Realtek High Definition Audio</SpeakerForCommunication>
        <DefaultSpeaker>Polycom CX5100</DefaultSpeaker>
        <ContentCameraId>USB\VID_046D&amp;PID_0843&amp;MI_00\7&amp;17446CF2&amp;0&amp;0000</ContentCameraId>
        <ContentCameraInverted>false</ContentCameraInverted>
        <ContentCameraEnhancement>true</ContentCameraEnhancement>
    </Devices>
    <Theming>
        <ThemeName>Custom</ThemeName>
        <CustomThemeImageUrl>file name</CustomThemeImageUrl>
        <CustomThemeColor>
            <RedComponent>100</RedComponent>
            <GreenComponent>100</GreenComponent>
            <BlueComponent>100</BlueComponent>
        </CustomThemeColor>
    </Theming>
    <CoordinatedMeetings enabled="true"> 
        <TrustedAccounts>room@contoso.com</TrustedAccounts>
        <Settings> 
            <Audio default="false" enabled="false" />
            <Video default="false" enabled="true" /> 
            <Whiteboard default="false" enabled="false"/>
        </Settings> 
    </CoordinatedMeetings>
</SkypeSettings>
```

변수 값이 잘못된 형식이거나, 요소가 순서를 틀리거나, 요소가 선택되지 않은 경우 또는 다른 오류가 발견되면 XML 파일이 잘못 *형식이 됩니다.* 잘못된 형식의 XML 파일을 처리하는 동안 오류가 발생하는 지점까지 찾은 설정이 적용된 다음 파일의 나머지는 무시됩니다. XML의 알 수 없는 요소는 무시됩니다. 매개 변수를 생략하면 디바이스에서 변경되지 않은 상태로 남아 있습니다. 매개 변수 값이 유효하지 않은 경우 이전 값은 변경되지 않습니다.

## <a name="manage-console-settings-with-an-xml-configuration-file"></a>XML 구성 파일을 사용하여 콘솔 설정 관리

시작 시 Microsoft Teams 회의실 콘솔이 SkypeSettings.xml XML 파일을 찾으면 XML 파일로 표시된 구성 설정을 적용한 다음 XML 파일을 `C:\Users\Skype\AppData\Local\Packages\Microsoft.SkypeRoomSystem_8wekyb3d8bbwe\LocalState` 삭제합니다.
  
기업에서 사용하는 Microsoft Teams Rooms 장치 수 및 구성하기 위해 관리하기로 선택하는 방법에 따라 XML 구성 파일을 저장하는 여러 가지 방법이 있습니다. 파일이 콘솔에 푸시된 후 다시 시작하여 구성 변경 내용을 처리합니다. XML 구성 파일은 성공적으로 처리된 후 삭제됩니다. Microsoft Teams 회의실 장치에 대해 제안된 관리 방법은 다음에서 설명하고 있습니다.
  
- [Microsoft Teams 회의실에 대한 그룹 정책 구성](rooms-operations.md#GroupPolicy)
- [PowerShell을 사용하여 원격 관리](rooms-operations.md#RemotePS) 및 [파일 항목 구성](https://technet.microsoft.com/library/cc772536%28v=ws.11%29.aspx)

파일을 전송하고 콘솔 장치에서 다시 시작을 트리거하는 데 사용할 수 있는 한 원하는 방법을 자유롭게 사용할 수 있습니다. 디바이스의 로컬 사용자 계정에서 파일을 읽을 수 있어야 합니다. 가급적이면 소유하고 해당 사용자에게 전체 권한을 부여합니다. 파일 사용 권한이 올바르게 설정되지 않으면 소프트웨어가 설정을 적용하지 못하고, 성공적으로 처리될 때 파일을 삭제하지 못할 수 있으며, 잠재적으로 충돌할 수 있습니다.
  
## <a name="custom-theme-images"></a>사용자 지정 테마 이미지

<a name="Themes"> </a>

사용자 지정 테마 이미지 파일을 폴더에 배치해야 `C:\Users\Skype\AppData\Local\Packages\Microsoft.SkypeRoomSystem_8wekyb3d8bbwe\LocalState` 합니다. 변수에 파일 이름 및 확장명을 \<CustomThemeImageUrl\> 입력합니다.
  
이미지 파일은 정확히 3840X1080 픽셀로 jpg, jpeg, png 및 bmp 파일 형식 중 하나에 있어야 합니다. 조직에서 사용자 지정 이미지를 원하는 경우 그래픽 디자이너에서 사용자 지정 테마 [사진 찍기 서식](../downloads/ThemingTemplateMicrosoftTeamsRooms_v2.1.psd)파일을 사용할 수 있습니다. 다양한 사용자 인터페이스 요소가 테마 이미지의 나머지를 상대로 하는 위치와 본체 및 디스플레이에 나타나는 영역에 대한 자세한 정보가 포함되어 있습니다.
  
테마 이미지를 인식하려면 디바이스 시작 시 XML 구성 파일을 업데이트해야 합니다. 새 XML 파일이 처리되고 삭제되면 테마 그래픽 파일이 디렉터리에서 삭제됩니다.
  
## <a name="locate-the-content-camera-usb-instance-path"></a>콘텐츠 카메라 USB 인스턴스 경로 찾기

인스턴스 경로를 찾으면 다음을 실행합니다.

1. Microsoft Teams 회의실 콘솔의 Windows 설정으로 이동합니다.
2. 관리자 암호를 입력합니다.
3. 명령 프롬프트에서 디바이스 `devmgmt.msc` 관리자를 표시하려면 입력합니다.
4. Device **Manager에서** **이미징** 디바이스 노드를 살펴보고 콘텐츠 카메라를 찾습니다.
5. 카메라를 마우스 오른쪽 단추로 클릭하고 속성을 **열 수 있습니다.**
6. 세부 **정보 탭을** 선택하고 드롭다운에서 디바이스 인스턴스 **경로** 속성을 찾습니다.
7. 표시된 값은 XML 구성 파일에 설정할 디바이스 인스턴스 경로입니다. XML에서 경로를 지정할 때 앰퍼탠드(&)를 으로 바랜 다음 `&amp;`
