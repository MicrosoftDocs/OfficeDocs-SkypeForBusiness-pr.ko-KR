# <a name="get-started-with-teams-for-healthcare-organizations"></a>건강 조직의 팀 시작 하기

Microsoft 팀은 다음을 포함 하 여 병원 및 기타 의료 기관에 유용한 여러 기능을 제공 합니다.

- 가상 방문 및 전자 의료 기록 (EHR) 통합
- 팀 정책 패키지
- 보안 메시지
- 팀 서식 파일
- 세심 한 조정 및 공동 작업

이 기능은 의료에 대 한 Microsoft 클라우드의 일부입니다. 이 솔루션을 사용 하 여 Azure, Dynamics 365, [의료의 Microsoft 클라우드에서](https://docs.microsoft.com/industry/healthcare)microsoft 365의 기능을 함께 제공 하는 방법에 대해 자세히 알아보세요.

Microsoft 팀의 상태 팀 공동 작업을 향상 시키기 위해 의료 컬렉션을 사용 하는 방법에 대해 자세히 알아보려면 다음 비디오를 시청 하세요.

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4Hqan]

> [!NOTE]
> 이 섹션의 콘텐츠는 조직에 이미 팀을 배포한 것으로 가정 합니다. 아직 팀을 롤아웃하기 않은 경우 먼저 [Microsoft 팀을 배포 하는 방법을](../../How-to-roll-out-teams.md)읽어 보세요.

## <a name="virtual-visits-and-electronic-healthcare-record-ehr-integration"></a>가상 방문 및 전자 의료 기록 (EHR) 통합

Microsoft 팀의 complete meeting platform을 사용 하 여 환자으로 가상 방문을 예약 하 고, 관리 하 고, 수행할 수 있습니다.

- 조직에서 이미 전자 메일 상태 레코드 또는 EHR를 사용 하는 경우에는 Microsoft 팀을 통합 하 여 보다 원활한 환경을 만들 수 있습니다. Microsoft 팀 EHR (전자적 상태 레코드) 커넥터를 사용 하면 clinicians에서 가상 환자를 쉽게 시작 하거나 EHR 시스템에서 직접 팀의 다른 공급자와 상담을 할 수 있습니다. 자세한 내용은 [팀과의 가상 방문-EHR에 통합](ehr-admin.md)을 참조 하세요.
- 지원 되는 EHR을 사용 하지 않는 경우 팀에서 Microsoft 예약 및 예약 앱을 사용할 수 있습니다. 자세한 내용은 [Microsoft 팀의 예약 앱 및 가상 방문](../../bookings-app-admin.md)을 참조 하세요.

![Microsoft 팀의 가상 방문](../../media/virtual-visits-teams.png)

## <a name="teams-policy-packages"></a>팀 정책 패키지

팀에서 다른 역할이 수행할 수 있는 작업을 정의 하는 팀 정책 패키지를 적용 합니다. 예를 들어 다음에 대 한 정책을 지정 합니다.

- 등록 된 nurses, 청구 nurses, 의사, 사교 작업자 등의 임상 근로자가 채팅, 통화, 교대 관리, 모임에 대 한 모든 액세스 권한을 가질 수 있도록 합니다.
- 의료 조직의 정보 근로자 (예: IT 직원, informatics 스태프, 재무 담당자, 규정 준수 책임자)는 채팅, 통화, 모임에 대 한 모든 권한을 가질 수 있습니다.
- 환자 채팅방 장치에 대 한 설정을 제어 합니다.

자세한 내용은 [팀 정책 패키지에 대 한 의료](../../policy-packages-healthcare.md)관련 정보를 참조 하세요.

## <a name="secure-messaging"></a>보안 메시지

보안 메시징은 다음의 몇 가지 새로운 기능을 포함 하 여 상태 팀 내에서 공동 작업을 지원 합니다.

- 메시지 보낸 사람이 메시지에 대 한 특별 한 우선 순위를 설정할 수 있으므로 메시지를 읽을 때까지 받는 사람이 반복적으로 알림을 받습니다.
- 메시지 보낸 사람은 읽음 확인을 요청할 수 있으므로 메시지를 받는 사람이 보낸 메시지를 읽었을 때 알림을 받습니다.

이러한 기능을 함께 사용 하면 긴급 메시지를 더욱 쉽게 확인 하 고 메시지를 받아서 읽음을 확신할 수 있습니다. 이러한 기능을 사용 하는 새로운 상태 팀은 환자 단위로 만들 수 있습니다. 이러한 기능은 정책 기반 이며, 개인 또는 전체 팀에 게 할당 될 수 있습니다.

자세한 내용은 [의료 기관에 대 한 보안 메시징 정책 시작](messaging-policies-hc.md)을 참조 하세요.

보안 메시징과 관련 하 여 의료 기관에서 다른 테 넌 트를 페더레이션 하 여 더 다양 한 테 넌 트 간 통신을 허용 하는 기능도 있습니다. ( [Microsoft 팀에서 외부 액세스 (페더레이션) 관리](../../manage-external-access.md)를 참조 하세요.)

## <a name="teams-templates-for-healthcare-organizations"></a>건강 보험 조직을 위한 팀 템플릿

팀을 만들기 위한 새 템플릿이 병원 설정에 적용 되도록 개발 되었고 더 많은 내용이 곧 제공 될 예정입니다. 이를 통해 의료 근로자가 다양 한 부서나 wards의 환자을 조정 하는 데 사용 하는 팀을 쉽게 만들 수 있습니다. 자세한 내용은 [의료 조직의 팀 서식 파일 시작](healthcare-templates.md)을 참조 하세요. 팀은 cardiology 등의 내부 부서를 위해 시작 하거나 주의를 wards 하 고 더 많은 서식 파일을 개발할 수 있습니다.

## <a name="care-coordination-and-collaboration"></a>세심 한 조정 및 공동 작업

건강 팀을 함께 사용 하 여 주의를 기울여 Microsoft 팀과 공동으로 작업할 수 있습니다.

![건강 보험: 팀의 건강 팀 공동 작업](../../media/teams-healthcare-collaborate-in-teams.png)

Microsoft 팀은 의사, clinicians, nurses, 기타 직원이 Microsoft 팀의 포함 된 공동 작업 기능으로 효율적으로 공동 작업할 수 있도록 합니다.

- 상태 팀 및 정보 근로자를 위해 팀과 채널을 설정 합니다. 탭에서 채널을 사용 하 여 작업을 구조화 하는 방법으로, 정보 원본을 고정 하는 탭의 추가 도움말을 사용할 수 있습니다.
- 채팅, 메시지 게시 및 통신 팀은 주의가 필요한 다양 한 환자에 대해 지속적인 대화를 나눌 수 있습니다.
- 상태 팀의 구성원과 통화 하 고 모임에 참가 합니다. 개별 모임을 설정 하거나 채널 모임을 사용 하 여 팀의 오디오, 비디오, 화면 공유, 녹음/녹화 기능을 통해 매일 모임을 관리할 수 있습니다.
- 파일 및 문서를 저장 하 고 공유 합니다. 건강 팀은 Office 문서에서 작동 하 고 공동 작업 하는 단일 가상화 된 팀의 일부입니다.

또한 팀에서 다음과 같은 작업을 할 수 있습니다.

- 목록을 공유 하 고 목록 앱을 사용 하 여 정보 추적
- Tasks 앱을 사용 하 여 작업 추적 및 모니터링
- 승인 앱으로 승인 간소화
- 교대 근무 앱을 사용 하 여 일정 만들기, 관리 및 공유

### <a name="share-lists-and-track-information-with-the-lists-app"></a>목록을 공유 하 고 목록 앱을 사용 하 여 정보 추적

> [!NOTE]
> 2020 년 10 월 30 일에 효력을 환자 앱이 회수 되 고 팀의 [목록 앱](https://support.microsoft.com/office/get-started-with-lists-in-teams-c971e46b-b36c-491b-9c35-efeddd0297db) 으로 대체 됩니다. 목록을 사용 하는 경우 의료 기관에서 팀은 팀 모임에서 일반 환자 모니터링과의 interdisciplinary에 이르기까지 시나리오에 대 한 환자 목록을 만들 수 있습니다.

팀의 목록 앱은 팀에서 정보를 추적 하 고 작업을 구성 하는 데 도움이 됩니다. 앱은 모든 팀 사용자를 위해 사전 설치 되어 있으며 모든 팀과 채널에서 탭으로 사용할 수 있습니다. 목록은 미리 정의 된 서식 파일에서 만들거나 Excel로 데이터를 가져와 처음부터 만들 수 있습니다.

상태 팀은 환자 서식 파일을 사용 하 여 시작할 수 있습니다. 환자의 요구 및 상태를 추적 하는 목록을 만들 수 있습니다. 팀에서 Excel 스프레드시트의 기존 환자 데이터를 가져와 목록을 만들 수 있습니다. 이러한 목록은 라운드 및 환자 모니터링과 같은 시나리오에 사용 하 여 주의를 조정할 수 있습니다.

예를 들어, 충전 nurse 모든 건강 팀 구성원이 포함 된 팀에 환자 목록을 만듭니다. 반올림 하는 동안 상태 팀은 모바일 장치에서 팀에 액세스 하 고 목록에서 환자 정보를 업데이트 하 여 팀의 모든 구성원이 동기화 상태를 유지할 수 있습니다. 상태 팀에서 주요 상태 성능 메트릭에 대해 설명 하 고 평가 하는 데 사용 되는 반올림 세션에서는 대규모 디스플레이 화면에서 팀을 사용 하 여이 정보를 공유할 수 있습니다. 사이트를 구성 하지 않은 상태 팀 구성원은 원격으로 참가할 수 있습니다.

### <a name="track-and-monitor-tasks-with-the-tasks-app"></a>Tasks 앱을 사용 하 여 작업 추적 및 모니터링

팀에서 [작업](https://support.microsoft.com/office/use-the-tasks-app-in-teams-e32639f3-2e07-4b62-9a8c-fd706c12c070) 을 사용 하 여 전체 상태 팀에 대 한 할 일 항목을 추적 합니다. 상태 팀은 팀을 실행 중인 모든 장치에서 언제 든 지 작업을 만들고 할당 하 고 예약 하 고, 작업을 분류 하 고, 상태를 업데이트할 수 있습니다.

자세한 내용은 [Microsoft 팀에서 조직의 작업 앱 관리](../../manage-tasks-app.md) 를 참조 하세요.

### <a name="streamline-approvals-with-the-approvals-app"></a>승인 앱으로 승인 간소화

[승인을](https://support.microsoft.com/office/what-is-approvals-a9a01c95-e0bf-4d20-9ada-f7be3fc283d3) 사용 하 여 팀과 함께 모든 요청과 프로세스를 합리화 하세요. 팀워크의 허브에서 직접 승인을 만들고, 관리 하 고, 공유 합니다. 같은 장소에서 승인 흐름 시작 채팅, 채널 대화 또는 승인 앱 자체를 보낼 수 있습니다. 승인 유형을 선택 하 고 세부 정보를 추가 하 고 파일을 첨부 하 고 승인자를 선택 하기만 하면 됩니다. 제출 된 승인자는 알림을 받고 요청을 검토 하 고 작동할 수 있습니다.

조직의 승인 앱을 허용 하 고 팀에 추가할 수 있습니다. 앱을 관리 하는 방법에 대해 자세히 알아보려면 [Microsoft 팀 관리 센터에서 앱 관리](../../manage-apps.md)를 참조 하세요.

### <a name="create-manage-and-share-schedules-with-the-shifts-app-and-firstline-worker-integration"></a>교대 앱 및 Firstline Worker 통합으로 일정 만들기, 관리 및 공유

Microsoft 팀은 교대 근무 시간을 교대 하는 앱 및 Firstline Worker와 통합 되며,이를 사용 하 여 이동 인력 배치 기능을 조정할 수 있습니다. 예를 들어 교대 근무 Nurse 관리자는 직원을 위해 일정을 설정 하 고 조정할 수 있으며, nurses에서는 일정을 확인 하 고 교대 근무를 전환할 수 있습니다. 팀에는 조직의 Firstline Worker에 할당할 수 있는 기본 제공 Firstline Worker 앱 설정 정책이 포함 됩니다. 기본적으로 정책에는 활동, 교대, 채팅 및 통화 앱이 포함 됩니다. 이 정책은 해당 앱에 대 한 동작을 제어 하며, 예를 들어 팀에서 빠르게 액세스할 수 있도록 앱 바에 이동 앱을 고정 합니다.

자세한 내용은 [Microsoft 팀에서 조직의 교대 근무 앱 관리](../shifts/manage-the-shifts-app-for-your-organization-in-teams.md)를 참조 하세요.
