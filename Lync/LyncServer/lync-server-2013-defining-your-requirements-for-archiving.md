# <a name="defining-your-requirements-for-archiving-in-lync-server-2013"></a>Lync Server 2013에서 보관에 대한 요구 사항 정의

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**마지막으로 수정한 주제:** 2012-10-09_

조직이 준수 규정을 따라야 하는 경우 보관을 배포 하 여 Lync Server 2013 인스턴트 메시징 (IM) 및 회의 (모임)에 대 한 보관 지원을 사용할 수 있습니다. 보관할 수 있는 콘텐츠 형식에 대 한 자세한 내용은 계획 설명서의 [Lync Server 2013에서 보관 개요](lync-server-2013-overview-of-archiving.md) 를 참조 하세요.

보관을 구현 하려면 먼저 조직의 보관 요구 사항에 맞는 방법을 결정 해야 합니다. 이를 위해서는 다음을 확인 해야 합니다.

  - **보관을 배포 하는 경우** 초기 Lync Server 2013 배포의 일부로 보관을 배포 하거나 기존 배포에 추가할 수 있습니다. 토폴로지 작성기를 사용 하 여 보관을 배포 하 여 토폴로지에 추가한 다음 토폴로지를 게시 합니다.

  - **내부 또는 외부 통신을 보관할지 여부**. 내부 통신 (내부 사용자 간 통신), 외부 통신 (내부 네트워크 외부에 있는 사용자를 하나 이상 포함 하는 통신) 또는 두 가지 모두에 대해 보관을 사용 하도록 설정할 수 있습니다. 조직 전체에 대해 이러한 옵션을 지정 하거나 특정 사이트 및 풀에 대해 지정할 수 있습니다. 기본적으로 두 옵션이 모두 사용 하도록 설정 되어 있지 않습니다.
    
    <div>
    

    > [!NOTE]  
    > Microsoft Exchange 통합을 사용 하 여 보관 된 데이터를 저장 하는 경우 Exchange 설정이 Lync 통신을 보관할지 여부를 제어 합니다. 배포에 여러 포리스트가 포함 된 경우 Lync Server와 Exchange 간에 설정을 동기화 해야 합니다. 내부 또는 외부 통신을 위한 보관을 제어 하는 것은 Lync 정책에만 사용할 수 있습니다. Exchange 통합 보관의 경우 두 파일 모두 보관 되거나 보관 되지 않습니다.

    
    </div>

  - **보관을 사용 하는 이유** 전역 수준에서 전체 배포에 대 한 보관을 사용 하거나 사용 하지 않도록 설정할 수 있으며, 특정 사이트 및 사용자의 보관을 사용 하거나 사용 하지 않도록 설정할 수 있습니다. 이러한 각 수준에서 IM 세션 보관 (피어 투 피어), 회의 (모임, 단체 세션) 또는 둘 다를 사용할지 여부를 지정 합니다. 기본적으로 보관을 사용할 수 없습니다.

  - **조직의 사용자에 게 얼마나 중요 한 보관이 되는지 설명**합니다. 조직에서 업무에 중요 한 보관 인 경우에는 Lync Server 2013을 중요 모드로 실행 하도록 지정할 수 있으며, 보관이 실패 하는 경우 IM 및 회의 세션을 차단 합니다. 예를 들면 다음과 같습니다.
    
      - 보관 서비스가 일시적으로 데이터베이스 큐로 메시지를 보내거나 데이터베이스에 메시지를 삽입할 수 없는 경우에는 보관 지원이 복원 될 때까지 배포에서 IM 및 회의 기능이 모두 차단 됩니다.
    
      - 회의 사용자가 파일을 업로드 했지만 보관 파일 저장소에 복사할 수 없는 경우에는 문제가 해결 될 때까지 배포에서 회의 기능이 차단 되지만 메신저 기능은 차단 되지 않습니다.
    
    전역 수준, 사이트 수준 및 풀 수준에서이 옵션을 구성할 수 있습니다. 기본적으로 중요 모드는 사용 하도록 설정 되어 있지 않습니다.

  - **Microsoft Exchange 통합 사용 여부** 이 옵션은 보관 저장소를 Exchange 2013 저장소와 통합 하 여 Lync Server 보관 데이터 및 Exchange 2013 아카이브된 데이터를 Exchange에 함께 저장 합니다. 사서함이 원본 위치 유지에 저장 되어 있는 경우 Exchange 2013에 있는 사용자의 보관 데이터 저장소에 Microsoft Exchange 통합을 사용할 수 있습니다. Exchange 2013 배포를 보유 하 고 있지 않거나이를 통합 하지 않으려는 경우 또는 Exchange 2013에 있지 않은 Lync 사용자가 있는 경우 SQL Server를 사용 하 여 Lync 통신에서 보관 된 데이터를 저장 하 여 별도의 보관 데이터베이스를 배포할 수 있습니다. 전역 수준, 사이트 수준 및 풀 수준에서 Microsoft Exchange 통합 옵션을 구성할 수 있습니다. 기본적으로 Microsoft Exchange 통합은 사용 하도록 설정 되어 있지 않습니다.

  - **보관 된 데이터를 관리 하는 방법**입니다. 보관 데이터베이스는 장기 보존을 위한 것이 아니며 Lync Server 2013에서 보관 된 데이터에 대 한 e-검색 (검색) 솔루션을 제공 하지 않으므로 데이터를 다른 저장소로 옮겨야 합니다. Lync Server는 보관 된 데이터를 내보내는 데 사용할 수 있는 세션 내보내기 도구를 제공 하 고 보관 된 데이터의 검색 가능한 성적 증명서를 만듭니다. 전역 정책과 만드는 각 사이트 및 사용자 정책에 대해 데이터 제거를 사용 하도록 설정 하 고 다음 옵션 중 하나를 지정할 수 있습니다.
    
      - 내보낸 보관 데이터를 모두 제거 하 고 특정 일 수 후에 보관 데이터를 저장 합니다. 지정할 수 있는 최소 날짜 수는 1 일입니다. 지정할 수 있는 최대 일 수는 2562 일입니다.
    
      - 내보낸 보관 데이터만 제거 합니다. 이 옵션은 세션 내보내기 도구에서 내보내고 안전한 것으로 표시 된 모든 레코드를 제거 합니다.
    
    전역 수준, 사이트 수준 및 풀 수준에서이 옵션을 구성할 수 있습니다. 기본적으로 제거는 사용할 수 없습니다.

다음 메서드를 사용 하 여 보관을 제어 합니다.

  - **보관 정책**. 하나 이상의 보관 정책을 사용 하 여 내부 및 외부 통신의 보관을 사용 하거나 사용 하지 않도록 설정 합니다. 기본적으로 보관을 사용 하도록 설정 되어 있지 않습니다. 기본 전역 정책을 사용 하 여 배포에서 내부 통신, 외부 통신 또는 둘 다에 대해 보관을 사용 하거나 사용 하지 않도록 설정 합니다. 전역 정책은 삭제할 수 없습니다. 특정 사이트에 대 한 내부 및 외부 통신의 보관을 사용 하거나 사용 하지 않도록 하나 이상의 선택적 사이트 정책을 지정할 수 있습니다. 또한 특정 사용자 및 사용자 그룹의 보관을 사용 하거나 사용 하지 않도록 설정 하는 사용자 정책을 하나 이상 지정할 수도 있습니다. 사용자 수준 정책은 사이트 정책 보다 우선 합니다. 사이트 수준 정책은 전역 수준 정책 보다 우선 합니다. 사용자 수준 정책은 정책을 사용 하도록 구성 된 특정 사용자만 구현할 수 있습니다. 그룹 인스턴트 메시지와 회의는 참가자 중 한 명 이상에 대 한 정책이 보관을 사용 하도록 구성 된 경우에만 보관 됩니다.
    
    <div>
    

    > [!NOTE]  
    > Microsoft Exchange 통합을 사용 하는 경우 Exchange 2013 정책은 Exchange 2013 서버에 속한 모든 사용자에 대해 Lync Server 보관 정책을 재정의 합니다.

    
    </div>

  - **구성을 보관**합니다. 앞에서 설명한 대로 보관 정책을 사용 하 여 구성 된 내부 및 외부 통신을 보관 하는 경우를 제외 하 고는 하나 이상의 보관 구성을 사용 하 여이 항목에서 설명 하는 대부분의 보관 옵션을 지정 합니다. 이전 글머리 기호) 보관 구성은 기본 전역 구성과 선택적 사이트 및 풀 구성을 포함 합니다. 전역 구성은 삭제할 수 없습니다. 풀 수준 구성은 사이트 수준 구성 보다 우선 합니다. 사이트 수준 구성은 전역 수준 구성 보다 우선 합니다.

요구 사항 분석의 일부로 전역 보관 구성 및 전역 보관 정책을 구성 하는 방법을 결정 해야 합니다. 또한 사이트 수준 보관 구성, 풀 수준 보관 구성, 사이트 수준 보관 정책, 사용자 수준 보관 정책에 대 한 요구 사항을 결정 해야 합니다.

하나 이상의 프런트 엔드 풀 또는 Standard Edition 서버에 대해 보관을 배포 하는 경우 배포의 다른 모든 프런트 엔드 풀 및 Standard Edition 서버에서 사용 하도록 설정 해야 합니다. 이 작업은 통신을 보관 해야 하는 사용자가 다른 풀에서 호스트 되는 그룹 메신저 대화 또는 모임에 초대 될 수 있기 때문에 수행 해야 합니다. 대화 또는 모임이 호스팅되는 풀에서 보관을 사용할 수 없는 경우에는 모든 컨퍼런스 데이터가 보관 되지 않을 수 있습니다. 보관은 계속 해 서 사용할 수 있는 사용자 및 모든 메신저 메시지에 적용 되지만, 회의 콘텐츠 및 이벤트는 보관 되지 않을 수 있습니다.

<div>


> [!NOTE]  
> 조직의 보안 표준을 유지 하면서 관리 작업을 위임할 수 있도록 Lync Server 2013&nbsp;는 RBAC (역할 기반 액세스 제어)를 사용 합니다. RBAC를 사용 하는 경우 사용자를 미리 정의 된 관리 역할에 할당 하 여 관리 권한을 부여 받습니다. Lync 보관 정책 및 보관 구성을 구성 하려면 사용자가 CsArchivingAdministrator 역할에 할당 되어야 합니다 (다른 컴퓨터에서 원격으로 저장 되지 않고 보관을 배포 하는 서버에서 직접 구성 되지 않은 경우). RBAC에 대 한 자세한 내용은 계획 문서에서 <A href="lync-server-2013-planning-for-role-based-access-control.md">Lync Server 2013의 역할 기반 액세스 제어 계획</A> 을 참조 하세요. 배포 보관에 필요한 사용자 권한, 사용 권한 및 역할 목록은 계획 문서와 배포 설명서 모두에서 제공 되는 <A href="lync-server-2013-deployment-checklist-for-archiving.md">Lync Server 2013에서 보관에 대 한 배포 검사 목록을</A>참조 하세요.<BR>Microsoft Exchange 통합을 사용 하는 경우 Exchange 정책 구성에 적절 한 관리자 권한 및 사용 권한이 필요 합니다. 자세한 내용은 Exchange 2013 설명서를 참조 하세요.
