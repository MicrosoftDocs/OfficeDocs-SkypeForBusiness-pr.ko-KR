# <a name="associate-subnets-with-network-sites-for-cac-in-lync-server-2013"></a>Lync Server 2013에서 CAC의 네트워크 사이트와 서브넷 연결

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**마지막으로 수정한 주제:** 2012-10-20_

네트워크의 모든 서브넷은 특정 네트워크 사이트에 연결 되어 있어야 합니다. 이는 끝점이 위치한 네트워크 사이트를 확인 하는 데 서브넷 정보를 사용 하기 때문입니다. 한 세션에서 두 당사자의 위치를 알고 있는 경우, call 허용 제어 (CAC)는 전화를 걸기에 충분 한 대역폭이 있는지 확인할 수 있습니다.

네트워크 사이트와 서브넷을 연결 하기 위한 특별 한 요구 사항이 통화 허용 제어에 없습니다. 토폴로지의 서브넷과 네트워크 사이트 간에 연결을 만들려면 [Lync Server 2013에서 서브넷을 네트워크 사이트와 연결](lync-server-2013-associate-a-subnet-with-a-network-site.md)의 절차를 따릅니다. 전화 허용 제어에 대 한 예제 네트워크 토폴로지에 있는 네트워크 사이트 및 해당 서브넷을 보려면 계획 설명서의 [Lync Server 2013에서 통화 허용 제어에 대 한 요구 사항 수집 예제](lync-server-2013-example-of-gathering-your-requirements-for-call-admission-control.md) 를 참조 하세요.
