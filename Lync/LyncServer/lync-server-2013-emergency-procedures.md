# <a name="emergency-procedures-in-lync-server-2013"></a>Lync Server 2013의 응급 절차

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**마지막으로 수정한 주제:** 2014-08-18_

비상 프로시저는 앞에서 설명한 표준 절차를 벗어나는 주문형 프로시저 이며 변경 관리 기능 ( [MOF](http://go.microsoft.com/fwlink/p/?linkid=40939) 에 대 한 "변경 관리" 기능에 대 한 설명을 참조)으로 처리 되어야 합니다. 이러한 절차는 일반적으로 다음 중 하 나와 관련이 있습니다.

  - 확인 된 시스템 문제를 해결 하기 위해 문제 해결 노력의 일부로 변경을 구현 합니다.

  - 확인 된 보안 위협 (예: 소프트웨어 업데이트)을 해결 합니다.

비즈니스에 미치는 영향을 평가 하 여 변경 사분면 지침에 정의 된 대로 공식적인 변경 관리 절차를 통해 임시 변경 사항을 구현 하거나 channeled 해야 하는지 여부를 결정 해야 합니다.

긴급 변경 절차를 구현 하는 것이 좋습니다. 예를 들어 긴급 변경 내용이 표시 되는 경우 혼란 상태를 줄이기 위해 이러한 유형의 절차와 도움말을 처리 하는 방법을 문서화 하세요.
