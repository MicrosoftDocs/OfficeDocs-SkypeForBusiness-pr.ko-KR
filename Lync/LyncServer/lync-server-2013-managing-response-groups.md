# <a name="managing-response-groups-in-lync-server-2013"></a>Lync Server 2013의 응답 그룹 관리

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**마지막으로 수정한 주제:** 2018-02-01_

응답 그룹은 헬프 데스크와 같은 특정 영역에 대 한 통화를 큐에 대기 시킬 수 있는 통화 관리 기능이 며, *상담원*이라는 지정 된 사용자 그룹에 게 전화를 라우팅합니다.

응답 그룹을 관리 하려면 에이전트가 해당 시간에 응답할 때까지 호출에 발생 하는 상황을 정의 하는 에이전트 그룹, 큐 및 워크플로를 구성 합니다.

<div>


> [!NOTE]  
> 응답 그룹 배포의 단일 풀에 300 개 이상의 워크플로가 있는 경우 Lync Server Management Shell cmdlet을 사용 하 여 워크플로를 만드는 것이 좋습니다. 응답 그룹 구성 도구를 사용 하 여 300 개 이상의 워크플로가 있는 풀에 대 한 워크플로를 만드는 경우 웹 페이지를 로드 하는 데 시간이 오래 걸립니다. 큐를 통해 워크플로와 간접적으로 연결 된 에이전트의 수에도 페이지 로드에 비례하여 효과가 적용 됩니다.



</div>

이 섹션의 항목에서는 배포에서 응답 그룹 응용 프로그램을 사용자 지정 하 고 유지 관리 하기 위해 수행할 수 있는 작업에 대 한 단계별 절차를 제공 합니다.
