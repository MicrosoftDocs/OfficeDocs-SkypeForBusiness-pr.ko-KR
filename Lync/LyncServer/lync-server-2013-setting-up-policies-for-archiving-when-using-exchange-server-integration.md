# <a name="setting-up-policies-for-archiving-in-lync-server-2013-when-using-exchange-server-integration"></a>Exchange Server 통합을 사용 하는 경우 Lync Server 2013에서 보관용 정책 설정

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**마지막으로 수정 된 항목:** 2012-10-09_

Exchange 2013에 있는 사용자의 사서함이 보류 In-Place 있으면 Exchange In-Place 보류 정책은 해당 사용자에 대 한 보관을 제어 합니다. 배포에 Microsoft Exchange 통합을 사용 하는 경우 Exchange 2013 정책은 Exchange 2013에 있는 사용자에 대 한 Lync Server 보관 정책을 재정의 합니다. Exchange 보관 정책 구성에 대 한 자세한 내용은 Exchange 2013 설명서를 참조 하십시오. Lync Server 2013에 있는 사용자에 대 한 사용자 정책을 설정 하는 방법에 대 한 자세한 내용은 배포 설명서의 [Lync server 2013에서 보관용 사용자 정책 설정을](lync-server-2013-setting-up-user-policies-for-archiving-in-lync-server.md) 참조 하십시오. 정책이 작동 하는 방식에 대 한 자세한 내용은 계획 설명서, 배포 설명서 또는 작업 설명서의 [Lync Server 2013에서 보관이 작동 하는 방식을](lync-server-2013-how-archiving-works.md) 참조 하세요.

<div>


> [!NOTE]
> 같은 포리스트에 Exchange 2013 및 Lync Server 2013을 배포 하는 경우 Exchange 2013 In-Place 보류 정책이 보관 됩니다. Exchange 2013 및 Lync Server 2013를 별도의 포리스트에 배포 하는 경우 <A href="lync-server-2013-deployment-checklist-for-archiving.md">배포 검사 목록에서 Lync server 2013의 보관을 위한</A>"lync Server 및 Microsoft Exchange를 다른 포리스트에서 배포"를 참조 하세요.
