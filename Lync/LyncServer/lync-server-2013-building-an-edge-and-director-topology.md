# <a name="building-an-edge-and-director-topology-in-lync-server-2013"></a>Lync Server 2013에서 에지 및 디렉터 토폴로지 구성

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**마지막으로 수정한 주제:** 2012-09-08_

토폴로지 빌드에는 다음과 같은 계획 및 배포 작업이 포함 됩니다.

  - **계획을 수립할**   때는 조직의 적절 한 토폴로지를 정의 하 고 배포에 필요한 구성 요소를 식별 해야 합니다. 계획 프로세스의 표준 단계는 다음과 같습니다. Microsoft Lync Server 2013, Lync Server 2013와 함께 제공 되는 계획 도구를 사용 하면 계획 프로세스를 쉽게 시작할 수 있을 뿐만 아니라 요구 사항과 계획이 완료 됨에 따라 쉽게 변경할 수 있는 기능을 포함 합니다.

  - **배포**   토폴로지 작성기를 사용 하 여 정의한 토폴로지는 Lync server 2013 서버 배포에 반드시 필요 합니다. 계획 노력의 일부로 토폴로지 작성기를 사용 하 여 토폴로지 정의 및 게시를 완료 하지 않은 경우에는이를 완료 하 고 Edge 서버를 배포 하기 전에 토폴로지를 게시 해야 합니다.

하나 이상의 내부 풀을 구축할 때까지 Edge 서버 구성 요소를 배포할 수 없으며 내부 풀을 배포 하기 위해 토폴로지 작성기를 설치 해야 합니다. 이 섹션에서는 내부 풀에 대 한 설치 프로세스의 일부 이기 때문에 토폴로지 작성기 설치를 다루지 않습니다.

이러한 도구에 대 한 자세한 내용은 [Lync Server 2013에서 외부 사용자 액세스에 대 한 배포 검사 목록을](lync-server-2013-deployment-checklist-for-external-user-access.md)참조 하세요.

<div>


> [!NOTE]  
> 이전에 edge 토폴로지를 비롯 한 토폴로지 작성기를 사용 하 여 전체 토폴로지를 정의한 경우 lync server <A href="lync-server-2013-define-your-edge-topology.md">2013에서 edge 토폴로지 정의</A> 를 건너뛰고이 섹션의 <A href="lync-server-2013-publish-your-topology.md">lync server 2013 작업에서 토폴로지를 게시할</A> 수 있지만 <A href="lync-server-2013-export-your-topology-and-copy-it-to-external-media-for-edge-installation.md">lync server 2013 토폴로지 내보내기를 완료 하 고 edge 설치 작업을 위해 외부 미디어에 복사</A> 하는 것이 필요 합니다.
