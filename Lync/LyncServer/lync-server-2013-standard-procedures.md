# <a name="standard-procedures-in-lync-server-2013"></a>Lync Server 2013의 표준 절차

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**마지막으로 수정 된 항목:** 2014-08-18_

여러 리소스를 통해 조직에서 필요한 표준 절차를 정의 하 고 수행 하는 방법을 확인할 수 있습니다. 각 조직이 고유 하기 때문에 일상적인 요구 사항에 맞게 이러한 리소스를 추가로 사용자 지정 하 고 조정 해야 할 수 있습니다. 표준 운영 절차 변경 및 문서를 가끔씩 수정 해야 하는 경우 변경 내용이 적용 되 면 Microsoft 운영 프레임 워크의 서비스 관리 기능에 정의 된 대로 변경 관리 프로세스에서 각 변경 내용이 관리 작업 수행 방법 및 시기에 어떤 영향을 주는지 식별 해야 합니다. 변경 관리 기능을 사용 하 여 절차 설명서를 업데이트 하 고 제어 합니다. 운영 작업은 매일, 매주, 매월 및 필요에 따라 수행 되는 작업을 관리 가능한 작업으로 구분 하는 것이 좋습니다. 일일 작업은 시스템 작동에 매우 중요 한 업무에 중점을 둔 것 이며, 월별 작업은 시스템의 장기간 상태를 확인 하는 데 집중 됩니다. 수행 해야 하는 작업은 다음 범주로 구분 될 수 있습니다.

  - [Lync Server 2013의 일별 작업](lync-server-2013-daily-tasks.md)

  - [Lync Server 2013의 주간 작업](lync-server-2013-weekly-tasks.md)

  - [Lync Server 2013의 월별 작업](lync-server-2013-monthly-tasks.md)

  - [Lync Server 2013의 필요에 따라 작업](lync-server-2013-as-needed-tasks.md)

작업 관리에 대 한 설명서를 준비할 때 검사 목록을 사용 하 여 필요한 작업이 적절 한 시간에 수행 되도록 합니다. 작업 검사 목록을 준비 하는 방법에 대 한 자세한 내용은 작업 검사 목록에 있는 예제 검사 목록을 참조 하십시오.

변경 관리는 시스템 관리가 완료 되는 위치를 자주 사용 합니다. 작업이 표준 절차에 따라 수행 되는 경우 해당 작업은 시스템 관리 기능의 일부입니다. 작업에 대 한 표준 프로시저가 없는 경우에는 변경 관리 기능을 사용 하 여 처리 해야 합니다.
