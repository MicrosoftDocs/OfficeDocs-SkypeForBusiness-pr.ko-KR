# <a name="standard-procedures-in-lync-server-2013"></a>Lync Server 2013의 표준 절차

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**마지막으로 수정한 주제:** 2014-08-18_

여러 리소스를 사용 하 여 조직에 필요한 표준 절차와 이러한 프로시저를 수행 하는 방법을 정의할 수 있습니다. 각 조직이 고유 하기 때문에 일상적인 요구 사항에 맞게 해당 리소스를 추가로 사용자 지정 하 고 조정 해야 할 수 있습니다. 표준 운영 절차 변경 및 문서를 가끔씩 수정 해야 하는 경우 변경 내용이 적용 되 면 Microsoft 운영 프레임 워크의 서비스 관리 기능에 정의 된 대로 변경 관리 프로세스에서 각 변경 내용이 관리 작업 수행 방법 및 시기에 미치는 영향을 식별 해야 합니다. 변경 관리 기능을 사용 하 여 절차 문서를 업데이트 하 고 제어 합니다. 작업을 매일, 매주, 매월, 필요한 기준에 따라 수행 하는 관리 가능한 작업으로 구분 하는 것이 좋습니다. 일일 작업은 시스템 작동에 매우 중요 한 측면에 중점을 두기 때문에 월간 작업은 시스템의 장기간 상태를 확인 하는 데 초점을 두어야 합니다. 수행 해야 하는 작업은 다음 범주로 나눌 수 있습니다.

  - [Lync Server 2013의 일별 작업](lync-server-2013-daily-tasks.md)

  - [Lync Server 2013의 주별 작업](lync-server-2013-weekly-tasks.md)

  - [Lync Server 2013의 월별 작업](lync-server-2013-monthly-tasks.md)

  - [Lync Server 2013의 필요에 따라 작업](lync-server-2013-as-needed-tasks.md)

작업 관리에 대 한 문서를 준비할 때는 검사 목록을 사용 하 여 필요한 작업이 적절 한 시간에 수행 되도록 합니다. 작업 검사 목록을 준비 하는 방법에 대 한 자세한 내용은 작업 검사 목록에 있는 샘플 검사 목록을 참조 하세요.

일반적으로 변경 관리는 시스템 관리가 완료 되는 위치를 차지 합니다. 표준 절차가 적용 되는 작업은 시스템 관리 함수의 일부입니다. 작업에 대 한 표준 절차가 없는 경우에는 변경 관리 기능을 사용 하 여 처리 해야 합니다.
