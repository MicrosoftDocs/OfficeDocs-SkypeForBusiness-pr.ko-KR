# <a name="planning-for-pstn-connectivity-in-lync-server-2013"></a>Lync Server 2013의 PSTN 연결 계획

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**마지막으로 수정한 주제:** 2012-09-21_

엔터프라이즈급 VoIP 솔루션은 서비스 품질 (QoS)에 동의 하지 않고 PSTN (공공 교환 전화 네트워크)에 대 한 통화를 제공 해야 합니다. 전화를 걸고 받는 사용자는 기본 기술을 인식 하지 못합니다. 사용자 관점에서 보면 엔터프라이즈 음성 인프라와 PSTN 간의 통화는 다른 전화 통화 처럼 보일 수 있습니다.

Lync Server 2013는 다음 옵션을 사용 하 여 안정적이 고 확장 가능한 PSTN 연결을 제공 합니다.

  - **SIP Trunks** 인터넷 통신 서비스 공급자 (itsp)

  - PSTN 게이트웨이에 대 한 **직접 SIP 연결**

  - PBX로 **직접 SIP 연결**

해당 크기, 지역 및 기존 음성 인프라에 따라 엔터프라이즈는 다양 한 위치에서 하나, 두 가지 또는 세 가지 옵션을 모두 사용할 수 있습니다.
