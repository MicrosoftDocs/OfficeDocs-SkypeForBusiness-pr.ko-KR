# <a name="new-role-based-access-control-features-in-lync-server-2013"></a>Lync Server 2013의 새 역할 기반 액세스 제어 기능

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**마지막으로 수정한 주제:** 2012-09-20_

Lync Server 2013는 두 가지 주요 방법으로 RBAC (역할 기반 액세스 제어) 기능을 향상 시킵니다. 이제 사용자 지정 역할을 만들 수 있으며, 여기에는 지정 된 일련의 cmdlet에 대 한 권한만 있습니다. 이러한 사용자 지정 역할에는 cmdlet의 스크립트를 실행할 수 있는 권한도 주어 집니다.

또한 Lync Server 2013에는 미리 정의 된 새 역할이 두 개 포함 되어 있습니다.

  - *응답 그룹 관리자* 역할이 지정 된 사용자는 조직의 특정 응답 그룹 대기열을 관리할 수 있지만, 다른 대기열 또는 전체적으로 응답 그룹 응용 프로그램에 대 한 관리 권한을 가질 필요는 없습니다.

  - *영구 채팅 관리자* 역할이 지정 된 사용자는 조직의 특정 영구 채팅방을 관리할 수 있지만, 다른 채팅방 또는 지속적인 채팅 기능에 대 한 관리 권한은 없을 것입니다.
