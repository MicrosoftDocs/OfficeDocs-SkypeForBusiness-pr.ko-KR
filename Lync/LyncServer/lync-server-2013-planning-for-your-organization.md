<div>

## <a name="in-this-section"></a>이 섹션의 내용

  - [Lync server 2013 배포 방법 결정](lync-server-2013-deciding-how-to-deploy-microsoft-lync.md) lync server 2013에 대 한 이러한 기본 배포 시나리오 중에서 선택할 수 있는 지침을 제공 합니다.

  - [Lync Server 2013에 대 한 계획 프로세스를 시작](lync-server-2013-beginning-the-planning-process.md) 하면 온-프레미스 배포 계획을 시작 하는 방법과 계획 설명서가 토폴로지 작성기에서 작동 하는 방식을 이해 하는 데 도움이 됩니다.

  - [계획 하기 전에 알아야 할 토폴로지 기본 사항 Lync server 2013](lync-server-2013-topology-basics-you-must-know-before-planning.md) 사이트, 서버 풀, 고가용성 및 재해 복구를 지 원하는 토폴로지를 비롯 한 lync server 토폴로지의 기본 사항에 대해 설명 합니다.

  - [초기 계획을 통해 Lync server 2013에 대 한 결정](lync-server-2013-initial-planning-decisions.md) 은 배포할 lync server의 작업 및 기능을 결정 하기 위해 답을 받아야 하는 질문을 안내 합니다.

  - [Lync Server 2013의 클라이언트](lync-server-2013-clients.md) 는 컴퓨터에 설치 된 클라이언트 소프트웨어, 웹 기반 클라이언트 및 모바일 장치를 포함 하 여 조직의 사용자에 게 배포할 수 있는 다양 한 유형의 클라이언트 소프트웨어에 대해 설명 합니다.

  - [Lync Server 2013의 참조 토폴로지에서](lync-server-2013-reference-topologies.md) 는 세 가지 일반적인 조직 유형에 적합 한 토폴로지 디자인을 보여 주는 세 가지 샘플 토폴로지를 보여 주고 이러한 토폴로지를 디자인할 때의 많은 결정에 대 한 이유를 설명 합니다.
