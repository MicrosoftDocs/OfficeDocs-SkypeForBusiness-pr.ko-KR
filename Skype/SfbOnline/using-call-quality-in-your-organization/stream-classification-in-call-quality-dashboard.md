# <a name="stream-classification-in-call-quality-dashboard"></a>통화 품질 대시보드의 스트림 분류

Microsoft 팀 및 비즈니스용 Skype Online에 대 한 CQD (통화 품질 대시보드)는 Microsoft 팀과 비즈니스용 Skype 서비스를 사용 하 여 만든 통화 품질에 대 한 통찰력을 얻을 수 있도록 합니다. 이 항목에서는 미디어 스트림의 품질 분류에 대 한 자세한 정보를 제공 합니다. CQD 및이를 사용 하는 방법에 대해 자세히 알아보려면 [통화 품질 대시보드 켜기 및 사용](turning-on-and-using-call-quality-dashboard.md)을 참조 하세요.

## <a name="classifier-definitions"></a>분류자 정의

CQD의 스트림은 사용 가능한 주요 품질 메트릭의 값에 따라 좋음, 나쁨 또는 미분류로 분류 됩니다. 스트림을 분류 하는 데 사용 되는 메트릭과 조건은 아래 표에 나와 있습니다. CQD의 "부적절 한 분류를 담당 하는 메트릭은 차원을 파악 하는 데 사용 될 수 있습니다. 이러한 차원에 대 한 자세한 내용은 [통화 품질 대시보드에서 사용할 수 있는 차원 및 측정값](dimensions-and-measures-available-in-call-quality-dashboard.md) 을 참조 하세요.

### <a name="audio-classifier"></a>오디오 분류자

오디오 스트림은 다음 조건 중 하나 이상이 충족 되는 경우 불량으로 표시 됩니다.

|**미터**|**조건**|**볼**|
|:-----|:-----|:-----|
|오디오 성능 저하 평균|> 1.0|스트림에 대 한 평균 네트워크 평균 의견 점수가 저하 됩니다. 네트워크 손실 및 지터로 수신 된 오디오의 품질에 영향을 주는 정도를 나타냅니다.|
|라운드트립|> 500|RFC3550에 지정 된 대로 평균 네트워크 전파 왕복 시간을 밀리초 단위로 계산 합니다.|
|패킷 손실 율|> 0.1|스트림의 평균 패킷 손실 율입니다.|
|지터|> 30|스트림의 평균 지터 (밀리초)입니다.|
|숨겨진 샘플 평균 비율|> 0.07|패킷 손실 치유에서 생성 된 숨겨진 샘플이 총 오디오 프레임 수에 해당 하는 오디오 프레임 수의 평균 비율입니다.|

### <a name="video-classifier"></a>비디오 분류자

비디오 스트림은 다음 순서 대로 첫 번째 사용 가능한 메트릭의 값에 따라 좋음 또는 불충분으로 표시 됩니다.

|**단계만 #**|**미터**|**조건**|**조건이 True 인 경우의 분류**|**If 조건이 거짓이 면 분류**|**메트릭을 사용할 수 없는 경우의 분류**|**볼**|
|:-----|:-----|:-----|:-----|:-----|:-----|:-----|
|raid-1|비디오 로컬 프레임 손실 평균|> 50% |저하|적합|2 단계를 진행 합니다.|사용자에 게 표시 되는 것 처럼 손실 된 비디오 프레임의 평균 백분율입니다. 여기에는 네트워크 손실에서 복구 된 프레임이 포함 됩니다.|
|2|비디오 프레임 속도 평균|< 7|저하|적합|3 단계로 진행|비디오 스트림에 대해 초당 받은 평균 프레임 수로, 세션 기간 동안 계산 됩니다.|
|3-4|비디오 게시물 (FECPLR)|> 0.15|저하|적합|않은|FEC가 모든 비디오 스트림 및 코덱에 대해 집계 된 후의 패킷 손실 속도입니다.|

### <a name="vbss-classifier"></a>VBSS 분류자

VBSS 스트림은 다음 순서 대로 첫 번째 사용 가능한 메트릭의 값에 따라 좋음 또는 불충분으로 표시 됩니다.

|raid-1|비디오 로컬 프레임 손실 평균|> 50% |저하|적합|2 단계를 진행 합니다.|사용자에 게 표시 되는 것 처럼 손실 된 비디오 프레임의 평균 백분율입니다. 여기에는 네트워크 손실에서 복구 된 프레임이 포함 됩니다.|

### <a name="application-sharing-classifier"></a>응용 프로그램 공유 분류자

다음 조건 중 하나 이상이 충족 되는 경우 응용 프로그램 공유 스트림은 불량으로 표시 됩니다.

**미터**|**조건**|**볼**|
|:-----|:-----|:-----|
|Spoiled 타일 백분율 합계|> 36|원격 피어로 전송 되는 대신 삭제 되는 타일의 백분율입니다 (예: MCU에서 뷰어로). 클라이언트와 서버 간의 대역폭 제한으로 인해 버려진 (또는 spoiled) 타일이 발생할 수 있습니다.|
|AppSharing RDP 타일 처리 지연 평균|> 400|회의 서버에서 RDP 스택의 타일 처리 평균 지연 시간 (밀리초)입니다.|
|AppSharing 상대 단방향 평균|> 1.75|응용 프로그램 공유 스트림의 끝점 간 단방향 지연 평균을 초 단위로 표시할 수 있습니다.|

## <a name="unclassified-streams"></a>분류 되지 않은 스트림

CQD에서, ICE 연결에 실패 하거나 스트림 분류를 계산 하는 데 필요한 모든 메트릭이 보고 되지 않으면 스트림이 분류 되지 않음으로 표시 됩니다.

ICE 연결 실패를 확인 하려면 "첫 번째 연결 Ice" 및 "두 번째 연결 아이스"의 크기를 확인 하 여 "실패" 값에 해당 합니다. 두 값 중 하나가 오류를 나타내는 경우 스트림은 분류 되지 않음으로 표시 됩니다.

분류 되지 않은 스트림에 대해 ICE 연결에 성공 하면 키 스트림 메트릭이 보고 되지 않았으므로 스트림은 비 미분류 것으로 간주 될 수 있습니다. 다음과 같은 이유로 인해 이러한 메트릭이 보고 되지 않을 수 있습니다.

- **체감 품질 보고서가 수신 되지 않음** -분류에 사용 된 메트릭은 통화 종료 시 보낸 체감 품질 보고서에 보고 됩니다. 이 보고서가 생성 되지 않은 경우 (예: 일부 타사 종점이 체감 품질를 보내지 못하거나 네트워크 중단 때문에 전송 되지 않음) CQD는 스트림을 분류할 수 없습니다.

> [!TIP]
> "체감 품질 기록 사용 가능" 차원은 체감 품질 보고서가 스트림에 수신 되었는지 여부를 확인 하는 데 사용할 수 있습니다. 체감 품질 보고서가 끝점 중 하나에서 수신 되 면이 차원에 "True" 값이 지정 됩니다. 메트릭에 대 한 가장 정확한 보고를 위해서는 두 끝점의 체감 품질 보고서가 필요 합니다.

- **짧은 호출** -짧은 호출에는 키 스트림 메트릭을 계산 하는 데 충분 한 미디어 작업이 없을 수 있습니다. 이러한 메트릭이 없으면 CQD는 스트림을 분류할 수 없습니다.

> [!TIP]
> "기간 (초)", "기간 (분)", "기간 5 초 또는 그 보다 작음" 및 "기간 60 초 이상"을 사용 하 여 스트림의 기간을 확인할 수 있습니다. "Avg Call Duration" 측정을 사용 하 여 스트림 집합의 평균 기간을 계산할 수도 있습니다.

- **패킷 사용률 낮음** -"짧은 통화" 시나리오와 마찬가지로 키 스트림 메트릭의 계산에 충분 한 패킷 이용률이 필요 합니다. 이러한 메트릭이 없으면 CQD는 스트림을 분류할 수 없습니다.
    - 일반적으로 낮은 패킷 사용률 시나리오는 사용자가 모임에 참가 하 여 프레 젠 터를 듣기만 하는 경우 (대부분의 통화에 대해 마이크를 음소거 할 수 있습니다)에 발생 합니다. 이러한 시나리오에서 한 오디오 스트림은 클라이언트에 대 한 인바운드 패킷 사용률을 가지 지만, 다른 하나는 패킷 사용률 (클라이언트의 아웃 바운드 없음)을 거의 갖지 않습니다. 이 시나리오에서는 스트림 지속 시간이 시간 보다 길어질 수 있지만, 마이크가 음소거 상태 이기 때문에 클라이언트에서 서버로의 스트림 사용률이 매우 낮기 때문에이로 인해 분류 되지 않은 스트림이 발생 하 게 됩니다.

> [!TIP]
> "패킷 사용률" 차원과 "Avg Packet 사용률" 측정값을 사용 하 여 스트림의 패킷 활동을 확인할 수 있습니다.
