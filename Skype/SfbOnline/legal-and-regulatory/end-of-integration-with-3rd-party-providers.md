# <a name="end-of-life-program-for-the-integration-of-skype-for-business-with-third-party-audio-conferencing-providers"></a>타사 오디오 회의 공급자와 비즈니스용 Skype 종료 프로그램 

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

Microsoft는 타사 오디오 회의 공급자(ACP)와 비즈니스용 Skype 통합을 위한 수명 종료 프로그램을 시작했다고 발표했습니다. 

수명 종료 프로그램은 2021년 7월 31일로 종료됩니다. 프로그램이 끝났을 때 타사 비즈니스용 Skype 오디오 회의 공급자와의 통합이 중지되어 다음 변경 내용이 이 날짜(2021년 7월 31일)에 관찰됩니다.

- 타사 ACP 비즈니스용 Skype 전화 접속 번호를 통해 모든 모임에 참가하려고 시도하는 참가자는 더 이상 모임 모임에 비즈니스용 Skype 없습니다.
 
- 타사 ACP 서비스에 사용하도록 설정된 사용자는 더 이상 새 모임 초대에 자동으로 전화 접속 정보가 비즈니스용 Skype 없습니다.

수명 종료 프로그램의 시작을 발표하는 일환으로 다음과 같은 변경이 적용되고 수명이 종료될 때까지 계속 적용됩니다. 

- 타사 ACP 비즈니스용 Skype 사용하도록 구성된 사용자가 없는 고객은 타사 ACP 서비스를 사용하도록 사용자를 구성할 수 없습니다.

- 타사 ACP 비즈니스용 Skype 사용자로 구성된 기존 고객은 수명이 끝나는 동안 계속 새 사용자를 추가할 수 있습니다. 2021년 7월 31일에 적용될 변경 사항이 적용될 예정이기 때문에 타사 ACP 서비스를 비즈니스용 Skype 사용자에 대한 추가 설정은 권장하지 않습니다.

## <a name="preparing-for-this-change"></a>이 변경에 대한 준비

이 변경에 대비하기 위해 영향을 받는 조직은 2021년 7월 31일 이전에 해당 지원되는 사용자에게 이 계획된 업데이트를 알릴 수 있도록 권장합니다. 

2021년 7월 31일 이후 사용자는 온라인 모임을 중단하지 않고 비즈니스용 Skype 계속 사용할 수 있습니다. 그러나 조직에서는 사용자가 Microsoft에서 제공하는 오디오 회의에 대해 사용자가 전화 접속 또는 전화 접속 오디오 회의를 요구하는 비즈니스용 Skype Microsoft Teams. Microsoft 오디오 회의에 대한 자세한 내용은 오디오 회의 [를 참조합니다.](https://products.office.com/skype-for-business/audio-conferencing) 

조직의 원하는 종료 상태에 따라 다음과 같은 세 가지 경로가 있습니다.

- Microsoft 오디오 회의로 마이그레이션합니다. 
- 타사 오디오 회의 공급자를 별도로 계속 사용합니다. 
- 전화 접속 회의 사용을 모두 중지합니다.

### <a name="path-1-migrate-to-microsoft-audio-conferencing"></a>경로 #1: Microsoft 오디오 회의로 마이그레이션   

2021년 7월 31일 이전에 Microsoft 오디오 회의로 마이그레이션하고 마이그레이션을 완료하기로 결정한 조직은 이 날짜 중 또는 이후에 서비스 영향을 경험하지 않습니다. Microsoft Audio Conferencing으로 마이그레이션하면 조직에 다음과 같은 변경 사항이 도입됩니다. 

- 서비스는 다른 모든 서비스 또는 Microsoft 365 Office 365 청구됩니다. 

- 표준 구독을 구매한 경우 사용자당 월별 구독 비용에 요금 전화 접속 비용이 포함됩니다. 

- 각 조직 및 해당 사용자에게 새 전화 접속 전화 번호 집합이 제공됩니다. Microsoft 오디오 회의 서비스의 지리적 범위를 확인한 경우 오디오 회의 및 통화 계획에 대한 국가 및 지역 가용성을 [참조합니다.](/microsoftteams/country-and-region-availability-for-audio-conferencing-and-calling-plans/country-and-region-availability-for-audio-conferencing-and-calling-plans)
 
- 타사 ACP를 사용하도록 설정된 사용자가 이미 예약한 모임은 Microsoft 오디오 회의 전화 접속 정보를 포함하기 위해 자동으로 다시 예약됩니다.
 
- 각 모임의 회의 ID는 동적으로, 각 모임에 자체 전용 회의 ID가 있습니다. 동적 회의 아이디는 백-백 모임을 위한 향상된 보안 및 향상된 환경을 제공합니다.

- 서비스의 모든 사용은 오디오 회의 서비스 사용 약관에 따라 다를 수 있습니다. 

Microsoft Audio Conferencing으로 마이그레이션하는 것은 간단하며, 서비스에 대한 라이선스를 취득한 후 몇 단계만 수행하면 됩니다. Microsoft 오디오 회의로 마이그레이션하는 방법에 대한 자세한 내용은 다음을 참조합니다.

- [오디오 회의를 시도하거나 Microsoft 365 또는 Office 365](../audio-conferencing-in-office-365/try-or-purchase-audio-conferencing-in-office-365.md)
 
**요약:**

- 2021년 7월 31일 이전의 Microsoft 오디오 회의로 마이그레이션하고 마이그레이션을 완료하는 조직은 이 날짜 중 또는 이후에 서비스에 미치는 영향이 없습니다.

- Microsoft 오디오 회의로 마이그레이션하는 방법을 자세히 알아보하려면 에서 오디오 [회의](../audio-conferencing-in-office-365/try-or-purchase-audio-conferencing-in-office-365.md)시도 또는 구매를 Microsoft 365 또는 Office 365. 

### <a name="path-2-continue-to-separately-use-a-third-party-audio-conferencing-provider"></a>경로 #2: 타사 오디오 회의 공급자를 별도로 계속 사용합니다.

2021년 7월 31일 이후에 타사 ACP를 계속 사용하기로 결정한 조직은 타사 ACP 전화 접속 정보를 더 이상 사용하여 모임의 오디오 부분에 참여할 수 비즈니스용 Skype 있습니다. 

일부 참가자가 타사 ACP를 통해 VoIP를 통해 참가하고 다른 참가자를 참가하게 하여 모임에서 오디오 조각화가 비즈니스용 Skype 방지하기 위해 이러한 조직에서는 사용자의 모임에서 VoIP 사용을 사용하지 않도록 설정하는 것이 좋습니다. 이렇게 하면 모든 참가자가 타사 ACP를 사용하여 모임의 오디오 부분에 참가해야 하며, 모임의 다른 모든 워크로드(예: 채팅 또는 화면 공유)는 계속 지원될 수 비즈니스용 Skype. 

- 주어진 이끌이의 모든 모임에서 VoIP를 사용하지 않도록 설정하기 위해 회의 정책의 AllowIPAudio 매개 변수를 Set-CsConferencingPolicy cmdlet을 통해 false로 설정합니다. 자세한 내용은 [Set-CsConferencingPolicy 를 참조하세요.](/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)
 
예정과 2021년 7월 31일을 기점으로 타사 ACP의 전화 접속 정보는 더 이상 모임 초대에 자동으로 비즈니스용 Skype 않습니다. 사용자가 모임의 일부로 이 정보를 계속 포함하려면 해당 비즈니스용 Skype 모임 초대에 대한 전화 접속 정보를 수동으로 추가해야 합니다. 

2021년 7월 31일에는 타사 ACP 전화 접속 정보를 제거하기 위해 사용자의 기존 모임이 자동으로 다시 예약되지 않습니다. 사용자의 모임에 VoIP를 사용하도록 설정하기로 결정한 조직은 사용자에 대한 타사 ACP 통합을 사용하지 않도록 설정하고 모임 마이그레이션 서비스를 사용하여 모임 일정을 다시 변경하여 기존 모임에서 타사 오디오 회의 전화 접속 정보를 제거하고 이미 예약된 모임에서 오디오 조각화 방지를 고려해야 합니다. 

- 주어진 이끌이에 대한 타사 오디오 회의의 통합을 사용하지 않도록 설정하기 위해 Remove-CsUserAcp cmdlet을 사용 합니다. 자세한 내용은 [Remove-CsUserAcp 을 참조하세요.](/powershell/module/skype/remove-csuseracp?view=skype-ps) 

- 타사 오디오 회의 공급자와의 통합을 사용 안 하여 사용자의 모임을 자동으로 다시 조정하려면 "사용자에 대해 모임 마이그레이션을 수동으로 실행하려면 어떻게 하세요?"를 참조하세요. 에서 [MMS(모임 마이그레이션 서비스)를 설정합니다.](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md) 

**요약:**

- 타사 ACP를 2021년 7월 31일 이후로 계속 사용하기로 결정한 조직은 타사 ACP를 사용하여 모임에 참가할 수 비즈니스용 Skype 수 있으며 새 모임에는 타사 ACP 전화 접속 정보가 포함되어 있지 않습니다. 

- VoIP를 통해 참가하는 참가자와 타사 ACP를 통해 오디오가 조각화되지 않도록 하기 위해 2021년 7월 31일 이전의 영향을 받는 모든 사용자의 모든 모임에서 VoIP를 사용하지 않도록 설정하는 것이 좋습니다. 

    - 주어진 이끌이의 모든 모임에서 VoIP를 사용하지 않도록 설정하기 위해 사용자의 회의 정책의 AllowIPAudio 매개 변수를 Set-CsConferencingPolicy cmdlet을 통해 false로 설정합니다. 자세한 내용은 [Set-CsConferencingPolicy 를 참조하세요.](/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)
 
- 조직에서 모든 모임에서 VoIP를 사용하지 않도록 설정하지 않는 경우 사용자가 타사 ACP와의 비즈니스용 Skype 온라인 통합을 사용하지 않도록 설정하고 모임을 다시 예약하여 오디오 조각화 방지를 위해 타사 ACP 전화 접속 정보를 제거하는 것이 좋습니다.

    - 주어진 이끌이에 대한 타사 오디오 회의 통합을 사용하지 않도록 설정하려면 [Remove-CsUserAcp](/powershell/module/skype/remove-csuseracp?view=skype-ps) cmdlet을 사용 합니다. 

    - 모임을 자동으로 다시 조정하려면 "사용자에 대해 모임 마이그레이션을 수동으로 실행하려면 어떻게 하세요?"를 참조하세요. 에서 [MMS(모임 마이그레이션 서비스)를 설정합니다.](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md)

### <a name="path-3-stop-using-dial-in-conferencing-altogether"></a>경로 #3: 전화 접속 회의 사용을 모두 중지합니다.

전화 접속 회의 사용을 완전히 중지하기로 결정한 조직(Microsoft 또는 타사 ACP에서 제공되지 않습니다)은 VoIP를 완전히 사용하여 모임의 오디오 부분을 비즈니스용 Skype 수 있습니다. 

이러한 조직에서는 사용자가 타사 오디오 회의 공급자를 사용하지 않도록 설정하고 모임 마이그레이션 서비스를 사용하여 해당 모임을 자동으로 다시 조정하여 전화 접속 회의 정보를 제거해야 합니다. 

- 주어진 이끌이에 대한 타사 오디오 회의의 통합을 사용하지 않도록 설정하기 위해 Remove-CsUserAcp cmdlet을 사용 합니다. 자세한 내용은 [Remove-CsUserAcp 을 참조하세요.](/powershell/module/skype/remove-csuseracp?view=skype-ps) 

- 타사 오디오 회의 공급자와의 통합을 사용 안 하여 사용자의 모임을 자동으로 다시 조정하려면 "사용자에 대해 모임 마이그레이션을 수동으로 실행하려면 어떻게 하세요?"를 참조하세요. 에서 [MMS(모임 마이그레이션 서비스)를 설정합니다.](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md) 

**요약:** 

- 2021년 7월 31일 이전의 오디오 회의 사용을 중지하기로 결정한 조직은 영향을주지 않습니다.

- 주어진 이끌이에 대한 타사 오디오 회의의 통합을 사용하지 않도록 설정하기 위해 Remove-CsUserAcp cmdlet을 사용 합니다. 자세한 내용은 [Remove-CsUserAcp 을 참조하세요.](/powershell/module/skype/remove-csuseracp?view=skype-ps) 

- 타사 오디오 회의 공급자와의 통합을 사용 안 하도록 설정한 후 사용자의 모임을 자동으로 다시 조정하려면 "사용자에 대해 모임 마이그레이션을 수동으로 실행하려면 어떻게 하세요?"를 참조하세요. 에서 [MMS(모임 마이그레이션 서비스)를 설정합니다.](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md)
