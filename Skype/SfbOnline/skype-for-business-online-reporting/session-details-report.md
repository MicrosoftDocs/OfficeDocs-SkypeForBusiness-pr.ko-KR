# <a name="session-details-report"></a>세션 정보 보고서

새 Office 365 **보고서** 대시보드는 조직의 Office 365 제품 전반에 걸친 활동 개요를 보여 줍니다. 각 제품의 활동에 대 한 세부적인 통찰력을 제공 하기 위해 개별 제품 수준 보고서를 드릴스루할 수 있습니다. 예를 들어 **비즈니스용 Skype 세션 세부 정보** 보고서를 사용 하 여 개별 사용자의 통화 환경에 대 한 세부 정보를 확인할 수 있습니다.
  
사용할 수 있는 보고서에 대 한 자세한 내용은 [보고서 개요](https://support.office.com/article/0d6dfb17-8582-4172-a9a9-aed798150263) 를 확인 하세요.
  
이 보고서는 다른 비즈니스용 Skype 보고서와 함께 조직 전체의 세션 세부 정보를 포함 하는 활동에 대 한 세부 정보를 제공 합니다. 이러한 세부 정보는 조직에 대 한 기타 비즈니스 의사 결정과 [통신 크레딧을](/microsoftteams/what-are-communications-credits)설정 하는 데 매우 유용 합니다.
  
> [!NOTE]
> Microsoft 365 관리 센터에 관리자로 로그온 하면 모든 비즈니스용 Skype 보고서를 볼 수 있습니다. 
  
## <a name="how-to-get-to-the-skype-for-business-session-details-report"></a>비즈니스용 Skype 세션 정보 보고서에 액세스 하는 방법

1. 관리 센터로 이동 > **보고서**
    
2. 왼쪽 메뉴에서 **보고서** 를 선택한 다음 **사용 현황을**클릭 합니다.
    
3. **보고서 선택**아래의 목록에서 **비즈니스용 Skype 세션 정보**를 클릭 합니다.
    
    > [!TIP]
    > 이 보고서가 나열 되지 않으면 **비즈니스용 Skype 관리 센터** > 에서**세션 세부 정보**를**보고** > 하세요. 
  
    > [!IMPORTANT]
    > 사용 중인 Office 365 구독에 따라 여기에 표시 된 모든 제품 및 보고서가 표시 되지 않을 수 있습니다. 
  
## <a name="interpret-the-skype-for-business-session-details-report"></a>비즈니스용 Skype 세션 정보 보고서 해석

표시 되는 각 열을 확인 하 여 사용자의 비즈니스용 Skype 세션 정보에 대 한 보기를 얻을 수 있습니다.
  
보고서의 모양은 다음과 같습니다.
  
![비즈니스용 Skype 세션 정보 보고서 대시보드.](../images/3d87ab39-6eaa-46b5-b5f9-7f54dc987ae0.png)
  
***
![번호 1](../images/sfbcallout1.png)<br/>**별칭을 기준으로 사용자 검색** -단일 사용자를 검색 하 고 아래 표에 있는 사용자의 모든 세션 정보를 표시할 수 있습니다. 
***
![번호 2](../images/sfbcallout2.png)<br/>시작 날짜를 입력할 수 있도록 **날짜/시간을 입력** 합니다. 달력을 사용 하 여 날짜를 선택 하거나 수동으로 날짜를 입력할 수 있습니다. 이 필드는 채워야 합니다.
***
![번호 3](../images/sfbcallout3.png)<br/>**날짜 시간에 Enter 키** 를 사용 하 여 끝 날짜를 입력 합니다. 달력을 사용 하 여 날짜를 선택 하거나 수동으로 날짜를 입력할 수 있습니다. 종료 날짜를 설정 하지 않은 경우 기본값은 시작 날짜 로부터 30 일입니다.
***
![번호 4](../images/sfbcallout4.png)<br/>다음 표는 사용자 당 모든 세션 세부 정보를 분석 한 결과를 보여 줍니다. 비즈니스용 Skype에 할당 된 모든 사용자 및 해당 세션 정보를 보여 줍니다. 표에 열을 추가 하거나 제거할 수 있습니다. <br/><br/>테이블에는 각 세션에 대해 다음과 같은 열이 있습니다.
*    **대화 상자 id** 는 SIP 세션의 고유 식별자에 대 한 id입니다.
*    **미디어 유형 설명** 세션이 전화 회의 인지 또는 P2P 세션과 사용 되는 미디어 유형 (오디오/비디오/응용 프로그램 공유) 중 어디에 있는지 설명 합니다.
*    **시작 시간은** 세션이 시작 된 시간입니다.
*    **종료 시간은** 세션이 종료 된 시간입니다.
*    In **URI** 는 세션을 시작한 사용자 또는 서비스의 URI입니다. 사용자가 PSTN 휴대폰에서 세션을 시작한 경우 비어 있을 수 있습니다.
*    **Uri로** 는 세션 시작의 대상인 사용자 또는 서비스의 URI입니다. 회의의 경우이는 이끌이의 URI입니다. 세션 대상이 PSTN 전화 번호 인 경우 비어 있을 수 있습니다.
*    **클라이언트 버전에서** 사용자 에이전트 및 세션을 시작한 사용자 또는 서비스에서 사용 하는 클라이언트 버전을 알려 줍니다.
*    **클라이언트 버전에** 대해 사용자 에이전트 및 세션 초기화의 대상인 사용자 또는 서비스에서 사용 하는 클라이언트 버전을 알려 줍니다.
*    회의 **url** 은 세션이 컨퍼런스 통화 인 경우 회의의 SIP url입니다. 동일한 컨퍼런스 통화에 참여 하는 모든 사용자에 게 동일한 컨퍼런스 URL이 포함 됩니다. 
*    **보낸 사람 전화 번호** 는 해당 되는 경우 세션의 대상인 전화 번호입니다. 전화 번호의 마지막 자리는 사용자 개인 정보 보호를 위해 ' x '로 바뀔 수 있습니다.
*    전화 **번호** 는 해당 되는 경우 세션의 대상인 전화 번호입니다. 전화 번호의 마지막 자리는 사용자 개인 정보 보호를 위해 ' x '로 바뀔 수 있습니다.
*    **보낸 사람 끝점 Id** 는 보낸 사람 사용자가 사용 하는 끝점의 고유 GUID입니다. 사용자가 동일한 끝점에서 여러 세션을 통신 하 고 있는지 여부를 식별 하는 데 사용 됩니다. 사용자가 PSTN 휴대폰을 사용 하거나 서비스에서 세션이 시작 된 경우 비어 있을 수 있습니다.
*    **끝점 Id** 는 to 사용자가 사용 하는 끝점의 고유 GUID입니다. 사용자가 동일한 끝점에서 여러 세션을 통신 하 고 있는지 여부를 식별 하는 데 사용 됩니다. 사용자가 PSTN 전화기를 사용 하 고 있거나 서비스에서 세션이 시작 되었거나 세션을 설정 하지 않은 경우 비어 있을 수 있습니다.
*    컨퍼런스 **인스턴스** 는 회의 URL을 사용 하는 회의 인스턴스의 고유 GUID입니다. 되풀이 모임에는 동일한 회의 URL이 있지만 모임의 각 인스턴스에는 서로 다른 컨퍼런스 인스턴스가 있습니다.
*    **Uri** 대신 세션을 만들고 있는 대리인의 uri입니다. <br/> **Uri** 는 세션의 설정을 참조 한 사용자의 uri입니다.
*    **응답 코드** 는 세션이 성공적으로 설정 되었는지 여부를 나타내는 세션의 성립에 대 한 SIP 응답 코드입니다.

각 세션에는 시나리오에 따라 다른 데이터를 사용할 수 있는 하위 테이블이 있습니다. 다음은 From 및 To 사용자 또는 서비스에 대 한 하위 테이블에서 사용할 수 있는 탭 목록입니다.
*    세션 탭 컴퓨터 및 운영 체제에 대 한 데이터를 표시 합니다.
*    MEDIALINES 탭에서는 네트워크 연결 정보와 장치 정보가 표시 됩니다.
*    오디오 스트림 탭에는 세션과 관련 된 오디오 스트림에 대 한 네트워크 성능 데이터가 표시 됩니다.
*    Audio CLIENTEVENTS 탭은 오디오 환경에 영향을 주는 클라이언트 감지 문제에 대 한 데이터를 표시 합니다.
*    Audio 신호 탭에는 세션의 오디오 신호 처리에 대 한 데이터가 표시 됩니다.
*    APPSHARINGSTREAMS 탭에는 세션과 관련 된 응용 프로그램 공유 또는 데스크톱 공유 스트림에 대 한 네트워크 성능 데이터가 표시 됩니다.
*    VIDEOCLIENTEVENTS 탭은 비디오 환경에 영향을 주는 클라이언트 감지 문제에 대 한 데이터를 표시 합니다.
*    VIDEOSTREAMS 탭에는 세션과 관련 된 비디오 스트림에 대 한 네트워크 성능 데이터가 표시 됩니다.
*    TRACEROUTES 탭에서는 세션 중에 traceroute를 통해 수집 된 네트워크 홉이 표시 됩니다. 세션에 사용 되는 실제 미디어 경로는 다를 수 있으며,이 데이터는 세션에 오디오가 있는 경우에만 사용할 수 있습니다.
*    FEEDBACKREPORTS 탭에서는 세션의 사용자가 제공 하는 호출 설문 조사 데이터의 끝이 표시 됩니다.
***

## <a name="want-to-see-other-skype-for-business-reports"></a>다른 비즈니스용 Skype 보고서를 보 시겠습니까?

- [비즈니스용 Skype 활동 보고서](activity-report.md) 사용자가 피어 투 피어를 사용 하는 시간과 회의 세션에서 참여 한 시간을 확인할 수 있습니다.
    
- [비즈니스용 Skype 장치 사용 현황 보고서](device-usage-report.md) 비즈니스용 Skype 앱이 설치 되어 있고 메신저 및 모임에 사용 되 고 있는 Windows 기반 운영 체제와 모바일 장치를 포함 하는 장치를 볼 수 있습니다.
    
- [비즈니스용 Skype 컨퍼런스 이끌이 활동 보고서](conference-organizer-activity-report.md) 사용자가 IM, 오디오/비디오, 응용 프로그램 공유, 웹, 전화 접속/외부, 타사, Microsoft의 전화 접속/종료 등을 사용 하는 회의를 구성 하는 방법을 확인할 수 있습니다.
    
- [비즈니스용 Skype 컨퍼런스 참가자 활동 보고서](conference-participant-activity-report.md) 참가 하 고 있는 메신저, 오디오/비디오, 응용 프로그램 공유, 웹 및 전화 접속/종료 회의 회의가 얼마나 많은 지 확인할 수 있습니다.
    
- [비즈니스용 Skype 피어 투 피어 활동 보고서](peer-to-peer-activity-report.md) 사용자 들이 메신저 대화, 오디오/비디오, 응용 프로그램 공유, 파일 전송 중 어느 것을 사용 하 고 있는지 확인할 수 있습니다.
    
- [비즈니스용 SKYPE PSTN 사용 보고서](pstn-usage-report.md) 인바운드/아웃 바운드 통화에 소요 된 시간과 이러한 통화에 대 한 비용을 확인할 수 있습니다.

- [비즈니스용 Skype 사용자 차단 보고서](users-blocked-report.md) 조직의 사용자에 게 PSTN 통화를 차단 하는 것을 볼 수 있습니다.

- [비즈니스용 SKYPE PSTN 분 풀 보고서](pstn-minute-pools-report.md) 조직 내 현재 달 중 사용 된 시간 (분)을 확인할 수 있습니다.
