# <a name="troubleshoot-statistics-manager-for-skype-for-business-server"></a>비즈니스용 Skype 서버 통계 관리자 문제 해결
 
**요약:** 이 항목을 읽고 통계 관리자를 배포할 때 문제를 비즈니스용 Skype 서버.
  
이 항목에서는 응용 프로그램 이벤트 로그에 표시될 수 있는 이벤트와 이벤트를 해결하기 위해 수행할 수 있는 적절한 작업에 대해 설명하여 Statistics Manager 배포 문제를 해결하는 방법에 대해 설명합니다. 이 항목에는 다음 섹션이 포함되어 있습니다.
  
- [에이전트 이벤트](troubleshoot.md#BKMK_Agent)
    
- [수신기 이벤트](troubleshoot.md#BKMK_Listener)
    
- [웹 사이트 문제](troubleshoot.md#BKMK_Website)
    
## <a name="agent-events"></a>에이전트 이벤트
<a name="BKMK_Agent"> </a>

- **1000** — 프로세서 제한기(Job Object)를 설정하지 못했습니다. 알 수 없는 이유
    
- **1001** — 프로세스 제한이 프로세스에서 허용되지 않습니다(이미 Job 개체 내부에 있는 경우).
    
    에이전트는 자동으로 메모리 Windows 제한하기 위해 작업 개체 내에서 실행됩니다. 에이전트가 시작되지 않는 경우 이러한 이벤트 항목이 이벤트 로그에 있는 경우 서버에서 Job 개체를 인스턴스화할 수 없습니다. 이를 해결하기 위해 구성 파일에서 값을 변경하여 상한값을 제거할 수 있습니다.
    
  ```console
  C:\Program Files\Skype for Business Server StatsMan Agent\PerfAgent.exe.config
  ```

    "MaxProcessMemoryMB"를 검색하고 값을 다음과 같이 "0"으로 변경합니다.
    
  ```xml
  <setting name="MaxProcessMemoryMB" serializeAs="String"> <value>300</value> </setting>
  ```

    > [!NOTE]
    > 이 변경이 진행된 경우 에이전트는 일반적으로 100MB의 메모리를 사용하나 기본값인 \< 300MB로 강제로 제한되지는 않습니다. 이 변경이 진행되는 경우 에이전트가 호스트 컴퓨터의 많은 양의 메모리를 소비하지 않도록 메모리 사용량을 면밀하게 모니터링하는 것이 좋습니다. 
  
- **2000** — 클라이언트 초기화 실패
    
- **2001**— 원본 IP의 서비스에 대한 연결을 만들 수 없음
    
    에이전트가 수신기 컴퓨터에 연결할 수 없는 경우 다음을 확인합니다.
    
    1. 수신기 컴퓨터에서 수신기 서비스가 실행 중인지 확인합니다. 그렇지 않은 경우 Redis가 해당 서버에서 실행 중인지 확인한 다음 수신기 서비스를 다시 시작합니다.
        
        수신기 컴퓨터에서 Statistics Manager 이벤트 로그를 확인하여 통계 관리자 수신기 서비스 자체에 문제가 없는지 확인합니다.
        
    2. 텔넷과 같은 연결 도구를 사용하여 에이전트 컴퓨터에서 올바른 포트의 수신기 연결을 확인합니다.
        
        그렇지 않은 경우 수신기 컴퓨터가 연결된 네트워크 유형(개인/공용/도메인)에 대해 수신기 컴퓨터에서 들어오는 방화벽 규칙을 사용하도록 설정되어 있는지 확인합니다. 수신기 컴퓨터가 도메인에 가입되어 있지 않은 경우 네트워크가 공용으로 나열될 수 있으며, 이 경우 통계 관리자를 사용하여 설치된 방화벽 규칙은 기본적으로 적용되지 않습니다.
    
- **4000** - 수신기에서 서버 정보를 다운로드하지 못했습니다(알 수 없는 이유).
    
  - **4001** - 수신기 토폴로지에서 서버를 찾을 수 없습니다.
    
    이 오류는 서버가 수신기에 연결되지만 서버가 수신기 캐시의 토폴로지에 추가되지 않은 경우 발생합니다. 해결 옵션:
    
  - 토폴로지 가져오기 지침을 따라야 합니다. [토폴로지 가져오기 를 참조합니다.](deploy.md#BKMK_ImportTopology) 
    
  - 에이전트가 토폴로지에 나열되지 않은 서버(예: SQL AlwaysOn 클러스터의 노드)에 있는 경우 토폴로지 가져오기의 지침에 따라 에이전트를 수동으로 [추가해야 합니다.](deploy.md#BKMK_ImportTopology)
    
  - **4002** — 잘못된 수신기 암호
    
    에이전트가 사용하려는 암호화된 암호가 수신기 자체의 서비스 암호와 일치하지 않습니다. 에이전트를 제거하고 올바른 서비스 암호를 사용하여 다시 설치합니다.
    
  - **4003** — 인증서 지문 불일치
    
    설치 시 에이전트에게 제공된 인증서 지문이 수신기에서 현재 사용 중인 인증서의 지문과 일치하지 않습니다. 따라서 연결이 거부됩니다. 에이전트를 제거하고 올바른 인증서 지문을 사용하여 다시 설치합니다.
    
  - **4004** — 잘못된 응답 또는 HttpStatusCode
    
    수신기에서 예상 상태로 응답하지 않습니다. 
    
  - 연결이 프록시로 설정되어 있는 경우 프록시 구성을 확인합니다.
    
  - 수신기 컴퓨터의 StatsMan 로그에서 구성에 문제가 있는지 확인합니다.
    
  - **4005** - XML을 직렬화할 수 없습니다.
    
    수신기 서버의 서버 정보가 손상되거나 에이전트와 수신기 컴퓨터 간에 버전이 불일치할 수 있습니다. 버전이 일치하는지 확인하고 수신기 이벤트 로그에서 문제가 있는지 검사합니다.
    
## <a name="listener-events"></a>수신기 이벤트
<a name="BKMK_Listener"> </a>

- **10000** — 시작 실패 알 수 없는 이유(복구할 수 없는 경우 서비스가 중지/충돌하여 발생)
    
  - **10001** — 구성 문제
    
    일반적으로 [listener_install_location]\PerfAgentListener.exe.config 파일을 수정하여 응용 프로그램에서 읽을 수 없는 경우 발생합니다.
    
  - **10002** - HTTP 수신기 초기화 오류
    
    이 이벤트는 일반적으로 설치 중에 URL ACL이 제대로 설정되지 않은 경우 또는 SSL Cert가 유효하지 않은 경우 기록됩니다. 구성의 인증서가 유효한지 확인 있는 경우 통계 관리자 배포의 지침에 따라 수신기 [를 다시 설치합니다.](deploy.md#BKMK_Deploy)
    
  - **10003** - 재배포 실패
    
  - **10004** - 인프라 오류 캐싱
    
  - **10007** — 설정(다시 저장)
    
    수신기에서 Redis에 연락하거나 캐시에서 잘 형성된 데이터를 검색할 수 없는 경우 시작할 수 없습니다. Redis 서비스가 서버에 올바르게 시작 및 구성되었는지 확인합니다.
    
  - **10005** — 서버 정보 검색/구문 분석
    
    Redis 캐시의 토폴로지 정보가 잘못되었습니다. 먼저 Redis 및 수신기 다시 시작을 시도합니다. 오류가 지속되면 토폴로지 [가져오기에서](deploy.md#BKMK_ImportTopology) 토폴로지 데이터를 다시 을 참조합니다.
    
- **10100** — Redis PING 중지
    
  - **10101** — Redis PING 계속 중지(60초마다)
    
  - **30100** — Redis PING 중지 복원
    
    수신기에서 Redis에 연결할 수 없는 경우 기록됩니다. Redis가 시작된 후 수신기 및 Redis 간의 네트워크 연결을 사용할 수 있는지 확인합니다.
    
- **10200** — Redis Write outage
    
  - **10201** — Redis 쓰기가 계속되었습니다(60초마다).
    
  - **30100** — Redis Write outage resolved
    
    수신기에서 Redis 캐시에 쓸 수 없는 경우 기록됩니다. Redis가 시작된 후 수신기 및 Redis 간의 네트워크 연결을 사용할 수 있는지 확인합니다.
    
- **30000** - 성공적으로 시작
    
    수신기 시작 시 기록됩니다.
    
- **22000** — 통계 관리자 에이전트의 초기화가 성공했습니다.
    
- **23000** — EventLogQueryManager 초기화 성공(처음 또는 실패한 후)
    
- **24000** — serverinfo 초기화 성공(최초 또는 실패 후)
    
- **25000** - 게시에 실패한 후(또는 처음 성공한 게시물) 수신기 온라인
    
- **5000** — 데이터를 게시하기 위해 오프라인으로 수신기 시작
    
- **5001** — 수신기가 연장된 기간 동안 계속 오프라인 상태입니다.
    
    이러한 이벤트는 문제를 모니터링/경고/지우는 데 유용할 수 있습니다.
    
## <a name="website-issues"></a>웹 사이트 문제
<a name="BKMK_Website"> </a>

- Chrome의 반복적인 로그인 프롬프트 - 버전 1.1에서 해결된 버그입니다. Chrome 브라우저에 반복되는 로그인 프롬프트가 표시될 경우 최신 버전의 통계 관리자로 업그레이드해야 합니다. 실행 중인 웹 사이트의 버전을 확인:
    
  - 파일 탐색기에서 열기(기본 디렉터리)
    
  - 마우스 오른쪽 단추를 StatsManHubWebSite.dll 해당 속성을 볼 수 있습니다.
    
  - KHI 가로 보기 또는 카운터 세부 정보 보기에서 컴퓨터를 찾을 수 없는 경우 해당 컴퓨터가 사이트 및 풀의 구성원인지 확인합니다. 그렇지 않은 경우 해당 보기에 나타나지 않습니다. 토폴로지에서 서버의 사이트 및 풀을 정의하는 데 대한 자세한 내용은 토폴로지 [가져오기 를 참조하세요.](deploy.md#BKMK_ImportTopology)
    
  - 제품 버전이 설명 세부 정보에 표시됩니다.
