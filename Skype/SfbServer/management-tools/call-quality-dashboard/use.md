# <a name="use-call-quality-dashboard-for-skype-for-business-server"></a>비즈니스용 Skype 서버용 통화 품질 대시보드 사용

**요약:** 통화 품질 대시보드를 사용 하는 방법에 대해 알아봅니다. 통화 품질 대시보드는 비즈니스용 Skype 서버용 도구입니다.

CQD (통화 품질 대시보드)를 통해 IT 전문가는 집계 데이터를 사용 하 여 사용자 그룹에 대 한 통계를 비교 하 여 추세와 패턴을 식별 함으로써 미디어 품질 문제를 확인할 수 있습니다. CQD는 개별 통화 문제를 해결 하는 데 중점을 두고 있지 않지만 여러 사용자에 게 적용 되는 문제 및 해결 방법에 대해서는 다루지 않습니다.

## <a name="call-quality-dashboard-user-guide"></a>통화 품질 대시보드 사용자 안내서

CQD는 체감 품질 (환경 품질) 데이터를 기준으로 신속 하 게 보고서를 만들고 구성 하는 웹 포털입니다. CQD는 체감 품질 메트릭 데이터베이스의 데이터를 집계 하기 위해 SSAS 큐브를 배포 하 고 관리자가 보고서를 만들고 수정 하거나 조사를 실시간으로 수행할 수 있도록 합니다. Excel을 사용 하 여 큐브에 직접 연결할 수 있지만 포털은 체감 품질 데이터를 포함 하는 여러 워크플로에 최적화 되어 있습니다. 데이터에는 다음이 포함 됩니다.

- 빠른 액세스를 위해 캐시 된 보고서 데이터
- 정보 공유 및 게시에 대 한 보고서 페이지의 상세 링크
- 효율적인 보고서 편집 및 만들기 및 보고서 설명 용 편집 가능한 메타 데이터

또한 CQD는 사용자가 사용자 지정 대시보드에서 사용할 큐브 데이터에 프로그래밍 방식으로 액세스할 수 있도록 하는 웹 Api를 제공 합니다.

### <a name="feature-overview"></a>기능 개요

통화 품질 대시보드를 방문 하면 다음 화면이 표시 됩니다.

![CQD 사용](../../media/1e061858-db6f-452b-9ae4-eab507220371.png)

1. "요약 창"은 "보고서 집합"에 대 한 컨텍스트를 찾을 수 있는 위치입니다.
2. 요약 PaneReport에서 "편집"을 클릭 하 여 수준 속성 (Y 축 높이 포함)을 설정 합니다.
3. 이동 경로를 통해 보고서 집합 계층 구조 내의 현재 위치를 확인할 수 있습니다.
4. 하위 보고서가 있는 보고서가 파란색 링크로 표시 됩니다. 하위 보고서를 드릴 다운 하는 링크를 클릭 합니다.

막대형 차트 및 추세선 위로 마우스를 이동 하 여 자세한 값을 표시 합니다. 포커스가 있는 보고서에는 "편집", "클론", "Delete", "다운로드" 등의 작업 메뉴가 표시 됩니다.

### <a name="default-reports"></a>기본 보고서

통화 품질 대시보드 포털에 처음 액세스 하는 경우 기본 보고서 집합이 자동으로 만들어집니다. 이러한 보고서는 종종 시스템 보고서 라고도 합니다. 이러한 보고서를 자유롭게 수정 또는 삭제 하거나 새 형제 및 하위 보고서를 만들어 확장할 수 있습니다.

최상위 수준에서 "오디오 스트림 월 단위 추세" 보고서에는 모든 오디오 스트림에 대 한 월 추세가 표시 됩니다. 가로 막대형 차트의 막대 위로 마우스를 이동 하면 가로 막대형 차트로 표시 되는 데이터에 대 한 자세한 보기를 표시할 수 있습니다. 월 단위 추세 보고서의 제목을 클릭 하 여 관리 및 비관리 호출 간에 보고서가 분할 되는 "관리 되는 vs 관리 되지 않는 오디오 스트림" 보고서로 이동 합니다. 관리 통화는 유선 연결을 통해 회사 방화벽 내에서 생성 된 통화입니다. 관리 되지 않는 호출에는 회사 방화벽 외부에서 생성 된 호출과 Wi-fi를 통해 생성 되는 모든 통화가 포함 됩니다.

다른 최상위 수준의 보고서를 "사용자가 보고 한 통화 품질 평가 등급 히스토그램" 이라고 합니다. 통화 품질 등급은 통화의 품질을 나타내기 위해 통화 종료 시 비즈니스용 Skype 사용자가 제공 하는 번호입니다. 등급 번호는 1부터 5 까지의 값 이며, 1은 가장 저렴 하 고 5입니다. 히스토그램은 한 달에 표시 된 등급이 있는 오디오 통화 수를 보여 줍니다.

보고서의 제목을 클릭 하 여 데이터에 대 한 더 많은 필터가 포함 된 보고서로 이동 합니다. 시스템 보고서에서 각 하위 보고서는 해당 상위 보고서에서 사용할 수 있는 데이터의 하위 집합을 표시 합니다. 문제 해결 모델은 간단 합니다. 문제를 제안 하는 데이터 나 추세가 관련 된 하위 보고서를 조사 하 여 문제 공간을 서서히 좁히는 것입니다. 하위 보고서를 만드는 기능을 사용 하면 특정 데이터 추세의 원인에 대 한 자신의 추측을 조사할 수 있습니다.

### <a name="create-and-edit-reports"></a>보고서 만들기 및 편집

보고서의 동작 메뉴에서 "편집"을 클릭 하 여 보고서 편집기를 표시 합니다. 각 보고서는 큐브로 쿼리를 통해 지원 됩니다. 보고서는 해당 쿼리에서 반환 된 데이터를 시각화 한 것입니다. 보고서 편집기를 사용 하 여 이러한 쿼리 및 보고서의 표시 옵션을 편집할 수 있습니다. 보고서 편집기를 열면 다음이 표시 됩니다.

![CQD 사용](../../media/e8969625-e6f9-4d67-873f-93e78dd12b35.png)

1. 왼쪽 창에서 치수, 측정값 및 필터가 선택 됩니다. 기존 값 중 하나 위에 커서를 놓으면 값을 제거할 수 있는 "x" 단추가 표시 됩니다. 제목 옆에 있는 "더하기 (+)" 단추를 클릭 하 여 새 차원, 측정값 또는 필터를 추가할 수 있는 대화 상자를 엽니다.
2. 차트 사용자 지정에 대 한 옵션이 맨 위에 표시 됩니다.
3. 보고서의 미리 보기는 보고서 편집기에서 사용할 수 있습니다.
4. 아래쪽에 있는 입력란을 사용 하 여 자세한 보고서 설명을 만들 수 있습니다.

### <a name="sparklines-in-tables"></a>테이블의 스파크 라인

시작 날짜를 차원으로 추가 하면 데이터는 테이블 양식에서 추세와 같이 렌더링 되 고 표 셀 안에 표시 될 수 있는 가로 막대형 차트와 스파크 라인 막대형 차트와 스파크 라인으로 마우스 포인터를 이동 하 여 개별 월의 값을 표시 합니다.

![CQD 사용](../../media/fe6b18d7-b8cf-472a-9c93-0f7703f5a700.png)

막대형 차트와 스파크 라인을 표시 하려면 보고서 편집기의 맨 위에 있는 "스파크 라인 표시" 확인란을 선택 해야 합니다. 이렇게 하면 추세 옵션이 선택 되 고 월을 클릭 하 고 위쪽 및 아래 화살표 키를 사용 하 여 시작 날짜 1을 기준으로 할 수 있는 마지막 차원으로 이동 합니다.

### <a name="settings"></a>설정

설정 메뉴에는 시스템 상태, 페이지 정보 등의 유용한 페이지 링크가 포함 되어 있으며 대시보드의 오른쪽 위 모서리에 있습니다.

![CQD 사용](../../media/0e9ae123-e231-4fea-94e1-5788e8f3e1d3.png)

설명 및 타임 스탬프 표시 여부는 개별 사용자에 게 적용 되며,이 설정은 개인의 대시보드 버전에만 영향을 미치며, 보고서 집합 또는 다른 사용자에 게 표시 되는 내용은 수정 하지 않습니다. 캐시를 지우면 모든 쿼리가 큐브에 있는 데이터를 다시 로드 하는 반면, 기본값을 복원 하면 사용자가 만들었거나 수정한 모든 보고서가 삭제 되 고 시스템 보고서 집합, 즉 사용자가 처음 로그인 할 때 표시 되는 항목이 만들어집니다.

사용자 대시보드 링크는 사용자가 CQD의 다른 사용자를 보고 해당 보고서를 찾아볼 수 있는 페이지를 보여 줍니다. 보고서 집합을 공유 하려면 URL 표시줄에 링크를 복사 하 고 다른 CQD 사용자와 공유 합니다. 이 링크는 다른 사용자가 사용자의 이름 아래에 있는 사용자 대시보드 링크 페이지에 표시 되는 것과 동일한 링크입니다.

## <a name="cqd-streams"></a>CQD 스트림

CQD 스트림은 정상, 나쁨 또는 분류 되지 않은 것으로 간주 됩니다. CQM 1.5에서 이제 다음 CQD 정의를 사용 합니다.

- 불량 스트림은 임계값을 초과 하는 불량 통화 메트릭의 조합입니다.
- 한 번의 스트림이 불량 한 경우에는 통화 중 두 스트림이 불량으로 플래그가 지정 됩니다. 회의에서 각 참가자는 고유한 통화로 계산 되며 다른 사용자와 독립적으로 보고 됩니다.
- 분류 되지 않은 스트림은 품질 메트릭 없는 스트림 (즉, 가상 트랜잭션이나 짧은 호출)입니다.
- 유효한 스트림 = 비 모바일 클라이언트
- 분류자는 수정할 수 없습니다.

**불량 통화 정의/분류자**

|미터|임계값|
|:-----|:-----|
|DegradationAvg |1.0 보다 큼 (-1 네트워크 SPECIALIST) |
|왕복이 |500 보다 큼 |
|PacketLossRate |0.1 보다 큼 (10%) |
|JitterInterArrival |30 개 이상 |
|RatioConcealedSamplesAvg |0.07 보다 큼 |
|||

JPDR definition = 낮은 통화 정의는 RatioConcealedSamplesAvg를 뺀 값

## <a name="where-is-callercallee"></a>호출자/호출 수신자는 어디에 있나요?

CQD는 호출자/호출 수신자 필드를 사용 하지 않고, 호출자와 호출 수신자 사이에 단계가 있으므로 "First" 및 "Second"를 대신 사용 합니다.

 **첫 번째** 서버가 스트림에 관련 되어 있으면 항상 서버 끝점 (예: AV MCU 또는 중재 서버)이 됩니다.

 **두 번째** 는 서버 서버 스트림이 아닌 경우 항상 클라이언트 끝점입니다.

**첫 번째 및 두 번째 분류의 예**

|끝점 1 UAType|끝점 2 UUAType|기본|두번째|
|:-----|:-----|:-----|:-----|
|2 (AVMCU) |4 (비즈니스용 Skype) |끝점 1 |끝점 2 |
|2 (AVMCU) |1 (mMediationServer) |끝점 2 |끝점 1 |
|4 (비즈니스용 Skype) |4 (비즈니스용 Skype) |MediaLine의 발신자 |MMediaLine의 호출 수신자 |
|||||

두 종점이 모두 같은 형식인 경우 CQD는 호출자를 먼저 입력 하 고 호출 수신자를 두 번째로 만듭니다. 끝점 이름에 대 한 자세한 내용은 [이 블로그](https://blogs.technet.com/b/jenstr/archive/2015/05/22/call-quality-dashboard-tips-and-tricks.aspx)를 참조 하세요.

## <a name="accounting-for-vpn"></a>VPN에 대 한 계정

Vpn 솔루션이 VPN 플래그를 정확 하 게 설정 하는 것으로 알려진 경우 모두 설정 됩니다. 그렇지 않으면 다음 방법 중 하나를 사용 합니다.

- VPN (기본 설정) 이라는 네트워크 유형을 만든 다음이 새로운 VPN NetworkType VPN 서브넷을 연결 합니다.
- VPN 이라는 건물을 만든 다음이 건물과 VPN 서브넷을 연결 합니다.

## <a name="query-fundamentals"></a>쿼리 기본 사항

올바른 형식의 쿼리는 다음 세 가지 매개 변수를 모두 포함 합니다.

- 측정할
- 차원
- Filter

제대로 구성 된 쿼리의 예로는 "건물에 적합 하지 않은 스트림 [측정] [측정값]을 [x]에 대 한 서브넷 [차원]에 따라 표시 됩니다."

## <a name="what-does-union-do"></a>통합은 어떤 작업을 수행 하나요?

Union을 사용 하면 AND 연산자를 사용 하 여 조건을 필터링 할 수 있습니다. OR 연산 같은 결과를 얻기 위해 여러 필터 조건을 함께 결합할 수 있는 시나리오가 있습니다.

예: 빌딩에서 모든 스트림을 가져오려면 UNION에서 병합 된 데이터 집합의 고유한 뷰를 제공 합니다. 통합을 사용 하려면 공용 텍스트를 통합 하려는 두 필터 조건의 UNION 필드에 삽입 합니다.

## <a name="operational-processes"></a>운영 프로세스

관리 되는 스트림을 먼저 검토 하 고 재구성 합니다. 이 영역의 품질은 컨트롤 내에서 100% 여야 하며, 따라서 재구성 하는 것이 가장 쉽습니다.

### <a name="managed-streams"></a>관리 스트림

관리 되는 스트림은 다음 순서로 검토 하 고 재구성 합니다.

1. 서버-서버
2. 서버-유선-내부
3. 유선-유선-내부

### <a name="unmanaged-streams"></a>관리 되지 않는 스트림

다음 순서 대로 관리 되지 않는 스트림을 검토 하 고 재구성 합니다.

1. 서버-Wifi-인사이드
2. 서버-유선 외부
3. 서버-Wifi-외부
4. 유선 외부-직접 연결
5. 유선 외부 릴레이
6. 기타 관리 되지 않는
