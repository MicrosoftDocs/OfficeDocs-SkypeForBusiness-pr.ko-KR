# <a name="use-call-quality-dashboard-for-skype-for-business-server"></a>비즈니스용 Skype 서버용 통화 품질 대시보드 사용
 
**요약:** 통화 품질 대시보드를 사용 하는 방법에 대해 알아봅니다. 통화 품질 대시보드는 비즈니스용 Skype 서버용 도구입니다.
  
CQD IT 전문가는 집계 데이터를 사용 하 여 미디어 품질 문제를 발생 하는 환경에서 포커스 영역을 식별할 수 있습니다. It 전문가는 여러 사용자 그룹의 통계를 비교 하 고 경향과 패턴을 확인할 수 있습니다. 개별 전화 문제를 해결 하는 데 중점을 두고 있지는 않지만 지정 된 환경의 여러 사용자에 게 적용 되는 문제 및 솔루션을 식별 합니다.
  
## <a name="call-quality-dashboard-user-guide"></a>통화 품질 대시보드 사용자 안내서

CQD (통화 품질 대시보드)는 체감 품질 (환경 품질) 데이터를 기준으로 신속 하 게 보고서를 만들고 구성 하는 웹 포털입니다. CQD는 체감 품질 메트릭 데이터베이스의 데이터를 집계 하기 위해 SSAS 큐브를 배포 합니다. 이렇게 하면 사용자가 보고서를 만들고 수정 하 고 실시간으로 조사를 수행할 수 있습니다. Excel을 사용 하 여 큐브에 직접 연결할 수 있지만 포털은 체감 품질 데이터를 포함 하는 여러 워크플로에 최적화 되어 있습니다. 이 데이터에는 빠른 액세스에 대 한 보고서 데이터 캐싱, 정보 공유 및 게시에 대 한 보고서 페이지의 상세 링크, 능률적인 보고서 편집 및 만들기, 보고서 설명 용 편집 가능한 메타 데이터 등이 포함 됩니다. 또한 CQD는 사용자가 사용자 지정 대시보드에서 사용할 큐브 데이터에 프로그래밍 방식으로 액세스할 수 있도록 하는 웹 Api를 제공 합니다. 
  
### <a name="feature-overview"></a>기능 개요

사용자가 통화 품질 대시보드를 방문 하면 s/그에 따라 다음과 같이 표시 됩니다.
  
![CQD 사용](../../media/1e061858-db6f-452b-9ae4-eab507220371.png)
  
1. "요약 창"은 "보고서 집합"에 대 한 컨텍스트를 찾을 수 있는 위치입니다. 
    
2. 요약 창에서 "편집"을 클릭 하 여 보고서 집합 수준 속성 (Y 축 높이 포함)을 설정할 수 있습니다.
    
3. 이동 경로는 사용자가 보고서 집합 계층 구조 내에서 현재 위치를 식별 하는 데 도움이 됩니다. 
    
4. 하위 보고서가 있는 보고서는 파란색 링크로 표시 됩니다. 링크를 클릭 하면 하위 보고서로 드릴 다운 합니다. 
    
가로 막대형 차트와 추세선 위로 마우스를 이동 하면 자세한 값이 표시 됩니다. 포커스가 있는 보고서에는 "편집", "클론", "삭제", "다운로드" 등의 작업 메뉴가 표시 됩니다. 
  
### <a name="default-reports"></a>기본 보고서

사용자가 먼저 통화 품질 대시보드 포털에 액세스 하면 기본 보고서 집합이 자동으로 만들어집니다. 이러한 보고서는 종종 시스템 보고서 라고도 합니다. 사용자는이 보고서를 자유롭게 수정 하거나 삭제할 수 있습니다. 일반적으로 새 형제 및 하위 보고서를 만들어 사용자를 확장 합니다. 
  
최상위 수준에서 "오디오 스트림 월 단위 추세" 보고서에는 모든 오디오 스트림에 대 한 월 추세가 표시 됩니다. 가로 막대형 차트의 막대 위로 마우스를 이동 하면 가로 막대형 차트로 표시 되는 데이터에 대 한 자세한 보기가 표시 됩니다. 오디오 스트림 월 단위 추세 보고서의 제목을 클릭 하면 관리 및 비관리 호출 간에 보고서가 분할 되는 "관리 되는 vs 관리 되지 않는 오디오 스트림" 보고서로 이동 합니다. 관리 통화는 유선 연결을 통해 회사 방화벽 내에서 생성 된 통화입니다. 관리 되지 않는 호출에는 회사 방화벽 외부에서 생성 된 호출과 Wi-fi를 통해 생성 되는 모든 통화가 포함 됩니다.
  
다른 최상위 수준 보고서는 "사용자가 보고 한 통화 품질 평가 등급 히스토그램" 이라고 합니다. 통화 품질 등급은 통화의 품질을 나타내기 위해 통화 종료 시 비즈니스용 Skype 사용자가 제공 하는 번호입니다. 평점 번호는 1부터 5 까지의 값 이며 1이 가장이 고 최고 5 일입니다. 히스토그램은 한 달에 표시 된 등급이 있는 오디오 통화 수를 보여 줍니다. 
  
일반적으로 보고서의 제목을 클릭 하면 데이터에 대 한 추가 필터를 사용 하 여 보고서로 이동 합니다. 시스템 보고서에서 각 하위 보고서는 해당 상위 보고서에서 사용할 수 있는 데이터의 하위 집합을 표시 합니다. 이는 문제 해결을 위한 개념적인 간단한 모델을 제공 합니다. 문제가 있는 데이터 또는 추세가 제한 되는 하위 보고서를 조사 하 여 문제 공간 범위를 좁힐 수 있습니다. 새 하위 보고서를 만들 수 있는 기능을 통해 사용자는 특정 데이터 추세의 provenance에 대 한 가설이으로 자신의 고유 하 게 조사 하 게 됩니다.
  
### <a name="creating-and-editing-reports"></a>보고서 만들기 및 편집

보고서의 작업 메뉴에서 "편집"을 클릭 하면 사용자에 게 보고서 편집기가 표시 됩니다. 각 보고서는 큐브로 쿼리를 통해 지원 됩니다. 보고서는 해당 쿼리에서 반환 된 데이터를 시각화 한 것입니다. 보고서 편집기는 이러한 쿼리를 편집 하는 사용자 인터페이스 및 보고서의 표시 옵션입니다. 사용자가 보고서 편집기를 열면 s/그에 게 표시 되는 항목은 다음과 같습니다.
  
![CQD 사용](../../media/e8969625-e6f9-4d67-873f-93e78dd12b35.png)
  
1. 왼쪽 창에서 치수, 측정값 및 필터가 선택 됩니다. 기존 값 중 하나를 가리키면 값이 제거 될 수 있는 "x" 단추가 표시 됩니다. 제목 옆에 있는 "더하기" 단추를 클릭 하면 새 차원, 측정값 또는 필터를 추가 하는 대화 상자가 열립니다. 
    
2. 차트 사용자 지정에 대 한 옵션이 맨 위에 표시 됩니다.
    
3. 보고서의 미리 보기는 보고서 편집기에서 사용할 수 있습니다. 
    
4. 아래쪽에 있는 입력란을 사용 하 여 자세한 보고서 설명을 만들 수 있습니다. 
    
### <a name="sparklines-in-tables"></a>테이블의 스파크 라인

시작 날짜를 차원으로 추가 하면 데이터는 테이블 양식에서 추세와 같이 렌더링 되 고 표 셀 안에 표시 될 수 있는 가로 막대형 차트와 스파크 라인 가로 막대형 차트 위로 마우스 포인터를 이동 하면 스파크 라인에 개별 월의 값이 표시 됩니다. 
  
![CQD 사용](../../media/fe6b18d7-b8cf-472a-9c93-0f7703f5a700.png)
  
막대형 차트와 스파크 라인을 표시 하려면 보고서 편집기의 맨 위에 있는 "스파크 라인 표시" 확인란을 선택 해야 합니다. 이 작업을 수행 하면 추세 옵션이 선택 되 고 월을 클릭 하면 위쪽 및 아래 방향 화살표를 사용 하 여 시작 날짜를 기준으로 할 수 있는 마지막 차원으로 이동 합니다. 
  
### <a name="settings"></a>설정

설정 메뉴에는 대시보드의 오른쪽 위 모서리에 있는 시스템 상태 및 페이지에 대 한 유용한 페이지로 연결 되는 링크가 있습니다.
  
![CQD 사용](../../media/0e9ae123-e231-4fea-94e1-5788e8f3e1d3.png)
  
설명 및 타임 스탬프 표시 여부는 개별 사용자에 게 적용 되는 것 이며,이 설정은 보고서 집합을 수정 하거나 다른 사용자에 게 표시 되지 않는 개별 대시보드의 버전에만 영향을 줍니다. 캐시를 지우면 모든 쿼리가 큐브에 있는 데이터를 다시 로드 하는 반면, 기본값을 복원 하면 사용자가 만들었거나 수정한 모든 보고서가 삭제 되 고 시스템 보고서 집합 (사용자에 게 처음 로그인 할 때 표시 되는 항목)이 만들어집니다.
  
사용자 대시보드 링크는 사용자가 CQD의 다른 사용자를 보고 해당 보고서를 찾아볼 수 있는 페이지를 보여 줍니다. 보고서 집합을 공유 하는 경우 URL 표시줄에 링크를 복사 하 여 다른 CQD 사용자와 공유 하면 됩니다. 이 링크는 다른 사용자가 사용자 이름 아래의 사용자 대시보드 링크 페이지에 표시 되는 것과 동일 합니다.

## <a name="cqd-streams"></a>CQD 스트림

CQD 스트림은 정상, 나쁨 또는 분류 되지 않습니다. CQM 1.5에서 이제 다음 CQD 정의를 사용 합니다. 
  
- 불량 스트림은 임계값을 초과 하는 불량 통화 메트릭의 조합입니다.
    
- 한 번의 스트림이 불량 한 경우에는 통화 중 두 스트림이 불량으로 플래그가 지정 됩니다. 회의에서 각 참가자는 고유한 통화로 계산 되며 다른 사용자와 독립적으로 보고 됩니다.
    
- 분류 되지 않은 스트림은 품질 메트릭이 없는 스트림 (즉, 가상 트랜잭션, 짧은 호출)입니다.
    
- 유효한 스트림 = 비 모바일 클라이언트
    
- 분류자는 수정할 수 없습니다.
    
**불량 통화 정의/분류자**

|**미터**|**임계값**|
|:-----|:-----|
|DDegradationAvg  <br/> |1.0 보다 큼 (-1 네트워크 SPECIALIST)  <br/> |
|왕복이  <br/> |500 보다 큼  <br/> |
|PacketLossRate  <br/> |1 보다 큼 (10%)  <br/> |
|JitterInterArrival  <br/> |30 개 이상  <br/> |
|RRatioConcealedSamplesAvg  <br/> |07 개 이상  <br/> |
   
JPDR definition = 낮은 통화 정의는 RatioConcealedSamplesAvg를 뺀 값 
  
## <a name="where-is-callercallee"></a>호출자/호출 수신자는 어디에 있나요?

CQD는 발신자/호출 수신자 필드를 사용 하지 않습니다. 이는 호출자와 호출 수신자 사이에 단계가 있으므로 "First" 및 "Second" 이름이 바뀌었습니다.
  
 **첫 번째** 항상 서버 종점 (예: AV MCU)이 됩니다. 서버가 스트림에 관련 된 경우) 조정 서버.
  
 **두 번째** 는 서버 서버 스트림이 아닌 경우 항상 클라이언트 끝점입니다.
  
**첫 번째 및 두 번째 분류의 예**

|**끝점 1 UAType**|**끝점 2 UUAType**|**기본**|**두번째**|
|:-----|:-----|:-----|:-----|
|2 (AVMCU)  <br/> |4 (비즈니스용 Skype)  <br/> |끝점 1  <br/> |끝점 2  <br/> |
|2 (AVMCU)  <br/> |1 (mMediationServer)  <br/> |끝점 2  <br/> |끝점 1  <br/> |
|4 (비즈니스용 Skype)  <br/> |4 (비즈니스용 Skype)  <br/> |MediaLine의 발신자  <br/> |MMediaLine의 호출 수신자  <br/> |
   
두 종점이 같은 형식인 경우 CQD는 호출자를 먼저 입력 하 고 호출 수신자를 초로 만듭니다. 자세한 내용은 [이 블로그](https://blogs.technet.com/b/jenstr/archive/2015/05/22/call-quality-dashboard-tips-and-tricks.aspx) 를 참조 하세요.
  
## <a name="accounting-for-vpn"></a>VPN에 대 한 계정

Vpn 솔루션이 VPN 플래그를 정확 하 게 설정 하는 것으로 알려진 경우 모두 설정 됩니다. 그렇지 않은 경우에는 다음 방법 중 하나를 사용 합니다.
  
- VPN (기본 설정) 이라는 네트워크 유형을 만든 다음이 새로운 VPN NetworkType VPN 서브넷을 연결 합니다.
    
- VPN 이라는 건물을 만든 다음이 건물과 VPN 서브넷을 연결 합니다.

## <a name="what-does-union-do"></a>통합은 어떤 작업을 수행 하나요?

Union은 AND 연산자를 사용 하 여 조건을 필터링 할 수 있습니다. 여러 필터 조건을 결합 하 여 결과를 달성 해야 하는 시나리오는 다음과 같습니다.
  
예: 빌드 통합에서 모든 스트림을 가져와야 하는 경우 병합 된 데이터 집합의 고유한 뷰를 제공 합니다. 통합을 사용 하려면 공용 텍스트를 통합 하려는 두 필터 조건의 UNION 필드에 삽입 합니다.

## <a name="operational-processes"></a>운영 프로세스

먼저 관리 되는 스트림을 검토 하 고 수정 합니다. 이 영역의 품질은 컨트롤 내에서 100% 여야 하며, 따라서 재구성 하는 것이 가장 쉽습니다. 
  
### <a name="managed-streams"></a>관리 스트림

관리 되는 스트림은 다음 순서로 검토 하 고 재구성 합니다. 
  
1. 서버-서버 
    
2.  서버-유선-내부
    
3. 유선-유선-내부 
    
### <a name="unmanaged-streams"></a>관리 되지 않는 스트림

다음 순서 대로 관리 되지 않는 스트림을 검토 하 고 재구성 합니다. 
  
1. 서버-Wifi-인사이드
    
2. 서버-유선 외부
    
3. 서버-Wifi-외부
    
4. 유선 외부-직접 연결
    
5. 유선 외부 릴레이
    
6. 기타 관리 되지 않는
