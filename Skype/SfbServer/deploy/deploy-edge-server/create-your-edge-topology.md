# <a name="create-your-edge-topology-for-skype-for-business-server"></a>비즈니스용 Skype 서버에 대 한 Edge 토폴로지 만들기
 
**요약:** 비즈니스용 Skype 서버에서 Edge 서버 토폴로지를 빌드, 게시 및 내보내기 하는 방법에 대해 알아봅니다.
  
토폴로지 작성기는 비즈니스용 Skype 서버의 모든 토폴로지 구성 요소에 사용 되는 것 처럼 Edge 서버 토폴로지를 작성 하는 데 필요한 도구입니다. 아래 단계를 수행 하기 전에 적어도 하나 이상의 프런트 엔드 풀 또는 Standard Edition server를 설정 해야 합니다.
  
이 문서의 다음 항목을 다룹니다.
  
- Edge 서버 토폴로지 빌드
    
- Edge 서버 토폴로지 게시
    
- Edge 서버 토폴로지 내보내기
    
  > [!NOTE]
  > 아래 단계를 수행 하려면 다음 도메인 그룹의 구성원 인 사용자로 아래에 언급 된 도메인 서버에 로그인 해야 합니다. 
  
- RTCUniversalServerAdmins
    
- 도메인 관리자
    
## <a name="build-your-edge-server-topology"></a>Edge 서버 토폴로지 빌드

첫 번째 배포 단계는 다음 세 가지 옵션 중 하나로 구성 되는 비즈니스용 Skype Server Edge 서버 토폴로지를 작성 하는 것입니다.
  
- 단일에 지 서버
    
- DNS 부하 분산 Edge 풀 (하나 이상의 서버)
    
- 하드웨어 부하 분산 된 Edge 풀 (하나 이상의 서버)
    
필요한 내용이 확실 하지 않은 경우 다음 단계를 시작 하기 전에 계획 설명서를 참조 하는 것이 좋습니다. 그렇지 않으면 시작 해 보겠습니다.
  
### <a name="defining-the-topology-for-a-single-edge-server"></a>단일 Edge 서버에 대 한 토폴로지 정의

1. 비즈니스용 Skype server Standard Edition server 또는 비즈니스용 Skype 서버 프런트 엔드 풀에 로그인 합니다.
    
2. 그런 다음 비즈니스용 **Skype 서버 토폴로지 작성기**를 엽니다.
    
3. 콘솔 트리에서 Edge 서버를 배포할 사이트를 확장 합니다.
    
4. **Edge 풀**을 마우스 오른쪽 단추로 클릭 한 다음 **새 edge 풀**을 클릭 합니다.
    
5. **새 Edge 풀 정의** 화면에서 **다음** 을 클릭 합니다.
    
6. **Edge 풀 FQDN 정의** 화면에서 edge 서버에서 사용할 내부 FQDN (정규화 된 도메인 이름)을 입력 하 고, **단일 컴퓨터 풀**을 선택 하 고 완료 되 면 **다음** 을 클릭 합니다.
    
7. **기능 선택** 화면에서 다음을 선택할 수 있습니다.
    
   - SIP 액세스 서비스, 비즈니스용 Skype 서버 웹 회의 서비스, 그리고 A/V Edge 서비스에 대해 동일한 FQDN과 IP 주소를 사용 하려고 할 수 있습니다. 그렇다면 **단일 FQDN 및 IP 주소 사용 확인란** 을 선택 하 고 아래 9 단계에 유의 하세요.
    
   - 페더레이션을 사용 하도록 계획 하는 경우 **이 Edge 풀에 페더레이션 사용 (포트 5061)** 확인란을 선택 합니다.
    
8. **다음**을 클릭 하 고 나면 **IP 옵션** 화면에서 직접 찾을 수 있습니다. 옵션은 다음과 같습니다.
    
   - 내부 인터페이스에 IPv4 사용
    
   - 내부 인터페이스에서 IPv6 사용
    
   - 외부 인터페이스에서 IPv4 사용
    
   - 외부 인터페이스에서 IPv6 사용
    
   IPv4 또는 IPv6 주소를 사용 하 고 있는지에 관계 없이 Edge 서버에서 해당 주소를 내부적으로 또는 외부적으로 적용 하는 것이 좋습니다 (10 단계에이 점에 유의 하 여 유지 해야 함). 또한 외부 IP 주소에 대 한 NAT (network address translation) 주소를 사용 하도록 Edge 서버 또는 Edge 풀을 구성할 수 있습니다. 이 **Edge 풀의 외부 IP 주소를 NAT에서 번역 됨** 확인란을 선택 하 여이 작업을 수행할 수 있습니다. 준비 되 면 **다음** 을 클릭 합니다.
    
9. 외부 Fqdn 화면에서 선택한 항목은 위의 7 단계에서 선택한 항목에 따라 달라 집니다.
    
   - **단일 FQDN 및 IP 주소 사용** 확인란을 선택한 경우에는 **SIP 액세스** 상자에 단일 외부 FQDN을 입력 해야 합니다. 그런 다음 모든 edge 서비스에 대해 서로 다른 포트 번호를 입력 해야 모든 사용자가 독립적으로 연결 될 수 있습니다. **SIP 액세스** 에 지 서비스, **웹 회의** 에 지 서비스에 대 한 444, **A/V** edge 서비스에 대 한 443을 5061 하는 것이 좋습니다. 완료 되 면 **다음** 을 클릭 합니다.
    
   - **단일 FQDN 및 IP 주소 사용** 확인란을 선택 하지 않은 경우에는 **SIP 액세스** 에 지 서비스, **웹 회의** 에 지 서비스, 그리고 **a/V** Edge 서비스에 대 한 세 개의 외부 fqdn을 입력 해야 합니다. 완료 되 면 **다음** 을 클릭 합니다.
    
10. 이제 **내부 IP 주소 정의** 화면이 표시 됩니다. 여기서는 8 단계에서 다시 선택한 항목에 따라 **내부 IPv4 주소** 및 **내부 IPv6 주소** 텍스트 상자에 Edge 서버의 IP 주소를 입력 합니다. 준비 되 면 **다음** 을 클릭 합니다.
    
11. **외부 IP 주소 정의** 화면에는 이전 선택 사항에 따라 몇 가지 옵션이 있습니다.
    
    - 모든 서비스에 단일 FQDN을 사용 하 고 있을 수 있습니다. 이 경우에는 사용 중인 외부 IPv4 또는 IPv6 주소를 **SIP Access** 텍스트 상자에 입력 하 고 **다음**을 클릭 합니다.
    
    - 서비스에 대해 세 개의 개별 Fqdn과 IP 주소를 사용 하도록 선택 했을 수 있습니다. 그러한 경우에는 **SIP 액세스** 에 지 서비스, **웹 회의** 에 지 서비스, **A/V** Edge 서비스에 대 한 외부 IPv4 또는 IPv6 주소를 입력 하 고 **다음**을 클릭 합니다.
    
    > [!NOTE]
    > 이전에 IPv6 주소 지정을 사용 하도록 선택 하지 않은 경우이 대화 상자가 표시 되지 않습니다. 그 이유는 다음 단계로 이동 하는 것입니다. 
  
12. 8 단계에서 NAT를 다시 사용 하도록 선택한 경우에는 이제 **공용 IP 주소** 텍스트 상자로 화면을 받게 됩니다. A/V Edge 서비스에 대해 설정한 공용 IPv4 및/또는 IPv6 주소를 입력 해야 NAT에서 번역할 수 있습니다. 그런 후 **다음**을 클릭 합니다.
    
13. 다음 화면은 **다음 홉을 정의**합니다. **다음 홉 풀** 상자에서 프런트 엔드 풀 또는 독립 실행형 풀 일 수 있는 내부 풀의 이름을 선택 합니다. 해당 환경에 디렉터가 있는 경우에는 디렉터를 선택 해야 합니다. 그런 후 **다음**을 클릭 합니다.
    
14. **프런트 엔드 풀 연결** 화면에서이 Edge 서버와 연결 하려면 프런트 엔드 풀 및 스탠더드 버전 서버를 포함 하 여 하나 이상의 내부 풀을 지정 해야 합니다. 이 Edge 서버를 사용 하 여 지원 되는 외부 사용자와 통신 하는 데 사용할 내부 풀의 이름을 선택 하기만 하면 됩니다. **다음**을 클릭 합니다.
    
    > [!NOTE]
    > 여기에서 염두에 두어야 할 사항은 내부 풀이나 독립 실행형 서버가 이미 다른 비즈니스용 Skype 서버 Edge 서버를 사용 하 고 있는 경우 여러 연관을 가질 수 없다는 것입니다. 해당 상황에 해당 하는 내부 풀 또는 독립 실행형 서버를 선택 하는 경우에는 다른 Edge 서버에 대 한 경고가 표시 되며 계속할지 여부를 결정할 수 있습니다. 이 새로운 연결을 계속 하면 다른 Edge 서버와의 연결이 중지 됩니다. 
  
15. 다음 화면에서 **마침을** 클릭 합니다.
    
16. 이제이 업데이트 된 기술을 게시 하 고 [비즈니스용 Skype 서버에 edge 서버 배포](deploy-edge-servers.md) 의 지침에 따라 여기에서 edge Server에 배포할 수 있습니다.
    
### <a name="defining-the-topology-for-a-dns-load-balanced-edge-server-pool"></a>DNS 부하 분산 Edge 서버 풀에 대 한 토폴로지 정의

1. 비즈니스용 Skype server Standard Edition server 또는 비즈니스용 Skype Server 프런트 엔드 서버에 로그인 합니다.
    
2. 그런 다음 비즈니스용 **Skype 서버 토폴로지 작성기**를 엽니다.
    
3. 콘솔 트리에서 Edge 서버를 배포할 사이트를 확장 합니다.
    
4. **Edge 풀**을 마우스 오른쪽 단추로 클릭 한 다음 **새 edge 풀**을 클릭 합니다.
    
5. **새 Edge 풀 정의** 화면에서 **다음** 을 클릭 합니다.
    
6. **Edge 풀 FQDN 정의** 화면에서 edge 풀에서 사용할 내부 FQDN (정규화 된 도메인 이름)을 입력 하 고, **여러 컴퓨터 풀**을 선택 하 고 완료 되 면 **다음** 을 클릭 합니다.
    
7. **기능 선택** 화면에서 다음을 선택할 수 있습니다.
    
    - SIP 액세스 서비스, 비즈니스용 Skype 서버 웹 회의 서비스, 그리고 A/V Edge 서비스에 대해 동일한 FQDN과 IP 주소를 사용 하려고 할 수 있습니다. 그렇다면 **단일 FQDN 및 IP 주소 사용 확인란** 을 선택 하 고 아래 9 단계에 유의 하세요.
    
    - 페더레이션을 사용 하도록 계획 하는 경우 **이 Edge 풀에 페더레이션 사용 (포트 5061)** 확인란을 선택 합니다.
    
8. **다음**을 클릭 하 고 나면 **IP 옵션** 화면에서 직접 찾을 수 있습니다. 옵션은 다음과 같습니다.
    
    - 내부 인터페이스에 IPv4 사용
    
   - 내부 인터페이스에서 IPv6 사용
    
   - 외부 인터페이스에서 IPv4 사용
    
   - 외부 인터페이스에서 IPv6 사용
    
     IPv4 또는 IPv6 주소를 사용 하 고 있는지 여부와 관계 없이 Edge 서버에서 해당 주소를 내부적으로 또는 외부적으로 적용 하는 것이 좋습니다 (11 단계에서이를 염두에 두어야 함). 또한 외부 IP 주소에 대 한 NAT (network address translation) 주소를 사용 하도록 Edge 서버 또는 Edge 풀을 구성할 수 있습니다. 이 **Edge 풀의 외부 IP 주소를 NAT에서 번역 됨** 확인란을 선택 하 여이 작업을 수행할 수 있습니다. 준비 되 면 **다음** 을 클릭 합니다.
    
9. 외부 Fqdn 화면에서 선택한 항목은 위의 7 단계에서 선택한 항목에 따라 달라 집니다.
    
   - **단일 FQDN 및 IP 주소 사용** 확인란을 선택한 경우에는 **SIP 액세스** 상자에 단일 외부 FQDN을 입력 해야 합니다. 그런 다음 모든 edge 서비스에 대해 서로 다른 포트 번호를 입력 해야 모든 사용자가 독립적으로 연결 될 수 있습니다. **SIP 액세스** 에 지 서비스, **웹 회의** 에 지 서비스에 대 한 444, **A/V** edge 서비스에 대 한 443을 5061 하는 것이 좋습니다. 완료 되 면 **다음** 을 클릭 합니다.
    
   - **단일 FQDN 및 IP 주소 사용** 확인란을 선택 하지 않은 경우에는 **SIP 액세스** 에 지 서비스, **웹 회의** 에 지 서비스, 그리고 **a/V** Edge 서비스에 대 한 세 개의 외부 fqdn을 입력 해야 합니다. 완료 되 면 **다음** 을 클릭 합니다.
    
10. 이제 **이 풀의 컴퓨터 정의** 화면에 도달 했습니다. **추가** 단추를 클릭 합니다.
    
11. 이제 **내부 IP 주소 정의** 화면이 표시 됩니다. 여기서는 8 단계에서 다시 선택한 항목에 따라 **내부 IPv4 주소** 및 **내부 IPv6 주소** 텍스트 상자에 Edge 서버의 IP 주소를 입력 합니다. 준비 되 면 **다음** 을 클릭 합니다.
    
12. **외부 IP 주소 정의** 화면에는 이전 선택 사항에 따라 몇 가지 옵션이 있습니다.
    
    - 모든 서비스에 단일 FQDN을 사용 하 고 있을 수 있습니다. 이 경우에는 사용 중인 외부 IPv4 또는 IPv6 주소를 **SIP Access** 텍스트 상자에 입력 하 고 **다음**을 클릭 합니다.
    
    - 서비스에 대해 세 개의 개별 Fqdn과 IP 주소를 사용 하도록 선택 했을 수 있습니다. 그러한 경우에는 **SIP 액세스** 에 지 서비스, **웹 회의** 에 지 서비스, **A/V** Edge 서비스에 대 한 외부 IPv4 또는 IPv6 주소를 입력 하 고 **다음**을 클릭 합니다.
    
    > [!NOTE]
    > 이전에 IPv6 주소 지정을 사용 하도록 선택 하지 않은 경우이 대화 상자가 표시 되지 않습니다. 그 이유는 다음 단계로 이동 하는 것입니다. 
  
13. **마침을**클릭 합니다. 방금 만든 Edge 서버가 이제 **이 풀의 컴퓨터 정의** 대화 상자에 나열 됩니다.
    
14. **이 풀의 컴퓨터 정의** 화면에서 **추가** 단추를 다시 클릭 하 고이 풀에서 사용할 모든 Edge 서버를 추가할 때까지 11 ~ 13 단계를 반복 합니다. 이 작업이 완료 되 면 **다음**을 클릭 합니다.
    
15. 8 단계에서 NAT를 다시 사용 하도록 선택한 경우에는 이제 **공용 IP 주소** 텍스트 상자로 화면을 받게 됩니다. A/V Edge 서비스에 대해 설정한 공용 IPv4 및/또는 IPv6 주소를 입력 해야 NAT에서 번역할 수 있습니다. 그런 후 **다음**을 클릭 합니다.
    
16. 다음 화면은 **다음 홉을 정의**합니다. **다음 홉 풀** 상자에서 프런트 엔드 풀 또는 독립 실행형 풀 일 수 있는 내부 풀의 이름을 선택 합니다. 해당 환경에 디렉터가 있는 경우에는 디렉터를 선택 해야 합니다. 그런 후 **다음**을 클릭 합니다.
    
17. **프런트 엔드 풀 연결** 화면에서이 Edge 서버와 연결 하려면 프런트 엔드 풀 및 스탠더드 버전 풀을 포함 하 여 하나 이상의 내부 풀을 지정 해야 합니다. 이 Edge 서버를 사용 하 여 지원 되는 외부 사용자와 통신 하는 데 사용할 내부 풀의 이름을 선택 하기만 하면 됩니다. **다음**을 클릭 합니다.
    
    > [!NOTE]
    > 여기에서 염두에 두어야 할 사항은 내부 풀이나 독립 실행형 서버가 이미 다른 비즈니스용 Skype 서버 Edge 서버를 사용 하 고 있는 경우 여러 연관을 가질 수 없다는 것입니다. 해당 상황에 해당 하는 내부 풀 또는 독립 실행형 서버를 선택 하는 경우에는 다른 Edge 서버에 대 한 경고가 표시 되며 계속할지 여부를 결정할 수 있습니다. 이 새로운 연결을 계속 하면 다른 Edge 서버와의 연결이 중지 됩니다. 
  
18. 다음 화면에서 **마침을** 클릭 합니다.
    
19. 이제이 업데이트 된 기술을 게시 하 고 [비즈니스용 Skype 서버에 edge 서버 배포](deploy-edge-servers.md) 의 지침에 따라 여기에서 edge Server에 배포할 수 있습니다.
    
### <a name="defining-the-topology-for-a-hardware-load-balanced-edge-server-pool"></a>하드웨어 부하 분산 Edge 서버 풀에 대 한 토폴로지 정의

1. 비즈니스용 Skype server Standard Edition server 또는 비즈니스용 Skype Server 프런트 엔드 서버에 로그인 합니다.
    
2. 그런 다음 비즈니스용 **Skype 서버 토폴로지 작성기**를 엽니다.
    
3. 콘솔 트리에서 Edge 서버를 배포할 사이트를 확장 합니다.
    
4. **Edge 풀**을 마우스 오른쪽 단추로 클릭 한 다음 **새 edge 풀**을 클릭 합니다.
    
5. **새 Edge 풀 정의** 화면에서 **다음** 을 클릭 합니다.
    
6. **Edge 풀 FQDN 정의** 화면에서 edge 풀에서 사용할 내부 FQDN (정규화 된 도메인 이름)을 입력 하 고, **여러 컴퓨터 풀**을 선택 하 고 완료 되 면 **다음** 을 클릭 합니다.
    
7. **기능 선택** 화면에서 다음을 선택할 수 있습니다.
    
   - SIP 액세스 서비스, 비즈니스용 Skype 서버 웹 회의 서비스, 그리고 A/V Edge 서비스에 대해 동일한 FQDN과 IP 주소를 사용 하려고 할 수 있습니다. 그렇다면 **단일 FQDN 및 IP 주소 사용 확인란** 을 선택 하 고 아래 9 단계에 유의 하세요.
    
   - 페더레이션을 사용 하도록 계획 하는 경우 **이 Edge 풀에 페더레이션 사용 (포트 5061)** 확인란을 선택 합니다.
    
8. **다음**을 클릭 하 고 나면 **IP 옵션** 화면에서 직접 찾을 수 있습니다. 옵션은 다음과 같습니다.
    
   - 내부 인터페이스에 IPv4 사용
    
   - 내부 인터페이스에서 IPv6 사용
    
   - 외부 인터페이스에서 IPv4 사용
    
   - 외부 인터페이스에서 IPv6 사용
    
     IPv4 또는 IPv6 주소를 사용 하 고 있는지 여부와 관계 없이 Edge 서버에서 해당 주소를 내부적으로 또는 외부적으로 적용 하는 것이 좋습니다 (11 단계에서이를 염두에 두어야 함).
    
     > [!NOTE]
     > 하드웨어 부하 분산 장치를 사용 하는 경우 다른 두 토폴로지 옵션의 경우와 달리 **Edge 풀의 외부 IP 주소가 NAT에서 번역 되**는 옵션을 선택 **하지 않아야 합니다** . 이는 **지원 되지**않습니다.
  
9. 외부 Fqdn 화면에서 선택한 항목은 위의 7 단계에서 선택한 항목에 따라 달라 집니다.
    
   - **단일 FQDN 및 IP 주소 사용** 확인란을 선택한 경우에는 **SIP 액세스** 상자에 단일 외부 FQDN을 입력 해야 합니다. 그런 다음 모든 edge 서비스에 대해 서로 다른 포트 번호를 입력 해야 모든 사용자가 독립적으로 연결 될 수 있습니다. **SIP 액세스** 에 지 서비스, **웹 회의** 에 지 서비스에 대 한 444, **A/V** edge 서비스에 대 한 443을 5061 하는 것이 좋습니다. 완료 되 면 **다음** 을 클릭 합니다.
    
   - **단일 FQDN 및 IP 주소 사용** 확인란을 선택 하지 않은 경우에는 **SIP 액세스** 에 지 서비스, **웹 회의** 에 지 서비스, 그리고 **a/V** Edge 서비스에 대 한 세 개의 외부 fqdn을 입력 해야 합니다. 완료 되 면 **다음** 을 클릭 합니다.
    
10. 이제 **이 풀의 컴퓨터 정의** 화면에 도달 했습니다. **추가** 단추를 클릭 합니다.
    
11. 이제 **내부 IP 주소 정의** 화면이 표시 됩니다. 여기서는 8 단계에서 다시 선택한 항목에 따라 **내부 IPv4 주소** 및 **내부 IPv6 주소** 텍스트 상자에 Edge 서버의 IP 주소를 입력 합니다. 준비 되 면 **다음** 을 클릭 합니다.
    
12. **외부 IP 주소 정의** 화면에는 이전 선택 사항에 따라 몇 가지 옵션이 있습니다.
    
    - 모든 서비스에 단일 FQDN을 사용 하 고 있을 수 있습니다. 이 경우에는 사용 중인 외부 IPv4 또는 IPv6 주소를 **SIP Access** 텍스트 상자에 입력 하 고 **다음**을 클릭 합니다.
    
    - 서비스에 대해 세 개의 개별 Fqdn과 IP 주소를 사용 하도록 선택 했을 수 있습니다. 그러한 경우에는 **SIP 액세스** 에 지 서비스, **웹 회의** 에 지 서비스, **A/V** Edge 서비스에 대 한 외부 IPv4 또는 IPv6 주소를 입력 하 고 **다음**을 클릭 합니다.
    
    > [!NOTE]
    > 이전에 IPv6 주소 지정을 사용 하도록 선택 하지 않은 경우이 대화 상자가 표시 되지 않습니다. 그 이유는 다음 단계로 이동 하는 것입니다. 
  
13. **마침을**클릭 합니다. 방금 만든 Edge 서버가 이제 **이 풀의 컴퓨터 정의** 대화 상자에 나열 됩니다.
    
14. **이 풀의 컴퓨터 정의** 화면에서 **추가** 단추를 다시 클릭 하 고이 풀에서 사용할 모든 Edge 서버를 추가할 때까지 11 ~ 13 단계를 반복 합니다. 이 작업이 완료 되 면 **다음**을 클릭 합니다.
    
15. 다음 화면은 **다음 홉을 정의**합니다. **다음 홉 풀** 상자에서 프런트 엔드 풀 또는 독립 실행형 풀 일 수 있는 내부 풀의 이름을 선택 합니다. 해당 환경에 디렉터가 있는 경우에는 디렉터를 선택 해야 합니다. 그런 후 **다음**을 클릭 합니다.
    
16. **프런트 엔드 풀 연결** 화면에서이 Edge 서버와 연결 하려면 프런트 엔드 풀 및 스탠더드 버전 풀을 포함 하 여 하나 이상의 내부 풀을 지정 해야 합니다. 이 Edge 서버를 사용 하 여 지원 되는 외부 사용자와 통신 하는 데 사용할 내부 풀의 이름을 선택 하기만 하면 됩니다. **다음**을 클릭 합니다.
    
    > [!NOTE]
    > 여기에서 염두에 두어야 할 사항은 내부 풀이나 독립 실행형 서버가 이미 다른 비즈니스용 Skype 서버 Edge 서버를 사용 하 고 있는 경우 여러 연관을 가질 수 없다는 것입니다. 해당 상황에 해당 하는 내부 풀 또는 독립 실행형 서버를 선택 하는 경우에는 다른 Edge 서버에 대 한 경고가 표시 되며 계속할지 여부를 결정할 수 있습니다. 이 새로운 연결을 계속 하면 다른 Edge 서버와의 연결이 중지 됩니다. 
  
17. 다음 화면에서 **마침을** 클릭 합니다.
    
18. 이제이 업데이트 된 기술을 게시 하 고 [비즈니스용 Skype 서버에 edge 서버 배포](deploy-edge-servers.md) 의 지침에 따라 여기에서 edge Server에 배포할 수 있습니다.
    
## <a name="publish-your-edge-server-topology"></a>Edge 서버 토폴로지 게시

위의 단계를 완료 한 후에는이 새로운 토폴로지를 게시 하 여 비즈니스용 Skype 서버에 지 서버 또는 Edge 풀로 내보낼 수 있도록 하겠습니다. 다음 단계를 따르세요.
  
1. **토폴로지 작성기** 를 시작 합니다 (이전 절차에서 아직 시작 되지 않은 경우).
    
2. **토폴로지 작성기**의 콘솔 트리에서 **비즈니스용 skype 서버** 를 마우스 오른쪽 단추로 클릭 한 다음 **비즈니스용 skype server 토폴로지 작성기**를 클릭 합니다.
    
3. 마법사의 **시작** 페이지에서 **다음**을 클릭 합니다.
    
4. **다른 데이터베이스 만들기** 페이지에서 **다음**을 클릭 합니다.
    
5. 상태가 데이터베이스를 성공적으로 생성 했음을 나타내는 경우 다음을 수행 합니다.
    
    - 로그를 보려면 **로그 보기**를 클릭 합니다.
    
    - 마법사를 닫으려면 **마침을**클릭 합니다.
    
## <a name="export-your-edge-server-topology"></a>Edge 서버 토폴로지 내보내기

성공적으로 배포 하려면 비즈니스용 Skype 서버 배포 마법사에서 중앙 관리 저장소 데이터에 액세스 해야 합니다. 도메인 또는 포리스트의 내부 서버에는 일반적으로 간단 합니다. Edge 서버는 도메인 외부에 있으므로 일반적으로 실제 미디어에서 토폴로지 파일을 Edge 서버 위치에 수동으로 내보내야 합니다. 이 내보내기는 PowerShell을 통해 수행 됩니다.
  
1. **비즈니스용 Skype 서버 관리 셸을**시작 합니다.
    
2. **비즈니스용 Skype 서버 관리 셸에서**다음을 실행 합니다.
    
   ```
   Export-CsConfiguration -FileName <ConfigurationFilePath.zip>
   ```

3. 내보낸 파일을 외부 미디어 (예: Edge 서버의 위치에서 연결할 수 있는 USB 드라이브 또는 네트워크 공유)에 복사 합니다.
