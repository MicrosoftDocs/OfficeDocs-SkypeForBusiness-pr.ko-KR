# <a name="create-your-edge-topology-for-skype-for-business-server"></a>사용자용 에지 토폴로지 비즈니스용 Skype 서버
 
**요약:** 에지 서버 토폴로지 작성, 게시 및 내보내기 방법을 비즈니스용 Skype 서버.
  
토폴로지 작성기에서는 에지 서버 토폴로지 작성기에서 토폴로지 구성 요소에 사용되는 경우처럼 에지 서버 토폴로지 작성기에서 비즈니스용 Skype 서버. 아래 단계를 수행하기 전에 하나 이상의 프런트 엔드 풀 또는 프런트 엔드 서버를 설정해야 Standard Edition 합니다.
  
이 문서에서는 다음 항목에 대해 다듬습니다.
  
- 에지 서버 토폴로지 빌드
    
- 에지 서버 토폴로지 게시
    
- 에지 서버 토폴로지 내보내기
    
  > [!NOTE]
  > 아래 단계를 수행하려면 아래 언급된 도메인 서버에 다음 도메인 그룹의 구성원인 사용자로 로그인해야 합니다. 
  
- RTCUniversalServerAdmins
    
- Domain Admins
    
## <a name="build-your-edge-server-topology"></a>에지 서버 토폴로지 빌드

첫 번째 배포 단계는 다음 비즈니스용 Skype 서버 중 하나로 구성된 에지 서버 토폴로지의 구축입니다.
  
- 단일 에지 서버
    
- DNS 부하가 균형 조정된 에지 풀(하나 이상의 서버)
    
- 하드웨어 부하가 균형이 잡힌 에지 풀(하나 이상의 서버)
    
필요한 것이 확실하지 않은 경우 이러한 단계를 수행하기 전에 계획 설명서를 참조하는 것이 좋습니다. 그렇지 않은 경우 시작해보아야 합니다.
  
### <a name="defining-the-topology-for-a-single-edge-server"></a>단일 에지 서버에 대한 토폴로지 정의

1. 비즈니스용 Skype 서버 Standard Edition 또는 비즈니스용 Skype 서버 풀에 로그온합니다.
    
2. 그런 다음 토 **폴로지 비즈니스용 Skype 서버 를 습니다**.
    
3. 콘솔 트리에서 에지 서버를 배포할 사이트를 확장합니다.
    
4. 에지 **풀을 마우스** 오른쪽 단추로 클릭한 다음 새 에지 **풀을 클릭합니다**.
    
5. 새 에지 풀 **정의** 화면에서 **다음을 클릭합니다** .
    
6. 에지 풀 **FQDN** 정의 화면에서 에지 서버에서 사용할 내부 FQDN(정식 도메인 이름)을 입력하고 단일 컴퓨터 풀을 선택하고 완료되면 다음을 클릭합니다.
    
7. 기능 **선택 화면** 에서 다음을 선택할 수 있습니다.
    
   - SIP 액세스 서비스, 비즈니스용 Skype 서버 웹 회의 서비스 및 A/V 에지 서비스에 대해 동일한 FQDN 및 IP 주소를 사용하려는 경우도 있습니다. 그렇다면 단일 **FQDN 및 IP** 주소 사용 확인란을 선택해야 합니다(아래 9단계에 유의하세요).
    
   - 페더전을 사용하도록 설정하려면 이 에지 풀에 페더전 사용(포트 **5061) 확인란** 을 선택합니다.
    
8. 다음을 클릭하면 **IP** 옵션 화면이 표시됩니다. 옵션은 다음과 같습니다.
    
   - 내부 인터페이스에서 IPv4 사용
    
   - 내부 인터페이스에서 IPv6 사용
    
   - 외부 인터페이스에서 IPv4 사용
    
   - 외부 인터페이스에서 IPv6 사용
    
   이러한 주소는 IPv4 또는 IPv6 주소를 사용 중이든, 내부적으로 또는 외부적으로 에지 서버에 해당 주소를 적용하는지 여부에 따라 매우 설명적인 설명입니다(10단계에 대해 이 점에 유의해야 합니다). 또한 외부 IP 주소에 NAT(Network Address Translation) 주소를 사용하기 위해 에지 서버 또는 에지 풀을 구성할 수도 있습니다. 이 작업을 위해 이 에지 풀의 외부 IP 주소가 **NAT** 로 변환됩니다. 확인란을 선택하여 이 작업을 할 수 있습니다. 준비 **되면 다음** 을 클릭합니다.
    
9. 외부 FQDNs 화면에서 선택은 위의 7단계에서 선택한 대로 선택됩니다.
    
   - 단일 **FQDN 및 IP** 주소 사용 확인란을 선택한 경우 **SIP** 액세스 상자에 단일 외부 FQDN을 입력해야 합니다. 그런 다음 각 에지 서비스에 대해 서로 다른 포트 번호를 입력하여 모두 독립적으로 연결할 수 있도록 해야 합니다. **SIP** 액세스 에지 서비스의 경우 5061, 웹 회의 에지 서비스의 경우 444, **A/V** 에지 서비스의 경우 443을 권장합니다. 완료되면 **다음** 을 선택합니다.
    
   - 단일 **FQDN 및 IP** 주소 사용 확인란을 선택하지 않은 경우 **이제 SIP** 액세스 에지 서비스, 웹 회의 에지 서비스 및 **A/V** 에지 서비스에 대한 외부 FQDN 세  개를 입력해야 합니다. 완료되면 **다음** 을 선택합니다.
    
10. 이제 내부 IP 주소 정의 **화면이 표시됩니다** . 8단계에서 다시 선택한 선택에 따라 내부 **IPv4** 주소 및 내부 **IPv6** 주소 텍스트 상자에 에지 서버의 IP 주소를 입력합니다. 준비 **되면 다음** 을 클릭합니다.
    
11. 외부 IP 주소 정의 **화면** 에서 이전 선택에 따라 몇 가지 옵션이 있습니다.
    
    - 모든 서비스에 대해 단일 FQDN을 사용할 수 있습니다. 그렇다면 외부 IPv4 또는 IPv6 주소(사용 중)를 **SIP 액세스** 텍스트 상자에 입력하고 다음을 **클릭합니다**.
    
    - 서비스에 세 개의 개별 FQDN 및 IP 주소를 사용하기로 선택한 것일 수 있습니다. 이 경우 **SIP** 액세스 에지 서비스, 웹 회의 에지 서비스 및 **A/V** 에지 서비스의 외부 IPv4 또는 IPv6 주소를 입력하고 다음을 **클릭합니다**.
    
    > [!NOTE]
    > 이전에 IPv6 주소 지정을 사용하도록 설정하고 할당하지 않은 경우 이 대화 상자가 표시되지 않습니다. 이는 정상적인 일입니다. 다음 단계로 이동하기만 합니다. 
  
12. 8단계에서 NAT를 다시 사용하기로 선택한 경우 이제 공용 IP 주소 텍스트 상자가 있는 화면 **이** 표시됩니다. NAT에서 변환하려면 A/V 에지 서비스에 대해 설정한 공용 IPv4 및/또는 IPv6 주소를 입력해야 합니다. **다음** 을 클릭합니다.
    
13. 다음 화면은 **다음 홉 정의입니다**. 다음 **홉 풀** 상자에서 프런트 엔드 풀 또는 독립 실행형 풀일 수 있는 내부 풀의 이름을 선택합니다. 환경에 있는 감독이 있는 경우 감독을 선택해야 합니다. **다음** 을 클릭합니다.
    
14. 프런트 엔드  풀 연결 화면에서 이 에지 서버와 연결하려면 프런트 엔드 풀 및 Standard Edition 서버를 포함하여 하나 이상의 내부 풀을 지정해야 합니다. 이 에지 서버를 사용하여 지원되는 외부 사용자와 통신할 내부 풀의 이름을 선택하기만 합니다. **다음** 을 클릭합니다.
    
    > [!NOTE]
    > 여기서 유의해야 할 점은 내부 풀 또는 독립 실행형 서버가 이미 다른 비즈니스용 Skype 서버 에지 서버를 사용하고 있는 경우 여러 개의 연결은 사용할 수 없습니다. 해당 상황에 있는 내부 풀 또는 독립 실행형 서버를 선택하면 다른 에지 서버에 대한 경고가 표시되고 계속할지 여부를 결정할 수 있습니다. 이 새 연결을 진행하면 다른 에지 서버에 대한 연결이 중지됩니다. 
  
15. 다음 **화면** 에서 마친을 클릭합니다.
    
16. 이제 이 업데이트된 기술을 게시하고 여기에서 에지 서버 배포의 비즈니스용 Skype 서버 따라 에지 서버에 [](deploy-edge-servers.md) 배포할 수 있습니다.
    
### <a name="defining-the-topology-for-a-dns-load-balanced-edge-server-pool"></a>DNS 부하가 균형 조정된 에지 서버 풀에 대한 토폴로지 정의

1. 비즈니스용 Skype 서버 Standard Edition 또는 비즈니스용 Skype 서버 프런트 엔드 서버에 로그온합니다.
    
2. 그런 다음 토 **폴로지 비즈니스용 Skype 서버 를 습니다**.
    
3. 콘솔 트리에서 에지 서버를 배포할 사이트를 확장합니다.
    
4. 에지 **풀을 마우스** 오른쪽 단추로 클릭한 다음 새 에지 **풀을 클릭합니다**.
    
5. 새 에지 풀 **정의** 화면에서 **다음을 클릭합니다** .
    
6. 에지 풀 **FQDN** 정의 화면에서 에지 풀에서 사용할 내부 FQDN(정식 도메인 이름)을 입력하고 다중 컴퓨터 풀을 선택하고 완료되면 다음을 클릭합니다.
    
7. 기능 **선택 화면** 에서 다음을 선택할 수 있습니다.
    
    - SIP 액세스 서비스, 비즈니스용 Skype 서버 웹 회의 서비스 및 A/V 에지 서비스에 대해 동일한 FQDN 및 IP 주소를 사용하려는 경우도 있습니다. 그렇다면 단일 **FQDN 및 IP** 주소 사용 확인란을 선택해야 합니다(아래 9단계에 유의하세요).
    
    - 페더전을 사용하도록 설정하려면 이 에지 풀에 페더전 사용(포트 **5061) 확인란** 을 선택합니다.
    
8. 다음을 클릭하면 **IP** 옵션 화면이 표시됩니다. 옵션은 다음과 같습니다.
    
    - 내부 인터페이스에서 IPv4 사용
    
   - 내부 인터페이스에서 IPv6 사용
    
   - 외부 인터페이스에서 IPv4 사용
    
   - 외부 인터페이스에서 IPv6 사용
    
     이러한 주소는 IPv4 또는 IPv6 주소를 사용 중이든, 내부적으로 또는 외부적으로 에지 서버에 해당 주소를 적용하는지 여부를 잘 알 수 있습니다(11단계에 대해 이 점에 유의해야 합니다). 또한 외부 IP 주소에 NAT(Network Address Translation) 주소를 사용하기 위해 에지 서버 또는 에지 풀을 구성할 수도 있습니다. 이 작업을 위해 이 에지 풀의 외부 IP 주소가 **NAT** 로 변환됩니다. 확인란을 선택하여 이 작업을 할 수 있습니다. 준비 **되면 다음** 을 클릭합니다.
    
9. 외부 FQDNs 화면에서 선택은 위의 7단계에서 선택한 대로 선택됩니다.
    
   - 단일 **FQDN 및 IP** 주소 사용 확인란을 선택한 경우 **SIP** 액세스 상자에 단일 외부 FQDN을 입력해야 합니다. 그런 다음 각 에지 서비스에 대해 서로 다른 포트 번호를 입력하여 모두 독립적으로 연결할 수 있도록 해야 합니다. **SIP** 액세스 에지 서비스의 경우 5061, 웹 회의 에지 서비스의 경우 444, **A/V** 에지 서비스의 경우 443을 권장합니다. 완료되면 **다음** 을 선택합니다.
    
   - 단일 **FQDN 및 IP** 주소 사용 확인란을 선택하지 않은 경우 **이제 SIP** 액세스 에지 서비스, 웹 회의 에지 서비스 및 **A/V** 에지 서비스에 대한 외부 FQDN 세  개를 입력해야 합니다. 완료되면 **다음** 을 선택합니다.
    
10. 이제 이 풀의 컴퓨터 정의 **화면에 도달** 했습니다. 추가 **단추를** 클릭합니다.
    
11. 이제 내부 IP 주소 정의 **화면이 표시됩니다** . 8단계에서 다시 선택한 선택에 따라 내부 **IPv4** 주소 및 내부 **IPv6** 주소 텍스트 상자에 에지 서버의 IP 주소를 입력합니다. 준비 **되면 다음** 을 클릭합니다.
    
12. 외부 IP 주소 정의 **화면** 에서 이전 선택에 따라 몇 가지 옵션이 있습니다.
    
    - 모든 서비스에 대해 단일 FQDN을 사용할 수 있습니다. 그렇다면 외부 IPv4 또는 IPv6 주소(사용 중)를 **SIP 액세스** 텍스트 상자에 입력하고 다음을 **클릭합니다**.
    
    - 서비스에 세 개의 개별 FQDN 및 IP 주소를 사용하기로 선택한 것일 수 있습니다. 이 경우 **SIP** 액세스 에지 서비스, 웹 회의 에지 서비스 및 **A/V** 에지 서비스의 외부 IPv4 또는 IPv6 주소를 입력하고 다음을 **클릭합니다**.
    
    > [!NOTE]
    > 이전에 IPv6 주소 지정을 사용하도록 설정하고 할당하지 않은 경우 이 대화 상자가 표시되지 않습니다. 이는 정상적인 일입니다. 다음 단계로 이동하기만 합니다. 
  
13. **마침** 을 클릭합니다. 방금 만든 에지 서버가 이제 이 풀의 컴퓨터 정의 대화 상자에 **나열** 됩니다.
    
14. 이 풀의 컴퓨터  정의 화면에서 추가 단추를 다시 클릭하고 이  풀에 사용하려는 모든 에지 서버를 추가할 때까지 11~13단계를 반복합니다. 완료되면 다음을 **클릭합니다**.
    
15. 8단계에서 NAT를 다시 사용하기로 선택한 경우 이제 공용 IP 주소 텍스트 상자가 있는 화면 **이** 표시됩니다. NAT에서 변환하려면 A/V 에지 서비스에 대해 설정한 공용 IPv4 및/또는 IPv6 주소를 입력해야 합니다. **다음** 을 클릭합니다.
    
16. 다음 화면은 **다음 홉 정의입니다**. 다음 **홉 풀** 상자에서 프런트 엔드 풀 또는 독립 실행형 풀일 수 있는 내부 풀의 이름을 선택합니다. 환경에 있는 감독이 있는 경우 감독을 선택해야 합니다. **다음** 을 클릭합니다.
    
17. 프런트 엔드  풀 연결 화면에서 이 에지 서버와 연결하려면 프런트 엔드 풀 및 Standard Edition 풀을 포함하여 하나 이상의 내부 풀을 지정해야 합니다. 이 에지 서버를 사용하여 지원되는 외부 사용자와 통신할 내부 풀의 이름을 선택하기만 합니다. **다음** 을 클릭합니다.
    
    > [!NOTE]
    > 여기서 유의해야 할 점은 내부 풀 또는 독립 실행형 서버가 이미 다른 비즈니스용 Skype 서버 에지 서버를 사용하고 있는 경우 여러 개의 연결은 사용할 수 없습니다. 해당 상황에 있는 내부 풀 또는 독립 실행형 서버를 선택하면 다른 에지 서버에 대한 경고가 표시되고 계속할지 여부를 결정할 수 있습니다. 이 새 연결을 진행하면 다른 에지 서버에 대한 연결이 중지됩니다. 
  
18. 다음 **화면** 에서 마친을 클릭합니다.
    
19. 이제 이 업데이트된 기술을 게시하고 여기에서 에지 서버 배포의 비즈니스용 Skype 서버 따라 에지 서버에 [](deploy-edge-servers.md) 배포할 수 있습니다.
    
### <a name="defining-the-topology-for-a-hardware-load-balanced-edge-server-pool"></a>하드웨어 부하가 균형이 조정된 에지 서버 풀에 대한 토폴로지 정의

1. 비즈니스용 Skype 서버 Standard Edition 또는 비즈니스용 Skype 서버 프런트 엔드 서버에 로그온합니다.
    
2. 그런 다음 토 **폴로지 비즈니스용 Skype 서버 를 습니다**.
    
3. 콘솔 트리에서 에지 서버를 배포할 사이트를 확장합니다.
    
4. 에지 **풀을 마우스** 오른쪽 단추로 클릭한 다음 새 에지 **풀을 클릭합니다**.
    
5. 새 에지 풀 **정의** 화면에서 **다음을 클릭합니다** .
    
6. 에지 풀 **FQDN** 정의 화면에서 에지 풀에서 사용할 내부 FQDN(정식 도메인 이름)을 입력하고 다중 컴퓨터 풀을 선택하고 완료되면 다음을 클릭합니다.
    
7. 기능 **선택 화면** 에서 다음을 선택할 수 있습니다.
    
   - SIP 액세스 서비스, 비즈니스용 Skype 서버 웹 회의 서비스 및 A/V 에지 서비스에 대해 동일한 FQDN 및 IP 주소를 사용하려는 경우도 있습니다. 그렇다면 단일 **FQDN 및 IP** 주소 사용 확인란을 선택해야 합니다(아래 9단계에 유의하세요).
    
   - 페더전을 사용하도록 설정하려면 이 에지 풀에 페더전 사용(포트 **5061) 확인란** 을 선택합니다.
    
8. 다음을 클릭하면 **IP** 옵션 화면이 표시됩니다. 옵션은 다음과 같습니다.
    
   - 내부 인터페이스에서 IPv4 사용
    
   - 내부 인터페이스에서 IPv6 사용
    
   - 외부 인터페이스에서 IPv4 사용
    
   - 외부 인터페이스에서 IPv6 사용
    
     이러한 주소는 IPv4 또는 IPv6 주소를 사용 중이든, 내부적으로 또는 외부적으로 에지 서버에 해당 주소를 적용하는지 여부를 잘 알 수 있습니다(11단계에 대해 이 점에 유의해야 합니다).
    
     > [!NOTE]
     > 다른 두 토폴로지 옵션과 달리 하드웨어 부하 부하를 해결할 때 에지  풀의 외부 IP 주소가 NAT로 변환되는 옵션을 선택하지 **말아야 합니다**. 이는 **지원되지 않습니다**.
  
9. 외부 FQDNs 화면에서 선택은 위의 7단계에서 선택한 대로 선택됩니다.
    
   - 단일 **FQDN 및 IP** 주소 사용 확인란을 선택한 경우 **SIP** 액세스 상자에 단일 외부 FQDN을 입력해야 합니다. 그런 다음 각 에지 서비스에 대해 서로 다른 포트 번호를 입력하여 모두 독립적으로 연결할 수 있도록 해야 합니다. **SIP** 액세스 에지 서비스의 경우 5061, 웹 회의 에지 서비스의 경우 444, **A/V** 에지 서비스의 경우 443을 권장합니다. 완료되면 **다음** 을 선택합니다.
    
   - 단일 **FQDN 및 IP** 주소 사용 확인란을 선택하지 않은 경우 **이제 SIP** 액세스 에지 서비스, 웹 회의 에지 서비스 및 **A/V** 에지 서비스에 대한 외부 FQDN 세  개를 입력해야 합니다. 완료되면 **다음** 을 선택합니다.
    
10. 이제 이 풀의 컴퓨터 정의 **화면에 도달** 했습니다. 추가 **단추를** 클릭합니다.
    
11. 이제 내부 IP 주소 정의 **화면이 표시됩니다** . 8단계에서 다시 선택한 선택에 따라 내부 **IPv4** 주소 및 내부 **IPv6** 주소 텍스트 상자에 에지 서버의 IP 주소를 입력합니다. 준비 **되면 다음** 을 클릭합니다.
    
12. 외부 IP 주소 정의 **화면** 에서 이전 선택에 따라 몇 가지 옵션이 있습니다.
    
    - 모든 서비스에 대해 단일 FQDN을 사용할 수 있습니다. 그렇다면 외부 IPv4 또는 IPv6 주소(사용 중)를 **SIP 액세스** 텍스트 상자에 입력하고 다음을 **클릭합니다**.
    
    - 서비스에 세 개의 개별 FQDN 및 IP 주소를 사용하기로 선택한 것일 수 있습니다. 이 경우 **SIP** 액세스 에지 서비스, 웹 회의 에지 서비스 및 **A/V** 에지 서비스의 외부 IPv4 또는 IPv6 주소를 입력하고 다음을 **클릭합니다**.
    
    > [!NOTE]
    > 이전에 IPv6 주소 지정을 사용하도록 설정하고 할당하지 않은 경우 이 대화 상자가 표시되지 않습니다. 이는 정상적인 일입니다. 다음 단계로 이동하기만 합니다. 
  
13. **마침** 을 클릭합니다. 방금 만든 에지 서버가 이제 이 풀의 컴퓨터 정의 대화 상자에 **나열** 됩니다.
    
14. 이 풀의 컴퓨터  정의 화면에서 추가 단추를 다시 클릭하고 이  풀에 사용하려는 모든 에지 서버를 추가할 때까지 11~13단계를 반복합니다. 완료되면 다음을 **클릭합니다**.
    
15. 다음 화면은 **다음 홉 정의입니다**. 다음 **홉 풀** 상자에서 프런트 엔드 풀 또는 독립 실행형 풀일 수 있는 내부 풀의 이름을 선택합니다. 환경에 있는 감독이 있는 경우 감독을 선택해야 합니다. **다음** 을 클릭합니다.
    
16. 프런트 엔드  풀 연결 화면에서 이 에지 서버와 연결하려면 프런트 엔드 풀 및 Standard Edition 풀을 포함하여 하나 이상의 내부 풀을 지정해야 합니다. 이 에지 서버를 사용하여 지원되는 외부 사용자와 통신할 내부 풀의 이름을 선택하기만 합니다. **다음** 을 클릭합니다.
    
    > [!NOTE]
    > 여기서 유의해야 할 점은 내부 풀 또는 독립 실행형 서버가 이미 다른 비즈니스용 Skype 서버 에지 서버를 사용하고 있는 경우 여러 개의 연결은 사용할 수 없습니다. 해당 상황에 있는 내부 풀 또는 독립 실행형 서버를 선택하면 다른 에지 서버에 대한 경고가 표시되고 계속할지 여부를 결정할 수 있습니다. 이 새 연결을 진행하면 다른 에지 서버에 대한 연결이 중지됩니다. 
  
17. 다음 **화면** 에서 마친을 클릭합니다.
    
18. 이제 이 업데이트된 기술을 게시하고 여기에서 에지 서버 배포의 비즈니스용 Skype 서버 따라 에지 서버에 [](deploy-edge-servers.md) 배포할 수 있습니다.
    
## <a name="publish-your-edge-server-topology"></a>에지 서버 토폴로지 게시

위의 단계를 완료한 후 이 새 토폴로지가 게시될 때입니다. 그러면 이 토폴로지도 비즈니스용 Skype 서버 에지 서버 또는 에지 풀로 내보낼 수 있습니다. 아래 단계를 수행하세요.
  
1. 토 **폴로지 작성** 기 시작(이전 절차에서 아직 시작하지 않은 경우).
    
2. 토 **폴로지 작성기** 의 콘솔 트리에서 마우스 오른쪽 단추로  비즈니스용 Skype 서버 토폴로지 **비즈니스용 Skype 서버 클릭합니다**.
    
3. 마법사의 **시작** 페이지에서 **다음** 을 클릭합니다.
    
4. **다른 데이터베이스 만들기** 페이지에서 **다음** 을 클릭합니다.
    
5. 상태가 데이터베이스 만들기가 성공했다고 표시되면 다음을 합니다.
    
    - 로그를 보려면 **로그 보기** 를 클릭합니다.
    
    - 마법사를 닫으려면 **마침** 을 클릭합니다.
    
## <a name="export-your-edge-server-topology"></a>에지 서버 토폴로지 내보내기

배포를 성공적으로 비즈니스용 Skype 서버 배포 마법사는 중앙 관리 저장소 데이터에 액세스해야 합니다. 도메인 또는 포리스트의 내부 서버의 경우 일반적으로 이 작업은 간단합니다. 에지 서버는 도메인 외부에 있으므로 일반적으로 실제 미디어에서 에지 서버 위치로 토폴로지 파일을 수동으로 내보내야 합니다. 이 내보내기는 PowerShell을 통해 수행됩니다.
  
1. 관리 **비즈니스용 Skype 서버 시작 합니다**.
    
2. 비즈니스용 Skype 서버 **관리 셸에서** 다음을 실행합니다.
    
   ```powershell
   Export-CsConfiguration -FileName <ConfigurationFilePath.zip>
   ```

3. 내보낼 파일을 외부 미디어(예: 에지 서버의 위치에서 연결할 수 있는 USB 드라이브 또는 네트워크 공유)에 복사합니다.
