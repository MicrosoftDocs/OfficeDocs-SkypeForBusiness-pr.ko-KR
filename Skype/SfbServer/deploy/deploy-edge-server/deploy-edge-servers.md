# <a name="deploy-edge-servers-in-skype-for-business-server"></a>비즈니스용 Skype 서버에 Edge 서버 배포
 
**요약:** 비즈니스용 Skype 서버 환경에 Edge 서버를 배포 하는 방법에 대해 알아봅니다.
  
다음 섹션에는 비즈니스용 skype 서버 설명서 [의 Edge 서버 배포](../../plan-your-deployment/edge-server-deployments/edge-server-deployments.md) 에 대 한 비즈니스용 Skype 서버 요금제를 검토 한 후 따라야 하는 단계가 포함 되어 있습니다. 배포 단계는 다음과 같습니다.
  
- 네트워크 인터페이스
    
- 설치용
    
- 인증
    
- Edge 서버 시작
    
## <a name="network-interfaces"></a>네트워크 인터페이스

계획에 명시 된 바와 같이 Edge 서버를 호스트 하는 외곽 네트워크 또는 경계 네트워크에 DNS 없이 네트워크 인터페이스를 DNS로 구성 하 게 됩니다.
  
### <a name="interface-configuration-with-dns-servers-in-the-perimeter-network"></a>경계 네트워크의 DNS 서버를 사용 하 여 인터페이스 구성

1. 각 Edge 서버에 대 한 두 개의 네트워크 어댑터, 내부 인터페이스 및 외부 인터페이스에 대해 각각 하나씩 설치 합니다.
    
    > [!NOTE]
    > 내부 및 외부 서브넷을 서로 라우팅할 수 없어야 합니다. 
  
2. 외부 인터페이스에서 다음 **중 하나** 를 구성 합니다.
    
   에서. 외부 경계 네트워크 서브넷에 있는 세 개의 고정 IP 주소와 외부 방화벽의 내부 인터페이스에 대 한 기본 게이트웨이를 가리킵니다. 경계 DNS 서버 쌍을 가리키도록 어댑터 DNS 설정을 구성 합니다.
    
   b. 외부 경계 네트워크 서브넷에 하나의 고정 IP 주소를 지정 하 고, 외부 방화벽의 내부 인터페이스에 대 한 기본 게이트웨이를 가리킵니다. 경계 DNS 서버 쌍을 가리키도록 어댑터 DNS 설정을 구성 합니다. 이 구성은 비즈니스용 [Skype 서버에 대 한 Edge 토폴로지 만들기](create-your-edge-topology.md) 문서에 설명 된 대로 포트 할당에 표준이 아닌 값이 포함 되도록 토폴로지를 구성한 경우에만 허용 됩니다.
    
3. 내부 인터페이스에서 내부 경계 네트워크 서브넷에 하나의 고정 IP를 구성 하 고 기본 게이트웨이를 설정 하지 마세요. 하나 이상의 DNS 서버를 가리키도록 어댑터 DNS 설정을 구성 하 고, 가능 하면 경계 DNS 서버 쌍을 만듭니다.
    
4. 내부 인터페이스에서 클라이언트, 비즈니스용 Skype Server 및 Exchange UM (통합 메시징) 서버가 있는 모든 내부 네트워크에 대 한 영구 고정 경로를 만듭니다.
    
### <a name="interface-configuration-without-dns-servers-in-the-perimeter-network"></a>경계 네트워크에 DNS 서버가 없는 인터페이스 구성

1. 각 Edge 서버에 대 한 두 개의 네트워크 어댑터, 내부 인터페이스 및 외부 인터페이스에 대해 각각 하나씩 설치 합니다.
    
    > [!NOTE]
    > 내부 및 외부 서브넷을 서로 라우팅할 수 없어야 합니다. 
  
2. 외부 인터페이스에서 다음 **중 하나** 를 구성 합니다.
    
   에서. 외부 경계 네트워크 서브넷의 세 가지 고정 IP 주소 또한 외부 인터페이스 (예: 인터넷 연결 라우터 또는 외부 방화벽을 기본 게이트웨이로 정의)에 대 한 기본 게이트웨이를 구성 해야 합니다. 외부 dns 서버를 가리키도록 어댑터 DNS 설정을 구성 하 고, 이상적으로는 외부 DNS 서버 쌍을 만듭니다.
    
   b. 외부 경계 네트워크 서브넷에서 하나의 고정 IP 주소. 또한 외부 인터페이스 (예: 인터넷 연결 라우터 또는 외부 방화벽을 기본 게이트웨이로 정의)에 대 한 기본 게이트웨이를 구성 해야 합니다. 외부 DNS 서버를 가리키도록 어댑터 DNS 설정을 구성 하거나 한 쌍의 외부 DNS 서버가 이상적입니다. 이 구성은 비즈니스용 [Skype 서버에 대 한 Edge 토폴로지 만들기](create-your-edge-topology.md) 문서에 설명 된 대로 포트 할당에 표준이 아닌 값이 포함 되도록 토폴로지를 구성한 경우에만 허용 됩니다.
    
3. 내부 인터페이스에서 내부 경계 네트워크 서브넷에 하나의 고정 IP를 구성 하 고 기본 게이트웨이를 설정 하지 마세요. 또한 어댑터 DNS 설정은 비워 둡니다.
    
4. 내부 인터페이스에서 클라이언트, 비즈니스용 Skype Server 및 Exchange UM (통합 메시징) 서버가 있는 모든 내부 네트워크에 대 한 영구 고정 경로를 만듭니다.
    
5. 각 Edge 서버의 호스트 파일을 편집 하 여 다음 홉 서버나 VIP (가상 IP)에 대 한 레코드를 포함 합니다. 이 레코드는 토폴로지 작성기에서 Edge 서버로 구성 된 디렉터, Standard Edition server 또는 프런트 엔드 풀이 됩니다. DNS 부하 분산을 사용 중인 경우 다음 홉 풀의 각 구성원에 대 한 회선을 포함 합니다.
    
## <a name="installation"></a>설치용

이러한 단계를 성공적으로 수행 하려면 [비즈니스용 Skype 서버에 대 한 Edge 토폴로지 만들기](create-your-edge-topology.md) 문서에 나와 있는 단계를 따라야 합니다.
  
1. 로컬 관리자 그룹의 계정으로 Edge 서버 역할을 구성한 서버에 로그온 합니다.
    
2. 이 컴퓨터에 대 한 Edge 서버 토폴로지 설명서의 끝에 복사 된 토폴로지 구성 파일이 필요 합니다. 구성 파일을 배치한 외부 미디어 (USB 드라이브 또는 공유 등)에 액세스 합니다.
    
3. **배포 마법사**를 시작 합니다.
    
4. 마법사가 열리면 **비즈니스용 Skype Server System 설치 또는 업데이트**를 클릭 합니다.
    
5. 마법사는 이미 설치 되어 있는 항목이 있는지 확인 하는 검사를 실행 합니다. 마법사를 처음 실행할 때 1 단계부터 시작 하는 것이 좋습니다 **. 로컬 구성 저장소를 설치 합니다.**
    
6. **중앙 관리 저장소의 로컬 복제본 구성** 대화 상자가 표시 됩니다. **파일에서 가져오기를 클릭 해야 합니다 (Edge 서버의 경우 권장)**.
    
7. 여기에서 이전에 내보낸 토폴로지의 위치로 이동한 다음 .zip 파일을 선택 하 고 **열기**를 클릭 한 후 **다음**을 클릭 합니다.
    
8. 배포 마법사에서 구성 파일을 읽고 XML 구성 파일을 로컬 컴퓨터에 씁니다.
    
9. **명령 실행** 프로세스가 완료 되 면 **마침을**클릭 합니다.
    
10. 배포 마법사에서 **2 단계를 클릭 합니다. 비즈니스용 Skype 서버 구성 요소를 설치 또는 제거**합니다. 그러면 마법사가 로컬 컴퓨터에 저장 된 XML 구성 파일에 지정 된 비즈니스용 Skype 서버에 지 구성 요소를 설치 합니다.
    
11. 설치가 완료 되 면 아래 **인증서** 섹션의 단계로 이동할 수 있습니다.
    
## <a name="certificates"></a>인증

Edge 서버의 인증서 요구 사항은 Edge 인증서 계획 문서에서 찾을 수 있습니다. 인증서를 설정 하는 단계는 다음과 같습니다.
  
> [!NOTE]
> 인증서 마법사를 실행 하는 경우 사용할 인증서 템플릿 형식에 대 한 올바른 권한이 있는 계정으로 로그인 해야 합니다. 기본적으로 비즈니스용 Skype 서버 인증서 요청은 웹 서버 인증서 템플릿을 사용 합니다. RTCUniversalServerAdmins 그룹의 구성원 인 계정으로 로그인 하 여이 템플릿을 통해 인증서를 요청 하는 경우 그룹에 해당 서식 파일을 사용 하기 위한 등록 권한이 할당 되었는지 확인 합니다. 
  
### <a name="internal-edge-interface-certificates"></a>내부에 지 인터페이스 인증서

 
### <a name="1-download-or-export-the-ca-certification-chain"></a>1. CA 인증서 체인 다운로드 또는 내보내기

 
#### <a name="nbspnbspnbsp-a-download-using-certsrv-web-site"></a>&nbsp;&nbsp;&nbsp;에서. Certsrv 웹 사이트를 사용 하 여 다운로드

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;칼럼. 내부 네트워크의 비즈니스용 Skype 서버에 로컬 관리자 그룹의 구성원으로 로그인 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;ii. **시작**을 열고 **실행** (또는 **검색** 및 **실행** ) 한 후 다음을 입력 합니다.
    
  ```console
  https://<NAME OF YOUR ISSUING CA SERVER>/certsrv
  ```

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;예를 들어:
    
  ```console
  https://ca01/contoso.com/certsrv
  ```

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iii. 발급 CA의 certsrv 웹 페이지에 있는 **작업 선택**에서 **CA 인증서, 인증서 체인 또는 CRL 다운로드**를 클릭 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iv. **Ca 인증서, 인증서 체인 또는 CRL 다운로드**에서 **Ca 인증서 체인 다운로드**를 클릭 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;variant. **파일 다운로드** 상자에서 **저장**을 클릭 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vi. P7b 파일을 서버의 하드 디스크 드라이브에 저장 한 다음 각 Edge 서버의 폴더에 복사 합니다.
    
### <a name="nbspnbspnbspb-export-using-mmc"></a>&nbsp;&nbsp;&nbsp;b. MMC를 사용 하 여 내보내기

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;칼럼. MMC를 사용 하 여 도메인에 연결 된 컴퓨터에서 CA 루트 인증서를 내보낼 수 있습니다. **시작** 및 **실행**으로 이동 하거나 **검색**을 열고 **MMC** 를 입력 하 여 엽니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;ii. MMC 콘솔에서 **파일**을 클릭 한 다음 **스냅인 추가/제거**를 클릭 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iii. **스냅인 추가/제거** 대화 상자 목록에서 **인증서**를 선택한 다음 **추가**를 클릭 합니다. 메시지가 표시 되 면 **컴퓨터 계정을**선택한 다음 **다음**을 선택 합니다. **컴퓨터 선택** 대화 상자에서 **로컬 컴퓨터**를 선택 합니다. **마침을**클릭 한 다음 **확인**을 클릭 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iv. **인증서 (로컬 컴퓨터)** 를 확장 합니다. **신뢰할 수 있는 루트 인증 기관**을 확장 합니다. **인증서**를 선택 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;variant. CA에서 발급 한 루트 인증서를 클릭 합니다. 인증서를 마우스 오른쪽 단추로 클릭 하 고 메뉴에서 **모든 작업** 을 선택한 다음 **내보내기를**선택 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vi. **인증서 내보내기 마법사** 가 열립니다. **다음**을 클릭 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vii. **파일 내보내기 형식** 대화 상자에서 내보낼 형식을 선택 합니다. 우리의 권장 사항은 **암호화 메시지 구문 표준-PKCS #7 인증서 (P7b)** 입니다. 이 옵션을 선택 하는 경우에는 **인증 경로에 모든 인증서 포함 가능** 확인란을 선택 하 고 루트 CA 인증서와 중간 인증서를 포함 하 여 인증서 체인도 내보내야 한다는 점에 유의 해야 합니다. **다음**을 클릭 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;viii. **내보낼 파일** 대화 상자의 파일 이름 항목에 내보낸 인증서에 대 한 경로와 파일 이름 (기본 확장명으로는. p7b)을 입력 합니다. 더 쉬운 경우에는 **찾아보기** 단추를 선택 하 여 내보낸 인증서를 저장할 위치로 이동한 후 내보낸 인증서의 이름을 여기에 입력 합니다. **저장**을 클릭 하 고 준비가 되 면 **다음** 을 클릭 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;ix. 작업 요약을 검토 하 고 **마침을** 클릭 하 여 인증서 내보내기를 완료 합니다. **확인** 을 클릭 하 여 내보내기가 성공적으로 완료 되었는지 확인 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;축. . P7b 파일을 각 Edge 서버에 복사 합니다.
    
### <a name="2-import-the-ca-certification-chain"></a>2. CA 인증서 체인 가져오기

&nbsp;&nbsp;&nbsp;에서. 각 Edge 서버에서 MMC를 엽니다 ( **시작** 및 **실행**또는 **검색**을 선택 하 고 **mmc** 를 입력 하 여 열기).
    
&nbsp;&nbsp;&nbsp;b. **파일** 메뉴에서 **스냅인 추가/제거**를 클릭 한 다음 **추가**를 선택 합니다.
    
&nbsp;&nbsp;&nbsp;c. **스냅인 추가/제거** 상자에서 **인증서**를 클릭 한 다음 **추가**를 클릭 합니다.
    
&nbsp;&nbsp;&nbsp;a. **인증서 스냅인** 대화 상자에서 **컴퓨터 계정을**클릭 하 고 **다음**을 클릭 합니다.
    
&nbsp;&nbsp;&nbsp;z.e.n.works. **컴퓨터 선택** 대화 상자에서 **로컬 컴퓨터: (이 콘솔이 실행 되 고 있는 컴퓨터)** 확인란이 선택 되어 있는지 확인 하 고 **마침을**클릭 합니다.
    
&nbsp;&nbsp;&nbsp;f. **닫기를**클릭 한 다음 **확인**을 클릭 합니다.

&nbsp;&nbsp;&nbsp;간체. 배포 중인 모든 Edge 서버에 대해이 작업을 수행 해야 합니다.
    
### <a name="3-create-the-certificate-request"></a>3. 인증서 요청 만들기

&nbsp;&nbsp;&nbsp;에서. Edge 서버 중 하나에 로그온 하 고 배포 마법사를 시작한 다음 **3 단계: 인증서 요청, 설치 또는 할당**에서 **실행** (또는 이미이 마법사를 실행 한 경우 **다시 실행**)을 클릭 합니다.

&nbsp;&nbsp;&nbsp;c. **지연 또는 즉시 요청** 페이지에서 Edge 환경에서 하나에 액세스할 수 있는 경우 **온라인 인증 기관에 즉시 요청 보내기를** 선택 하 고 **, 나중에 요청을 보낼** 수 있습니다.

&nbsp;&nbsp;&nbsp;f. 이름 및 보안 설정 페이지에서 다음을 수행 합니다.

 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;ii. 비트 **길이**에서 비트 길이를 선택 합니다 (기본값은 2048 이며, 더 높거나 더 안전 하 게 진행할 수 있지만 성능이 느려지는 것).

&nbsp;&nbsp;&nbsp;칼럼. **제목 이름/주체 대체 이름** 페이지에서 마법사가 자동으로 채워야 합니다.
    
&nbsp;&nbsp;&nbsp;j. **추가 주제 대체 이름 구성** 페이지에서 필요한 추가 주제 대체 이름을 추가 해야 합니다.
    
&nbsp;&nbsp;&nbsp;간체. **요청 요약** 페이지에서 요청을 생성 하는 데 사용할 인증서 정보를 찾습니다. 변경 작업을 수행 해야 하는 경우에는 이전 단계로 이동 합니다.
    
&nbsp;&nbsp;&nbsp;l-p. 그런 후 **다음** 을 클릭 하 여 CA에 제공 해야 하는 CSR 파일을 생성 합니다 ( **로그 보기** 를 클릭 하 여 인증서 요청에 대 한 로그를 확인 할 수도 있음).

  > [!NOTE]

&nbsp;&nbsp;&nbsp;넣기. 로컬 컴퓨터의 **인증서 개인 저장소** 에서 내보내려는 인증서를 마우스 오른쪽 단추로 클릭 합니다. 결과 메뉴에서 **모든 작업** 을 선택한 다음 **내보내기를**클릭 합니다.
    
&nbsp;&nbsp;&nbsp;칼럼. **인증서 내보내기 마법사**에서 **다음**을 클릭 합니다. **예, 개인 키를 내보냅니다를**선택 합니다. **다음**을 클릭 합니다.
    
&nbsp;&nbsp;&nbsp;j. **파일 내보내기 형식** 대화 상자에서 **개인 정보 교환-PKCS # 12 ()를 선택 합니다. PFX)** 를 선택한 후 다음을 선택 합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;칼럼. 가능 하면 인증 경로에 있는 모든 인증서를 포함 합니다.

   > [!NOTE]

&nbsp;&nbsp;&nbsp;간체. 개인 키를 보호 하기 위해 암호를 할당 하려는 경우 개인 키에 대 한 암호를 입력할 수 있습니다. 확인을 위해 암호를 다시 입력 하 고 **다음**을 클릭 합니다.

&nbsp;&nbsp;&nbsp;암. **인증서 내보내기 마법사 완료** 대화 상자에서 요약을 검토 한 다음 **마침을**클릭 합니다.
    
&nbsp;&nbsp;&nbsp;개. 성공적으로 내보내기 대화 상자에서 **확인을** 클릭 합니다.
