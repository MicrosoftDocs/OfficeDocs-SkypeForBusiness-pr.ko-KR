# <a name="deploy-edge-servers-in-skype-for-business-server"></a>에지 서버 배포 비즈니스용 Skype 서버
 
**요약:** 에지 서버를 사용자 환경으로 배포하는 비즈니스용 Skype 서버 대해 자세히 알아보십시오.
  
다음 섹션에는 비즈니스용 Skype 서버 에지 서버 배포 계획 설명서가 검토된 비즈니스용 Skype 서버 [단계가](../../plan-your-deployment/edge-server-deployments/edge-server-deployments.md) 포함되어 있습니다. 배포 단계는 다음과 같습니다.
  
- 네트워크 인터페이스
    
- 설치
    
- 인증서
    
- 에지 서버 시작
    
## <a name="network-interfaces"></a>네트워크 인터페이스

계획에서 에지 서버를 호스트하는 경계 네트워크에서 또는 경계 네트워크에 DNS를 사용하지 않고 DNS를 사용하여 네트워크 인터페이스를 구성할 수 있습니다.
  
### <a name="interface-configuration-with-dns-servers-in-the-perimeter-network"></a>경계 네트워크에 있는 DNS 서버와의 인터페이스 구성

1. 각 에지 서버에 대해 두 개의 네트워크 어댑터를 설치합니다. 하나는 내부 인터페이스용, 다른 하나는 외부 연결 인터페이스용입니다.
    
    > [!NOTE]
    > 내부 및 외부 서브넷은 서로 라우팅할 수 없습니다. 
  
2. 외부 인터페이스에서 다음 중 **하나를** 구성합니다.
    
   a. 외부 경계 네트워크 서브넷에 있는 세 개의 고정 IP 주소가 있으며 기본 게이트웨이가 외부 방화벽의 내부 인터페이스를 향하게 합니다. 한 쌍의 경계 DNS 서버를 향하도록 어댑터 DNS 설정을 구성합니다.
    
   b. 외부 경계 네트워크 서브넷의 고정 IP 주소 하나와 기본 게이트웨이가 외부 방화벽의 내부 인터페이스를 지정합니다. 한 쌍의 경계 DNS 서버를 향하도록 어댑터 DNS 설정을 구성합니다. 이 구성은 이전에 포트 할당에 표준이 아닌 값을 하도록 토폴로지가 구성한 경우만 허용됩니다. 이 [구성은](create-your-edge-topology.md) 포트 할당에 대한 에지 토폴로지 만들기 문서에서 비즈니스용 Skype 서버 있습니다.
    
3. 내부 인터페이스에서 내부 경계 네트워크 서브넷에 고정 IP를 하나 구성하고 기본 게이트웨이를 설정하지 않습니다. 어댑터 DNS 설정을 구성하여 하나 이상의 DNS 서버를 가급적 경계 DNS 서버 쌍으로 구성합니다.
    
4. 클라이언트, 서버 및 UM(통합 메시징) 서버가 있는 모든 비즈니스용 Skype 서버 Exchange 고정 경로를 내부 인터페이스에 만들 수 있습니다.
    
### <a name="interface-configuration-without-dns-servers-in-the-perimeter-network"></a>경계 네트워크에 DNS 서버가 없는 인터페이스 구성

1. 각 에지 서버에 대해 두 개의 네트워크 어댑터를 설치합니다. 하나는 내부 인터페이스용, 다른 하나는 외부 연결 인터페이스용입니다.
    
    > [!NOTE]
    > 내부 및 외부 서브넷은 서로 라우팅할 수 없습니다. 
  
2. 외부 인터페이스에서 다음 중 **하나를** 구성합니다.
    
   a. 외부 경계 네트워크 서브넷의 세 개의 고정 IP 주소입니다. 또한 인터넷 연결 라우터 또는 외부 방화벽을 기본 게이트웨이로 정의하는 등 외부 인터페이스에서 기본 게이트웨이를 구성해야 합니다. 외부 DNS 서버(이상적으로는 한 쌍의 외부 DNS 서버)를 지점으로 하도록 어댑터 DNS 설정을 구성합니다.
    
   b. 외부 경계 네트워크 서브넷의 고정 IP 주소 1개 또한 인터넷 연결 라우터 또는 외부 방화벽을 기본 게이트웨이로 정의하는 등 외부 인터페이스에서 기본 게이트웨이를 구성해야 합니다. 외부 DNS 서버 또는 이상적으로 한 쌍의 외부 DNS 서버를 하도록 어댑터 DNS 설정을 구성합니다. 이 구성은 이전에 포트 할당에 표준이 아닌 값을 하도록 토폴로지가 구성한 경우만 허용됩니다. 이 [구성은](create-your-edge-topology.md) 포트 할당에 대한 에지 토폴로지 만들기 문서에서 비즈니스용 Skype 서버 있습니다.
    
3. 내부 인터페이스에서 내부 경계 네트워크 서브넷에 고정 IP를 하나 구성하고 기본 게이트웨이를 설정하지 않습니다. 어댑터 DNS 설정도 비워 두면 됩니다.
    
4. 클라이언트, 서버 및 UM(통합 메시징) 서버가 있는 모든 비즈니스용 Skype 서버 Exchange 고정 경로를 내부 인터페이스에 만들 수 있습니다.
    
5. 다음 홉 서버 또는 VIP(가상 IP)에 대한 레코드를 포함하려면 각 에지 서버에서 호스트 파일을 편집합니다. 이 레코드는 토폴로지 작성기에서 Standard Edition 에지 서버 다음 홉 주소로 구성한 에지 서버 또는 프런트 엔드 풀인 Director가 됩니다. DNS 부하 분산을 사용하는 경우 다음 홉 풀의 각 구성원에 대한 줄을 포함합니다.
    
## <a name="installation"></a>설치

이러한 단계를 성공적으로 완료하려면 Create [your Edge topology for 비즈니스용 Skype 서버.](create-your-edge-topology.md)
  
1. 로컬 관리자 그룹에 있는 계정을 사용하여 에지 서버 역할에 대해 구성한 서버에 로그온합니다.
    
2. 이 컴퓨터의 에지 서버 토폴로지 설명서 끝에 복사한 토폴로지 구성 파일이 필요합니다. 구성 파일에 배치한 외부 미디어(예: USB 드라이브 또는 공유)에 액세스합니다.
    
3. 배포 **마법사를 시작 합니다.**
    
4. 마법사가 열리면 시스템 설치 **또는 비즈니스용 Skype 서버 클릭합니다.**
    
5. 마법사가 검사를 실행하여 이미 설치된 모든 것이 설치되어 있는지 확인합니다. 마법사를 처음 실행하는 것이기 때문에 1단계에서 시작할 **수 있습니다. 로컬 구성 저장소를 설치합니다.**
    
6. 중앙 **관리 저장소의 로컬 복제본 구성 대화 상자가** 나타납니다. 파일에서 가져오기(에지 서버에 **권장)를 클릭해야 합니다.**
    
7. 여기에서 이전에 내보냈던 토폴로지의 위치로 이동하고 .zip 파일을 선택하고 열기 를 클릭한 후 다음 을 **클릭합니다.**
    
8. 배포 마법사에서 구성 파일을 읽고 로컬 컴퓨터에 XML 구성 파일을 쓴다.
    
9. **명령 실행** 프로세스가 완료되면 **마침** 을 클릭합니다.
    
10. 배포 마법사에서 **2단계를 클릭합니다. 구성 요소 설치 비즈니스용 Skype 서버 제거합니다.** 그런 다음 마법사가 로컬 컴퓨터에 비즈니스용 Skype 서버 XML 구성 파일에 지정된 에지 구성 요소를 설치합니다.
    
11. 설치가 완료되면 아래 인증서 섹션의 단계로 이동할 **수** 있습니다.
    
## <a name="certificates"></a>인증서

에지 서버에 대한 인증서 요구 사항은 에지 인증서 계획 설명서에서 찾을 수 있습니다. 인증서 설정 단계는 아래와 같습니다.
  
> [!NOTE]
> 인증서 마법사를 실행하는 경우 사용할 인증서 템플릿 유형에 대한 올바른 사용 권한이 있는 계정으로 로그인해야 합니다. 기본적으로 인증서 비즈니스용 Skype 서버 웹 서버 인증서 템플릿을 사용하게 됩니다. RTCUniversalServerAdmins 그룹의 구성원인 계정으로 로그인하여 이 템플릿을 통해 인증서를 요청하는 경우 해당 서식 파일을 사용할 수 있는 등록 권한이 그룹에 할당되어 있는지 다시 한 번 확인하십시오. 
  
### <a name="internal-edge-interface-certificates"></a>내부 에지 인터페이스 인증서

 
### <a name="1-download-or-export-the-ca-certification-chain"></a>1. CA 인증 체인 다운로드 또는 내보내기

 
#### <a name="nbspnbspnbsp-a-download-using-certsrv-web-site"></a>&nbsp;&nbsp;&nbsp; a. certsrv 웹 사이트를 사용하여 다운로드

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;i. 로컬 비즈니스용 Skype 서버 그룹의 구성원으로 내부 네트워크에 로그인합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;ii. 시작 **,** **실행(또는** 검색 **및** **실행)을** 열고 다음을 입력합니다.
    
  ```console
  https://<NAME OF YOUR ISSUING CA SERVER>/certsrv
  ```

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;예를 들어:
    
  ```console
  https://ca01/contoso.com/certsrv
  ```

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iii. 발급 CA의 인증서 웹 페이지의 작업 선택에서 CA 인증서, 인증서 체인 또는 **CRL 다운로드를 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iv. CA 인증서, 인증서 체인 또는 **CRL** 다운로드에서 CA 인증서 체인 **다운로드를 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;v. 파일 **다운로드 상자에서** 저장을 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vi. .p7b 파일을 서버의 하드 디스크 드라이브에 저장한 다음 각 에지 서버의 폴더에 복사합니다.
    
### <a name="nbspnbspnbspb-export-using-mmc"></a>&nbsp;&nbsp;&nbsp;b. MMC를 사용하여 내보내기

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;i. MMC를 사용하여 도메인에 가입된 모든 컴퓨터의 CA 루트 인증서를 내보낼 수 있습니다. 시작 및 **실행으로** 이동하거나 검색을 **열고**  **MMC를** 입력하여 를 열 수 있습니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;ii. MMC 콘솔에서 파일 **을** 클릭한 다음 **스냅인 추가/제거를 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iii. 스냅인 추가 **또는** 제거 대화 상자 목록에서 인증서를 선택하고 추가를 **클릭합니다.** 메시지가 표시될 때 컴퓨터 계정 을 **선택하고** 다음 을 **선택합니다.** 컴퓨터 **선택 대화 상자에서** 로컬 **컴퓨터 를 선택합니다.** 마친 **후** 확인을 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iv. **인증서(로컬 컴퓨터)를 확장합니다.** **신뢰할 수 있는 루트 인증 기관** 을 확장합니다. 인증서를 **선택합니다.**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;v. CA에서 발급된 루트 인증서를 클릭합니다. 인증서를 마우스 오른쪽 단추로 클릭하고 메뉴에서 **모든** 작업을 선택한 다음 내보내기 를 **선택합니다.**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vi. 인증서 **내보내기 마법사가** 열립니다. **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vii. 파일 **형식 내보내기 대화 상자에서** 내보낼 형식을 선택합니다. 암호화 메시지 구문 **Standard - PKCS #7 인증서(P7b)가 권장됩니다.** 이 경우 루트 CA 인증서 및 모든  중간 인증서를 포함하여 인증서 체인도 내보내기 때문에 가능한 경우 인증 경로에 모든 인증서 포함 확인란을 선택해야 합니다. **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;viii. **내보낼** 파일 대화 상자의 파일 이름 항목에서 내보낼 인증서의 경로와 파일 이름(기본 확장명은 .p7b)을 입력합니다. 더 쉬운 경우 찾아보기 단추를 클릭하여 내보낼 인증서를 저장할 위치로 이동하고 여기에 내보낼 인증서의 이름을 지정합니다.  **저장을** 클릭하고 **준비가** 되면 다음을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;ix. 작업 요약을 검토하고 마친을 **클릭하여** 인증서 내보내기 작업을 완료합니다. **확인** 을 클릭하여 내보내기 성공을 확인합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;x. .p7b 파일을 각 에지 서버에 복사합니다.
    
### <a name="2-import-the-ca-certification-chain"></a>2. CA 인증 체인 가져오기

&nbsp;&nbsp;&nbsp;a. 각 에지 서버에서 MMC를 열고(시작 **및** 실행 **또는** 검색을 선택하고  **MMC를** 입력하여 열기).
    
&nbsp;&nbsp;&nbsp;b. 파일 **메뉴에서** **스냅인 추가/제거를 클릭한** 다음 추가 를 **선택합니다.**
    
&nbsp;&nbsp;&nbsp;c. 스냅인 추가 **또는** 제거 상자에서 인증서를 **클릭한** 다음 추가를 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;d. **인증서 스냅인** 대화 상자에서 **컴퓨터 계정** 을 클릭한 다음 **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;e. 컴퓨터 **선택** 대화 상자에서 로컬 **컴퓨터: (이** 콘솔이 실행되고 있는 컴퓨터) 확인란이 선택되어 있으며 마침을 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;f. **닫기** 를 클릭한 다음 **확인을 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;g. 콘솔 트리에서 **인증서(로컬 컴퓨터)를** 확장하고 신뢰할 **수** 있는 루트 인증 기관을 마우스 오른쪽 단추로 클릭하고 모든 작업으로 이동한 다음 가져오기 를 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;h. 나타나는 마법사의 가져오기 텍스트  상자에 인증서의 파일 이름(이전 섹션에서 .p7b 파일에 지정한 이름)을 지정합니다. **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;i. 신뢰할 수 있는  루트 인증 기관을 선택해야 하기에 모든 인증서를 다음 저장소에 저장에 라디오 단추를 떠 두십시오. **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;j. 요약을 검토하고 **마친을** 클릭하여 가져오기 작업을 완료합니다.
    
&nbsp;&nbsp;&nbsp;k. 배포하는 모든 에지 서버에 대해 이 방법을 실행해야 합니다.
    
### <a name="3-create-the-certificate-request"></a>3. 인증서 요청 만들기

&nbsp;&nbsp;&nbsp;a. 에지 서버 중 하나에 로그온하고 배포 마법사를 시작한 다음 **3단계:** 인증서  요청, 설치 또는 할당에서 실행(또는 이 마법사를 이미 실행한 경우 다시 실행)을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;b. 인증서 요청 **페이지에서** 내부 에지 인증서가 **선택되어** 있으며 요청을 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;c. 지연된  요청 또는 즉시 요청  페이지에서 에지 환경에서 요청에 액세스할 수 있는 경우 온라인 인증 기관에 즉시 보내기 또는 지금 요청을 준비하지만 나중에 보내지 않습니다.를 선택합니다. 
    
&nbsp;&nbsp;&nbsp;d. 인증서 요청 **파일** 페이지에서 파일을 저장할 위치의 전체 부분과 파일 이름(예: c:\SkypeInternalEdgeCert.cer)을 입력합니다. **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;e. 대체 **인증서** 템플릿 지정 페이지에서 기본 WebServer 템플릿이 아니면 다른 템플릿을 사용하 고 선택한 인증 기관에 대체 인증서 템플릿 사용 **확인란을** 선택합니다. 그렇지 않으면 아무 것도 하지 않습니다.
    
&nbsp;&nbsp;&nbsp;f. 이름 및 보안 설정 페이지에서 다음을 수행합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   i. 이름 **에** 인증서의 표시 이름(예: 내부 에지)을 입력합니다.
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  ii. 비트 **길이에서** 비트 길이를 선택하십시오(기본값은 2048, 더 높게 이동하여 더 안전할 수 있지만 성능이 느려질 수 있습니다).
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  iii. 내보낼 수 있는 인증서가 필요한 경우 인증서 개인 키를 내보낼 수 있는 것으로 **표시 확인란을 선택해야** 합니다.
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  iv. **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;g. 조직 **정보 페이지에서** 조직 및 OU(조직 구성 단위)의 이름을 입력합니다. 부서나 부서(예: IT)를 입력할 수 있습니다.
    
&nbsp;&nbsp;&nbsp;h. 지리적 **정보 페이지에서** 위치 정보를 입력합니다.
    
&nbsp;&nbsp;&nbsp;i. 주체 **이름/주체** 대체 이름 페이지에서 마법사에 의해 자동으로 채워야 합니다.
    
&nbsp;&nbsp;&nbsp;j. 추가 **주체 대체** 이름 구성 페이지에서 필요한 주체 대체 이름을 추가해야 합니다.
    
&nbsp;&nbsp;&nbsp;k. 요청 **요약 페이지에서** 요청을 생성하는 데 사용할 인증서 정보를 살펴 봐야 합니다. 변경해야 하는 경우 돌아가서 지금 변경합니다.
    
&nbsp;&nbsp;&nbsp;l. 그런 **다음 다음을** 클릭하여 CA에 제공해야 하는 CSR 파일을 생성합니다. 로그 보기를 클릭하여 인증서 요청에 대한 로그를 확인할 수도 있습니다. 
    
&nbsp;&nbsp;&nbsp;m. 요청이 생성되고 나면 보기를 클릭하여 인증서를 보고 **마친** 후 창을 닫을 수 있습니다.  CSR 파일의 내용을 CA에 제공해야 하여 다음 섹션에서 이 컴퓨터로 가져올 수 있는 인증서를 생성할 수 있습니다.
    

### <a name="4-import-the-certificate"></a>4. 인증서 가져오기

&nbsp;&nbsp;&nbsp;a. 로컬 Administrators 그룹의 구성원으로 마지막 절차에서 인증서를 요청한 에지 서버에 로그온합니다.
    
&nbsp;&nbsp;&nbsp;b. 배포 마법사에서 **3단계 옆의 인증서 요청, 설치** 또는 할당 , 다시 **실행을 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;c. 사용 **가능한 인증서 작업 페이지에서** 에서 **인증서 가져오기 를 클릭합니다. P7b, .pfx 또는 .cer 파일**.
    
&nbsp;&nbsp;&nbsp;d. 인증서 **가져오기** 페이지에서 이전 섹션에 있는 인증서의 전체 경로와 파일 이름을 입력합니다. 또는 찾아보기를 클릭하여 파일을 찾아서 선택할 수 있습니다. 
    
&nbsp;&nbsp;&nbsp;e. 에지 풀의 다른 구성원에 대한 인증서를 가져오는 중일 때 인증서에 개인  키가 포함되어 있는 경우 인증서의 개인 키가 포함된 인증서 파일 확인란을 선택하고 암호를 지정해야 합니다. **다음** 을 클릭하여 계속합니다.
    
&nbsp;&nbsp;&nbsp;f. 정보를 **확인한** 후  요약 페이지에서 다음을 클릭하고 인증서를 성공적으로 가져온 후 마친 다음을 클릭합니다. 
    
 
### <a name="5-export-the-certificate"></a>5. 인증서 내보내기

&nbsp;&nbsp;&nbsp;a. 이전에 인증서를 가져와 로컬 Administrators 그룹의 구성원으로 에지 서버에 로그온해야 합니다.
    
&nbsp;&nbsp;&nbsp;b. **시작,** **실행(또는** 검색 열기)을 클릭하고  **MMC 를 입력합니다.**
    
&nbsp;&nbsp;&nbsp;c. MMC 콘솔에서 파일 **을** 클릭하고 **스냅인 추가/제거를 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;d. 스냅인 **추가 또는** 제거 상자에서 인증서를 클릭하고 추가를 **클릭합니다.** 
    
&nbsp;&nbsp;&nbsp;e. 인증서 **스냅인** 대화 상자에서 컴퓨터 **계정 을 선택합니다.** **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;f. 컴퓨터 **선택 대화** 상자에서 로컬 컴퓨터: (이 콘솔이 실행되고 **있는 컴퓨터)를 선택합니다.** **마침** 을 클릭합니다. **확인을** 클릭하면 MMC 콘솔의 구성이 완료됩니다.
    
&nbsp;&nbsp;&nbsp;g. **인증서(로컬** 컴퓨터)를 두 번 클릭하여 인증서 저장소를 확장합니다. 개인 을 **두 번** 클릭한 다음 **인증서를 클릭합니다.**
    
  > [!NOTE]
  > 이 경우 로컬 컴퓨터의 인증서 개인 저장소에 인증서가 없습니다. 키가 없는 경우 가져온 인증서에 연결된 개인 키가 없는 경우 헌팅할 필요가 없습니다. 요청 및 가져오기 단계를 한 번 더 시도해 보시고 모든 권한이 있는지 확인된 경우 CA 관리자 또는 공급자에게 문의하세요. 
  
&nbsp;&nbsp;&nbsp;h. 로컬 **컴퓨터의** 인증서 개인 저장소에서 내보낼 인증서를 마우스 오른쪽 단추로 클릭합니다. 결과 **메뉴에서** 모든 작업을 선택한 다음 내보내기 를 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;i. **인증서 내보내기 마법사** 에서 **다음** 을 클릭합니다. 예, **개인 키를 내보낼 수 있습니다.를 선택합니다.** **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;j. 파일 **형식 내보내기** 대화 상자에서 개인 정보 Exchange **- PKCS#12(를 선택합니다. PFX)** 를 선택하고 다음을 선택합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   i. 가능한 경우 인증 경로에 모든 인증서를 포함합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   ii. 모든 확장 속성을 내보낼 수 있습니다.
    
   > [!NOTE]
   > **내보내기에** 성공하면 개인 키 **삭제를 선택하지 않습니다.** 인증서 및 개인 키를 이 에지 서버로 다시 이 에지 서버로 다시 전송해야 합니다.
  
&nbsp;&nbsp;&nbsp;k. 개인 키를 보호하기 위해 암호를 할당하려는 경우 개인 키의 암호를 입력할 수 있습니다. 암호를 다시 입력하여 확인한 후 다음 을 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;l. 파일 확장명 **.pfx를** 사용하여 내보낼 인증서의 경로와 파일 이름을 입력합니다. 풀의 다른 에지 서버에서 경로에 액세스할 수 있도록 해야 합니다. 또는 외부 미디어(예: USB 드라이브)를 통해 파일을 이동해야 합니다. 선택한 **경우** 다음을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;m. 인증서 내보내기 마법사 완료 대화 상자의 **요약을** 검토하고 마친 을 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;n. 내보내기 성공 대화 상자에서 **확인** 을 클릭합니다.
    
 
### <a name="6-assign-the-certificate"></a>6. 인증서 할당

&nbsp;&nbsp;&nbsp;a. 각 에지 서버의 배포 마법사에서 **3단계 옆에 있는 인증서 요청, 설치** 또는 할당 , 다시 **실행을 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;b. 사용 **가능한 인증서 작업 페이지에서** 기존 인증서 **할당을 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;c. **인증서 할당** 페이지의 목록에서 **에지 내부** 를 선택합니다.
    
&nbsp;&nbsp;&nbsp;d. 인증서 **저장소 페이지에서** 내부 에지(이전 섹션)에 대해 가져온 인증서를 선택합니다.
    
&nbsp;&nbsp;&nbsp;e. 인증서 **할당 요약 페이지에서** 설정을 살펴보고 **다음을** 클릭하여 인증서를 할당합니다.
    
&nbsp;&nbsp;&nbsp;f. 마법사 완료 페이지에서 **마침** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;g. 이 절차를 완료한 후 각 에지 서버에서 인증서 MMC 스냅인을 열고 인증서(로컬 **컴퓨터)** 및 개인을 확장하고 인증서를 클릭한 다음 내부 에지 인증서가 세부 정보 창에 나열되어 있는지 확인하는 것이 좋습니다.

&nbsp;&nbsp;&nbsp;c. 인증서 **요청 페이지에서** 외부 **에지** 인증서가 선택되어 있는지 확인한 후 다음 을 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;d. **지연 또는 즉시 요청** 페이지에서 **지금 요청을 준비하고 나중에 보냅니다** 를 클릭합니다.
    
&nbsp;&nbsp;&nbsp;e. 인증서 요청 **파일** 페이지에서 파일을 저장할 위치의 전체 부분과 파일 이름(예: c:\SkypeInternalEdgeCert.cer)을 입력합니다. **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;f. 대체 **인증서** 템플릿 지정 페이지에서 기본 WebServer 템플릿이 아니면 다른 템플릿을 사용하 고 선택한 인증 기관에 대체 인증서 템플릿 사용 **확인란을** 선택합니다.
    
&nbsp;&nbsp;&nbsp;g. 이름 및 보안 설정 페이지에서 다음을 수행합니다.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   i. 이름 **에** 인증서의 표시 이름(예: 외부 에지)을 입력합니다.
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  ii. 비트 **길이에서** 비트 길이를 선택하십시오(기본값은 2048, 더 높게 이동하여 더 안전할 수 있지만 성능이 느려질 수 있습니다).
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  iii. 내보낼 수 있는 인증서가 필요한 경우 인증서 개인 키를 내보낼 수 있는 것으로 **표시 확인란을 선택해야** 합니다.
    
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; iv. **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;h. 조직 **정보 페이지에서** 조직 및 OU(조직 구성 단위)의 이름을 입력합니다. 부서나 부서(예: IT)를 입력할 수 있습니다.
    
&nbsp;&nbsp;&nbsp;i. 지리적 **정보 페이지에서** 위치 정보를 입력합니다.

&nbsp;&nbsp;&nbsp;k. **주체 대체 이름(SANS)에 대한 SIP** 도메인 설정 페이지에서 도메인 확인란을 확인하여 sip를 추가합니다.\<sipdomain> 항목의 제목을 주체 대체 이름 목록으로 입력합니다.
    
&nbsp;&nbsp;&nbsp;l. 추가 **주체 대체** 이름 구성 페이지에서 필요한 주체 대체 이름을 추가해야 합니다.
    
&nbsp;&nbsp;&nbsp;m. 요청 **요약 페이지에서** 요청을 생성하는 데 사용할 인증서 정보를 살펴 봐야 합니다. 변경해야 하는 경우 돌아가서 지금 변경합니다.
    
&nbsp;&nbsp;&nbsp;n. 준비가 되면 다음을  클릭하여 CA에 제공해야 하는 CSR 파일을 생성합니다. 로그  보기를 클릭하여 인증서 요청에 대한 로그를 확인할 수도 있습니다.

&nbsp;&nbsp;&nbsp;p. (선택 사항) CSR의 콘텐츠를 제출할 때 다음과 같이 특정 정보를 요구할 수 있습니다(CAS는 크게 다르기 때문에 필요하지 않을 수 있음).
    
  - **서버** 플랫폼으로 Microsoft

### <a name="2-import-the-certificate"></a>2. 인증서 가져오기

&nbsp;&nbsp;&nbsp;a. 로컬 Administrators 그룹의 구성원으로 마지막 절차에서 인증서를 요청한 에지 서버에 로그온합니다.
    
&nbsp;&nbsp;&nbsp;b. 배포 마법사에서 **3단계 옆의 인증서 요청, 설치** 또는 할당 , 다시 **실행을 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;c. 사용 **가능한 인증서 작업 페이지에서** 에서 **인증서 가져오기 를 클릭합니다. P7b, .pfx 또는 .cer 파일**.
    
&nbsp;&nbsp;&nbsp;d. 인증서 **가져오기** 페이지에서 이전 섹션에 있는 인증서의 전체 경로와 파일 이름을 입력합니다. 또는 찾아보기를 클릭하여 파일을 찾아서 선택할 수 있습니다.  인증서에 개인 키가 포함되어 있는 경우 인증서 파일에 인증서의 개인 키가 포함되어 있는지 선택하고 개인 키의 암호를 입력해야 합니다. 준비되면 **다음을** 클릭합니다.
    
&nbsp;&nbsp;&nbsp;e. 인증서 **가져오기 요약 페이지에서** 요약 정보를 검토하고 다음 을 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;f. 명령 **실행** 페이지에서 로그 보기를 클릭하여 가져오기 완료 결과를 **검토할 수 있습니다.** **마친을** 클릭하여 인증서 가져오기 작업을 완료합니다.
    
&nbsp;&nbsp;&nbsp;g. 풀에 다른 에지 서버가 있는 경우 다음 두 절차도 따라야 합니다. 독립 실행형 에지 서버인 경우 외부 인증서를 사용할 수 있습니다.
    
 
### <a name="3-export-the-certificate"></a>3. 인증서 내보내기

&nbsp;&nbsp;&nbsp;a. 인증서를 가져온 에지 서버에 로컬 관리자로 로그온해야 합니다.
    
&nbsp;&nbsp;&nbsp;b. **시작,** **실행(또는** 검색 열기)을 클릭하고  **MMC 를 입력합니다.**
    
&nbsp;&nbsp;&nbsp;c. MMC 콘솔에서 파일 **을** 클릭한 다음 **스냅인 추가/제거를 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;d. 스냅인 **추가 또는** 제거 상자에서 인증서를 클릭하고 추가를 **클릭합니다.** 
    
&nbsp;&nbsp;&nbsp;e. 인증서 **스냅인** 대화 상자에서 컴퓨터 **계정 을 선택합니다.** **다음** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;f. 컴퓨터 **선택 대화** 상자에서 로컬 컴퓨터: (이 콘솔이 실행되고 **있는 컴퓨터)를 선택합니다.** **마침** 을 클릭합니다. **확인을** 클릭하면 MMC 콘솔의 구성이 완료됩니다.
    
&nbsp;&nbsp;&nbsp;g. **인증서(로컬** 컴퓨터)를 두 번 클릭하여 인증서 저장소를 확장합니다. **개인 을 두 번** 클릭한 다음 **인증서를 클릭합니다.**
    
   > [!NOTE]
   > 이 경우 로컬 컴퓨터의 인증서 개인 저장소에 인증서가 없습니다. 키가 없는 경우 가져온 인증서에 연결된 개인 키가 없는 경우 헌팅할 필요가 없습니다. 요청 및 가져오기 단계를 한 번 더 시도해 보시고 모든 권한이 있는지 확인된 경우 CA 관리자 또는 공급자에게 문의하세요. 
  
&nbsp;&nbsp;&nbsp;h. 로컬 **컴퓨터의** 인증서 개인 저장소에서 내보낼 인증서를 마우스 오른쪽 단추로 클릭합니다. **결과 메뉴에서** 모든 작업을 선택한 다음 내보내기 를 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;i. **인증서 내보내기 마법사** 에서 **다음** 을 클릭합니다. 예, **개인 키를 내보낼 수 있습니다.를 선택합니다.** **다음** 을 클릭합니다.
    
   > [!NOTE]
   > 예, **개인 키를 내보낼** 수 없는 경우 이 인증서의 개인 키가 내보내기 전에 내보내기로 표시되지 않은 것입니다. 이렇게 하기 전에 공급자에게 인증서를 다시 요청하고 개인 키를 내보내기로 설정해야 합니다.
  
&nbsp;&nbsp;&nbsp;j. 파일 형식 내보내기 대화 상자에서 개인 정보 Exchange - PKCS#12(를 선택합니다. PFX)를 선택하고 다음을 선택합니다.
    
 &nbsp;&nbsp;&nbsp;  i. 가능한 경우 인증 경로에 모든 인증서를 포함합니다.
    
 &nbsp;&nbsp;&nbsp;  ii. 모든 확장 속성을 내보낼 수 있습니다.
    
   > [!NOTE]
   > **내보내기에** 성공하면 개인 키 **삭제를 선택하지 않습니다.** 인증서 및 개인 키를 이 에지 서버로 다시 이 에지 서버로 다시 전송해야 합니다.
  
&nbsp;&nbsp;&nbsp;k. 개인 키를 보호하기 위해 암호를 할당하려는 경우 개인 키의 암호를 입력할 수 있습니다. 암호를 다시 입력하여 확인한 후 다음 을 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;l. 파일 확장명 **.pfx를** 사용하여 내보낼 인증서의 경로와 파일 이름을 입력합니다. 풀의 다른 에지 서버에서 경로에 액세스할 수 있도록 해야 합니다. 또는 외부 미디어(예: USB 드라이브)를 통해 파일을 이동해야 합니다. 선택한 **경우** 다음을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;m. 인증서 내보내기 마법사 완료 대화 상자의 **요약을** 검토하고 마친 을 **클릭합니다.**
    
&nbsp;&nbsp;&nbsp;n. 내보내기 성공 대화 상자에서 **확인** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;o. 이제 이전에 인증서 가져오기 섹션으로 돌아가 나머지 모든 에지 서버로 인증서를 가져온 다음 아래에서 할당을 진행해야 합니다.
    
 
### <a name="4-assign-the-certificate"></a>4. 인증서 할당

&nbsp;&nbsp;&nbsp;a. 각 **에지** 서버의 배포 마법사에서 **3단계 옆에 있는 인증서 요청, 설치** 또는 할당 , 다시 **실행을 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;b. 사용 **가능한 인증서 작업 페이지에서** 기존 인증서 **할당을 클릭합니다.**
    
&nbsp;&nbsp;&nbsp;c. 인증서 **할당 페이지에서** 목록에서 **Edge External를** 선택합니다.
    
&nbsp;&nbsp;&nbsp;d. 인증서 **저장소 페이지에서** 외부 에지(이전 섹션에서)에 대해 가져온 인증서를 선택합니다.
    
&nbsp;&nbsp;&nbsp;e. 인증서 **할당 요약 페이지에서** 설정을 살펴보고 **다음을** 클릭하여 인증서를 할당합니다.
    
&nbsp;&nbsp;&nbsp;f. 마법사 완료 페이지에서 **마침** 을 클릭합니다.
    
&nbsp;&nbsp;&nbsp;g. 이 절차를 완료한 후 각 서버에서 인증서 MMC 스냅인을 열고 인증서(로컬 **컴퓨터)** 및 개인을 확장하고 인증서를 클릭한 다음 내부 에지 인증서가 세부 정보 창에 나열되어 있는지 확인하는 것이 좋습니다.
    
   > [!NOTE]
   > 또한 역방향 프록시 서버에 대한 인증서를 설정해야 합니다. 
  
## <a name="starting-the-edge-servers"></a>에지 서버 시작

설치가 완료되면 배포의 각 에지 서버에서 서비스를 시작해야 합니다.
  
1. 각 에지 서버의 배포 마법사 **4단계:** 서비스 시작 옆의 실행을 **클릭합니다.** 
    
2. 서비스 **비즈니스용 Skype 서버 시작** 페이지에서 서비스 목록을 검토하고 **다음을** 클릭하여 서비스를 시작합니다.
    
3. 서비스가 시작된 후 마친을  클릭하여 마법사를 닫을 수 있습니다.
    
4. (선택 사항) 계속 **4단계:** 서비스 시작 아래에서 서비스 **상태를 클릭합니다.**
    
5.  각 **서버의 서비스 MMC에서** 모든 비즈니스용 Skype 서버 실행 중인지 확인해야 합니다.
