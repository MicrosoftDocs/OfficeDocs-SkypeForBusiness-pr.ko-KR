# <a name="plan-network-requirements-for-skype-for-business"></a>비즈니스용 Skype에 대 한 네트워크 요구 사항 계획

**요약:** 비즈니스용 Skype 서버를 구현 하기 전에 아래의 네트워크 구성 요소 고려 사항을 검토 하세요.

또한이 항목의 정보는 추가 세부 정보 및 깊이를 포함 하는 [Lync Server의 네트워크 계획, 모니터링 및 문제 해결](https://www.microsoft.com/download/details.aspx?id=39084) 백서에서 설명 합니다. 콘텐츠가 Lync 2010 및 Lync 2013를 명시적으로 참조 하는 동안 비즈니스용 Skype 서버에 대 한 고려 사항이 변경 되지 않습니다.

마찬가지로 네트워크에 wi-fi와 유선 액세스를 함께 사용 하는 경우 [wi-fi를 통한 Lync 2013 실시간 통신을 제공](https://www.microsoft.com/download/details.aspx?id=36494) 하는 것이 좋은 참조 이며 비즈니스용 Skype 서버에도 적용 됩니다.

## <a name="server-hardware"></a>서버 하드웨어
<a name="S_hard"> </a>

비즈니스용 Skype 서버 토폴로지에 있는 각 서버의 네트워크 어댑터는 초당 1 기가 비트 (Gbps)를 지원 해야 합니다. 일반적으로는 낮은 대기 시간 및 높은 대역폭 LAN (local area network)을 사용 하 여 비즈니스용 Skype 서버 토폴로지에 있는 모든 서버 역할을 연결 해야 합니다. LAN의 크기는 토폴로지의 크기에 따라 달라 집니다.

- Standard Edition 토폴로지에서 서버는 1Gbps 이더넷 또는 동급를 지 원하는 네트워크에 있어야 합니다.

- Enterprise Edition 토폴로지에서는 특히, A/V (오디오/비디오) 회의 및 응용 프로그램 공유를 지 원하는 경우 대부분의 서버가 1Gbps를 지 원하는 네트워크에 있어야 합니다.

공중 전화망(PSTN) 통합의 경우에는 T1/E1 회선 또는 SIP 트렁크를 사용하여 통합할 수 있습니다.

## <a name="audiovideo-network-requirements"></a>오디오/비디오 네트워크 요구 사항
<a name="AV_req"> </a>

비즈니스용 Skype 서버 배포에서 A/V (오디오/비디오)에 대 한 네트워크 요구 사항은 다음과 같습니다.

- DNS 부하 분산을 사용 하 여 단일에 지 서버 또는에 지 풀을 배포 하는 경우 NAT (network address translation)를 수행 하도록 _외부_ 방화벽을 구성할 수 있습니다. NAT를 수행 하도록 _내부_ 방화벽을 구성할 수는 없습니다. 자세한 내용은 [포트 및 방화벽 계획](../edge-server-deployments/edge-environmental-requirements.md#port-and-firewall-planning)을 참조 하십시오.

    > [!IMPORTANT]
    > 에 지 풀이 있고 하드웨어 부하 분산 장치를 사용 하는 경우에는에 지 서버에서 공용 IP 주소를 사용 해야 하며, 서버 또는 NAT 가능 장치 (예: 방화벽 기기 또는 LAN 스위치)에 대해 NAT를 사용할 수는 없습니다. 자세한 내용은 [비즈니스용 Skype 서버의에 지 서버 시나리오](../edge-server-deployments/scenarios.md)를 참조 하세요.

- 조직에서 QoS(서비스 품질) 인프라를 사용하는 경우 미디어 하위 시스템은 이러한 기존 인프라 내에서 작동하도록 설계됩니다.

- IPsec(인터넷 프로토콜 보안)를 사용하는 경우 A/V 트래픽에 사용되는 포트 범위에서 IPSec를 사용하지 않도록 설정하는 것이 좋습니다. 자세한 내용은 [IPsec 예외](#ipsec-exceptions)를 참조 하십시오.

최적의 미디어 품질을 제공 하려면 다음을 수행 합니다.

- 최대 사용 기간 동안 속도 (초당 65 킬로 비트) 및 비디오 스트림 당 500 Kbps의 처리량을 지원 하도록 네트워크 링크를 구축 합니다. 양방향 오디오 또는 비디오 세션에서는 두 개의 스트림을 사용 하므로 간단한 오디오/전화 연결을 위해서는 각 스트림에 130Kbps가 필요 합니다. 비디오는 또한 업스트림 및 다운스트림 연결을 수행 하는 데 1000 Kbps의 총을 사용 합니다.

- 트래픽의 예기치 않은 스파이크를 처리 하 고 시간이 지나면서 사용량이 증가 함에 따라 비즈니스용 Skype 서버 미디어 끝점은 다양 한 네트워크 조건에 적응할 수 있으며, 적절 한 품질을 유지 하면서도 오디오 및 비디오 처리량의 3 배를 지원 합니다. 네트워크가 프로 비전 되는 경우에도이 적응성에서 문제를 마스킹하는 것은 아닙니다. 프로 비전 된 네트워크에서 비즈니스용 Skype 서버 미디어 끝점이 동적으로 다양 한 네트워크 조건을 처리할 수 있는 기능 (예: 일시적인 높은 패킷 손실)이 줄었습니다.

- 프로 비전 비용이 매우 저렴 하 고 어려운 네트워크 링크의 경우 낮은 트래픽 양에 대해 프로 비전을 고려해 야 할 수 있습니다. 이 시나리오에서는 비즈니스용 Skype 서버 미디어 끝점이 트래픽 양과 최대 트래픽 수준 간의 차이를 회복 력 음성 품질이 약간 저하 될 수도 있습니다. 또한 사용량을 줄이고 트래픽이 갑자기 최고를 흡수 하도록 할 수도 있습니다.

- 소규모 용어로 올바르게 프로 비전 할 수 없는 링크 (예: WAN 링크를 매우 잘못 사용 하는 사이트)의 경우 특정 사용자에 대해 비디오를 사용 하지 않도록 설정 하는 것이 좋습니다.

- 네트워크를 프로 비전 하 여 최대 부하에 150 밀리초 (ms) 까지의 대기 시간 (밀리초)을 보장 합니다. 대기 시간은 비즈니스용 Skype 서버 미디어 구성 요소에서 줄일 수 없는 단일 네트워크 장애가 며 취약 한 점을 찾아서 제거 하는 것이 중요 합니다.

- 바이러스 백신 소프트웨어를 실행 하는 서버의 경우 예외 목록에서 비즈니스용 Skype 서버가 실행 되는 모든 서버를 포함 하 여 최적의 성능 및 오디오 품질을 제공 합니다.

## <a name="conferencing-network-requirements"></a>회의 네트워크 요구 사항
<a name="Conf_req"> </a>

IIS (인터넷 정보 서비스) 서버에서 전화 회의 콘텐츠를 다운로드 하는 데 사용 되는 대역폭은 콘텐츠의 크기에 따라 달라 집니다. 실제 사용량을 모니터링 하 고 대역폭 계획을 적절히 조정 하도록 선택할 수 있습니다.

## <a name="network-bandwidth-requirements-for-media-traffic"></a>미디어 트래픽에 대 한 네트워크 대역폭 요구 사항
<a name="Conf_req"> </a>

네트워크 계획에서 중요 한 부분은 네트워크에서 비즈니스용 Skype 서버에서 생성 된 미디어 트래픽을 처리할 수 있는지 확인 하는 것입니다. 이 섹션에서는 이러한 미디어 트래픽을 계획하는 데 도움을 줍니다.

비즈니스용 Skype 서버에서는 많은 해결 방법이 지원 됩니다. 이를 통해 비즈니스용 Skype 서버가 다양 한 네트워크 대역폭과 클라이언트 기능을 받을 수 있습니다. 비즈니스용 Skype 서버에 대 한 기본 가로 세로 비율은 16:9입니다. 레거시 4:3 가로 세로 비율은 웹캠에 대해 여전히 지원 되며,이는 16:9 가로 세로 비율로 캡처를 허용 하지 않습니다.

끝점에서는 오디오 또는 비디오 패킷을 지속적으로 스트리밍하지 않습니다. 시나리오에 따라 패킷이 스트림에 대해 전송되는 빈도를 나타내는 스트림 작업 수준이 다릅니다. 스트림 작업은 미디어 및 시나리오에 따라 달라지면 사용되는 코덱과는 관련이 없습니다. 피어 투 피어 시나리오의 경우:

- 끝점은 사용자가 말할 때만 오디오 스트림을 보냅니다.

- 두 참가자 모두 오디오 스트림을 받습니다.

- 비디오를 사용 하는 경우 두 끝점에서 통화 중에 비디오 스트림을 보내고 받습니다.

- 모든 참가자가 오디오 스트림을 받습니다.

## <a name="managing-quality-of-service"></a>서비스 품질 관리
<a name="man_QOS"> </a>

QoS (서비스 품질)는 오디오 및 비디오 통신을 위한 최적의 최종 사용자 환경을 제공 하는 데 도움이 되는 일부 조직에서 사용 되는 네트워킹 기술입니다. QoS는 대역폭이 제한 되는 네트워크에서 가장 자주 사용 되는 것으로, QoS는 사용 가능한 대역폭에 대해 경쟁 되는 많은 수의 네트워크 패킷이 있어 관리자가 오디오 또는 비디오 데이터를 운반 하는 패킷에 더 높은 우선 순위를 할당할 수 있습니다. 이러한 패킷에 더 높은 우선 순위를 부여 하면 오디오 및 비디오 통신은 파일 전송, 웹 검색, 데이터베이스 백업 등의 작업을 수행 하는 네트워크 세션 보다 더 빠르고 중단 될 수 있습니다. 이는 파일 전송 또는 데이터베이스 백업에 사용 되는 네트워크 패킷에 "최상의 작업" 우선 순위를 할당 하기 때문입니다.

> [!NOTE]
> 일반적으로 QoS는 내부 네트워크의 통신 세션에만 적용 됩니다. QoS를 구현할 때 인터넷 이나 다른 네트워크에서 지원 되지 않을 수 있는 특정 방식으로 패킷 표시를 지원 하도록 서버 및 라우터를 구성 합니다. 다른 네트워크에서 서비스 품질을 지 원하는 경우에도 QoS는 서비스를 구성한 것과 동일한 방식으로 구성 하는 것은 아닙니다. MPLS을 사용 하는 경우에는 MPLS 공급자와 함께 작업 해야 합니다.

비즈니스용 Skype 서버에는 QoS가 필요 하지 않지만 권장 됩니다. 네트워크에서 패킷 손실 문제가 발생 하는 경우 사용 가능한 해결 방법은 더 많은 대역폭을 추가 하거나 QoS를 구현 하는 것입니다. 더 이상 대역폭을 추가할 수 없는 경우에는 QoS를 구현 하 여 문제를 해결 해야 할 수 있습니다.

비즈니스용 skype 서버는 QoS에 대 한 완전 한 지원을 제공 합니다. 즉, 이미 QoS를 사용 중인 조직은 비즈니스용 Skype 서버를 기존 네트워크 인프라에 쉽게 통합할 수 있습니다. 이 작업을 수행 하려면 다음 단계를 수행 해야 합니다.

- [Windows를 기반으로 하지 않는 장치에 대해 비즈니스용 Skype 서버에서 QoS를 사용 하도록 설정](../../manage/network-management/qos/enabling-qos-for-devices-that-are-not-based-on-windows.md)합니다. 기본적으로 다른 운영 체제를 실행하는 컴퓨터 및 기타 장치(예: iPhone)에 대해서는 QoS를 사용하지 않도록 설정됩니다. 비즈니스용 Skype 서버를 사용 하 여 장치에 대 한 서비스 품질을 사용 하거나 사용 하지 않도록 설정할 수 있지만 일반적으로이 제품을 사용 하 여 이러한 장치에서 사용 되는 DSCP 코드를 수정할 수는 없습니다.

- 전화 [회의, 응용 프로그램 및 중재 서버에 대 한 서비스 품질 정책 및 포트 범위를 구성](../../manage/network-management/qos/configuring-port-ranges-for-your-conferencing-application-and-mediation-servers.md)합니다. 오디오 및 비디오와 같은 다른 패킷 유형에 대해 고유한 포트 집합을 예약 해야 합니다. 비즈니스용 Skype 서버를 사용 하 여 속성 값을 True 또는 False로 설정 하 여 QoS를 사용 하거나 사용 하지 않도록 설정 합니다. 대신 포트 범위를 구성한 다음 그룹 정책을 만들고 적용 하 여 QoS를 사용 하도록 설정 합니다. 나중에 QoS를 사용 하지 않기로 결정 한 경우 해당 하는 그룹 정책 개체를 제거 하 여 QoS를 "사용 하지 않도록 설정할 수 있습니다."

- 에 [지 서버에 대 한 포트 범위 및 서비스 품질 정책 구성](../../manage/network-management/qos/configuring-port-ranges-for-your-edge-servers.md) 반드시 필요한 것은 아니지만 다른 서버와 동일한 포트 범위를 사용하도록 에지 서버를 구성할 수 있습니다. QoS 정책만 구성 하는 것은에 지 서버의 내부에만 수행 해야 합니다. QoS는 인터넷이 아닌 내부 네트워크에서 사용 하도록 설계 되었기 때문입니다.

- [비즈니스용 Skype 서버에서 클라이언트에 대 한 포트 범위 및 서비스 품질 정책 구성](../../manage/network-management/qos/configuring-port-ranges-for-your-skype-clients.md) 이러한 포트 범위는 클라이언트 컴퓨터에만 적용 되며 일반적으로 서버에 구성 된 포트 범위와는 다릅니다. 비즈니스용 Skype 서버는 windows 10 이외의 Windows 운영 체제에 대해 QoS를 지원 하지 않습니다.


> [!NOTE]
> Windows Server 2012 또는 Windows Server 2012 R2를 사용 하는 경우 해당 플랫폼에서 QoS를 관리 하는 데 사용할 수 있는 새로운 Windows PowerShell cmdlet 집합에 관심이 있을 수 있습니다. 자세한 내용은 [Windows PowerShell의 Network QoS cmdlet](https://go.microsoft.com/fwlink/p/?LinkId=285379)를 참조 하세요.

또한 QoS는 추가 세부 정보 및 깊이를 포함 하는 [Lync Server를 사용한 네트워크 계획, 모니터링 및 문제 해결](https://www.microsoft.com/download/details.aspx?id=39084) 백서에서 설명 합니다. 콘텐츠가 Lync 2010 및 Lync 2013를 명시적으로 참조 하는 동안 비즈니스용 Skype 서버에 대 한 고려 사항이 변경 되지 않습니다.
