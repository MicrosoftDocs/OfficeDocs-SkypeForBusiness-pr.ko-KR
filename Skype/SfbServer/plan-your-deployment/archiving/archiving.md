# <a name="plan-for-archiving-in-skype-for-business-server"></a>2016년 8월에 보관 비즈니스용 Skype 서버
 
**요약:** 이 항목을 통해 보관을 계획하는 방법을 비즈니스용 Skype 서버.
  
특정 유형의 통신을 보존하도록 요구하는 회사 및 기타 조직에서 따라야 할 업계 및 정부 규정이 점차 증가하고 있습니다. 조직에 이러한 요구 사항이 있는 경우 조직의 보관을 비즈니스용 Skype 서버 IM(인스턴트 메시징) 및 회의(모임) 통신을 보관하여 일부 규정 준수 요구 사항을 지원할 수 있습니다.
  
## <a name="archiving-components"></a>보관 구성 요소

비즈니스용 Skype 서버 구성 요소는 다음과 같습니다.
  
- **보관 에이전트**. 보관 에이전트(통합 데이터 수집 에이전트라고도 알려지음)는 모든 프런트 엔드 풀 및 Enterprise Edition 서버에 자동으로 설치되고 Standard Edition 활성화됩니다. 보관 에이전트는 자동으로 활성화되어도 보관을 사용하도록 설정하고 적절하게 구성할 때까지 실제로 메시지가 캡처되지 않습니다. 기본적으로 보관은 사용하지 않도록 설정됩니다.
    
- **보관 데이터 저장소**. 데이터 저장소를 비즈니스용 Skype 서버 데이터베이스로 구현할 수도 비즈니스용 Skype 서버 SQL Server, Exchange 저장소와 통합된 Exchange 있습니다. 
    
또한 보관에는 파일 저장소가 필요하지만 보관은 프런트 엔드 서버 또는 서버와 동일한 파일 Standard Edition 사용 합니다.

  
## <a name="determine-your-organizations-requirements-for-archiving"></a>보관에 대한 조직 요구 사항 확인

보관을 구현하려면 다음을 결정하여 보관에 대한 조직의 요구 사항을 충족하는 방법을 결정해야 합니다.
  
- **어떤 저장소 옵션을 사용할지.** 다음 두 가지 방법 중 하나를 사용하여 저장소를 구현하거나 두 가지 방법을 조합하여 사용할 수 있습니다.
    
  - **Exchange 저장소.** Exchange 배포가 있는 경우 보관된 비즈니스용 Skype 서버 및 Exchange 보관을 통합하여 보관된 비즈니스용 Skype 서버 Exchange 보관 데이터를 Exchange. Microsoft Exchange 통합 옵션을 사용하도록 설정하면 사서함이 보관된 데이터에 Exchange Server Exchange 저장소를 사용하지만 사서함이 보류된 In-Place 사용합니다. 기본적으로 Microsoft Exchange 통합은 사용하도록 설정되어 있지 않습니다.
    
  - **비즈니스용 Skype 서버 저장소.** Exchange 또는 사서함이 In-Place Hold를 설정하지 않은 사용자가 있는 경우 또는 배포의 일부 또는 모든 사용자에 대해 Microsoft Exchange 통합을 사용하지 않을 경우 비즈니스용 Skype 서버 보관 데이터베이스를 배포할 수 SQL Server.
    
- **보관을 배포하는 경우.** 보관은 초기 비즈니스용 Skype 서버 배포의 일부로 배포하거나 기존 배포에 추가할 수 있습니다. 데이터베이스의 비즈니스용 Skype 서버 저장소(SQL Server)를 사용하려면 토폴로지 작성기에서 토폴로지에 데이터베이스를 추가한 다음 토폴로지 다시 게시합니다. 모든 사용자가 Exchange 사서함을 In-Place 유지로 설정한 경우 토폴로지 업데이트가 필요 없지만 Microsoft Exchange 통합을 사용하도록 설정하여 보관된 데이터를 Exchange. 
    
- **조직의 어느 사이트 및 사용자에게 보관이 필요한지.** 전체 조직 및 특정 사이트, 풀, 사용자 및 사용자 그룹에 대해 선택적으로 보관 설정을 구성할 수 있습니다.
    
- **보관할 콘텐츠** 보관을 전역 수준에서 또는 특정 사이트 및 사용자에 대해 지정하는 경우 각 수준에서 다음 유형의 콘텐츠를 사용하도록 설정할지 여부를 지정합니다. 
    
  - 피어 투 피어 인스턴트 메시지
    
  - 단체 인스턴트 메시지인 회의(모임)
    
  - 업로드된 콘텐츠(예: 참고 파일) 및 이벤트 관련 콘텐츠(예: 입장, 퇴장, 업로드 공유 및 표시 여부 변경)를 비롯한 회의 콘텐츠
    
  - 회의 중 공유된 화이트보드 및 설문
    
- **보관할 수 없는 콘텐츠** 다음과 같은 유형의 콘텐츠를 보관할 수 없습니다. 
    
  - 피어 투 피어 파일 전송
    
  - 피어 투 피어 인스턴스 메시지 및 회의에 대한 오디오/비디오
    
  - 피어 투 피어 인스턴스 메시지 및 회의에 대한 데스크톱 및 응용 프로그램 공유
    
    비즈니스용 Skype 서버 영구 채팅 대화도 보관하지 않습니다. 영구 채팅 대화를 보관하려면 영구 채팅 서버와 함께 배포할 수 있는 구성 요소인 준수 서비스를 사용하도록 설정하고 구성해야 합니다. 자세한 내용은 [Plan for Persistent Chat Server in 비즈니스용 Skype 서버 2015을 참조하세요.](../../plan-your-deployment/persistent-chat-server/persistent-chat-server.md)

    > [!NOTE] 
    > 영구 채팅은 비즈니스용 Skype 서버 2015에서 사용할 수 있지만 2019년 8월에는 더 이상 비즈니스용 Skype 서버 없습니다. 동일한 기능을 사용할 수 Teams. 자세한 내용은 업그레이드 시작을 [Microsoft Teams 참조하세요.](/microsoftteams/upgrade-start-here) 영구 채팅을 사용해야 하는 경우 이 기능이 필요한 사용자를 Teams 또는 비즈니스용 Skype 서버 2015를 계속 사용할 수 있습니다. 
    
- **보관된 재료의 보존 기간.** 보관 데이터베이스는 장기 보존을 위한 것이 아니며 비즈니스용 Skype 서버 데이터를 위한 전자 검색(검색) 솔루션을 제공하지 않습니다. 따라서 데이터를 다른 저장소로 이동해야 합니다. 비즈니스용 Skype 서버 보관된 데이터를 내보내고 보관된 데이터의 검색 가능한 기록을 만드는 데 사용할 수 있는 세션 내보내기 도구를 제공합니다. 
    
     글로벌 정책 및 만드는 각 사이트 및 사용자 정책에 대해 보관된 데이터 및 내보낼 데이터를 삭제하는 시간을 지정할 수 있습니다. 데이터 삭제에 대한 자세한 내용은 [Manage purging of archived data in 비즈니스용 Skype 서버.](../../manage/archiving/purging-of-archived-data.md) 세션 내보내기 도구를 사용하는 데 대한 자세한 내용은 에서 보관된 [데이터 내보내기 를 비즈니스용 Skype 서버.](../../manage/archiving/export-archived-data.md)
    
- **내부 또는 외부 통신을 보관할지 여부를 입니다.** 내부 통신(내부 사용자 간의 통신), 외부 통신(내부 네트워크 외부의 사용자를 한명 이상이 포함된 통신)에 대해 보관을 사용하도록 설정하거나 둘 다에 대해 보관을 사용하도록 설정할 수 있습니다. 전체 조직에 대해 이러한 옵션을 지정하거나 특정 사이트 및 풀에 대해 이러한 옵션을 지정할 수 있습니다. 기본적으로 두 옵션 모두 사용하도록 설정되지 않습니다.
    
    > [!NOTE]
    > 내부 또는 외부 통신에 대한 보관 제어는 정책에 비즈니스용 Skype 있습니다. 통합 Exchange 보관의 경우 내부 및 외부 통신이 모두 보관되거나 보관되지 않습니다. 
  
- **중요 모드를 구현할지 여부를 입니다.** 조직에 보관이 필요한 경우 중요 모드를 구성하면 보관을 방지하는 비즈니스용 Skype 서버 경우 IM 및 회의 세션이 차단됩니다. 예: 
    
  - 저장소 서비스에 비즈니스용 Skype 서버 문제입니다. 이 경우 보관을 사용하도록 설정된 사용자에 대해 IM이 차단됩니다.
    
  - 사용할 수 없는 파일 공유 또는 저장소 서비스에 문제가 있습니다. 이 경우 오류 시점에 풀에 호스팅된 모든 활성 회의가 제한 모드로 전환되고 새로운 회의를 활성화할 수 없습니다.
    
    오류가 수정된 다음에는 IM 및 회의 모두 자동으로 복구됩니다.
    
## <a name="choose-archiving-deployment-and-configuration-options"></a>보관 배포 및 구성 옵션 선택

보관은 서버를 배포할 때 각 프런트 엔드 서버에 자동으로 설치되지만 구성하기 전까지는 보관이 사용하도록 설정되지 않습니다. 보관을 구성하는 방식은 배포 방법에 따라 결정됩니다. 보관은 다음 방법 중 하나를 사용하여 배포할 수 있습니다.
  
- Microsoft Exchange 저장소 사용
    
- 저장소 비즈니스용 Skype 서버 사용
    
> [!NOTE]
> 비즈니스용 Skype 서버 보관 데이터베이스를 모두 구현하고 Microsoft Exchange 통합을 사용하도록 설정하는 경우 Exchange 정책은 비즈니스용 Skype 서버 보관 정책을 다시 설정하지만 Exchange에 있으며 사서함이 In-Place Hold에 있는 사용자에 한합니다. 비즈니스용 Skype 보관은 Microsoft Exchange In-Place 보류 정책에 따라 다를 수 있습니다. 
  
한 프런트 엔드 풀 또는 Standard Edition Server에 대해 보관을 배포하는 경우 배포의 다른 모든 프런트 엔드 풀 및 Standard Edition 서버에 대해 보관을 사용하도록 설정해야 합니다. 대화 또는 모임이 호스팅되는 풀에서 보관을 사용하도록 설정하지 않은 경우 모든 회의 데이터가 보관되지 않을 수 있습니다. 보관은 IM 메시지에 대해 계속 작동하지만 회의 콘텐츠 및 이벤트는 보관되지 않을 수 있습니다.
  
> [!NOTE]
> 조직의 보안 표준을 유지하면서 관리 작업을 위임할 수 있도록 비즈니스용 Skype 서버 RBAC(역할 기반 액세스 제어)를 사용합니다. RBAC를 사용할 경우 미리 정의한 관리 역할에 사용자를 할당하여 관리 권한이 부여됩니다. 비즈니스용 Skype 정책 및 구성을 구성하려면 사용자를 CsArchivingAdministrator 역할에 할당해야 합니다(보관이 배포된 서버에서 다른 컴퓨터에서 원격으로 수행되는 것이 아니라 직접 구성을 수행하지 않는 한). 보관 배포에 필요한 사용자 권한, 사용 권한 및 역할 목록은 [Deploy archiving for 비즈니스용 Skype 서버.](../../deploy/deploy-archiving/deploy-archiving.md) 
  
> [!NOTE]
> Microsoft Exchange 통합을 사용하는 경우 Exchange 정책을 구성하려면 적절한 관리자 권한이 필요합니다. 자세한 내용은 설명서를 Exchange 참조하십시오. 
  
### <a name="microsoft-exchange-storage"></a>Microsoft Exchange 저장소

 Microsoft Exchange 통합을 선택하는 경우 Exchange 정책 및 구성을 사용하여 정책 및 구성을 비즈니스용 Skype 서버. Microsoft Exchange 통합 옵션은 전역 수준, 사이트 수준 및 풀 수준에서 구성할 수 있습니다. 배포에 여러 포리스트가 포함된 경우 포리스트와 포리스트 간 설정을 비즈니스용 Skype 서버 Exchange. 다음을 결정해야 합니다.
  
- IM, 회의 또는 둘 다를 보관할지 여부
    
- 오류 발생 시 IM 및 회의 세션을 차단하는 중요 모드를 구현할지 비즈니스용 Skype 서버 여부 
    
- 보관된 데이터 저장을 위해 Exchange Microsoft Exchange 통합 옵션 선택
    
보관을 지원하도록 Exchange In-Place 정책 및 설정을 구성하는 방법에 대한 자세한 내용은 Exchange 설명서를 참조하십시오.
  
### <a name="skype-for-business-server-storage"></a>비즈니스용 Skype 서버 저장소

저장소를 비즈니스용 Skype 서버 경우 비즈니스용 Skype 서버 정책 및 구성을 사용하여 보관을 사용하도록 설정하고 구현하는 방법을 제어할 수 있습니다. 비즈니스용 Skype 서버 저장소는 SQL Server 사용 있으므로 토폴로지에 적절한 SQL Server 데이터베이스를 추가한 다음 보관 정책을 구성해야 합니다. 
  
### <a name="add-storage-databases-to-your-topology"></a>토폴로지에 저장소 데이터베이스 추가

토폴로지에 SQL Server 데이터베이스를 추가할 때 보관 데이터베이스를 다음 중 한 가지와 함께 사용할 수 있습니다.
  
- 모니터링 데이터베이스
    
- Enterprise Edition 프런트 엔드 풀의 백 엔드 데이터베이스
    
> [!NOTE]
> 보관 데이터베이스를 호스팅하는 서버가 다른 데이터베이스를 호스팅할 수 있습니다. 단, 보관 데이터베이스를 다른 데이터베이스와 함께 배치하려는 경우 많은 사용자의 메시지를 보관하면 보관 데이터베이스에 필요한 디스크 공간이 매우 커질 수 있다는 점에 유의해야 합니다. 따라서 보관 데이터베이스는 백 엔드 데이터베이스와 함께 배치하지 않는 것이 좋습니다. 
  
보관 데이터베이스를 모니터링 데이터베이스, 백 엔드 데이터베이스 또는 이 두 데이터베이스 모두에 함께 사용하면 데이터베이스의 일부 또는 전체에 대해 단일 SQL 인스턴스를 사용할 수도 있으며, 각 데이터베이스에 대해 별도의 SQL 인스턴스를 사용할 수도 있습니다. 각 SQL 인스턴스에는 단일 백 엔드 데이터베이스, 단일 모니터링 데이터베이스 및 단일 보관 데이터베이스만 포함될 수 있습니다.
  
모든 서버 역할 및 데이터베이스의 함께 사용에 대한 자세한 내용은 [Topology Basics for 비즈니스용 Skype 서버.](../../plan-your-deployment/topology-basics/topology-basics.md) 저장소 데이터베이스를 포함하기 위해 토폴로지 업데이트에 대한 자세한 내용은 에서 [새 토폴로지 만들기 및 게시를 비즈니스용 Skype 서버.](../../deploy/install/create-and-publish-new-topology.md)
  
### <a name="determine-archiving-options-and-user-policies"></a>보관 옵션 및 사용자 정책 결정

다음을 결정해야 합니다.
  
- 내부 및 외부 통신에 대해 보관을 사용할지 여부
    
- IM, 회의 또는 둘 다를 보관할지 여부
    
- 오류 발생 시 IM 및 회의 세션을 차단하는 중요 모드를 구현할지 비즈니스용 Skype 서버 여부 
    
- 특정 사용자 및 그룹에 대해 정책을 사용하도록 설정할지 여부
    
비즈니스용 Skype 서버 보관 옵션은 다음 수준에서 지정할 수 있습니다. 
  
- **전역 수준 옵션 입니다.** 이 구성은 기본 보관 구성으로, 전체 배포에 적용됩니다. 이 매크로는 배포할 때 비즈니스용 Skype 서버 내부 및 외부 통신 모두에 대해 보관을 사용하지 않도록 설정됩니다. 이 옵션은 삭제할 수 없습니다. 삭제 옵션을 선택하면 전역 옵션이 기본 설정으로 다시 설정됩니다.
    
- **사이트 수준 옵션**. 사이트에 대한 사이트 수준 보관 옵션을 만들고 구성하여 하나 이상의 특정 사이트에 대해 보관을 사용하도록 설정하거나 사용하지 않도록 설정할 수 있습니다. 만든 모든 사이트 수준 보관 옵션을 삭제할 수 있습니다. 사이트 수준 보관 옵션은 옵션에 지정된 사이트에 한해 전역 옵션을 다시 지정합니다. 
    
    예를 들어 전역 구성에서 내부 및 외부 통신에 대해 보관을 사용하도록 설정하고 외부 통신에 대해 보관을 사용하지 않도록 설정하는 사이트 구성을 만드는 경우 해당 사이트에 대해 내부 통신만 보관됩니다. 또 다른 예로, 전역 구성에서 IM에만 보관을 사용하도록 설정하고 IM 및 회의 모두에 대해 보관을 사용하도록 설정하는 사이트 구성을 만드는 경우 회의는 조직의 나머지가 아니라 사이트에만 보관됩니다.
    
- **풀 수준 옵션.** 개별 풀에 대해 풀 수준 구성을 만들고 구성하여 하나 이상의 특정 풀에 대한 보관 설정을 지정할 수 있습니다. 풀 수준 보관 구성은 만들 때만 존재합니다. 풀 수준 보관 구성을 수정하고 삭제할 수 있습니다. 풀 수준 보관 구성은 만들 수 있는 모든 사이트 보관 구성 및 전역 구성을 다시 구성합니다. 
    
    예를 들어 전역 구성에서만 IM에 대해 보관을 사용하도록 설정했다고 가정하고 IM 및 회의 모두에 대해 보관을 사용하도록 설정하는 사이트 수준 구성을 만든 다음 IM에만 보관을 사용하도록 설정하는 풀 수준 구성을 만들 수 있습니다. 풀 수준 구성에 지정된 풀에 있는 사용자를 제외한 모든 사이트의 사용자에 대해 IM 및 회의에 대한 통신이 보관됩니다. 조직의 다른 모든 사용자에 대해 보관은 IM에만 사용하도록 설정됩니다.
    
- **사용자 보관 정책**. 지정된 사용자 및 사용자 그룹에 대한 사용자 수준 보관 정책을 만들고 구성하고 적용하여 하나 이상의 특정 사용자 및 사용자 그룹에 대해 보관을 사용하도록 설정하거나 사용하지 않도록 설정할 수 있습니다. 만든 사용자 수준 보관 정책을 삭제할 수 있으며 보관 정책이 적용되는 사용자 및 사용자 그룹을 변경할 수 있습니다. 사용자 수준 보관 정책은 글로벌 정책 및 사이트 정책을 다시 설정하지만 정책이 적용되는 사용자 및 사용자 그룹에 한해 적용됩니다. 
    
    예를 들어 전역 구성에서 내부 및 외부 통신에 대해 보관을 사용하지 않도록 설정하고, 내부 및 외부 통신에 대해 보관을 사용하도록 설정하는 사이트 수준 정책을 만든 다음 외부 통신에 대해 보관을 사용하지 않도록 설정하는 사용자 수준 정책을 만드는 경우를 가정해 보겠습니다. 사용자 수준 정책을 적용하는 사용자를 제외한 모든 사이트 사용자에 대한 외부 및 내부 통신에 대한 통신은 보관됩니다. 이러한 사용자에 대한 내부 통신만 보관됩니다.
    
보관을 배포할 때 초기 보관 구성을 설정하는 방법에 대한 자세한 내용은 [Deploy archiving for 비즈니스용 Skype 서버.](../../deploy/deploy-archiving/deploy-archiving.md) 배포 후 보관 관리에 대한 자세한 내용은 [Manage archiving in 비즈니스용 Skype 서버.](../../manage/archiving/archiving.md) 
  
## <a name="archiving-configuration-tools"></a>보관 구성 도구

 대부분의 보관 옵션은 제어판을 사용하여 비즈니스용 Skype 서버 제어합니다. 그러나 관리 셸에서만 사용할 수 있는 몇 가지 비즈니스용 Skype 서버 있습니다. 이러한 옵션에는 중복 메시지 보관 및 보관된 데이터 내보내기가 포함됩니다. 비즈니스용 Skype 서버 제어판 및 비즈니스용 Skype 서버 관리 셸을 사용하여 보관 정책을 관리하는 데 대한 자세한 내용은 [Manage archiving in 비즈니스용 Skype 서버.](../../manage/archiving/archiving.md)
  
## <a name="access-archived-data"></a>보관된 데이터에 액세스

보관된 데이터에 대한 액세스는 데이터가 저장된 위치에 따라 달라집니다. 
  
- **Microsoft Exchange 저장소 입니다.** Exchange 통합 옵션을 선택하는 경우 비즈니스용 Skype 서버 보관 콘텐츠를 Exchange 저장소에 보관 콘텐츠를 저장하여 Exchange 사용자의 사서함을 In-Place 보류합니다. 보관된 데이터는 일반적으로 사용자에게 표시되지 않는 복구 가능한 항목 폴더에 저장되고 검색 관리 역할이 있는 사용자만 Exchange **수** 있습니다. Exchange 배포된 경우 SharePoint 검색과 함께 페더전된 검색을 사용할 수 있습니다. Exchange 저장, 보존 및 검색에 대한 자세한 내용은 Exchange 및 SharePoint 참조하십시오.
    
- **비즈니스용 Skype 서버 저장소입니다.** 비즈니스용 Skype 서버 보관 데이터베이스를 설정하는 경우 비즈니스용 Skype 서버 사서함이 비즈니스용 Skype 서버에 있지 않은 사용자와 사서함을 보류하지 않은 사용자에 대해 비즈니스용 Skype 서버 Exchange 보관 데이터베이스에 보관 콘텐츠를 In-Place 저장합니다. 이 데이터는 검색할 수 없지만 다른 도구를 사용해서 검색할 수 있는 형식으로 내보낼 수 있습니다. 보관 데이터베이스에 저장된 데이터를 내보내는 데 대한 자세한 내용은 에서 보관된 데이터 [내보내기 비즈니스용 Skype 서버.](../../manage/archiving/export-archived-data.md)
