# <a name="plan-for-group-call-pickup-in-skype-for-business"></a>비즈니스용 Skype의 그룹 통화 선택 계획
 
사용자가 원래 다른 사용자를 위해 의도한 통화에 응답할 수 있도록 하는 비즈니스용 Skype Enterprise Voice 그룹 통화 선택 계획
  
그룹 통화 Pickup을 사용하면 자신의 휴대폰에서 동료에게 걸려오는 전화에 응답할 수 있습니다. 이렇게 하면 통화 수신 그룹 번호로 전화를 걸면 다른 사용자가 수신 전화에 응답할 수 있으므로 사용자 줄의 가용성이 향상됩니다. 그룹 통화 수신이 배포되면 음성 메일로 라우팅되는 수신 전화 수가 크게 감소할 수 있습니다. 이는 조직 외부의 고객의 통화에 특히 유용합니다.
  
그룹 통화 선택 기능은 열려 있는 사무실 환경의 업무 단위에 특히 적합합니다. 수신 전화는 의도한 대상에서만 울리기 때문에 중단되지 않습니다. 그러나 링을 듣는 다른 사용자는 그룹 번호로 전화를 걸기만 하여 전화를 걸 수 있습니다. 
  
사용자가 열린 사무실 레이아웃에 있지 않은 환경이나 공통 책임을 공유하는 사용자가 지리적으로 분산되어 있는 환경에서 팀 통화는 가장 적합한 솔루션을 제공합니다. 그룹 통화 수신과 팀 통화의 주요 차이점은 그룹 통화 Pickup을 사용할 경우 수신 전화가 의도한 대상에서만 벨이 울리지만 누구나 그룹 번호로 전화를 걸고 전화를 걸 수 있습니다. 팀 통화를 통해 통화가 모든 팀 구성원의 전화에서 울리며, 팀의 모든 사용자는 전화를 통해 통화에 응답할 수 있습니다. 그룹 통화 선택과 팀 통화 간의 추가적인 차이점은 그룹 통화 Pickup은 관리자가 비즈니스용 Skype 서버를 통해 관리하다는 것입니다. 팀 통화를 사용하면 최종 사용자가 비즈니스용 Skype 클라이언트를 사용하여 기능을 관리합니다. 따라서 그룹 통화 Pickup을 사용하여 이러한 통화 관리 측면을 중앙 집중화할 수 있습니다.
  
그룹 통화 선택은 통화 파크 응용 프로그램을 통해 구축됩니다. 그룹 통화 선택을 배포할 때 통화 Pickup 그룹 번호로 지정된 별도의 내선 번호 범위로 통화 파크 파크 번호 테이블을 구성합니다. 통화 걸기 그룹 번호와 마찬가지로, 통화 선택 그룹 번호는 사용자에게 할당된 사용자나 전화가 없는 가상 내선 번호입니다. 그룹 통화 선택을 배포하는 각 프런트 엔드 풀에는 하나 이상의 통화 선택 그룹 번호 범위가 있습니다. 그룹 번호 범위는 비즈니스용 Skype 서버 배포에서 전역적으로 고유해야 합니다. 
  
> [!NOTE]
> 통화 파기 번호 테이블에서 그룹 통화 선택 번호로 지정된 번호 범위는 비즈니스용 Skype 서버 제어판을 사용하여 관리하거나 볼 수 없습니다. 통화 파기 번호 테이블의 모든 번호 범위를 보는 유일한 방법은 비즈니스용 Skype 서버 관리 셸을 사용하는 것입니다. 마찬가지로 그룹 통화 선택 번호를 추가, 수정 또는 제거하는 유일한 방법은 비즈니스용 Skype 서버 관리 셸을 사용하는 것입니다. 
  
통화 선택 그룹 번호를 구성한 후 사용자를 통화 Pickup 그룹에 할당합니다. 통화 선택 그룹에 할당된 사용자는 다른 사용자가 전화를 걸 수 있습니다. 통화를 Pickup 그룹에 할당된 사용자에게 전화를 걸면 통화를 발견한 다른 사용자가 수동으로 통화 Pickup 그룹 번호로 전화를 걸면 해당 통화에 응답할 수 있습니다. 통화를 선택한 사용자는 그룹의 구성원이 아니어도 됩니다. 다른 사용자가 전화를 걸면 원래 호출된 번호로 알림이 전송됩니다.
  
> [!NOTE]
> 사용자는 하나의 통화 Pickup 그룹의 구성원일 수 있습니다. 
  
> [!NOTE]
> 비즈니스용 Skype 서버 배포의 모든 사용자가 통화 Pickup 그룹 구성원에게 전화를 걸 수 있는 경우 전화를 거는 사람은 전화를 걸 올바른 통화 Pickup 그룹 번호를 알고 있어야 합니다. 
  
그룹의 여러 전화가 울리는 경우 사용자가 통화에 응답하기 위해 통화 Pickup 그룹 번호로 전화를 걸면 가장 긴 벨이 울리는 통화에 응답합니다.
  
동시 전화 울림 설정은 그룹 통화 Pickup이 있는 사용자에 대해 작동됩니다. 즉, 그룹 통화 Pickup이 있는 사용자에게 걸린 통화가 구성된 모든 대상에 대해 벨이 울리며 다른 사용자가 통화에 응답할 수 있습니다. 이 규칙의 예외는 사용자가 모든 팀 구성원에게 전화를 걸도록 동시 벨 울림을 구성한 경우입니다.
  
그룹 통화 Pickup을 사용하여 다음 유형의 통화에 응답할 수 없습니다.
  
- 전용선 호출
    
- 친구 및 가족 개인 정보 보호 관계가 할당된 연락처의 통화
    
    > [!TIP]
    > 통화 선택 그룹의 구성원인 사용자는 연락처를 비즈니스용 Skype 클라이언트에서 개인 연락처로 표시하여 그룹 통화 수집을 통해 특정 통화가 검색되지 않도록 할 수 있습니다. 연락처를 개인 연락처로 표시하기 위해 연락처의 개인 정보 관계를 친구 및 가족으로 설정하세요. 개인 정보 관계가 Friends 및 Family로 설정된 연락처로부터의 수신 전화는 그룹 통화 Pickup을 사용하여 검색할 수 없습니다. 
  
- 오디오/비디오 통화의 비디오 부분 
    
    > [!NOTE]
    > 사용자가 오디오/비디오 통화에 응답하면 사용자는 오디오만 수신합니다. 통화를 하는 사람 또는 통화에 응답한 사람이 전화를 에스컬레이터하여 비디오를 추가할 수 있습니다. 
  
- 팀 통화 구성원에게 라우팅된 동시 전화 울림 통화
    
- 대리인에게 라우팅된 통화
    
- 응답 그룹으로 라우팅된 통화
    
다음 유형의 사용자는 그룹 통화 Pickup에 참가할 수 없습니다. 즉, 그룹 통화 선택 그룹에 포함되지 말고 그룹 통화 Pickup을 사용하도록 설정된 사용자에 대한 통화를 선택할 수 없습니다.
  
- 하이브리드 배포에서 온라인에 있는 사용자
    
- 비즈니스용 Skype 서버 2015 풀 또는 Lync Server 2013용 누적 업데이트가 있는 Lync Server 2013 풀에 있지 않은 사용자: 온-프레미스 배포의 2013년 2월
    
통화 선택 그룹의 구성원에게 전화를 응답하는 사용자가 없는 경우 통화는 클라이언트 설정에 지정된 것으로 라우팅됩니다. 즉, 통화가 음성으로 이동되거나 클라이언트 설정에 지정된 다른 대상으로 전달됩니다.
  
## <a name="deployment-and-requirements"></a>배포 및 요구 사항

그룹 통화 선택은 통화 파크 응용 프로그램을 배포할 Enterprise Voice 자동으로 배포됩니다. 통화 선택 그룹 번호로 지정된 별도의 번호 범위로 통화 파킹된 통화 번호 테이블을 구성한 다음 사용자에게 Pickup 그룹에 통화를 할당하고 사용자가 그룹 통화 선택을 사용하도록 설정하여 그룹 통화 선택을 사용하도록 설정할 수 있습니다.
  
## <a name="clients-supported-for-group-call-pickup"></a>그룹 통화 Pickup에 지원되는 클라이언트

다음 클라이언트를 사용하여 그룹 통화 Pickup 구성원에 대한 통화에 응답할 수 있습니다.
  
- 비즈니스용 Skype
    
- Lync 2013
    
- Lync 2010
    
- Lync Phone Edition
    
> [!NOTE]
> 사용자는 이러한 클라이언트를 사용하여 그룹 통화 Pickup 구성원에 대한 통화에 응답할 수 있지만 사용자는 Lync Server 2013용 누적 업데이트가 있는 비즈니스용 Skype 서버 풀 또는 Lync Server 2013 풀에 있어야 합니다. 
  
다음 클라이언트 및 장치는 그룹 통화 Pickup 구성원에 대한 통화를 선택할 수 없습니다.
  
- Lync Mobile
    
- Windows 8 및 Windows RT용 Lync 앱
    
- Lync for iPad
    
- 아날로그 전화
    
- PSTN(Public Switched Telephone Network) 번호가 있는 전화
    
## <a name="capacity-planning"></a>용량 계획

다음 표에서는 용량 계획 요구 사항의 기준으로 사용할 수 있는 그룹 통화 Pickup 사용자 모델에 대해 설명합니다.
  
> [!IMPORTANT]
> 그룹 통화 Pickup은 통화 파크 응용 프로그램을 기반으로 합니다. 재해 복구 용량 계획을 위해 페어링된 풀의 각 풀은 두 풀 모두에서 그룹 통화 선택을 비롯한 통화 파크 서비스에 대한 작업을 처리할 수 있습니다. 
  
**그룹 통화 선택 사용자 모델**

|**메트릭**|**프런트 엔드  <br/>  풀당(프런트 엔드 서버 8대)**|**Standard Edition Server별**|
|:-----|:-----|:-----|
|그룹당 권장되는 사용자 수  <br/> |50  <br/> |50  <br/> |

|그룹 통화 Pickup을 사용하도록 설정된 풀당 최대 사용자 수  <br/> |25,000  <br/> |3,000  <br/> |
|풀당 분당 그룹 통화 수신을 사용하도록 설정된 총 사용자에 대한 최대 수신 전화 속도  <br/> |500  <br/> |60  <br/> |

> 프런트 엔드 서버가 8대 미만인 프런트 엔드 풀의 경우 메트릭을 선형으로 계산합니다. 예를 들어 프런트 엔드 풀에 프런트 엔드 서버가 하나인 경우 최대 부하를 표에 표시된 값의 1/8로 계산합니다. 
  
> [!NOTE]
> 풀당 최대 사용자 수를 초과하지 않는 한 그룹 및 그룹 수에 따라 권장되는 사용자 수를 늘리거나 줄이면 됩니다. 예를 들어, 그룹 통화 선택을 사용하도록 설정된 사용자 수가 여전히 사용자 모델 최대값 내에 있기 때문에 Standard Edition 서버는 그룹당 25명이 있는 120개 그룹을 사용할 수 있습니다. 즉, 120개 그룹에서 25명 사용자가 그룹 통화 선택을 사용하도록 설정된 경우 3,000명입니다.
