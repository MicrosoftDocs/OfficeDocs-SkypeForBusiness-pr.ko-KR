# <a name="plan-for-mobility-for-skype-for-business-server"></a>비즈니스용 Skype 서버에 대 한 이동성 계획
 
비즈니스용 Skype 서버에 대 한 이동성 구현 계획
  
비즈니스용 Skype 서버를 사용 하 여 모바일 장치에서 비즈니스용 Skype 서버 기능을 제공 하는 이동성 기능을 배포할 수 있습니다. 이 문서에서는 모바일 기능에 대 한 세부 정보를 제공 하 고 배포를 계획 하는 데 도움을 줍니다.
  
비즈니스용 Skype 서버에 대 한 모바일 클라이언트는 비즈니스용 Skype (비즈니스 용) 및 Lync 클라이언트 (2010으로 다시 전환)를 지원할 수 있습니다. 배포 된 사용자는 지원 되는 iOS, Android, Windows Phone 모바일 장치를 사용 하 여 비즈니스용 Skype 서버 배포에 연결 하 여 엔터프라이즈 음성 기능을 비롯 한 다양 한 기능을 활용할 수 있습니다. 아래에는 일부 목록이 포함 되어 있으며 자세한 내용은 비즈니스용 [Skype에 대 한 데스크톱 클라이언트 기능 비교](clients-and-devices/desktop-feature-comparison.md) 를 확인할 수도 있습니다.
  
- 메시지 보내기 및 받기
    
- 현재 상태 보기
    
- 연락처 보기
    
- 클릭 하 여 전화 회의에 참가
    
- 직장을 통한 통화
    
- 단일 번호 도달
    
- 음성 메일
    
- 부재 중 전화 알림
    
- VoIP (Voice over IP)
    
- 참석자 비디오 (H. 264)
    
- 모임 콘텐츠 보기 (PowerPoint 및 데스크톱/응용 프로그램 공유)
    
이러한 모든 작업은 통합 커뮤니케이션 웹 API 또는 작업을 통해 수행 됩니다. S e e는 Lync Server 2013에서 처음 소개 되었으며, 아직 비즈니스용 Skype Server에 사용 중입니다. Lync 2010 클라이언트와 통신 하는 추가 기능, 그리고 해당 이동성 서비스 (MCX)가 있습니다. 이러한 서비스는 Lync Server 2010 및 2013 클라이언트와 비즈니스용 Skype 클라이언트를 허용 하 고 비즈니스용 skype 서버 배포에 액세스 하는 데 사용 됩니다.
  
> [!NOTE]
> 이전 모바일 클라이언트에 대 한 MCX (Mobility Service) 지원은 더 이상 비즈니스용 Skype 서버 2019에서 사용할 수 없습니다. 현재 모든 비즈니스용 Skype 모바일 클라이언트는 이미 통합 커뮤니케이션 웹 API (인스턴트 메시징 (IM), 현재 상태, 대화 상대 지원)를 사용 합니다. MCX를 사용 하는 레거시 클라이언트가 있는 사용자는 현재 클라이언트로 업그레이드 해야 합니다.
  
이동성을 구현한 후에도 이러한 기능을 모두 사용할 수 있는 반면 일부 장치에서는 약간 다르게 작동할 수 있다는 점을 기억해 야 합니다. [비즈니스용 Skype에 대 한 모바일 클라이언트 기능 비교](clients-and-devices/mobile-feature-comparison.md)에서 어떤 장치에서 작동 하는지 설명 하는 웹 사이트를 얻었습니다. 또한 [클라이언트 및 장치 계획](clients-and-devices/clients-and-devices.md)에 몇 가지 유용한 장치 및 OS 정보가 있습니다.
  
이동성은 자동 검색 기능을 사용 하 여 사용자가 Url에 입력을 요구 하지 않아도 클라이언트가 비즈니스용 Skype Server web services를 찾을 수 있도록 합니다. 일부 문제 해결을 수행 해야 하는 경우에는 Url 수동 입력이 여전히 지원 됩니다.
  
푸시 알림은 비즈니스용 Skype 앱이 백그라운드에서 실행 되 고 있지 않은 경우 (또는 백그라운드에서 실행 되는 응용 프로그램을 지원 하지 않는 모바일 장치)에도 지원 됩니다. 장치 또는 앱이 비활성 상태일 때 발생 하는 이벤트에 대 한 푸시 알림을 모바일 장치에 보냅니다. 휴대폰을 사용할 수 없는 경우에는 메신저 메시지가 표시 되지 않으며,이로 인해 푸시 알림이 전송 됩니다 (이는 앱이 백그라운드에서 실행 되는 경우와 같이 알림 또는 알림으로 제공 됨). 푸시 알림을 사용 하면 사용자가 IM 또는 음성 통화를 놓치지 않습니다.

## <a name="mobility-components"></a>모바일 구성 요소
<a name="MobilityComponents"> </a>

비즈니스용 Skype 서버용 모바일 기능을 구성 하는 4 가지 서비스는 다음과 같습니다.
  
- **통합 커뮤니케이션 웹 API (c)**
    
    비즈니스용 Skype 서버용 모바일 및 웹 클라이언트를 사용 하 여 실시간으로 통신 하는 서비스를 제공 합니다. 비즈니스용 Skype 서버를 배포 하는 경우 내부 및 외부 웹 서비스에서 생성 된 가상 디렉터리입니다. 이 가상 디렉터리의 가상 구성 요소를 사용 하 여 인천 wa 클라이언트의 통화를 수락 합니다. 클라이언트 앱은 다음에 대 한 REST (representational state transfer) 인터페이스를 통해 통신 합니다.
    
  - 늘어
    
  - 상대가
    
  - 인스턴트 메시지 (IM)
    
  - P
    
  - 영상 회의
    
  - 협동
    
    (A) P-GET 기반 채널을 사용 하 여 수신 전화, 받는 메신저 대화 또는 클라이언트 앱에 대 한 메시지 등의 이벤트를 보냅니다.
    
- **모바일 서비스 (MCX)**
    
    모바일 장치에서 IM, 현재 상태, 대화 상대와 같은 비즈니스용 Skype 서버 기능을 지원 합니다. 모바일 장치에서 비즈니스용 Skype Server 기능을 지원 하기 위해 만든 각 풀의 모든 프런트 엔드 서버에 모바일 서비스가 설치 되어 있습니다. 비즈니스용 Skype Server 2015을 설치 하면 프런트 엔드 서버의 내부 및 외부 웹 사이트에 새 가상 디렉터리 (Mcx)가 만들어집니다.
    
    > [!NOTE]
    > 이전 모바일 클라이언트에 대 한 MCX (Mobility Service) 지원은 더 이상 비즈니스용 Skype 서버 2019에서 사용할 수 없습니다. 현재 모든 비즈니스용 Skype 모바일 클라이언트는 이미 통합 커뮤니케이션 웹 API (인스턴트 메시징 (IM), 현재 상태, 대화 상대 지원)를 사용 합니다. MCX를 사용 하는 레거시 클라이언트가 있는 사용자는 현재 클라이언트로 업그레이드 해야 합니다.
  
- **자동 검색 서비스**
    
    네트워크 위치에 관계 없이 사용자의 위치를 식별 하 고 모바일 장치 및 기타 Skype for Business 클라이언트를 사용 하 여 리소스 (비즈니스용 Skype Server Web Services, Mcx URL 또는 XWA URL의 내부 및 외부 URL)를 찾습니다. 자동 검색은 하드 코드 된 호스트 이름 (네트워크 내부의 사용자에 게 lyncdiscoverinternal, 네트워크 외부 사용자를 위한 lyncdiscover) 및 사용자의 SIP 도메인을 사용 합니다. HTTP 또는 HTTPS 중 하나를 사용 하는 클라이언트 연결을 지원 합니다. 
    
    자동 검색 서비스는 모바일 장치에서 비즈니스용 Skype Server 기능을 지원 하기 위해 만든 각 풀의 모든 프런트 엔드 서버와 모든 디렉터에 설치 됩니다. 서비스를 설치 하면 프런트 엔드 서버 및 디렉터에 있는 내부 및 외부 웹 사이트에 새 자동 검색 (virtual directory)이 만들어집니다.
    
- **푸시 알림 서비스**
    
    비즈니스용 Skype Online 데이터 센터에 있는 클라우드 기반 서비스입니다. 백그라운드에서 실행 되는 비즈니스용 Skype 클라이언트가 없는 휴대폰에서는 새 이벤트가 발생할 때 부재 중 이벤트 (푸시 알림)의 알림이 모바일 장치로 전송 됩니다. 모바일 서비스는 푸시 알림 서비스 (MPNS)에 알림을 전송 하 고이를 모바일 장치에 보냅니다. 그러면 사용자가 모바일 장치에서 알림에 응답 하 여 앱을 정품 인증할 수 있습니다. 이 기능을 위해서는 Edge 서버가 필요 합니다.
    
## <a name="supported-topologies"></a>지원되는 토폴로지
<a name="SupportedTopos"> </a>

토폴로지 계획을 위해 다음과 같은 지원 되는 비즈니스용 Skype 서버 응용 프로그램이 있습니다.
  
- 모바일 표준 버전
    
- Mobility Enterprise Edition
    
이 기능은 비즈니스용 Skype 서버에 지 서버 또는 Lync Server 2013 Edge 서버에서 사용할 수 있어야 합니다.
  
모바일 서비스는 두 개의 네트워크 인터페이스를 사용 하 여 중재 서버 역할을 collocated 때 프런트 엔드 서버에서 지원 되지만 해당 인터페이스를 구성 하는 적절 한 단계를 수행 해야 합니다. 중재 서버로 통신 하는 특정 인터페이스에 IP 주소를 할당 하 고 프런트 엔드 서버로 통신 하는 네트워크 IP 인터페이스를 지정 해야 합니다. 기본 **설정 된 모든 IP 주소** 를 사용 하는 대신 각 서비스에 대 한 올바른 IP 주소를 선택 하 여 토폴로지 작성기에서이 작업을 수행할 수 있습니다.
  
## <a name="technical-requirements"></a>기술 요구 사항
<a name="TechRequirements"> </a>

모바일 사용자에 게 발생할 수 있는 다양 한 모바일 응용 프로그램 시나리오를 계획 하는 것이 중요 합니다. 예를 들어 누군가가 3G 네트워크를 통해 연결 하 여 작업 외부에서 모바일 앱을 사용할 수 있는 경우, 회사 Wi-fi 네트워크로 전환 하 여 작업에 도달 하면 됩니다. 건물을 떠나는 동안 4G 전환할 수 있습니다. 지금 계획을 수립 하면 이러한 네트워크 전환을 지원 하 고 일관 된 사용자 환경을 보장할 수 있습니다.
  
### <a name="dns-configuration"></a>DNS 구성

모바일 서비스 Mcx와 함께 DNS를 같은 방식으로 사용 합니다. 자동 검색을 사용 하면 모바일 장치에서 DNS를 사용 하 여 리소스를 찾습니다. DNS 조회 중에 내부 DNS 레코드 (lyncdiscoverinternal)와 연결 된 FQDN에 대 한 연결이 시도 되었습니다 ( 내부 도메인 이름]). 내부 DNS 레코드를 연결 하는 데 사용할 수 없는 경우에는이 시간이 외부 DNS 레코드에 대 한 두 번째 연결이 시도 됩니다 (lyncdiscover. [ sipdomain]). 2 개를 사용 하는 이유는 무엇 인가요? 네트워크 내부의 모바일 장치에서 내부 자동 검색 URL을 사용할 수 있습니다. 외부 모바일 장치에서 외부 자동 검색 URL을 사용 합니다. 두 경우 모두 자동 검색 서비스는 모바일 서비스 (Mcx 및 Xwa)를 포함 하 여 사용자의 홈 풀에 대 한 모든 웹 서비스 Url을 반환 합니다.
  
외부 자동 검색 요청이 비즈니스용 Skype 서버용으로 구성한 리버스 프록시를 통과 하는 것으로 예상 됩니다. 그러나 내부 모바일 서비스 URL과 외부 모바일 서비스 URL은 모두 외부 웹 서비스 FQDN과 연결 됩니다. 따라서 모바일 장치가 네트워크 내부 또는 외부에 있든 상관 없이,이 장치는 항상 역방향 프록시를 통해 외부의 비즈니스용 Skype 서버 모바일 서비스에 연결 됩니다.
  
> [!NOTE]
> 방금 설명한 대로 모든 이동성 서비스 트래픽 (내부 및 외부)은 리버스 프록시를 거칩니다. 그러나 내부 트래픽이 인터페이스를 벗어날 때 문제가 발생 하는 경우에만 같은 인터페이스를 사용해 볼 수 있습니다. 이는 위장 (TCP 패킷 스푸핑 이라고 하는 공식적) 보안 규칙을 위반할 수 있습니다. 모바일 기능을 사용 하려면 **헤어 고정** 을 허용 해야 합니다.
  
> [!NOTE]
> 이 작업을 수행할 준비가 되 면 방화벽에서 분리 된 역방향 프록시를 사용 하도록 선택할 수도 있습니다 (보안을 위해 스푸핑 방지는 항상 방화벽에서 적용 되어야 함). 이 방법으로 연결는 방화벽의 외부 인터페이스가 아닌 역방향 프록시의 외부 인터페이스에서 발생할 수 있습니다. 이렇게 하면 역방향 프록시에서 규칙을 완화 하는 동안 방화벽에서 스푸핑을 제대로 감지 하 고 모바일 기능을 이용할 수 있습니다. 
  
> [!NOTE]
> 이 경로로 이동 하는 경우 가능 하면 DNS 호스트 또는 CNAME 레코드를 사용 하 여 연결 동작 (방화벽이 아님)에 대 한 리버스 프록시를 정의 해야 합니다. 
  
회사 네트워크 내부와 외부의 사용자를 지원 하기 위해 몇 가지 작업을 구성 해야 합니다.
  
다음은 내부 및 외부 웹 Fqdn에 대 한 규칙입니다.
  
- 자동 검색을 위한 새 CNAME 또는 A (호스트, if IPv6, AAAA) DNS 레코드
    
- Wi-fi 네트워크를 통해 푸시 알림을 지원 하려면 새 방화벽 규칙을 선택 합니다.
    
- 자동 검색을 위한 내부 서버 인증서 및 역방향 프록시 인증서의 제목 대체 이름입니다.
    
- 프런트 엔드 서버 하드웨어 부하 분산 구성 변경 원본 선호도.
    
다음은 모바일 서비스 및 자동 검색 서비스를 지 원하는 데 필요한 토폴로지 요구 사항입니다.
  
- 프런트 엔드 풀 내부 웹 FQDN은 프런트 엔드 풀 외부 웹 FQDN과 구분 되어야 합니다.
    
- 내부 웹 FQDN은 회사 네트워크 내에서 확인 하 고 액세스할 수 있어야 합니다.
    
- 외부 웹 FQDN은 인터넷에서 확인 하 고 액세스할 수 있어야 합니다.
    
- 회사 네트워크 내의 사용자의 경우 모바일 서비스 URL을 외부 웹 FQDN으로 처리 해야 합니다. 이 요구 사항은 모바일 서비스에 대 한 것 이며,이 URL에만 적용 됩니다.
    
- 회사 네트워크 외부의 사용자의 경우 요청이 프런트 엔드 풀 또는 디렉터의 외부 웹 FQDN으로 이동 해야 합니다.
    
자동 검색을 지 원하는 경우 각 SIP 도메인에 대해 다음 DNS 레코드를 설정 해야 합니다.
  
- 조직의 네트워크 내에서 연결 하는 모바일 사용자를 지 원하는 내부 DNS 레코드
    
- 인터넷에서 연결 하는 모바일 사용자를 지 원하는 외부 또는 공용 DNS 레코드
    
내부 자동 검색 URL은 내부 네트워크 외부에서 주소를 지정할 수 없습니다. 외부 자동 검색 URL은 네트워크 내에서 주소를 지정할 수 없습니다. 그러나 외부 URL에 대해이 방법을 사용할 수 없는 경우에는 내부 URL이 항상 먼저 시도 되므로 모바일 클라이언트 기능이 영향을 받지 않을 수 있습니다.
  
### <a name="port-and-firewall-requirements"></a>포트 및 방화벽 요구 사항

이 작업의 대부분은 다른 설명서에서 다루고 있지만, 특히 이동성을 위해 Survivable Branch 기기 (SBA)에 속한 사용자가 있는 경우 엔터프라이즈 Wi-fi 네트워크에서 다음 포트를 열어야 합니다.
  
- UcwaSipExternalListeningPort에는 5088이 필요 합니다.
    
- UcwaSipPrimaryListeningPort에는 5089이 필요 합니다.
    
### <a name="certificate-requirements"></a>인증서 요구 사항

비즈니스용 Skype 모바일 클라이언트에 자동 검색을 사용 하는 경우 모바일 클라이언트에서 보안 연결을 지원 하도록 인증서의 SAN (주체 대체 이름) 목록을 수정 해야 합니다. 현재 위치에 이미 인증서가 있는 경우 여기에 설명 된 SAN 항목을 사용 하 여 새 인증서를 요청 하 고 할당 해야 합니다. 자동 검색 서비스를 실행 하는 각 프런트 엔드 서버와 디렉터 (환경 내에 있는 경우)에 대해이 작업을 수행 해야 합니다. 또한 조직의 모든 SIP 도메인에 대 한 SAN 항목을 추가 하는 역방향 프록시 인증서의 SAN 목록을 수정 하는 것이 좋습니다.
  
이는 내부 CA (인증 기관)에서 새 인증서를 요청 하는 데 도움이 되지만, 공개 인증서가 더 복잡 하 고 다시 요청 하는 데 더 많은 비용이 소요 될 수 있으며,이는 많은 수의 SIP 도메인을 새 공개 인증서에 추가 하는 데 많은 비용이 듭니다. 이 경우 지원 되지만 권장 되는 접근 방법이 **없습니다**. 포트 80에서 초기 자동 검색 서비스 요청을 수행 하도록 리버스 프록시를 구성할 수 있는데,이는 HTTPS (및 443이 기본 구성 인 포트 443가 아닌 HTTP를 사용 하는 경우)입니다. 그러면 들어오는 요청이 프런트 엔드 풀 또는 디렉터에서 포트 8080로 리디렉션됩니다. 이렇게 하면이 소통량이 요청에 대해 HTTPS를 사용 하 고 있지 않으므로 인증서를 변경할 필요가 없습니다. 그러나이 경우에도이 방법을 사용 하는 것이 좋습니다.
  
### <a name="windows-and-iis-requirements"></a>Windows 및 IIS 요구 사항

비즈니스용 Skype 서버 환경용으로 지원 되는 Windows Server 버전이 있어야 합니다. 결과적으로, 이동성 요구를 충족 하는 IIS 8 또는 IIS 8.5도 있어야 합니다. 기본 ASP.NET 설정에 대 한 변경 사항이 필요 하지만 모바일 서비스 설치 관리자가 자동으로 수행 합니다.
  
### <a name="hlb-requirements"></a>HLB 요구 사항

프런트 엔드 풀에 대 한 HLB를 포함 하는 비즈니스용 Skype 서버에 대 한 토폴로지를 사용 하는 경우 (둘 이상의 프런트 엔드 서버를 포함 하는 토폴로지) 웹 서비스 트래픽에 대 한 외부 Vip (웹 서비스 가상 Ip)를 구성 해야 합니다. 원본에 해당 합니다. 원본 선호도는 세션 상태를 유지 하기 위해 단일 클라이언트의 여러 연결이 동일한 서버로 전송 되도록 하는 데 도움이 됩니다.
  
내부 Wi-fi 네트워크를 통해서만 비즈니스용 Skype 모바일 클라이언트를 지원 하려는 경우 외부 웹 서비스 Vip에 대해 설명한 대로 원본에 대 한 내부 웹 서비스 Vip를 구성 해야 합니다. 이러한 상황에서는 HLB의 내부 웹 서비스 Vip에 대 한 source_addr (또는 TCP) 선호도를 사용 해야 합니다.
  
이에 대 한 자세한 내용은 [비즈니스용 Skype 문서에 대 한 부하 분산 요구 사항을](network-requirements/load-balancing.md) 검토 하세요.
  
### <a name="reverse-proxy-requirements"></a>리버스 프록시 요구 사항

비즈니스용 Skype mobile 클라이언트에 대 한 자동 검색을 지원 하려면 현재 게시 규칙을 다음과 같이 업데이트 해야 합니다.
  
- 역방향 프록시 인증서의 SAN 목록을 업데이트 하 고 초기 자동 검색 서비스 요청에 대해 HTTPS를 사용 하는 경우에는 lyncdiscover에 대 한 웹 게시 규칙을 업데이트 해야 합니다. \<sipdomain\>. 이는 일반적으로 프런트 엔드 풀의 외부 웹 서비스 URL에 대 한 게시 rul과 함께 사용 됩니다.
    
- 초기 자동 검색 서비스 요청에 대해 HTTP를 사용 하 여 역방향 프록시 인증서에 대 한 SAN 목록을 업데이트 하지 않도록 결정 한 경우 (권장 하지 않음) 포트 HTTP/TCP 80에 대 한 새 웹 게시 규칙을 만들어야 합니다 (아직 없는 경우). 해당 규칙이 있으면이를 업데이트 하 여 lyncdiscover을 포함 합니다. \<sipdomain\> 항목.
    
## <a name="defining-your-mobility-needs"></a>이동성 요구 사항 정의

### <a name="do-you-want-to-use-automatic-discovery-for-skype-for-business-mobile-clients"></a>비즈니스용 Skype 모바일 클라이언트에 자동 검색을 사용 하 고 싶으세요?

자동 검색을 사용 하는 것이 좋습니다. 위의 기술 요구 사항 섹션에서 설명 하는 대로 새 내부 및 외부 DNS 레코드를 만들어야 할 수 있습니다. 자동 검색을 사용 하 여 비즈니스용 Skype 클라이언트는 모든 위치에서 자동으로 비즈니스용 Skype 서버 웹 서비스를 찾을 수 있습니다 (URL을 수동으로 입력할 필요는 없음).
  
필요에 맞게 수동 설정을 사용할 수 있습니다. 이러한 Url은 사용자가 모바일 장치에 입력 해야 합니다.
  
- 외부 액세스를 위한 **Https://\<extpoolfqdn\>/Autodiscover/autodiscoverservice.svc/Root** .
    
- 내부 액세스용 **Https://\<intpoolfqdn\>/Autodiscover/autodiscoverservice.svc/Root** .
    
이번에도 자동 검색을 사용 하는 것이 좋습니다. 문제 해결을 위해 수동 설정을 사용 하는 것이 좋습니다.
  
### <a name="are-you-going-to-support-push-notifications"></a>푸시 알림을 지원 하나요?

푸시 알림은 앱이 활성 상태가 아닌 동안 사용자에 게 이벤트를 알리는이 기능을 지 원하는 모바일 응용 프로그램에 사용 됩니다. Edge 서버는 비즈니스용 Skype Online 데이터 센터에 있는 클라우드 기반 Skype for Business Server 푸시 알림 서비스와 페더레이션 관계를 설정 해야 합니다. 푸시 알림을 사용 하도록 설정 하려면 cmdlet을 실행 해야 합니다.
  
> [!NOTE]
> 다른 사용자가 Lync Server 2010 클라이언트를 사용 하는 경우에는 엔터프라이즈 WiFi 네트워크에서 TCP 포트 5223 열기 아웃 바운드가 필요 합니다. 
  
### <a name="do-you-want-all-your-users-accessing-all-mobility-features-or-do-you-want-to-specify-the-users-who-can-access-these-features-instead"></a>모든 사용자가 모든 모바일 기능에 액세스 하도록 하 시겠습니까? 아니면 이러한 기능에 액세스할 수 있는 사용자를 지정 하 시겠습니까?

모든 사용자에 게 제공 되는 일부 기능에 대 한 표를 제공 하 고이를 기본적으로 설정 하거나 지정 하지 않도록 합니다. 전체 목록을 보려면 [CsMobilityPolicy](https://docs.microsoft.com/powershell/module/skype/new-csmobilitypolicy?view=skype-ps)을 검토 하세요.

### <a name="should-users-who-arent-enabled-for-enterprise-voice-be-able-to-use-click-to-join-to-join-conferences"></a>Enterprise Voice를 사용 하도록 설정 하지 않은 사용자가 클릭 하 여 회의 참가에 참가할 수 있으려면 어떻게 해야 하나요?

사용자가 모바일 기능에 액세스 하 고 작업을 통해 통화할 수 있도록 하려면 Enterprise Voice를 사용 하도록 설정 해야 합니다. 그러나이 기능을 사용 하도록 설정 하지 않은 경우에도 해당 모바일 장치에서 링크를 클릭 하 여 회의에 참석할 수 있지만 적절 한 음성 정책이 할당 되어 있는 경우에만 가능 합니다. 다음 중 하나를 수행 합니다.
  
- 이러한 사용자에 게 특정 음성 정책을 할당 하거나
    
- 전역 정책 또는 사이트 수준 정책이 있는지 확인 하 여 적용 합니다.
    
어떤 방법을 사용 하 든 지정 하는 음성 정책에 따라 사용자가 회의에 참가 하기 위해 전화를 걸 수 있는 위치를 정의 하는 PSTN (공개 전화 통신망) 및 경로가 필요 합니다.
  
> [!NOTE]
> 클릭 투 콜을 사용 하려는 모바일 사용자는 모바일 장치에서 해당 링크를 클릭할 때 비즈니스용 Skype 서버에서의 아웃 바운드 통화가 발생 하기 때문에 관련 PSTN 사용 레코드 및 음성 경로와 함께 음성 정책이 필요 합니다.
