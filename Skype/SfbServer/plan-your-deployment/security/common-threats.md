# <a name="common-security-threats-in-modern-day-computing"></a>최신 컴퓨팅의 일반적인 보안 위협
 
이 비즈니스용 Skype 서버 엔터프라이즈급 통신 시스템인 경우 인프라와 통신에 영향을 줄 수 있는 일반적인 보안 공격에 대해 잘 알고 있어야 합니다.
  
## <a name="compromised-key-attack"></a>손상된 키 공격

키는 비밀 정보를 암호화, 암호 해독 또는 유효성 검사하는 데 사용되는 암호 또는 번호입니다. PKI(공개 키 인프라)에는 다음 두 가지 중요한 키가 사용 중으로 간주되어야 합니다. 
  
- 각 인증서 소유자가 보유하는 개인 키
    
- 통신 파트너가 식별 및 세션 키를 교환한 후에 사용되는 세션 키
    
손상된 키 공격은 공격자가 개인 키 또는 세션 키를 확인할 때 발생합니다. 공격자가 키를 성공적으로 확인하면 공격자는 키를 사용하여 보낸 사람 없이 암호화된 데이터의 암호를 해독할 수 있습니다.
  
비즈니스용 Skype 서버 Windows Server 운영 체제의 PKI 기능을 사용하여 TLS(전송 계층 보안) 연결의 암호화에 사용되는 키 데이터를 보호합니다. 미디어 암호화에 사용되는 키는 TLS 연결을 통해 교환됩니다.
  
## <a name="network-denial-of-service-attack"></a>네트워크 서비스 거부 공격

서비스 거부 공격은 공격자가 유효한 사용자의 정상적인 네트워크 사용과 기능을 차단하는 것입니다. 공격자가 합법적인 사용자에 의해 서비스 사용을 압도하는 합법적인 요청으로 서비스를 넘치게 할 때 수행됩니다. 공격자는 서비스 거부 공격을 사용하여 다음을 할 수 있습니다.
  
- 공격을 받는 네트워크에서 실행 중인 응용 프로그램 및 서비스로 잘못된 데이터를 보내 정상적인 기능을 방해합니다.
    
- 많은 양의 트래픽을 보내 시스템을 오버로드하여 적절한 요청에 대한 시스템 응답을 중지시키거나 느리게 응답하도록 합니다.
    
- 공격 증거를 숨깁니다.
    
- 사용자가 네트워크 리소스에 액세스하지 못하도록 합니다.
    
## <a name="eavesdropping-sniffing-snooping"></a>도청(스니핑, 스누핑)

도청은 공격자가 네트워크의 데이터 경로에 액세스할 수 있으며 트래픽을 모니터링하고 읽을 수 있는 능력이 있는 경우 발생할 수 있습니다. 이를 스니핑 또는 스누핑이라고도 합니다. 트래픽이 일반 텍스트인 경우 공격자가 경로에 대한 액세스 권한을 얻을 때 트래픽을 읽을 수 있습니다. 예를 들어 데이터 경로에서 라우터를 제어하여 수행되는 공격이 있습니다. 
  
클라이언트 내에서 트래픽에 대한 기본 권장 비즈니스용 Skype 서버 설정은 클라이언트에서 서버로의 TLS와 신뢰할 수 있는 서버 간에 MTLS(상호 TLS)를 사용하는 것입니다. 이러한 보호 조치는 특정 대화가 발생하는 기간 내에 공격을 달성하기 어렵거나 불가능하게 만들 수 있습니다. TLS는 모든 당사자를 인증하고 모든 트래픽을 암호화합니다. 이렇게 해서 도청을 방지할 수는 없지만 암호화가 손상되지 않으면 공격자가 트래픽을 읽을 수 없습니다.
  
TURN(릴레이 NAT) 프로토콜을 사용한 트래버스에서는 트래픽을 암호화하도록 요구하지 않고 전송하는 정보가 메시지 무결성에 의해 보호됩니다. 도청이 열려 있는 경우 전송되는 정보(IP 주소 및 포트)는 패킷의 원본 및 대상 주소를 간단히 확인하여 직접 추출할 수 있습니다. A/V 에지 서비스는 TURN 암호를 포함한 몇 가지 항목에서 파생된 키를 사용하여 메시지의 메시지 무결성을 확인하여 데이터가 유효하도록 합니다. 이 키는 투명한 텍스트로 전송되지 않습니다. SRTP(Secure Real Time Protocol)를 사용하는 경우 미디어 트래픽도 암호화됩니다.
  
## <a name="identity-spoofing-ip-address-and-caller-id-spoofing"></a>ID 스푸핑(IP 주소 및 발신자 ID 스푸핑)

ID 스푸핑은 공격자가 권한이 부여되지 않은 유효한 사용자의 전화 번호(발신자 번호) 또는 네트워크, 컴퓨터 또는 네트워크 구성 요소의 IP 주소를 확인하여 사용할 때 발생합니다. 공격자가 일반적으로 전화 번호(발신자 번호) 또는 IP 주소로 식별되는 엔터티인 경우처럼 공격자가 작동할 수 있습니다.

이러한 비즈니스용 Skype 서버 내에서 IP 주소 스푸핑은 관리자가 다음 두 가지를 모두 수행한 경우만 재생됩니다.
  
- TCP(Transmission Control Protocol)만 지원하는 구성된 연결(TCP 통신은 암호화되지 않음) 권장되지 않습니다.
    
- 해당 연결의 IP 주소를 신뢰할 수 있는 호스트로 표시했습니다.
    
TLS는 모든 당사자를 인증하고 모든 트래픽을 암호화하기 때문에 TLS(전송 계층 보안) 연결의 문제보다 적습니다. TLS를 사용하는 경우 공격자가 특정 연결(예: 상호 TLS 연결)에 대해 IP 주소 스푸핑을 수행할 수 없습니다. 그러나 공격자는 여전히 사용하는 DNS 서버의 주소를 비즈니스용 Skype 서버 있습니다. 그러나 인증서를 비즈니스용 Skype 인증을 수행하기 때문에 공격자는 통신의 당사자 중 하나를 스푸핑하는 데 필요한 유효한 인증서가 없습니다.

반면 공급자, PSTN 게이트웨이 또는 다른 PBX 시스템 간에 SIP 트렁크를 설정하고 SIP를 설정한 경우 발신자 ID 스푸핑이 비즈니스용 Skype 서버. 이러한 경우 비즈니스용 Skype 서버 스푸핑을 방지하는 보호 기능을 제공하지 않습니다. 즉, 비즈니스용 Skype 다른 사용자에 대한 전화 번호 또는 표시 이름(역방향 번호 lookup이 적용되는 경우)을 표시하는 스푸핑된 발신자 번호를 사용하여 SIP 트렁크에서 전화를 받을 비즈니스용 Skype 있습니다. 이에 대한 보호는 공급자 쪽, PSTN 또는 PBX 게이트웨이에 적용해야 합니다.
  
## <a name="man-in-the-middle-attack"></a>메시지 가로채기(Man-in-the-Middle) 공격

공격자가 통신하는 두 사용자에 대한 지식 없이 공격자 컴퓨터를 통해 두 사용자 간의 통신을 재조정하는 경우 중간자 공격이 발생합니다. 공격자는 트래픽을 모니터링하여 읽은 후에 의도한 받는 사람에게 보낼 수 있습니다. 그러면 통신을 하는 각 사용자는 공격자에게 트래픽을 보내고 공격자로부터 트래픽을 받게 되며, 자신이 원하는 사용자와 통신하고 있다고 생각합니다. 공격자가 Active Directory 도메인 서비스를 수정하여 자신의 서버를 트러스트된 서버로 추가하거나, DNS(Domain Name System)를 수정하여 클라이언트가 공격자 컴퓨터를 통해 서버로 연결하도록 할 수 있는 경우에 이러한 현상이 발생합니다. 또한 두 클라이언트 간의 미디어 트래픽으로 중간자 공격이 발생할 수 있습니다. 그러나 비즈니스용 Skype 서버, 비디오 및 응용 프로그램 공유에서 스트림은 TLS를 통해 SIP(Session Initiation Protocol)를 사용하는 피어 간에 협상되는 암호화 키를 사용하여 SRTP로 암호화됩니다. 그룹 채팅과 같은 서버는 HTTPS를 사용하여 웹 트래픽 보안을 향상시킵니다.
  
## <a name="rtp-replay-attack"></a>RTP 재생 공격

재생 공격은 두 주체 간의 유효한 미디어 전송을 누군가가 가로채 악의적인 용도로 다시 전송하는 것입니다. 보안 신호 프로토콜과 함께 사용되는 SRTP는 수신자가 이미 받은 RTP 패킷의 인덱스를 유지 관리하고 각 새 패킷을 이미 인덱스에 나열되어 있는 패킷과 비교하도록 함으로써 재생 공격으로부터 전송 작업을 보호합니다.
  
## <a name="spim"></a>Spim

Spim은 원치 않는 상업용 인스턴트 메시지 또는 현재 상태 구독 요청입니다. 그 자체로 네트워크 손상은 아니며 최소한 성가신 것일 수 있으며, 리소스 가용성과 프로덕션을 줄일 수 있으며, 네트워크 손상으로 이어질 수 있습니다. 예를 들어 사용자가 요청을 보내 서로 Pimming하는 경우를 들 수 있습니다. 사용자가 이를 방지하기 위해 서로 차단할 수 있지만 페더미스를 통해 조율된 스피드 공격이 설정되면 파트너에 대해 페더니언스를 사용하지 않도록 설정하지 않는 한 이를 극복하기가 어려울 수 있습니다.
  
## <a name="viruses-and-worms"></a>바이러스 및 웜

바이러스는 일종의 코드 단위로, 자신과 비슷한 코드 단위를 계속해서 재생성합니다. 바이러스는 파일, 전자 메일, 프로그램 등의 호스트가 있어야 작동합니다. Aworm은 비슷한 코드 단위를 추가로 재현하는 것이 목적인 코드 단위이지만 호스트는 필요하지 않습니다. 바이러스와 웜은 다른 사용자가 URL을 보낼 때나 클라이언트 간에 파일을 전송할 때 주로 나타납니다. 컴퓨터에 바이러스가 있으면 사용자를 대신해 사용자의 ID로 인스턴트 메시지를 본래 수 있습니다.
  
## <a name="personally-identifiable-information"></a>개인 식별 정보

비즈니스용 Skype 서버 개인에 연결할 수 있는 공용 네트워크를 통해 정보를 공개할 수 있습니다. 정보 유형은 다음과 같은 두 가지 특정 범주로 구분할 수 있습니다.
  
- **향상된 현재 상태 데이터** 향상된 현재 상태 데이터는 사용자가 페더링 파트너 또는 조직 내의 연락처에 대한 링크를 통해 공유하거나 공유하지 않을 수 있는 정보입니다. 이 데이터는 공용 IM 네트워크의 사용자와 공유되지 않습니다. 클라이언트 정책 및 기타 클라이언트 구성을 통해 시스템 관리자에게 일부 제어 권한을 넣을 수 있습니다. 비즈니스용 Skype 서버 개인 사용자에 대해 향상된 현재 상태 개인 정보 보호 모드를 구성하여 비즈니스용 Skype 목록에 없는 사용자가 사용자의 현재 상태 정보를 볼 수 없습니다. 향상된 현재 상태 개인 정보 보호 모드에서는 Microsoft Office Communicator 2007 및 Microsoft Office Communicator 2007 R2의 사용자가 사용자의 현재 상태 정보를 볼 수 없습니다. 클라이언트 및 현재 상태 배포에 대한 자세한 내용은 [Deploy clients for 비즈니스용 Skype 서버](../../deploy/deploy-clients/deploy-clients.md) and Plan for instant messaging and presence in [비즈니스용 Skype 서버.](../../plan-your-deployment/instant-messaging-and-presence.md)
    
- **필수 데이터** 필수 데이터는 서버 또는 클라이언트의 적절한 작동에 필요하며 클라이언트 또는 시스템 관리의 제어를 에 있지 않습니다. 이 정보는 라우팅, 상태 유지 관리 및 신호 목적으로 서버 또는 네트워크 수준에서 필요한 정보입니다.
