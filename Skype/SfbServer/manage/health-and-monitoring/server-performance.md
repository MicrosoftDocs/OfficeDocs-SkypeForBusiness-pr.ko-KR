# <a name="server-performance-report-in-skype-for-business-server"></a>비즈니스용 Skype 서버의 서버 성능 보고서
 
**요약:** 비즈니스용 Skype 서버의 서버 성능 보고서에 대해 자세히 알아보세요.
  
서버 성능 보고서는 불량 통화의 최고 백분율이 발생 한 비즈니스용 Skype 서버 서버 목록을 제공 합니다. 다음 형식에 대 한 별도의 통계를 보고 하는 서버 유형별로 서버에서 하위 보고서를 분류 합니다.
  
- 중재 서버
    
- A/V 회의 서버
    
- A/V에 지 서버
    
- 게이트웨이 (중재 서버)
    
- 게이트웨이 (중재 서버 무시)
    
- 비디오 (A/V 회의 서버용 비디오 메트릭과 A/V Edge 서버 포함)
    
- 응용 프로그램 공유 (A/V 회의 서버 및 A/V에 지 서버에 대 한 응용 프로그램 공유 메트릭 포함)
    
이 보고서에는 기준 순위 지정으로 표시 되는 등급이 있다는 점에 유의 해야 합니다. 예를 들어, 최악의 서버의 1000 연결 된 통화에 대 한 한 가지 좋지 않은 호출이 있다고 가정 합니다. 이는 허용 되는 백분율이 며 1%입니다. 그러나 사용 하는 최악의 서버 (즉, 다른 서버 모두의 통화 백분율이 0.1% 미만이 되 면 서버 성능 보고서에 계속 표시 됩니다.)
  
## <a name="accessing-the-server-performance-report"></a>서버 성능 보고서에 액세스

서버 성능 보고서는 모니터링 보고서 홈 페이지에서 액세스할 수 있습니다. 다음 메트릭 중 하나를 클릭 하 여 [비즈니스용 Skype 서버에서 통화 목록 보고서](call-list-report-0.md) 로 드릴 다운할 수 있습니다.
  
- 통화 볼륨
    
- 통화 불량 백분율
    
또한 다음 메트릭을 클릭 하 여 서버 미디어 품질 추세 보고서로 드릴 다운할 수 있습니다.
  
- 추세가
    
## <a name="making-the-best-use-of-the-server-performance-report"></a>서버 성능 보고서를 최대한 활용 하기

서버 성능 보고서는 데이터를 필터링 하는 여러 가지 방법을 제공 합니다. 예를 들어 네트워크 종류 (유선 연결에서 발생 하는 통화와 무선 연결에서 발생 하는 전화) 및 액세스 형식 (방화벽 외부에서 생성 된 호출 및 방화벽 내부에서 발생 한 호출)에 대해 필터링 할 수 있습니다. 이러한 필터를 사용 하려면 서버 성능 보고서를 보는 것이 좋습니다. 예를 들어 잘못 된 통화 백분율이 3.24% 인 중재 서버가 있다고 가정 합니다. 무선 전화를 사용 하 고 있는 경우 같은 서버의 통화 백분율이 20%에 도달 했을 수 있습니다. 즉, 서버에서 무선 통화에 문제가 발생 하 여 서버가 유선 호출에 문제가 있지 않기 때문에 부분적으로 가려진 문제입니다.
  
## <a name="filters"></a>필터가

필터를 통해 더욱 세밀 하 게 대상 지정 된 데이터 집합을 반환 하거나 반환 된 데이터를 다양 한 방식으로 볼 수 있습니다. 예를 들어 서버 성능 보고서를 사용 하면 반환 된 데이터를 서버 유형 또는 네트워크 형식 (유선 또는 무선)으로 필터링 하는 등의 작업을 수행할 수 있습니다. 데이터 그룹화 방법을 선택할 수도 있습니다. 이 경우 데이터는 시간, 일, 주 또는 월로 그룹화 됩니다.
