# <a name="ip-phone-inventory-report-in-skype-for-business-server"></a>ip 전화 인벤토리 비즈니스용 Skype 서버 보고서
 
**요약:** 2013의 IP 전화 인벤토리 보고서에 비즈니스용 Skype 서버.
  
IP 전화 인벤토리 보고서는 조직에서 현재 사용 중인 IP 전화에 대한 정보를 보고합니다. IP 인벤토리 보고서는 지정된 보고 기간 동안 실제로 사용된 IP 전화의 자세한 목록을 제공합니다. 무엇보다 이 보고서를 통해 관리자는 교체해야 하는 오래된 오래된 휴대폰이 아직 사용 중이지 않습니다. 또한 조직에 거의 사용되지는 하는 고가의 전화가 존재하다는 사실을 관리자에게 경고할 수 있습니다. 이러한 유형의 정보는 새 휴대폰을 구입하거나 기존 휴대폰을 재배포해야 할 때가 될 수 있습니다. 예를 들어, 자신의 고가의 휴대폰을 거의 사용하는 사용자에게 휴대폰을 훨씬 자주 사용하는 사용자와 교체하도록 요청될 수 있습니다.
  
이 보고서는 실제 인벤토리 보고서로 사용되는 경우 몇 가지 제한이 있습니다. 한 가지 중요한 점은 IP 전화 보고서에 지정된 기간 동안 로그온한 모든 비즈니스용 Skype 서버 마지막 로그온 시간별로 정렬하여 나열합니다. 지정된 기간 동안 전화가 로그온되지 않은 경우 인벤토리 보고서에 나열되지 않습니다. 여기에는 기간이 시작되기 전에 로그온되어 지정된 시간 간격 동안 계속 로그온된 전화가 포함됩니다. 예를 들어 2015년 7월의 모든 전화 인벤토리를 확인했다고 가정해 보겠습니다. 또한 여러 개의 전화가 2015년 6월 30일 비즈니스용 Skype 서버 로그온되어 7월 1일 현재 계속 로그온되어 있는 경우를 가정해 565.06.15. 이러한 전화는 7월 1일의 인벤토리 보고서에 표시되지 않습니다.
  
또한 인벤토리 보고서에는 조직에서 더 이상 사용하지 사용하는 전화가 포함되어 있을 수 있습니다. 예를 들어 2015년 7월 1일 시스템에 로그온한 여러 Fabrikam 전화를 가정해 보겠습니다. 5일 후에 조직에서 Fabrikam 휴대폰을 모두 제거하고 새 Contoso 모델로 대체했습니다. Fabrikam 휴대폰은 7월 한 달 동안 시스템에 로그온했다는 것만으로도 "인벤토리" 보고서에 계속 표시됩니다.
  
또한 IP 전화 인벤토리 보고서는 다양한 유형의 전화에 대한 요약 합계를 보고하지 않습니다. 예를 들어 Polycom CX600 전화가 105개인 경우를 가정해 보겠습니다. 보고서에 이러한 전화 중 105개가 있는 것으로 알려지지는 않습니다. 대신 Polycom Cx600에 대해 105개의 개별 항목을 볼 수 있습니다. Polycom Cx600에 대한 항목이 105개인지를 알 수 있는 유일한 방법은 이러한 각 항목을 수동으로 계산하는 것입니다.
  
> [!TIP]
> 또는 데이터를 내보낼 수 Microsoft Excel 또는 Windows PowerShell 사용하여 계산을 할 수 있습니다. 
  
## <a name="accessing-the-ip-phone-inventory-report"></a>IP 전화 인벤토리 보고서 액세스

IP 전화 인벤토리 보고서는 모니터링 보고서 홈 페이지에서 액세스합니다. 사용자 URI 메트릭을 클릭하면 해당 사용자의 사용자 활동 보고서에 액세스할 수 있습니다. 피어 투 피어 통화에 대한 마지막 활동 메트릭을 클릭하면 피어 투 피어 세션 세부 정보 보고서로 진행됩니다. 회의에 대해 동일한 메트릭을 클릭하면 회의 정보 보고서로 진행됩니다.
  
## <a name="making-the-best-use-of-the-ip-phone-inventory-report"></a>IP 및 인벤토리 보고서를 전화 만들기

특정 종류의 전화(예: "Polycom CX600 휴대폰을 사용하는 사용자의 수")에 대한 사용 정보에만 관심이 있는 경우 해당 종류의 전화에 대해 필터링하여 IP 전화 Inventory Report에서 해당 정보를 직접 얻을 수 있습니다. 그러나 모든 전화(Polycom CX600을 사용하는 사용자 수, LG-Nortel IP8540을 사용하는 사용자 수 등)에 대한 요약 정보를 원할 경우 데이터를 내보내고 다른 응용 프로그램(예: Windows PowerShell)을 사용하여 해당 유형의 분석을 수행해야 합니다. 예를 들어 데이터를 콤보로 구분된 값 파일(C:\Data\IP_Phone_Inventory_Report.csv)로 내보낼 수 C:\Data\IP_Phone_Inventory_Report.csv. 이 경우 다음 두 명령을 사용하여 모든 휴대폰에 대한 요약 데이터를 제공할 수 있습니다.
  
```PowerShell
$phones = Import-Csv "C:\Data\IP_Phone_Inventory_Report.csv"
$phones |Group-Object Manufacturer, "Hardware version" | Select-Object Count, Name | Sort-Object Count -Descending
```

이 명령을 실행하면 다음과 같은 데이터가 반환됩니다.
  
<pre>
Count    Name
-----    ----
  267    POLYCOM, CX700
  267    POLYCOM, CX600
  166    POLYCOM, C
   68    Microsoft, CPE
   64    LG-Nortel, IP8540
   59    Aastra, 6725ip
   37    LG-Nortel, IP
   22    POLYCOM, CX3000
   11    Microsoft, CPE_A
    9    POLYCOM, CX500
    7    Aastra, 6721ip
</pre>

마찬가지로, 이 두 명령은 시스템에 로그온했지만 실제로 호출하는 데 사용되지 않은 전화(마지막 활동 메트릭 값이 비어 있습니다. 마지막 활동이 없는 경우)를 알려줄 수 있습니다.
  
```PowerShell
$phones = Import-Csv "C:\Data\IP_Phone_Inventory_Report.csv"
$phones | Where-Object {$_."Last activity" -eq ""}
```

이 경우 사용되지 않은 각 전화에 대해 이와 유사한 데이터가 반환됩니다.
  
<pre>
Manufacturer     : POLYCOM
Hardware version : CX600
MAC address      : 00-04-F2-00-01-76
User URI         : 422
User agent       : CPE/4.0.7423.1 OCPhone/4.0.7423.1 (Microsoft Lync 2010 (Beta) Phone Edition)
Last logon time  : 8/30/2010 4:44:48 PM
Last logoff time : 8/30/2010 5:59:07 PM
Last activity    :
</pre>

IP 전화 인벤토리 보고서를 사용하는 또 다른 흥미로운 방법은 IP 주소가 있는 경우전화 MAC 주소 텍스트 상자에 해당 주소를 입력하여 해당 전화를 마지막으로 사용한 사용자를 찾을 수 있습니다. 그러면 IP 전화 인벤토리 보고서는 해당 전화로 마지막으로 로그온한 사용자의 SIP 주소를 다시 보고합니다. 또는 사용자 SIP 주소(사용자 URI prefix 상자에)를 입력하여 해당 사용자가 사용한 모든 전화를 찾을 수 있습니다.
  
## <a name="filters"></a>필터

필터를 사용하면 여러 방식으로 반환된 데이터를 보거나 보다 세부적으로 대상화된 데이터 집합을 반환할 수 있습니다. 예를 들어 IP 전화 Inventory를 사용하면 특정 회사에서 제작한 휴대폰만 볼 수 있습니다. 또는 해당 휴대폰의 특정 버전을 볼 수도 있습니다. 또한 데이터의 그룹 지정 방식도 선택할 수 있습니다. 이 경우 등록은 시간, 일, 주 또는 월별 그룹화됩니다.
  
다음 표에는 IP 및 인벤토리 보고서에 사용할 수 있는 전화 나열되어 있습니다.

|**일 이전의 마지막 활동** <br/> | 다음 값 중 하나를 선택합니다. <br/>  [모두] <br/>  10  <br/>  20 <br/>  30 <br/> |
|**마지막 로그프 시간 전일** <br/> | 다음 값 중 하나를 선택합니다. <br/>  [모두] <br/>  10  <br/>  20 <br/>  30 <br/> |
|**사용자 URI 접두사** <br/> |IP 전화를 사용한 사용자의 SIP 주소입니다.  <br/> |
