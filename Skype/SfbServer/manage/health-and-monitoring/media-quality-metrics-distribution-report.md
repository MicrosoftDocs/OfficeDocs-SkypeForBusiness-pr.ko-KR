# <a name="the-media-quality-metrics-distribution-report-in-skype-for-business-server"></a>보고서의 미디어 품질 메트릭 배포 비즈니스용 Skype 서버 
 
**요약:** 미디어 품질 메트릭 배포 보고서에 대해 비즈니스용 Skype 서버.
  
미디어 품질 메트릭 분포 보고서를 사용하면 지터, 패킷 손실 등 QoE(체감 품질)의 분포 값을 표시하는 그래프를 확인할 수 있습니다. 예를 들어 사용자가 총 10건의 전화 통화를 한 경우 해당 10건의 통화가 다음 왕복 시간을 보고합니다.
  
|**통화 횟수**|**왕복 시간(밀리초)**|
|:-----|:-----|
|1  <br/> |50  <br/> |
|2  <br/> |50  <br/> |
|3   <br/> |50  <br/> |
|4   <br/> |50  <br/> |
|5   <br/> |50  <br/> |
|6   <br/> |50  <br/> |
|7   <br/> |50  <br/> |
|8   <br/> |4550  <br/> |
|9   <br/> |50  <br/> |
|10   <br/> |50  <br/> |
   
이러한 왕복 시간의 평균은 500밀리초(5000을 10으로 나눈)입니다. 500밀리초는 매우 큰 왕복 시간입니다. 따라서 네트워크 정체에 심각한 문제가 있는 것으로 생각할 수 있습니다. 왕복 시간이 길면 네트워크가 과부하될 수 있습니다.
  
물론, 실제로 통화의 90%에 대한 왕복 시간은 상당히 양호하며 단지 한 번의 불량 통화로 전체 결과가 왜곡되었습니다. 평균 왕복 시간만 보면 매우 잘못된 결론으로 이동하게 될 수 있습니다.
  
미디어 품질 메트릭 배포 보고서는 지정된 메트릭의 그래픽 배포(예: 왕복 시간)를 표시하여 잘못된 결론으로 이동하지 않도록 하는 데 도움이 됩니다. 이러한 그래프를 통해 9개의 아주 좋은 통화와 한 번의 불량 통화가 있었다는 것이 명확해지는 데 도움이 될 수 있습니다. 그러나 여전히 한 통화를 더 조사할 수 있습니다. 그러나 10개 통화 중 9개가 매우 양호했다는 사실은 적어도 이 시점이 아니라 네트워크를 대대적으로 변경할 이유가 없음을 나타났습니다.
  
## <a name="filters"></a>필터

필터를 사용하면 여러 방식으로 반환된 데이터를 보거나 보다 세부적으로 대상화된 데이터 집합을 반환할 수 있습니다. 다음 표에는 미디어 품질 메트릭 배포 보고서에 사용할 수 있는 필터가 나열됩니다.

|**이름**|**설명**|
|:-----|:-----|
|**시작** <br/> |시간 범위의 시작 날짜/시간입니다. 시간별 데이터를 보려면 다음과 같이 시작 날짜 및 시간을 입력합니다.  <br/> 2015/7/7 오후 1:00  <br/> 시작 시간을 입력하지 않으면 보고서가 자동으로 지정된 날짜의 오전 12시부터 시작됩니다. 일별 데이터를 보려면 날짜만 입력합니다.  <br/> 7/7/2015  <br/> 주 또는 월별로 보려면 데이터를 보려는 해당 주 또는 월에 속하는 날짜를 입력합니다. 주 또는 월의 첫 번째 날짜를 입력할 필요가 없습니다.  <br/> 7/3/2015  <br/> 주는 항상 일요일부터 토요일까지로 실행됩니다.  <br/> |
|**To** <br/> |시간 범위의 종료 날짜/시간입니다. 시간별 데이터를 보려면 다음과 같이 종료 날짜 및 시간을 입력합니다.  <br/> 2015/7/7 오후 1:00  <br/> 종료 시간을 입력하지 않으면 보고서가 자동으로 지정된 날짜의 오전 12시에 종료됩니다. 일별 데이터를 보려면 날짜만 입력합니다.  <br/> 7/7/2015  <br/> 주 또는 월별로 보려면 데이터를 보려는 해당 주 또는 월에 속하는 날짜를 입력합니다. 주 또는 월의 첫 번째 날짜를 입력할 필요가 없습니다.  <br/> 7/3/2015  <br/> 주는 항상 일요일부터 토요일까지로 실행됩니다.  <br/> |
|**최소 x축** <br/> |그래프의 X축에 표시할 최저값입니다.  <br/> |
|**최대 x축** <br/> |그래프의 X축에 표시할 최고 값입니다.  <br/> |
|**액세스 유형** <br/> | 통화가 시도되었을 때 클라이언트가 내부 네트워크 또는 외부 네트워크에 로그온되어 있는지를 나타냅니다. 다음 중 하나를 선택합니다. <br/>  [모두] <br/>  내부 <br/>  외부 <br/> |
|**VPN** <br/> | 통화가 시도되었을 때 외부 클라이언트가 VPN(가상 사설망) 연결을 사용 중이었는지를 나타냅니다. 다음 중 하나를 선택합니다. <br/>  [모두] <br/>  VPN <br/>  비 VPN <br/> |
|**네트워크 유형** <br/> | 통화가 시도되었을 때 클라이언트가 연결된 네트워크의 유형을 나타냅니다. 다음 중 하나를 선택합니다. <br/>  [모두] <br/>  유선 <br/>  무선 <br/> |
