# <a name="user-registration-report-in-skype-for-business-server"></a>비즈니스용 Skype 서버의 사용자 등록 보고서
 
**요약:** 비즈니스용 Skype 서버의 사용자 등록 보고서에 대해 자세히 알아보습니다.
  
사용자 등록 보고서는 사용자 로그온 활동에 대한 개요를 제공합니다. 특히 지정된 기간(시간별, 일별, 주별, 월별) 동안 비즈니스용 Skype 서버에 로그온한 사용자 수에 대한 정보를 제공합니다. 이 보고서에는 로그온한 사용자의 수만 표시됩니다. 즉, 누가 로그온했는지는 확인할 수 없습니다. 모니터링 보고서는 비즈니스용 Skype 서버를 사용하며 사용하지 않는 특정 사용자에 대한 정보는 제공하지 않습니다. 그러나 사용자 활동 보고서를 사용하면 대략적인 사용자 정보를 파악할 수는 있습니다.
  
사용자 등록 보고서는 사용자 로그온에 대한 정보를 제공할 때 두 가지 주요 기준을 사용하여 정보를 구분합니다. 첫째로, 이 보고서에서는 로그온이 두 가지 기본 범주(내부 로그온/외부 로그온)로 구별됩니다. 내부 로그온은 회사 네트워크에 연결되어 있는 동안 조직 방화벽 내에서 로그온한 사용자를 나타냅니다. 외부 로그온은 에지 서버를 통해 방화벽 외부에서 로그온한 사용자를 나타내며, 예를 들어 인터넷 카페에서 로그온한 사용자는 외부 로그온으로 계산됩니다. 방화벽 외부에서 로그온하는 사용자 수를 확인해야 하는 경우 사용자 등록 보고서에서 해당 정보를 파악할 수 있습니다.
  
사용자 등록 보고서는 지정된 기간 동안의 활성 사용자 수에 대한 정보도 제공합니다. 활성 사용자는 IM(인스턴트 메시징) 세션에 참가하거나, 비즈니스용 Skype 서버 모임에 참가하거나, 전화를 걸거나 받거나, 해당 기간 동안 비즈니스용 Skype 서버를 사용하는 사용자입니다. 활성 사용자는 로그온하기는 했지만 시스템을 실제로 사용한 적은 없는 사용자와는 다릅니다.
  
## <a name="accessing-the-user-registration-report"></a>사용자 등록 보고서 액세스

사용자 등록 보고서는 모니터링 보고서 홈 페이지에서만 액세스할 수 있습니다. 이 보고서는 다른 보고서에 연결되지 않습니다.
  
## <a name="making-the-best-use-of-the-user-registration-report"></a>효율적인 사용자 등록 보고서 활용

비즈니스용 Skype 서버를 배포한 후 일반적으로 묻는 질문은 사용자가 실제로 이 새 기술을 사용하고 있는지 어떻게 알 수 있나요? 이와 관련하여 몇 가지 제한이 있기는 하지만, 사용자 등록 보고서를 사용하면 해당 사항을 확인할 수 있습니다. 사용자가 비즈니스용 Skype 서버를 사용하고 있는지 여부를 확인하려면 두 가지 작업을 해야 합니다. 먼저 사용자 등록 보고서에서 고유 로그온 사용자 메트릭 값을 가져옵니다. 이 값은 비즈니스용 Skype 서버에 로그온한 개별 사용자 수를 알려합니다.
  
반면 총 로그온 메트릭은 모든 사용자가 비즈니스용 Skype 서버에 로그온한 총 수를 보여 주었습니다. 예를 들어 Ken Myer가 하루 동안 5번 비즈니스용 Skype 서버에 로그온했다고 가정해 보겠습니다. 이 경우 Ken Myer의 로그온은 총 로그온 메트릭에서는 5회의 개별 로그온 세션으로 계산되지만 고유 로그온 사용자 메트릭에서는 단일 로그온 사용자로 계산됩니다. 마찬가지로, 사용자가 여러 장치나 여러 위치에서 로그온하는 경우도 흔합니다. 예를 들어 사용자는 자신의 데스크톱 컴퓨터, 노트북 컴퓨터를 사용하여 로그온할 수 있으며 비즈니스용 Skype 서버에 자동으로 로그온하는 IP 전화를 사용할 수 있습니다. 이 예제의 경우 로그온 횟수는 3회이지만 고유 사용자는 1명입니다.
  
총 로그온과 고유 로그온 간의 차이를 보다 자세하게 파악하려면 아래 표에 나와 있는 지정된 기간 동안의 로그온을 살펴보십시오.
  
|**사용자**|**로그온 시간**|
|:-----|:-----|
|Ken Myer  <br/> |2015/7/7 오전 8:45  <br/> |
|Ken Myer  <br/> |2015/7/7 오전 8:46  <br/> |
|Pilar Ackerman  <br/> |2015/7/7 오전 9:17  <br/> |
|Ken Myer  <br/> |2015/7/7 오전 9:22  <br/> |
|Pilar Ackerman  <br/> |2015/7/7 오전 9:31  <br/> |
   
이 경우 총 로그온 횟수는 5회이지만 고유 로그온 사용자는 2명(3회 로그온한 Ken Myer, 2회 로그온한 Pilar Ackerman)입니다. 이처럼 로그온 횟수와 고유 로그온 사용자 수는 서로 다를 수 있습니다.
  
고유한 로그 로그의 수를 아는 것 외에도 비즈니스용 Skype 서버에 대해 사용하도록 설정된 총 사용자 수를 알아야 합니다. 이 값은 비즈니스용 Skype 서버 관리 셸을 열고 다음 명령을 실행하여 검색할 Windows PowerShell 있습니다.
  
```PowerShell
(Get-CsUser).Count
```

위 명령이 값 1,236을 반환하고 고유 로그온 사용자 메트릭이 평균 값 667을 반환하는 경우, 즉 사용자의 절반 이상이 매일 시스템에 실제로 로그온하고 있습니다(즉, 667을 1,236으로 나눈 약 54%).
  
> [!CAUTION]
> 로그온 메트릭은 지정된 기간 동안 실제로 로그온한 사용자를 기록합니다. 그러나 시스템에 이미 로그온되어 있는 사용자는 추적하지 않습니다. 예를 들어 고유 로그온 사용자 메트릭에서 로그온 667회를 표시하는데 사용자 수는 1,236명이면 사용자 중 절반 정도가 시스템에 로그온함을 의미합니다. 그러나 로그온 데이터 확인을 시작한 시간에 사용자 300명이 시스템에 이미 로그온한 상태라고 가정해 보겠습니다. 이는 실제로 1,000명에 가까운 사용자가 비즈니스용 Skype 서버에 로그온했다는 의미입니다. 즉, 사용자의 80%에 가깝게 로그온했습니다. 
  
고유 로그온 사용자 값은 고유 활성 사용자 메트릭과도 비교해야 합니다. 고유 활성 사용자 메트릭은 사용자가 전화 통화를 하거나, 비즈니스용 Skype 서버 모임에 참가하거나, IM 세션에 참가하는 등 실제로 비즈니스용 Skype 서버를 사용한 고유 사용자 수를 알 수 있습니다. 이는 사용자가 Windows를 시작할 때마다 비즈니스용 Skype 서버를 자동으로 시작하도록 구성할 수 있기 때문에 유용한 정보입니다. 따라서 매일 Windows에 로그온할 때 비즈니스용 Skype에 자동으로 로그온하는 사용자가 많은 수 있지만 해당 기간 동안 실제로 비즈니스용 Skype 서버를 사용하지는 않을 수 있습니다.
  
고유 활성 사용자 메트릭은 사용자가 일반적으로 하루가 끝날 때 Windows에서 로그오프하지 않는 조직에서 더 의미 있는 데이터를 제공합니다. 대신 컴퓨터를 잠그고 Windows 및 비즈니스용 Skype를 실행 중인 그대로 떠날 수 있습니다. 이러한 상황에서는 사용자가 로그온한 상태를 며칠씩 유지하고 로그오프하지는 않기 때문에 일별 로그온 수가 매우 적을 수 있습니다. 그러나 고유한 활성 사용자는 사용자가 비즈니스용 Skype를 적극적으로 사용하고 있는지 아니면 다른 비즈니스용 Skype 서버 클라이언트를 사용하고 있는지를 알 수 있습니다.
  
## <a name="filters"></a>필터

필터를 사용하면 여러 방식으로 반환된 데이터를 보거나 보다 세부적으로 대상화된 데이터 집합을 반환할 수 있습니다. 예를 들어 사용자 등록 보고서를 사용하면 모든 등록자 풀 및 에지 서버에 대한 데이터를 보거나 개별 풀의 데이터를 볼 수 있습니다. 또한 데이터의 그룹 지정 방식도 선택할 수 있습니다. 이 경우 등록은 시간, 일, 주 또는 월별로 그룹화됩니다.
  
다음 표에서는 사용자 등록 보고서에서 사용할 수 있는 필터를 보여 줍니다.

|**이름**|**설명**|
|:-----|:-----|
