# <a name="peer-to-peer-activity-diagnostic-report-in-skype-for-business-server"></a>비즈니스용 Skype 서버의 피어 투 피어 활동 진단 보고서
 
**요약:** 비즈니스용 Skype 서버의 피어 투 피어 활동 진단 보고서에 대해 자세히 알아보습니다.
  
피어 투 피어 활동 진단 보고서는 피어 투 피어 통신 세션의 성공 및 실패에 대한 정보를 제공합니다. 비즈니스용 Skype 서버는 각 오류 종류를 구분합니다.
  
- **예상 오류입니다.** 예상 오류는 일반적으로 가장 기술적 의미에서만 실패입니다. 예를 들어 누군가에게 전화를 걸었지만 부재 중이라 전화에 응답할 수 없다고 가정해 보겠습니다. 통화에 응답하지 않은 경우 기술적으로는 통화가 실패로 간주됩니다. 반면, 이 오류는 예상된 오류입니다. 비즈니스용 Skype 서버는 전화에 응답할 수 없는 경우 전화에 응답할 것으로 예상하지 않습니다. 마찬가지로 오프라인 상태이거나 인스턴트 메시징을 지원하지 않는 전화에만 로그온되어 있는 사용자에게 인스턴트 메시지를 보내면 예상되는 오류가 발생합니다.
    
- **예기치 않은 오류입니다.** 예기치 않은 오류는 이름에서 알 수 있는 정확한 오류, 즉 상황에 따라 발생할 것으로 예상되지 않는 오류입니다. 예를 들어 누군가와 통화에 응답할 수 있는 사람을 가정해 보겠습니다. 그러나 비즈니스용 Skype 서버가 통화를 음성 메일로 라우팅할 때 Exchange 통합 메시징에 대한 연결이 끊어지기 때문에 통화가 실패합니다. 예기치 않은 오류입니다. 통화가 항상 음성 메일로 라우팅될 수 있습니다. 일반적으로 예기치 않은 오류는 실제 오류입니다. 즉, 사용자 교육 또는 유사한 조치를 통해 해결할 수 없는 문제입니다.
    
성공, 예상 오류 및 예기치 않은 오류 메트릭은 총 세션 메트릭에 추가되지 않을 수 있습니다. 예를 들어 위 그림에는 다음과 같은 값이 있습니다.
  
|**성공**|**예상 오류**|**예기치 않은 오류**|**총 세션**|
|:-----|:-----|:-----|:-----|
|2024  <br/> |469  <br/> |16   <br/> |2521  <br/> |
   
이 경우 2024+469+16=총 세션 수는 2,509개가 되지만 총 세션 열에는 총 세션 수가 2,521개로 표시됩니다. "누락된" 12개 세션은 시스템에서 성공 또는 실패로 분류하지 못한 세션입니다. 타사 제품이 비즈니스용 Skype 서버에 익숙하지 않은 새로운 진단 코드를 도입하는 경우도 있습니다. 이와 같은 경우 해당 제품을 사용하여 전화를 걸고 해당 진단 코드를 보고하는 경우를 항상 성공, 예상 오류 또는 예기치 않은 오류로 분류할 수 있는 것은 아닙니다.
  
## <a name="accessing-the-peer-to-peer-activity-diagnostic-report"></a>피어 투 피어 활동 진단 보고서 액세스

피어 투 피어 진단 보고서는 모니터링 보고서 홈 페이지에서 액세스합니다. 다음 메트릭 중 하나를 클릭하여 비즈니스용 [Skype 서버에서](failure-distribution-report.md) 오류 배포 보고서에 액세스할 수 있습니다.
  
- 예기치 않은 오류량
    
- 예상 오류량
    
## <a name="making-the-best-use-of-the-peer-to-peer-activity-diagnostic-report"></a>피어 투 피어 활동 진단 보고서를 가장 효율적으로 활용

다양한 방법으로 피어 투 피어 활동 진단 보고서를 필터링할 수 있지만, 기본적으로 이러한 필터링 옵션은 보기에서 숨겨집니다. 사용 가능한 필터링 옵션을 보려면 보고서 창 오른쪽 위의 매개 변수 표시/숨기기 단추를 클릭합니다. 그러면 필터링 옵션을 사용할 수 있게 됩니다.
  
## <a name="filters"></a>필터

필터를 사용하면 여러 방식으로 반환된 데이터를 보거나 보다 세부적으로 대상화된 데이터 집합을 반환할 수 있습니다. 예를 들어 피어 투 피어 활동 진단 보고서를 통해 세션 형식(예: 인스턴트 메시징, 파일 전송 또는 응용 프로그램 공유)과 같은 항목으로 필터링할 수 있습니다. 또한 데이터의 그룹 지정 방식도 선택할 수 있습니다. 이 경우 통화는 시간, 일, 주 및 월별로 그룹이 지정됩니다.
  
다음 표에서는 피어 투 피어 활동 진단 보고서에 사용할 수 있는 필터를 보여 줍니다.
  
**피어 투 피어 활동 진단 보고서 필터**

|**이름**|**설명**|
|:-----|:-----|
|**시작** <br/> |시간 범위의 시작 날짜/시간입니다. 시간별 데이터를 보려면 다음과 같이 시작 날짜 및 시간을 입력합니다.  <br/> 2015/7/7 오후 1:00  <br/> 시작 시간을 입력하지 않으면 보고서가 자동으로 지정된 날짜의 오전 12시부터 시작됩니다. 일별 데이터를 보려면 날짜만 입력합니다.  <br/> 7/7/2015  <br/> 주 또는 월별로 보려면 데이터를 보려는 해당 주 또는 월에 속하는 날짜를 입력합니다. 주 또는 월의 첫 번째 날짜를 입력할 필요가 없습니다.  <br/> 7/3/2015  <br/> 주는 항상 일요일부터 토요일까지로 실행됩니다.  <br/> |
|**To** <br/> |시간 범위의 종료 날짜/시간입니다. 시간별 데이터를 보려면 다음과 같이 종료 날짜 및 시간을 입력합니다.  <br/> 2015/7/7 오후 1:00  <br/> 종료 시간을 입력하지 않으면 보고서가 자동으로 지정된 날짜의 오전 12시에 종료됩니다. 일별 데이터를 보려면 날짜만 입력합니다.  <br/> 7/7/2015  <br/> 주 또는 월별로 보려면 데이터를 보려는 해당 주 또는 월에 속하는 날짜를 입력합니다. 주 또는 월의 첫 번째 날짜를 입력할 필요가 없습니다.  <br/> 7/3/2015  <br/> 주는 항상 일요일부터 토요일까지로 실행됩니다.  <br/> |
