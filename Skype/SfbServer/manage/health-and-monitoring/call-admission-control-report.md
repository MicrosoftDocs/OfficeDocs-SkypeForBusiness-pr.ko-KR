# <a name="call-admission-control-report-in-skype-for-business-server"></a>비즈니스용 Skype 서버의 통화 제어 보고서
 
**요약:** 비즈니스용 Skype 서버에서 사용되는 통화 제어 보고서에 대해 자세히 알아보습니다.
  
통화 허용 제어 보고서는 통화 허용 제어에서 설정한 제한에 따라 수행된 피어 투 피어 및 회의 세션에 대한 정보를 제공합니다. 통화 허용 제어를 사용하면 관리자가 대역폭 제약 조건에 따라 통신 세션을 허용하거나 허용하지 않을 수 있습니다. 예를 들어 관리자는 음성 및 비디오 통화에 사용할 수 있는 대역폭의 양을 제한하는 정책을 만들 수 있습니다. 대역폭 제한에 도달한 경우 현재 통화 중 하나를 종료하고 필요한 네트워크 리소스를 확보할 때까지 새 음성 또는 화상 통화를 걸 수 없습니다.
  
## <a name="accessing-the-call-admission-control-report"></a>통화 허용 제어 보고서 액세스

통화 허용 제어 보고서는 모니터링 보고서 홈 페이지에서 액세스합니다. 통화 수락 제어 보고서에서 다음 보고서 중 하나로 드릴다운할 수 있습니다.
  
- 회의 정보 보고서 - 이 보고서에 액세스하려면 회의 세션에서 세부 정보 메트릭을 클릭합니다. 
    
- 피어 투 피어 세션 세부 정보 보고서 - 이 보고서에 액세스하려면 피어 투 피어 세션에 대한 세부 정보 메트릭을 클릭합니다.
    
## <a name="making-the-best-use-of-the-call-admission-control-report"></a>통화 수당 제어 보고서를 가장 잘 활용

대역폭이 부족하여 실패한 통화 목록을 얻습니다. 통화 범주 드롭다운 목록에서 통화가 거부된 통화를 선택합니다. 반환된 호출의 진단 ID는 대부분 5입니다.
  
세션을 설정하기 위한 대역폭이 부족합니다. PSTN 다시 라우팅을 시도합니다.
  
이는 통화 제어 제한으로 인해 VoIP 네트워크에서 통화를 할 수 없습니다.
  
## <a name="filters"></a>필터

다음 표에서는 피어 투 피어 세션(즉, 참가자가 두 명인 세션)에 대한 통화 제어 보고서에 제공된 정보를 나열합니다.
  
**피어 투 피어 세션에 대한 메트릭**

|**이름**|**이 항목에 대한 정렬 가능 여부**|**설명**|
|:-----|:-----|:-----|
|**자세한 정보** <br/> |아니요  <br/> |이 항목을 클릭하면 보고서에 지정된 세션에 대한 피어 투 피어 세션 세부 정보 보고서가 표시됩니다.  <br/> |
|**시작 사용자** <br/> |예  <br/> |세션을 시작한 사용자의 SIP 주소입니다.  <br/> |
|**대상 사용자** <br/> |예  <br/> |세션에 참가하도록 초대된 사용자의 SIP 주소입니다.  <br/> |
|**Modalities** <br/> |예  <br/> |세션 중에 사용된 통신 모달(예: 오디오 및 비디오)입니다.  <br/> |
|**초대 시간** <br/> |예  <br/> |초기 세션 초대가 보낸 사용자에게 전송된 날짜 및 시간입니다.  <br/> |
|**응답 시간** <br/> |예  <br/> |초대 수락을 받은 날짜 및 시간입니다.  <br/> |
|**종료 시간** <br/> |예  <br/> |세션이 종료된 날짜 및 시간입니다.  <br/> |
|**진단 ID** <br/> |예  <br/> |오류 문제를 해결할 때 종종 유용한 정보를 제공하는 SIP 메시지에 연결된 고유 식별자(ms-diagnostics 헤더 형식)입니다. 진단 헤더는 선택 사항이며(이러한 헤더를 포함하지 않는 SIP 세션도 가능함) 진단 ID는 일부 유형의 문제가 발생한 세션에 대해서만 보고됩니다.  <br/> |
