# <a name="microsoft-telephony-solutions"></a>Microsoft 전화 통신 솔루션

Microsoft는 Microsoft 클라우드에서 Teams로의 여정을 시작할 때 몇 가지 옵션을 지원하고 있습니다. 이 문서에서는 조직의 사용자에게 적합한 Microsoft 전화 통신 솔루션(클라우드의 전화 시스템 또는 Enterprise Voice-프레미스)과 조직에서 PSTN(Public Switched Telephone Network)에 연결하는 방법을 결정하는 데 도움이 됩니다.

조직에 올바른 결정을 내리는 데 도움이 될 수 있는 관련 기술 다이어그램과 함께 이 문서를 사용해야 합니다.

## <a name="private-branch-exchange-pbx-options"></a>PBX(Private Branch Exchange) 옵션

### <a name="phone-system-microsoft-365-or-office-365"></a>전화 시스템(Microsoft 365 또는 Office 365)
  
전화 시스템은 Microsoft Teams 및/또는 비즈니스용 Skype Online을 사용하여 Microsoft 365 또는 Office 365 클라우드에서 통화 제어 및 PBX(Private Branch Exchange) 기능을 사용하도록 설정하는 Microsoft의 기술입니다.

전화 시스템은 Teams 또는 비즈니스용 Skype Online 클라이언트 및 인증된 장치에서 작동합니다. 전화 시스템을 사용하면 기존 PBX 시스템을 Microsoft 365 또는 Office 365에서 직접 제공되고 회사의 클라우드 생산성 환경과 긴밀하게 통합된 기능 집합으로 바꿀 수 있습니다. 전화 시스템을 PSTN(Public Switched Telephone Network)에 연결하기 위해 Microsoft의 통화 플랜 또는 자체 전화 통신 회사를 선택할 수 있습니다.

자세한 내용은 [Microsoft 365 또는 Office 365의](/MicrosoftTeams/what-is-phone-system-in-office-365)전화 시스템 을 참조하세요.

### <a name="enterprise-voice-skype-for-business-server"></a>Enterprise Voice(비즈니스용 Skype 서버)

Enterprise Voice 는 비즈니스용 Skype 서버에서 통화 제어 및 PBX(Private Branch Exchange) 기능을 사용하도록 설정하기 위한 Microsoft의 기술입니다. 이 비즈니스용 Skype 옵션은 자체 전화 통신 사업자만 사용하여 Public Switched Telephone Network에 연결할 수 있습니다.

자세한 내용은 [Plan for Enterprise Voice in Skype for Business Server 을 참조하십시오.](../../SfbServer/plan-your-deployment/enterprise-voice-solution/enterprise-voice.md?bc=%2fSkypeForBusiness%2fbreadcrumb%2ftoc.json&toc=%2fSkypeForBusiness%2ftoc.json)

## <a name="connection-to-the-public-switched-telephone-network-pstn-options"></a>PSTN(Public Switched Telephone Network) 옵션에 대한 연결

다음을 통해 PSTN(Public Switched Telephone Network)에 연결할 수 있습니다.

- Microsoft 365 또는 Office 365에서 Microsoft 통화 플랜 사용 
- 자체 전화 통신 사업자 연결

### <a name="calling-plan-microsoft-365-or-office-365"></a>통화 플랜(Microsoft 365 또는 Office 365)

이 옵션은 Microsoft 365 또는 Office 365 전화 시스템을 PSTN(Public Switched Telephone Network)에 연결하여 전 세계 일반 전화 및 휴대폰으로 전화를 걸 수 있도록 합니다. 통화 플랜을 통해 Microsoft는 PSTN 통신 사업자입니다.

자세한 내용은 [Microsoft 365 또는 Office 365에](/MicrosoftTeams/calling-plans-for-office-365)대한 통화 플랜을 참조하세요.

### <a name="connect-your-own-telephony-carrier-microsoft-365-or-office-365-and-skype-for-business-on-premises"></a>자체 전화 통신 사업자 연결(Microsoft 365 또는 Office 365 및 비즈니스용 Skype On-premises)

이 옵션은 Microsoft 365 또는 Office 365의 전화 시스템 또는 비즈니스용 Skype Enterprise Voice 시스템과 전화 통신 네트워크에 연결합니다. 이 옵션을 사용하려면 지원되는 SBC(Session Border Controller)가 필요합니다. 경우에 따라 이 옵션을 사용하려면 추가 Microsoft 소프트웨어가 필요한 경우도 있습니다.

## <a name="which-solution-is-right-for-your-organization"></a>조직에 적합한 솔루션은 무엇입니까?

클라우드 올인원 솔루션, 통신사 연결 솔루션 또는 클라우드 내 모든 통신사와 타사 통신사 간의 혼합을 선택할 수 있습니다.

- 통화 플랜이 있는 전화 시스템(모두 클라우드)

- 직접 라우팅을 통해 통신 사업자 소유의 전화 시스템

- 비즈니스용 Skype 서버 또는 클라우드 커넥터 에디션을 통해 통신 사업자 소유의 전화 시스템

- Enterprise Voice 통신사와 함께 비즈니스용 Skype 서버에서 사용 가능

선택하는 솔루션은 현재 및 미래의 요구에 따라 다음을 선택해야 합니다.

- 원하는 기능(또는 필요한 경우)에 따라, 사내 배포에서 제공하는 기능을 유지할 수 있습니다.
- 사용자에 대해 배포할 클라이언트
- 클라우드로 이동하기 위한 계획
- 제3자 PBX 및 기타 전화 통신 장비와 상호 연결해야 하는지 여부

다음 질문을 고려하여 조직에 가장 적합한 솔루션을 결정하십시오.

- 기존 비즈니스용 Skype 서버 배포가 있나요?
- 사용자가 비즈니스용 Skype Online의 클라우드 또는 비즈니스용 Skype Online의 클라우드에 모두 있나요? 
- 사내 사용자를 클라우드로 이동하고 싶나요?
- Microsoft의 PSTN 통화 플랜을 사용자 지역에서 사용할 수 있나요?
- 현재 전화 통신 사업자에 대한 정보를 원하거나 유지해야 하나요?  예를 들어 기존 계약 때문에 현재 통신 사업자만 유지해야 하나요?
- 유지하려는 기존 레거시 PBX가 있나요?
- 현재 레거시 PBX가 비즈니스에 중요한 고유한 기능을 제공하나요?
- 일부 또는 모든 사용자에게 현재 전화 시스템에서 제공되지 않는 기능이 필요한가요?

다음에 유의하세요.

- 복잡한 환경을 위한 솔루션을 디자인하거나 다단계 마이그레이션을 관리해야 하는 경우를 위해 네 가지 옵션을 모두 함께 사용할 수 있습니다.
- 비즈니스용 Skype 서버 또는 클라우드 커넥터 에디션을 통해 자체 통신사가 있는 전화 시스템은 비즈니스용 Skype 서버 또는 클라우드 커넥터를 통해서만 배포할 수 있습니다. 비즈니스용 Skype 서버와 클라우드 커넥터의 동시 사용은 단일 회사에서 지원되지 않습니다.

## <a name="phone-system-with-calling-plan"></a>통화 플랜이 있는 전화 시스템


통화 플랜이 있는 전화 시스템은 다음 다이어그램과 같이 Teams 또는 비즈니스용 Skype Online 사용자를 위한 클라우드 내 모든 옵션입니다.

![통화 플랜이 있는 전화 시스템](../../sfbserver2019/media/msft-telephony-solutions-1.png)


- 사용이 허가되는 서비스 수준에 따라 전 세계 전화로 전화를 걸 수 있는 국내 또는 국제 통화 플랜이 추가된 Microsoft Phone System 
- PSTN 통화 플랜은 Microsoft 365 또는 Office 365에서 작동하기 때문에 이 옵션을 사용하려면 모든 사내 배포를 배포하거나 유지 관리하지 필요가 없습니다.
- 고객은 SBC에서 지원하는 제3자 PBX, 아날로그 장치 및 기타 제3자 전화 통신 장비와의 상호 연동성을 위해 직접 라우팅을 통해 지원되는 SBC를 연결할 수 있습니다.

| 인프라 요구 사항                   | 필수?|
| :----------------------------------------------------| ---------:|
| Microsoft 365 또는 Office 365에 대한 무정전 연결 필요 | 예 |
| 전 세계 사용 가능*                            | 아니요  |
| 지원되는 SBC(Session Border Controller)를 배포하고 유지 관리해야 합니다. | 아니요 |
| 타사 통신사와의 계약 필요      | 아니요   |
| 비즈니스용 Skype 서버 또는 클라우드 커넥터 에디션을 배포하고 유지 관리해야 합니다. | 아니요 |

\*통화 플랜을 사용할 수 있는 국가에 대한 자세한 내용은 오디오 회의 및 통화 플랜의 국가 및 지역 가용성을 [참조하세요.](/MicrosoftTeams/country-and-region-availability-for-audio-conferencing-and-calling-plans/country-and-region-availability-for-audio-conferencing-and-calling-plans)


다음과 같은 질문에 대해 예로 대답하면 이것이 적합한 해결 방법입니다.

- 통화 플랜은 사용자 지역에서 사용할 수 있습니다.
- 현재 PSTN 통신 사업자만 보유할 필요는 없습니다.
- PSTN(Public Switched Telephone Network)에 대한 Microsoft 관리 액세스를 사용하려는 경우
- 세션 경계 컨트롤러를 직접 관리하고 싶지는 않습니다.
- Teams 및/또는 비즈니스용 Skype Online에는 조직에 필요한 모든 기능이 있습니다.

자세한 내용은 [Microsoft 365 및 Office 365의](/MicrosoftTeams/what-is-phone-system-in-office-365) 전화 시스템 및 [Microsoft 365 또는 Office 365에](/MicrosoftTeams/calling-plans-for-office-365)대한 통화 플랜을 참조하세요.

## <a name="phone-system-with-own-carrier-via-direct-routing"></a>직접 라우팅을 통해 통신 사업자 소유의 전화 시스템

이 옵션은 Teams 사용자를 위한 거의 모든 전화 통신 통신 사업자에 클라우드의 Microsoft Phone System을 제공합니다.

![직접 라우팅을 통해 통신사와의 전화 시스템](../../sfbserver2019/media/msft-telephony-solutions-2.png)

- 추가 사내 소프트웨어 없이 지원되는 SBC를 Microsoft Phone System에 직접 연결합니다.  
- Microsoft Phone System에서 거의 모든 전화 통신 사업자 사용.
- 고객 또는 통신 사업자 또는 파트너가 구성하고 관리할 수 있습니다(통신 사업자 또는 파트너가 이 옵션을 제공하는지 요청).
- 타사 PBX 및 아날로그 장치와 같은 전화 통신 장비와 Microsoft Phone System 간의 상호 운영성을 구성합니다.

| 인프라 요구 사항                   | 필수?|
| :----------------------------------------------------| ---------:|
| Microsoft 365 또는 Office 365에 대한 무정전 연결 필요 | 예 |
| 전 세계에 사용 가능                            | 예  |
| 지원되는 SBC(Session Border Controller)를 배포하고 유지 관리해야 합니다. | 예 |
| 타사 통신사와의 계약 필요*      | 예   |
| 비즈니스용 Skype 서버 또는 클라우드 커넥터 에디션을 배포하고 유지 관리해야 합니다. | 아니요 |

\* 통화 플랜이 있는 전화 시스템에 있는 사용자를 위해 제3자 PBX, 아날로그 장치 또는 기타 전화 통신 장비에 대한 연결을 제공하는 옵션으로 배포되지 않는 한

다음과 같은 질문에 대해 예로 대답하면 이것이 적합한 해결 방법입니다.

- Teams를 전화 시스템과 함께 사용하려는 경우
- 현재 PSTN 통신 사업자도 유지해야 합니다.
- 라우팅을 혼합하려는 경우, 일부 통화는 통신사를 통해 통화 요금제로 이동함
- 오버헤드 페이지, 아날로그 장치 등의 제3자 PBX 및/또는 장비와 상호 연결해야 합니다.
- Teams에는 조직에 필요한 모든 기능이 있습니다.

자세한 내용은 [Microsoft 365 및 Office 365의](/MicrosoftTeams/what-is-phone-system-in-office-365) 전화 시스템 및 직접 라우팅 [계획을 참조하세요.](/MicrosoftTeams/direct-routing-plan)


## <a name="phone-system-with-own-carrier-via-skype-for-business-server-or-cloud-connector-edition"></a>비즈니스용 Skype 서버 또는 클라우드 커넥터 에디션을 통해 통신 사업자 소유의 전화 시스템

이 옵션은 비즈니스용 Skype Online 사용자를 위한 사내 전화 통신 네트워크에 연결하여 클라우드의 Microsoft Phone System을 제공합니다.

![비즈니스용 Skype 서버 또는 클라우드 커넥터 에디션을 통해 통신 사업자용 전화 시스템](../../sfbserver2019/media/msft-telephony-solutions-3.png)

 - 비즈니스용 Skype 서버 또는 비즈니스용 Skype 클라우드 커넥터 Edition을 통해 지원되는 SBC를 Microsoft Phone System에 연결합니다. 
- Microsoft Phone System에서 거의 모든 전화 통신 사업자 사용. 
- 비즈니스용 Skype 서버가 이미 있는 경우 이를 활용할 수 있습니다.  그렇지 않은 경우 더 밝은 버전인 Cloud Connector Edition을 배포할 수 있습니다.


| 인프라 요구 사항                   | 필수?|
| :----------------------------------------------------| ---------:|

다음과 같은 질문에 대해 예로 대답하면 이것이 적합한 해결 방법입니다.

- 사용자를 위해 비즈니스용 Skype Online을 사용하려는 경우
- PSTN 통화 플랜은 사용자 지역에서 사용할 수 없습니다.
